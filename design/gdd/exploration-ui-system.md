## 概述

探索UI系统负责探索阶段所有面向玩家的视觉交互界面——包括地图选择界面、节点图主界面、节点交互弹窗、行动力指示器、地图状态面板、Boss战前确认以及通关结算面板。探索UI系统不执行探索逻辑（那是探索系统的职责），而是将探索系统的地图状态和节点数据转化为玩家可理解、可操作的视觉界面，并将玩家的操作（选择地图、移动节点、进入事件/战斗）转化为探索系统的指令。

对玩家来说，探索UI是「修仙世界的地图」——他们在节点图上规划路线、在选择界面中决定去哪张地图、在事件弹窗中做出决策。UI的清晰度和交互流畅度直接决定了探索阶段的沉浸感和策略体验。

## 玩家幻想

探索UI的核心情感是**路径的可见性**——玩家能清晰地看到地图全貌、自己的位置、可选路径和剩余行动力。UI像一张真正的修仙地图，既展示了前方的未知，也记录了走过的足迹。

| 情感 | 触发场景 |
|------|---------|
| **路线的预览** | 在节点图上看到前方分叉——「左边是事件，右边是战斗，我选哪边？」 |
| **距离的感知** | 看到Boss节点还有多远、行动力还够不够——「3步到Boss，刚好够」 |
| **未知的好奇** | 迷雾节点暗示前方有内容——「那里面是什么？」 |
| **选择的确认** | 点击节点时看到节点类型和简短描述——「这个战斗值得打吗？」 |
| **地图的掌控** | 多张地图之间切换选择时的清晰概览——「这张图已经通了，换一张」 |

### 核心规则

```
│                                              │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐   │

│  │重入30 │  │重入60 │  │重入  │  │      │   │
│  │ 灵石  │  │ 灵石  │  │100   │  │      │   │
│  └──────┘  └──────┘  └──────┘  └──────┘   │
│                                              │
│  ┌──────┐  ┌──────┐                          │
│  │天南正│  │天南古│ ← 境界不足（灰色锁）     │

│  │ 未   │  │ 未   │                          │

│                                              │
│  [刷新地图列表]                               │
└──────────────────────────────────────────────┘
```

**界面规则：**
- 地图卡片网格排列（2~4列，根据屏幕宽度自适应）
- 每张卡片展示：
  - 地图名称
  - 难度标记（低/中/高/极高，用1~4颗⭐或颜色表示）
  - 关卡状态（未通关 / 已通关 / 首次通关⭐/ 未解锁🔒）
  - 当前境界下该地图的推荐等级
  - 已通关地图显示「重入费用」（如「重入 30 灵石」）
- 地图卡片交互：
  - 未通关+境界达标 → 点击直接进入（0灵石）
  - 已通关 → 点击弹出确认窗口（显示重入费用 + 灵石余额）
  - 未解锁 → 灰色锁标记，不可点击，悬停显示「需X境界」
  - 境界不足但未解锁 → 灰色+锁，悬停显示所需境界
- 顶部显示当前灵石余额，用于参考重入决策
- 底部显示当前境界/章节进度提示

**布局：**

```
┌──────────────────────────────────────────────┐
│  七玄门·低              AP: ██░░ 3/5  ≡    │
│                                              │
│         🏁 [入口] 📍 ←当前节点              │

│     ╱  ╲      ╱   ╲                        │
│  ❓[事件] ⚔[战斗] 🏪[商店]                 │
│    ╲    ╱      ╲    ╱                       │
│    ⚡[精英]    ❓[事件]                      │
│         ╲    ╱                             │
│         👑 [Boss]                           │
│                                              │
│  [▲返回地图选择]              [卡组查看]      │
└──────────────────────────────────────────────┘
```

**界面规则：**

| 元素 | 视觉表现 |
|------|---------|
| **当前节点** | 金色边框 + 玩家标记（人形图标）固定在节点上 |
| **可达节点** | 彩色正常显示 + 外发光（白色/蓝色脉冲） |
| **已访问节点** | 绿色边框 + ✓标记，稍暗 |
| **不可达节点** | 灰色 + 锁标记，不可点击 |
| **行动力不足的节点** | 灰色 + 红色斜线，悬停显示「行动力不足」 |
| **迷雾节点（未暴露）** | 深色覆盖 + ❓标记，不显示类型 |
| **Boss节点** | 金色/红色王冠图标 + 脉动光环 |
| **精英节点** | 闪电⚡图标 + 红色边框 |
| **事件节点** | ❓图标 + 蓝色边框 |
| **商店节点** | 🏪图标 + 绿色边框 |
| **传送节点** | ✦图标 + 紫色边框 |
| **行动力泉** | 💧图标 + 蓝色脉冲 |
| **渡劫台** | ⚡图标 + 金色脉冲 |
| **回复点** | 💚图标 + 柔和绿光 |
| **入口节点** | 🏁图标 + 固定基底色 |

**交互规则：**
- 点击可达节点 → 玩家标记移动到该节点 || 如果该节点需要确认（如战斗/商店/Boss），先弹确认框
- 不可达节点 → 点击无反应，悬停显示不可达原因
- 鼠标悬停节点 → 显示节点名称 + 简短描述 + 敌人详情（如果是战斗节点）
- 已访问节点 → 点击无反应（不可回退），悬停显示已探索标记
- 点击已暴露的未访问节点 → 如果同一层有其他节点，该节点高亮待选；确认移动后消耗行动力
- 节点间连线：已走过的路径高亮（金色实线），未走过的路径显示为暗色虚线

**节点间连线规则：**
- 已走过的路径：金色/白色实线，线条从源节点中心指向目标节点中心
- 未走过的路径：暗灰色虚线
- 不可达路径（被锁/未解锁）：红色斜线覆盖，不可点击

#### 3. 行动力指示器

位于节点图右上角（或左上角，与节点图不重叠）：

```
AP  ██████░░░░  5/7
```

- 以**格子/分段**展示（每格代表1点行动力）
- 已消耗的格子为暗色/灰色，剩余格子为亮色（蓝/绿）
- 数字显示 current/max
- 行动力 < 30% 时：指示器颜色变为黄色/红色闪烁
- 行动力归零：指示器变灰 + 闪烁效果
- 鼠标悬停显示详情：「剩余行动力：5/7 | 到Boss还需3步」

### 状态与转换

探索UI状态与探索系统状态同步：

| 状态 | 含义 | 交互可用性 |
|------|------|----------|
| **地图选择** | 选择地图界面 | 选择地图、查看地图信息 |
| **地图加载** | 加载动画中 | 不可操作（加载动画） |
| **探索进行中** | 节点图主界面 | 节点移动、查看节点、打开卡组查看 |
| **节点交互（弹窗）** | 节点确认弹窗打开 | 弹窗内操作（进入/取消），节点图背景可见但不可操作 |
| **Boss确认** | Boss战前确认弹窗 | 确认/取消，节点图不可操作 |
| **重入确认** | 重入地图费用确认 | 确认/取消 |
| **通关结算** | 通关奖励面板 | 查看奖励、确认返回 |
| **探索结束** | 结束摘要面板 | 确认返回 |
| **商店界面** | 商店节点中（跳转到商店系统UI） | 商店操作 |
| **战斗中** | 战斗节点中（跳转到战斗UI） | 战斗操作 |

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **探索系统** | 地图选择操作（进入/重入地图）、节点移动指令、节点交互（进入/取消） | 地图列表数据、节点图数据、节点状态、行动力数据 |

## 公式

探索UI系统不执行计算逻辑——所有数值计算由探索系统和相关系统完成。UI只负责展示数据。

### 1. 节点图标映射

```
node_type_to_icon(type: NodeType) → (icon, color, label):
  match type:
    入口:      return (🏁, "#8B7355", "入口")
    普通战斗:  return (⚔, "#CC4444", "战斗")
    精英战斗:  return (⚡, "#FF4444", "精英")
    Boss:      return (👑, "#FFD700", "Boss")
    事件:      return (❓, "#4488CC", "事件")
    商店:      return (🏪, "#44CC88", "商店")
    传送:      return (✦, "#9944CC", "传送")
    行动力泉:  return (💧, "#44AAFF", "灵泉")
    渡劫台:    return (⚡, "#FFAA00", "渡劫台")
    回复点:    return (💚, "#44CC66", "歇脚处")
```

### 2. 行动力格颜色

```
ap_bar_color(current, max) → Color:
  ratio = current / max
  if ratio > 0.5:  return BLUE        # >50% 蓝色
  if ratio > 0.2:  return YELLOW      # 20~50% 黄色
  return RED                           # ≤20% 红色+闪烁
```

## 边界情况

- **所有地图通关但境界不足**：地图选择界面显示所有已通关地图（可重入刷资源），顶部显示境界突破提示
- **灵石不足以重入任何地图**：地图选择界面所有已通关地图标注费用+灵石不足，点击弹出「灵石不足」提示
- **节点图中所有路径都通向同一个Boss**：标准情况——节点图设计保证汇聚到Boss层
- **玩家在探索中游戏最小化**：返回时仍处于当前节点位置（不自动推进）
- **行动力为0但Boss节点已在当前位置**：Boss节点仍可交互（点击Boss弹出确认）
- **首次通关奖励是一张卡牌但卡组已满**：结算面板中奖励卡牌以特殊方式展示（标注「卡组已满，超限进入弃牌流程」）
- **多张地图同时解锁时界面展示**：新地图解锁弹窗以列表形式展示，点击「前往查看」跳转到地图选择界面，新地图有⭐NEW标记
- **节点图中某层只有一个可达节点**：该节点自动为必经节点，UI显示为唯一连接（没有分支选择）
- **玩家尝试在行动力为0时移动**：节点图交互被阻止，点击节点弹出「行动力不足」提示
- **屏幕分辨率过低导致节点图拥挤**：节点间距自动调整（自适应布局），节点图标按比例缩小

## 验收标准

- **GIVEN** 玩家进入探索阶段，**WHEN** 加载完成，**THEN** 显示地图选择界面，已解锁地图彩色+可点击，未解锁地图灰色+锁
- **GIVEN** 地图选择界面，**WHEN** 已通关地图上有重入费用，**THEN** 卡片上显示「重入XX灵石」
- **GIVEN** 已通关地图，**WHEN** 点击，**THEN** 弹出重入确认窗口（费用+余额）
- **GIVEN** 灵石不足时，**WHEN** 点击已通关地图，**THEN** 弹出提示「灵石不足」
- **GIVEN** 选择地图进入，**WHEN** 地图加载完成，**THEN** 显示节点图+行动力指示器+入口高亮
- **GIVEN** 节点图，**WHEN** 点击可达节点，**THEN** 玩家标记移动到该节点，行动力-1
- **GIVEN** 节点图，**WHEN** 点击不可达节点，**THEN** 无反应，悬停显示不可达原因
- **GIVEN** 节点图，**WHEN** 鼠标悬停战斗节点，**THEN** 显示敌人信息预览
- **GIVEN** 行动力变更，**WHEN** 每次消耗，**THEN** 行动力指示器格数减少
- **GIVEN** 行动力≤30%，**WHEN** 查看指示器，**THEN** 颜色变为黄色/红色
- **GIVEN** 到达Boss节点，**WHEN** 点击，**THEN** 弹出Boss战确认窗口
- **GIVEN** Boss战确认窗口，**WHEN** 点击「进入战斗」，**THEN** 切换到战斗UI
- **GIVEN** 战斗胜利，**WHEN** 结算，**THEN** 弹出地图通关结算面板
- **GIVEN** 地图首次通关，**WHEN** 结算面板展示，**THEN** 显示额外首次通关奖励
- **GIVEN** 探索结束，**WHEN** 结算，**THEN** 弹出探索结束面板显示收集摘要
- **GIVEN** 突破新境界，**WHEN** 回到探索界面，**THEN** 弹出新地图解锁提示
- **GIVEN** 探索中打开卡组查看，**WHEN** 浏览完成，**THEN** 关闭后返回节点图
- **GIVEN** 迷雾节点暴露，**WHEN** 移动到邻近层，**THEN** 迷雾消散，节点类型显现
