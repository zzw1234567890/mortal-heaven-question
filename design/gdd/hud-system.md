## 概述

HUD系统（抬头显示系统）负责游戏中始终可见的全局信息展示——包括境界/修为条、灵石计数、卡组数量、行动力（探索中）、回合信息（战斗中）以及通知/提示系统。HUD是所有系统中唯一「始终活跃」的UI层，它不依赖当前场景（探索/战斗/商店），在任何状态下都提供核心状态信息。

对玩家来说，HUD是「修仙之路的仪表盘」——扫一眼就知道自己当前的综合状态。HUD的设计目标是：**信息可扫读，而非可阅读**——玩家不需要仔细看文字，通过颜色、图标和位置就能获取关键信息。

## 玩家幻想

HUD的核心情感是**安心的掌控感**——玩家不需要主动打开任何面板，就能感知到自己的核心状态。

| 情感 | 触发场景 |
|------|---------|
| **状态的一瞥** | 扫一眼右上角就知道灵石够不够、境界是多少 |
| **成长的刻度** | 看到修为条缓慢增长——「快了，快能突破了」 |
| **危机的预兆** | HP条变红、行动力变黄——「需要注意了」 |
| **变化的即时感** | 获得灵石时HUD上的数字跳动——「进账了！」 |
| **通知的不打扰** | 获得道具时角落轻提示——「知道了，不用停下手头的事」 |

## 详细设计

### 核心规则

#### 1. HUD整体布局

```
┌──────────────────────────────────────────────────┐
│  [境界+修为条]                    [灵石]  [卡组] │
│                                                    │
│   [中央主内容区 — 探索/战斗/商店 占据全屏]          │
│                                                    │
│                                      [行动力] 探索 │
│                                      [费用]  战斗  │
└──────────────────────────────────────────────────┘
```

| 位置 | 内容 | 可见场景 |
|:----:|------|---------|
| **左上** | 境界名称+修为进度条 | 所有场景 |
| **右上** | 灵石数量+卡组数量 | 所有场景（卡组仅在探索/商店显示） |
| **右下** | 行动力（探索中）、费用（战斗中） | 对应场景 |
| **中央顶部** | 通知/提示区域 | 按需出现 |
| **左下** | 设置/菜单按钮 | 所有场景（战斗中折叠为暂停） |

#### 2. 境界+修为条（左上）

```
┌────────────────┐
│ 🜁 金丹期·三层  │
│ ████████░░ 80% │ ← 修为进度（当前/上限）
│ ⬆↑ 敌方标记    │ ← 仅在战斗中有高境界敌人时
└────────────────┘
```

**展示规则：**
- 始终显示当前境界名称 + 进度条
- 进度条颜色：<50%蓝色，50~90%紫色，≥90%金色（接近突破）
- 鼠标悬停：显示具体数值（如 1800/2250）
- 修为满时：进度条金色脉动动画 + 「可突破！」文字提示
- 战斗中有高于玩家境界的敌人：境界名称旁显示⬆标记+敌人境界等级
- 化神期满修为：显示「可飞升」替代进度条
- 炼气·落难状态：境界名称显示「炼气·落难」，进度条有特殊光效（破碎感）

#### 3. 灵石+卡组计数（右上）

```
┌────────────────────┐
│ 🪙 1,250           │ ← 灵石计数
│ 📜 28/30           │ ← 卡组数量（仅在探索/商店可见）
│                    │
│  卡组满时：红色数字+闪烁 │
└────────────────────┘
```

**规则：**
- 灵石计数：始终显示，数字超过999时显示「1.2k」格式
- 卡组数量：探索和商店中显示，战斗中不显示（战斗中有独立的牌库计数）
- 卡组达到上限（如30/30）时：数字变为黄色
- 卡组超过上限（如32/30）时：数字变为红色+闪烁+「超限！」标记
- 灵石变更时：数字短暂跳动动画（+xx或-xx

#### 4. 通知/提示区域（中央顶部）

```
┌──────────────────────────────────┐
│  📦 获得道具：回血丹 ×1    [3s]  │ ← 自动消失
│  ⚔ 进入战斗区域              [3s]  │
│  💀 韩立阵亡                  [5s]  │ ← 重要通知停留更久
└──────────────────────────────────┘
```

**通知类型：**

| 类型 | 持续时间 | 颜色 | 示例 |
|------|:-------:|:----:|------|
| 获得道具 | 3s | 绿色 | 「获得 回血丹 ×1」 |
| 获得卡牌 | 3s | 金色 | 「新卡：青元剑诀」 |
| 灵石变更 | 2s | 金色 | 「+25 灵石」 |
| 修为变更 | 2s | 紫色 | 「+50 修为」 |
| 战斗事件 | 3s | 红色/蓝色 | 「韩立阵亡」「境界压制生效」 |
| 系统提示 | 4s | 白色 | 「修为已达瓶颈」 |
| 错误提示 | 4s | 红色闪烁 | 「灵石不足」「卡组已满」 |

**规则：**
- 通知最多同时显示3条（堆叠排列）
- 超过3条时，最早的自动消失
- 重要通知（系统提示、错误）优先级高，不会因为获得道具通知被挤掉
- 通知可点击关闭（鼠标点击立即消失）
- 通知不阻塞玩家操作（非弹窗）

#### 5. 探索中HUD（右下）

```
┌──────────────┐
│ 🏁 七玄门     │ ← 当前地图名
│ AP ██░░ 3/5   │ ← 行动力
│ 📍 层3/5      │ ← 当前层/总层数
└──────────────┘
```

详见探索UI系统——行动力指示器。

#### 6. 战斗中HUD（右下）

```
┌──────────────┐
│ ⚡ 5/5  +1临时 │ ← 费用
│ 📜 牌库:12    │ ← 牌库剩余
│ 💀 弃牌:8     │ ← 弃牌堆
│ 🔄 第3回合    │ ← 回合数
└──────────────┘
```

详见战斗UI系统——费用栏。

#### 7. 过渡提示

场景切换时的过渡信息：

| 场景切换 | 提示内容 | 持续时间 |
|---------|---------|:-------:|
| 探索→战斗 | 「进入战斗」+ 战斗名称 | 1s |
| 战斗→探索 | 无（直接过渡） | — |
| 探索→商店 | 「坊市」图标浮现 | 0.5s |
| 地图加载 | 「加载中...」进度指示 | 按加载时间 |
| 地图通关→地图选择 | 通关结算面板 | 手动关闭 |
| 新地图解锁 | 「新地图已解锁」+ 地图名称 | 3s自动/点击关闭 |

#### 8. 暂停菜单

按ESC或点击左下角暂停按钮打开：

```
┌──────────────────────────────┐
│  ⏸ 暂停                       │
│                              │
│  [继续游戏]                   │
│  [查看卡组]                   │
│  [系统设置]                   │
│  [保存并退出]                 │
│  [返回主菜单]                 │
│                              │
│  探索进度：七玄门 · 层3/5     │
└──────────────────────────────┘
```

- 暂停时冻结所有游戏计时
- 战斗中暂停→冻结回合计时（如有）
- 暂停菜单背景模糊

### 状态与转换

HUD在所有场景下始终可见，但不同场景下显示的内容不同：

| 场景 | HUD元素 |
|------|---------|
| **地图选择** | 左上境界+修为条、右上灵石、暂停按钮 |
| **探索进行中** | 左上境界+修为条、右上灵石+卡组、右下行动力+地图名+层数、暂停按钮 |
| **战斗中** | 左上境界+修为条（含敌方境界标记）、右上灵石（仅图标）、右下费用+牌库+弃牌堆+回合、暂停按钮（战斗中为暂停） |
| **商店中** | 左上境界+修为条、右上灵石+卡组、暂停按钮 |
| **事件中** | 左上境界+修为条、右上灵石、暂停按钮（事件背景HUD半透明） |

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **游戏状态管理器** | 订阅境界、修为、灵石变更信号 | 境界/修为/灵石数据 |
| **卡牌系统** | 订阅卡组变更信号 | 卡组数量 |
| **行动力系统** | 订阅行动力变更信号 | 行动力数据 |
| **境界系统** | 订阅境界变更信号 | 境界名称/层级/修为进度 |
| **资源系统** | 订阅灵石变更信号 | 灵石数据 |
| **战斗系统** | 订阅费用/牌库变更信号 | 费用/牌库数据 |
| **通知系统** | 通知显示请求 | 通知消息 |

## 公式

HUD不执行计算逻辑，仅展示数据。

## 边界情况

- **境界「炼气·落难」状态的HUD显示**：境界名称处显示「炼气·落难」，修为条有破碎特效；重新筑基后恢复正常
- **灵石达到9999+**：显示「9.9k」格式代替
- **同时收到多个通知**：最多同时显示3条堆叠，超出时自动丢弃最早的非重要通知
- **战斗中修为条变化**：修为在战斗中可能增加（部分战斗奖励），修为条实时更新
- **卡组超限时HUD警报**：卡组数量变红闪烁+「超限！」标记，直到玩家处理超限弃牌
- **卡组上限为0（尚未获得任何卡牌）**：显示「0/20」，不触发异常

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 通知显示持续时间 | 3s（普通）/5s（重要） | 2~6s | 来不及看清 | 通知堆积干扰 |
| 同时显示通知上限 | 3条 | 2~5条 | 重要通知可能被挤掉 | 通知区域太大干扰 |
| 灵石显示阈值（k格式） | ≥1000 | ≥1000 | — | — |
| 修为条脉动动画阈值 | ≥90% | 80~100% | 过早脉动失去突破感 | 太晚脉动玩家没注意 |

### 音效需求
- 通知弹出：轻快的提示音（不同通知类型音调不同）
- 灵石增加：碰撞声
- 修为条满：微弱的充能声
- 暂停打开：菜单弹出声

## 验收标准

- **GIVEN** 任何场景，**WHEN** 查看左上角，**THEN** 显示当前境界名称+修为进度条
- **GIVEN** 修为≥90%，**WHEN** 查看修为条，**THEN** 进度条金色脉动动画
- **GIVEN** 炼气·落难状态，**WHEN** 查看境界名称，**THEN** 显示「炼气·落难」
- **GIVEN** 任何场景，**WHEN** 查看右上角，**THEN** 显示灵石数量
- **GIVEN** 探索中，**WHEN** 查看右上角，**THEN** 显示卡组数量
- **GIVEN** 卡组超限，**WHEN** 查看HUD，**THEN** 卡组数字红色闪烁
- **GIVEN** 获得灵石，**WHEN** 变更完成，**THEN** HUD灵石数字跳动更新
- **GIVEN** 获得道具，**WHEN** 触发通知，**THEN** 顶部滑入通知，3s后消失
- **GIVEN** 同时多个通知，**WHEN** 触发，**THEN** 最多3条堆叠显示
- **GIVEN** 按ESC，**WHEN** 暂停菜单弹出，**THEN** 游戏暂停，背景模糊
- **GIVEN** 战斗中暂停，**WHEN** 回到游戏，**THEN** 战斗状态不变
