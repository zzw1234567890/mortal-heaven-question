## 概述 (Overview)

卡牌效果解析引擎是游戏的运行时核心，负责解析和执行全部222张卡牌的效果逻辑。它定义了效果的类型体系、触发时机、目标选择规则、结算顺序和堆叠规则。所有卡牌的行为（伤害、治疗、绑定、阵法激活、状态附加等）都通过此引擎统一解析和结算，确保效果执行的一致性、可预测性和可扩展性。

作为纯基础设施层，玩家不会直接感知引擎的存在——他们感受到的是"卡牌效果如描述那样稳定地发生了"。

## 玩家幻想 (Player Fantasy)

卡牌效果引擎是纯基础设施，玩家不会直接感受到它的存在。玩家感受到的是：每一张卡牌的效果都如描述精确发生——"击杀后攻击+1"就是每击杀一个敌人就永久+1攻击，没有歧义、没有意外。可预测性是策略可信的基石。

## 详细设计 (Detailed Design)

### 核心规则

#### 1. 效果类型体系

所有卡牌效果分为以下三大类：

**① 即时效果 (Instant Effects)** — 打出瞬间结算完毕，不持续存在
- 伤害（对单体/全体）
- 治疗/回复
- 临时费用增加
- 抽牌/弃牌
- 直接移除状态
- 临时属性变化（仅本回合）

**② 持续效果 (Persistent Effects)** — 绑定后持续存在直到条件解除
- 功法绑定（角色存活时永久生效）
- 法宝绑定（角色存活时生效，可能有冷却）
- 阵法光环（阵法部署期间生效）
- 持续buff/debuff（持续N回合）

**③ 触发式效果 (Triggered Effects)** — 满足条件时自动触发
- 回合开始/结束时触发
- 攻击时/被攻击时触发
- 击杀时触发
- 阵亡时触发
- 打出时触发
- 抽牌时触发

#### 2. 触发时机映射（与战斗系统8阶段对齐）

| 战斗阶段 | 可触发的效果类型 | 例子 |
|---------|-----------------|------|
| 阶段0：准备 | 回合开始触发、持续伤害/恢复结算 | 每回合掉1血的中毒 |
| 阶段1：抽牌 | 抽牌时触发 | 抽牌时获得临时属性 |
| 阶段2：出牌 | 出牌时触发（打出卡牌自身效果） | 所有卡牌的主效果 |
| 阶段3：布阵 | 阵法入场效果 | 阵法激活时光环生效 |
| 阶段4：攻击声明 | 攻击前触发 | 攻击前+攻击力 |
| 阶段5：攻击结算 | 命中/伤害/击杀时触发 | 击杀后攻击永久+1 |
| 阶段6：敌方行动 | 受击时触发 | 受击时反弹伤害 |
| 阶段7：结束 | 回合结束触发 | 回合结束时回血 |

#### 3. 效果结算顺序规则

```
同一时机触发的多个效果，结算顺序为：
  第1优先级：玩家主动出牌的效果（当前回合方优先）
  第2优先级：玩家已存在的持续效果（按激活时间从新到旧）
  第3优先级：敌方已存在的持续效果（按激活时间从新到旧）
  第4优先级：同时激活的同类效果 → 按卡牌ID排序
```

#### 4. 目标选择规则

| 目标类型 | 选择方式 | 说明 |
|---------|---------|------|
| 单体己方 | 玩家点击选择 | 功法/法宝绑定时选择目标角色 |
| 单体敌方 | 玩家点击选择 | 符箓攻击时选择目标敌人 |
| 全体己方 | 自动选择 | 全体治疗/群体buff（如掩月回春阵） |
| 全体敌方 | 自动选择 | AOE伤害 |
| 自身 | 强制绑定 | 卡牌只对自己生效 |
| 条件选择 | 自动匹配 | "HP最低的己方角色"等条件 |

#### 5. 效果堆叠规则

| 效果类型 | 堆叠规则 |
|---------|---------|
| 同名功法绑定不同角色 | 可叠加，每角色独立计算 |
| 同名功法绑定同角色 | 效果不可叠加，取最高值 |
| 同名法宝绑定不同角色 | 可叠加 |
| 同名buff/debuff | 不可叠加，刷新持续时间 |
| 不同来源的同功能效果 | 可叠加（加法叠加） |
| 本命加成（×1.5） | 乘法叠加，与其他加成互不干扰 |

#### 6. 状态效果系统

所有状态效果（buff/debuff）使用统一的接口：

```
状态 = {
  id: String（唯一标识）
  name: String
  type: Enum [增益, 减益, 特殊]
  duration: Integer（回合数，-1=永久）
  value: Float/FormulaRef（数值）
  stack_rule: Enum [独立, 刷新, 叠加_上限N]
  source_card_id: String（来源卡牌）
}
```

#### 7. 效果描述规范化

所有卡牌效果的文本描述必须使用统一模板：

| 模板 | 示例 |
|------|------|
| "对【目标】造成【N】点伤害" | "对单体敌方造成3点伤害" |
| "【目标】攻击+X" | "绑定角色攻击+2" |
| "【条件】时，触发【效果】" | "击杀敌方时，自身攻击永久+1" |
| "每回合【时机】，【效果】" | "每回合开始，回复1血" |
| "【概率】概率【效果】" | "30%概率冰冻目标一回合" |

### 状态与转换

效果引擎本身无状态，它处理的效果的状态流转由卡牌系统管理。

### 与其他系统的交互

| 系统 | 数据流入（引擎→目标） | 数据流出（目标→引擎） |
|------|----------------------|----------------------|
| **卡牌系统** | 效果查询（根据效果ID） | 卡牌模板定义中的效果字段 |
| **战斗系统** | 效果结算结果（伤害、状态变更） | 触发信号（阶段转换、事件） |
| **绑定系统** | 绑定/解绑结果 | 功法/法宝的绑定请求 |
| **阵法系统** | 阵法激活/失效事件 | 阵法的触发条件判定 |
| **状态系统** | buff/debuff的附加/移除 | 已存在的状态查询 |
| **AI系统** | 可选效果评分（AI决策用） | 出牌意图 |

## 公式 (Formulas)

### 效果数值缩放

```
effective_value = base_value × binding_multiplier
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| base_value | int | 按卡牌定义 | 效果基准数值 |
| binding_multiplier | float | 1.0 或 1.5 | 本命绑定系数 |

本命激活时 ×1.5，否则 ×1.0。稀有度差异通过 base_value 本身体现。

### 效果叠加规则（数值）

```
stacked = sum(all_sources) —— 不同来源加法叠加
stacked = max(duplicates) —— 同来源同类型只取最高
```

## 边界情况 (Edge Cases)

- **效果同时触发**：同一时机多效果触发时，按 主动出牌→己方持续→敌方持续→卡牌ID 顺序结算
- **目标已死亡**：若效果结算前目标已死亡（被先手击杀），效果不结算（浪费）
- **本命绑定冲突**：同一功法/法宝不可被多个角色本命绑定——第二绑定尝试自动降级为普通绑定
- **效果叠加溢出**：同一角色叠加多个同类型效果时，加法叠加无硬上限（由数值平衡约束）
- **状态持续时间归零**：持续N回合的效果在N回合结束时自动解除
- **免疫与效果**：当目标免疫某效果时，该效果不生效但不消耗卡牌（如丹药仍进弃牌堆）
- **效果触发链**：A触发B触发C时，按栈式结算（C先结算完→B→A），防止无限循环（设最大10层触发链深度）
- **概率效果**：概率型效果（如30%冰冻）在触发时随机判定，每次独立

## 调优参数 (Tuning Knobs)

| 参数 | 默认值 | 安全范围 | 说明 |
|------|:-----:|:--------:|------|
| 触发链最大深度 | 10层 | 5-20 | 防止无限循环的效果连锁 |
| 法术动画保底时间 | 0.3s | 0.1-1.0 | 效果结算动画最短播放时间 |
| 概率精度 | 整数百分比 | 5%步进 | 概率效果的随机粒度 |

## 视觉/音频需求 (Visual/Audio Requirements)

作为基础设施，引擎本身无视觉需求。但每个效果类型应有默认的播放规格：
- 伤害：命中特效 + 数字飘出（0.3s）
- 治疗：绿色光效 + 数字飘出（0.3s）
- 绑定：流光从卡牌飞向目标角色（0.4s）
- 阵法部署：地面展开阵图光效（0.5s）
- 状态附加：角色头顶图标闪现（0.2s）

## 验收标准 (Acceptance Criteria)

- **GIVEN** 玩家打出伤害型符箓，**WHEN** 确认目标，**THEN** 目标HP减少对应数值
- **GIVEN** 玩家打出功法卡并选择目标角色，**WHEN** 结算，**THEN** 目标角色获得功法效果
- **GIVEN** 角色有本命加成条件，**WHEN** 绑定对应功法/法宝，**THEN** 效果数值显示×1.5
- **GIVEN** 多个效果在同一时机触发，**WHEN** 结算，**THEN** 按优先级顺序依次执行
- **GIVEN** 目标在效果结算前已死亡，**WHEN** 该效果结算，**THEN** 效果不生效不报错
- **GIVEN** 概率效果（如30%冰冻），**WHEN** 独立测试100次，**THEN** 触发次数在20-40次之间
- **GIVEN** 效果触发链超过10层，**WHEN** 检查，**THEN** 第11层被终止并记录日志
