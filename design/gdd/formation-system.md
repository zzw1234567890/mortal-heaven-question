# 阵法系统 (Formation System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-22
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱1「自由组牌，策略为王」、支柱2「苟道成长，步步为营」

## 概述

阵法系统管理阵法卡的部署、激活判定、光环效果和失效逻辑。系统维护场上阵法区（最多3个阵法），每个阵法定义一组触发条件（阵营人数≥N、特定角色在场等），条件满足时自动激活提供光环效果，条件不满足时自动失效。系统同时处理阵法覆盖、阵眼角色阵亡导致的阵法失效、以及多阵法时角色归属的冲突解决。

对玩家来说，阵法是构筑策略的"放大器"——卡组和上了场的角色是基础战力，阵法让这些基础战力在有条件的情况下获得额外加成。玩家在组卡和上场时，不仅要考虑角色本身，还要考虑"我的阵容够不够触发这个阵法"。阵法将阵营标签变成了一个有奖励的目标，而非纯粹的身份标识。

暂无相关ADR。

## 玩家幻想

### 创意总监声明 (Creative Director's Vision)

阵法系统是修仙世界观的精髓——它不是一张卡牌被打出就完事，而是一个需要玩家用整个阵容去"供养"的光环。阵法的核心情感不是"我放了一个技能"，而是"我的阵容成型了"——当场上角色刚好满足条件时，阵法光环亮起的那一刻，你感觉自己的构筑有了灵魂。

我确立以下幻想方向，作为所有下游设计的创意北极星。

| 情感层 | 触发场景 | 强度 |
|--------|----------|------|
| **成型的满足** | 场上正道刚好3人，苍玄正道盟阵激活——"成了！" | 中频——流派成型的标志 |
| **阵眼的焦虑** | 核心角色残血，他是阵法的关键条件——"他不能倒！" | 高频——每场战斗的潜在压力 |
| **阵碎的空落** | 阵眼角色阵亡，阵法失效——"阵破了..." | 低频但沉重 |
| **多阵的困惑** | 场上3个阵法同时激活，角色触发哪个？ | 中频——需要手动指定的决策点 |

**关键区别：** 大多数卡牌游戏的"场地/光环"是永久性的。本游戏的阵法是**条件驱动**的——条件满足则激活，条件失去则失效。这让阵法成为了一个动态的、需要玩家在整场战斗中持续维护的系统，而非一次性的buff投掷。

## 详细设计

### 核心规则

#### 1. 阵法区

- 场上固定一个**阵法区**，最多同时部署**3个**阵法
- 阵法区独立于角色阵位——阵法不占用角色位置，角色也不占用阵法位
- 阵法区位于战场顶部 UI（敌方信息下方）

#### 2. 阵法部署流程（战斗阶段2：出牌阶段）

```
① 玩家打出阵法卡 → 扣除费用
② 检查阵法区是否有空位：
   - 有空位（阵法数<3）→ 部署到空位
   - 无空位（阵法数=3）→ 进入覆盖流程（步骤③）
③ 覆盖流程：
   a. 显示当前3个阵法
   b. 玩家选择要覆盖的旧阵法
   c. 旧阵法失效→进入弃牌堆
   d. 新阵法部署到该位置
④ 立即判定触发条件：
   - 条件满足 → 阵法激活，光环生效
   - 条件不满足 → 阵法未激活（灰色显示，等待条件满足）
⑤ 通知效果引擎：阵法部署事件
```

#### 3. 阵法激活与失效

**激活条件自动判定（无玩家手动开关）：**
- 部署时立即判定一次
- 每次"场上状态变更"时重新判定：
  - 角色上场/阵亡/离场/复活
  - 绑定变更（如青锋剑阵依赖绑定）
  - 回合开始/结束时（部分有条件判定的效果）
- 条件满足瞬间激活（无延迟）
- 条件失去瞬间失效（无缓冲）

**激活状态生命周期：**
```
未激活（灰色）→ 条件满足 → 激活（金色/彩色光环）
                                    ↓
                      条件不满足 → 失效（回到未激活）
```

**关键规则：**
- 激活状态的阵法和未激活的阵法**都占用一个阵法位**（3个上限包含未激活的）
- 未激活的阵法不产生任何效果，但可以被覆盖
- 玩家无法手动停用一个激活的阵法

#### 4. 覆盖规则

- 场上已有3个阵法时，打出新阵法→玩家选择覆盖目标
- 覆盖时旧阵法进入弃牌堆（不可恢复）
- 被覆盖的旧阵法如果有阵眼角色，该角色不再受旧阵法影响
- 新阵法在覆盖完成后立即判定条件

#### 5. 多阵法角色归属规则

**核心规则**：当角色满足场上多个阵法的条件时：
- 该角色**仅能享受1个阵法的效果加成**
- 玩家**手动指定**该角色归属哪个阵法（在阵法激活时弹出选择）
- 选择后本场战斗中锁定（或每回合可调整？需确认）

**归属优先级设计：**

我的推荐方案——**首次激活时选择，锁定后持续到被覆盖或该阵法失效**：

```
① 多阵法同时激活时，弹出归属选择面板
② 每个满足条件的角色显示其可归属的阵法列表
③ 玩家为每个角色选择归属
④ 选择后锁定——直到该角色归属的阵法失效（被覆盖或条件不满足）
⑤ 阵法失效后，角色回到"无归属"状态
⑥ 如果再次满足条件，重新选择
```

这比"每回合可调"更简洁，且增加了阵法的沉没成本——你选择了就要承担后果。

#### 6. 阵眼角色阵亡与阵法失效

- 当阵法的条件涉及**特定角色**（如"场上存在林渊"）且该角色阵亡时→阵法立即失效
- 当阵法的条件涉及**阵营人数**（如"场上≥3个正道"）且阵亡导致人数不足→阵法立即失效
- 阵眼角色阵亡后，阵法位保留（占用位），但阵法变为未激活状态
- 如果后续有其他角色上场补足了条件，阵法重新激活（无需重新打出）

#### 7. 阵法效果类型

| 效果类型 | 说明 | 示例 |
|---------|------|------|
| **属性增益** | 直接修改角色属性（HP/ATK/DEF） | 苍玄正道盟阵：HP上限+2，防御+1 |
| **每回合效果** | 每回合自动执行一次 | 玄冰回春阵：全体回复1血 |
| **条件触发** | 特定事件触发 | 魔域血海阵：击杀后攻击永久+2 |
| **概率效果** | 每回合概率判定 | 魅影迷魂阵：25%魅惑男性敌人 |
| **全局buff** | 影响战斗规则 | 万象忘尘阵：抽卡概率+15% |

#### 8. 阵法区UI

- 3个阵法位并排显示在战场上部
- 激活状态：彩色图标+旋转光效
- 未激活状态：灰色图标+锁标记
- 可覆盖时：高亮边框提示
- 归属显示：角色头像旁的阵法小角标（该角色归属哪个阵法）

#### 9. 同阵营梯度阵法

此类阵法的效果随场上同阵营角色数量增长，而非简单的二元（激活/未激活）判定：

**核心规则：**

- **条件门槛**：场上同阵营角色 ≥ 2 人即可激活
- **效果成长**：每增加 1 个同阵营角色，效果提升一个等级
- **生效范围**：效果作用于所有同阵营角色（或全队，视具体阵法定义）
- **封顶机制**：随阵法稀有度决定最大生效人数等级（蓝色 4 级封顶、紫色 5 级封顶、金色 6 级封顶）
- **失效条件**：同阵营人数降到 1 人（低于门槛）→ 梯度阵法失效
- **多阵营判定**：若场上存在多个阵营，取人数最多的阵营计算梯度；平局时取先入场的阵营

**与普通阵营阵法的区别：**

| 维度 | 普通阵营阵法（正魔盟阵） | 同阵营梯度阵法 |
|:----|:----------------------|:--------------|
| 激活方式 | ≥3 人 → 激活，否则不激活 | ≥2 人 → 激活，效果随人数递增 |
| 效果量 | 固定值（如 HP+2） | 梯度值（如 HP+1/2/3/4） |
| 策略含义 | "凑够门槛就行" | "人越多越好" |

**示例梯度阵法：**

| 阵法 | 门槛 | 梯度效果 | 封顶 |
|:----|:----|:--------|:----:|
| 青云合击阵 | ≥2 青云剑宗 | 每多 1 人，青云剑宗角色攻击+1 | 封顶+4（5人封顶） |
| 玄冰回春阵 | ≥2 玄冰宫 | 每多 1 人，全体每回合回复+1HP | 封顶+3（4人封顶） |
| 丹霞剑阵 | ≥2 丹霞谷 | 每多 1 人，全队防御+1 | 封顶+4（5人封顶） |
| 轮回殿轮转阵 | ≥2 轮回殿 | 每多 1 人，轮回殿角色暴击率+5% | 封顶+25%（6人封顶） |

### 状态与转换

#### 阵法状态

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **部署中（未激活）** | 刚打出，条件未满足 | 打出阵法卡 | 条件满足→激活；被覆盖→弃牌堆 |
| **激活** | 条件满足，光环生效 | 条件满足 | 条件不满足→未激活；被覆盖→弃牌堆 |
| **失效** | 条件失去 | 阵眼阵亡/人数不足 | 条件恢复→重新激活；被覆盖→弃牌堆 |
| **弃牌堆** | 从场上移除 | 被覆盖 | 不可恢复 |

#### 角色归属状态

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **无归属** | 未享受任何阵法加成 | 战斗开始、阵法失效 | 归属选择→已归属 |
| **已归属** | 被指定到一个阵法 | 玩家手动选择 | 归属阵法失效→无归属 |

## 公式

### 1. 阵法激活判定

```
is_active(formation) = check_condition(formation.requirement)
```

`formation.requirement` 格式：`{condition_type, tag_id / character_id, min_count}`

### 4. 同阵营梯度阵法效果等级

```
effect_level(formation, tag_id) = min(count_on_field(tag_id) - 1, formation.max_level)
effect_value(formation, tag_id) = formation.base_value × effect_level(formation, tag_id)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| tag_id | String | 阵营标签ID | 梯度的判定阵营 |
| count_on_field | int | [0,6] | 当前场上该阵营的角色数量 |
| max_level | int | [4,6] | 该阵法的最大效果等级（蓝色4/紫色5/金色6） |
| base_value | int | 按阵法定义 | 每级的基础效果增量 |
| effect_level | int | [1, max_level] | 当前效果等级（2人→1级、3人→2级，依此类推） |
| effect_value | int | [base_value, max_level × base_value] | 最终效果数值 |

**验证：** 青云合击阵，场上4个青云剑宗角色 → effect_level = min(4-1, 5) = 3 → effect_value = 1 × 3 = +3攻击

### 2. 覆盖选择

```
overwrite(formation_slot, new_formation):
  old_formation → discard_pile
  release_all_affiliated(new_formation.slot)
  deploy(new_formation)
  try_activate(new_formation)
```

### 3. 角色归属限制

```
max_affiliations(character) = 1
max_formations_on_field = 3
```

## 边界情况

- **3个阵法位全满时打出新阵法**：进入覆盖流程，玩家选择替换目标，旧阵法进弃牌堆
- **阵法全部未激活却满了3个位**：允许——玩家浪费了阵法位，但可以覆盖
- **同一个角色满足3个阵法的条件**：玩家为该角色指定归属阵法（仅1个生效），其余阵法的条件判定不受影响
- **阵眼角色阵亡后阵法失效，但位子保留**：阵法区仍然占用1格，直到被覆盖
- **阵眼角色被涅槃丹复活后重新上场**：阵法条件重新判定→如果阵眼在场，阵法可重新激活
- **覆盖后新阵法立即判定**：覆盖完成瞬间进行激活判定，如果条件满足立即激活
- **覆盖本命阵眼角色的阵法**：如果被覆盖的阵法的阵眼角色已经阵亡，覆盖不影响任何角色（因为阵法已失效）
- **跨阵营阵法如苍玄正邪盟阵**：要求正道+魔道各≥2，正道角色阵亡时→可能降至正1魔2→条件不满足→阵法失效
- **阵法归属的每回合重置**：角色归属锁定到阵法失效为止（不是每回合重新选），简化操作量
- **敌方是否使用阵法？**：敌方AI也可以部署和激活阵法（走同样的规则），但敌方阵法的归属判定由AI自动处理
- **梯度阵法效果超过封顶**：当场上同阵营人数超过封顶值时，效果取封顶值，不溢出（不产生负收益也不额外叠加）
- **梯度阵法的阵营人数下降**：若从4人降到2人，效果等级从3级降到1级，不立即失效（≥2门槛仍满足），威力减弱但不消失
- **梯度阵法从人数不足恢复到满足**：若从1人恢复到2人，效果重新激活从1级开始
- **多阵营平局时的梯度判定**：场上正道3人+魔道3人，梯度阵法取先入场的阵营计算；不可同时为两个阵营生效
- **梯度阵法的同阵营覆盖规则**：覆盖旧梯度阵法时，旧阵法的积累效果是否保留？——不保留，梯度效果是动态计算的，而非累加
- **梯度阵法与固定阵法的叠加**：场上同时存在苍玄正道盟阵（固定增益）和青云合击阵（梯度增益）→ 青云剑宗角色同时享受两个阵法的效果，加法叠加

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 阵法区上限 | 3个 | 2-4 | 阵法策略空间小 | 太多阵法难以管理 |
| 角色归属上限 | 1个 | 1-2 | 单一阵法策略 | 角色可叠加多阵法buff过强 |
| 阵法条件阵营人数 | ≥3人 | 2-4 | 条件太容易满足 | 凑不齐阵法永远无法激活 |
| 归属锁定方式 | 锁定到失效 | 锁定/每回合可调 | 选择压力适中 | 选择压力太小 |

## 验收标准

- **GIVEN** 场上无阵法，**WHEN** 打出阵法卡，**THEN** 阵法部署到阵法区（空位）
- **GIVEN** 场上已有3个阵法，**WHEN** 打出阵法卡，**THEN** 进入覆盖流程
- **GIVEN** 条件满足（如场上≥3正道），**WHEN** 部署对应阵法，**THEN** 阵法立即激活
- **GIVEN** 条件不满足，**WHEN** 部署阵法，**THEN** 阵法未激活（灰色）
- **GIVEN** 阵法未激活时场上状态变更（角色上场补足条件），**WHEN** 自动判定，**THEN** 阵法激活
- **GIVEN** 阵眼角色阵亡，**WHEN** 自动判定，**THEN** 依赖该角色的阵法立即失效
- **GIVEN** 场上正道角色从3人降到2人，**WHEN** 自动判定，**THEN** 苍玄正道盟阵失效
- **GIVEN** 角色满足多阵法条件，**WHEN** 归属选择，**THEN** 玩家手动指定该角色归属
- **GIVEN** 角色已归属某阵法，**WHEN** 该阵法失效，**THEN** 角色回到无归属
- **GIVEN** 覆盖旧阵法，**WHEN** 确认覆盖，**THEN** 旧阵法进入弃牌堆
- **GIVEN** 场上未激活的阵法，**WHEN** 检查阵法位占用，**THEN** 仍计为1/3
