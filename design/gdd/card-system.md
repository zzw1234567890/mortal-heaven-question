> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-23
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱1「自由组牌，策略为王」、支柱2「苟道成长，步步为营」、支柱3「机缘巧合，意外之喜」、支柱4「始于凡人，忠于原著」

## 概述 (Overview)

卡牌系统是《仙途问道》的核心数据基础设施，定义了游戏中全部卡牌的数据结构、类型体系、稀有度层次和收藏管理逻辑。系统支持6种卡牌类型——角色卡、功法卡、法宝卡、阵法卡、丹药卡、符箓卡——每种类型有独立的属性模板和行为规则。卡牌数据驱动所有下游系统（战斗、绑定、阵法、卡组编辑），是游戏内容的基石。

对玩家来说，卡牌系统意味着一个充满发现的收集世界：他们在探索中通过战利品三选一、商店交易、事件选择逐步构筑一副独一无二的卡组——每一局的卡组都是那条特定旅途的产物。通过拆解不需要的卡牌换取灵石、通过本命绑定加成发掘角色+功法+法宝之间的协同效应。约220-250张卡牌覆盖5种流派方向，每局游戏的卡牌收集旅程都独一无二。

## 玩家幻想 (Player Fantasy)

### 创意总监声明 (Creative Director's Vision)

卡牌系统是玩家在这个修仙世界中最直接、最亲密的交互对象。它既是你手中的兵器库，也是你一路走来所经历的一切的见证。在StS式渐进构筑模型下，卡牌系统必须传递的核心情感是：**我不是在抽奖，我是在一趟修仙旅途中用有限的资源和随机缘做出最好的选择。** 每一张卡牌都应该让你想起——"这张是我在碎星群岛遗迹里搏命拿到的"，而不是"这包开出来的"。

我以创意总监的身份确立以下幻想方向，作为所有下游设计（视觉效果、音效、UI、数值平衡）的创意北极星。

---

### 1. 核心情感：凡人旅途 (The Mortal's Journey)

**你在收集和使用卡牌时应该感受到的情感：**

这不是英雄收集神器的爽快感，而是一位谨慎的修士在一次充满变数的旅途中，利用途中的一切资源拼凑出一条生路的应变智慧。

| 情感层 | 触发场景 | 强度 |
|---------|----------|------|
| **应变的智慧** | 战利品三选一——"这三张都不错，但我现在最需要这张" | 常在——每次战利品选择 |
| **豁然开朗** | 发现两张卡牌间意想不到的协同（例如青云剑诀+青锋逐影剑都出现在了本局） | 高频——每次新组合 |
| **后怕式庆幸** | 在关键时刻抽到一张刚好能解局的卡 | 中频——每局1-2次 |
| **百感交集** | 在商店卖掉一张曾经帮你打过Boss的白卡，换取灵石 | 低频——但每次都有分量 |
| **意难平** | 这局没搜到流派的最后一块拼图，没能成型 | 低频——驱动的重玩欲望 |
| **旅途的独一性** | 每局结束后回顾——"这局的卡组和上次完全不同" | 常在——每局结束时 |

卡牌系统不应让玩家感觉"我又抽到了一张没用的垃圾卡"，而应让玩家感觉"这张卡现在用不上，但它的能力我记住了，下局如果路线不同可能就用得上"。

**关键区别：** 大多数卡牌Roguelike的乐趣来自"每次开箱都有新玩具"的新奇感。本游戏的独特之处在于：**每局都是一次独一无二的修仙旅途——途中遇到什么、选择什么、放弃什么，构成了这段旅途的故事。**

### 2. 基调定位：艰辛但充实 (Arduous but Rewarding)

**作为"凡人修仙"而非"天选之子"，卡牌收集的基调：**

基调必须是 **艰辛的**，但不是 **绝望的**。艰辛让每一次获得都有分量。

| 收集方式 | 基调 | 情感回报 |
|---------|------|---------|
| 战斗后掉落 | "险胜之后，战利品是应得的" | 胜任感 |
| 洞府奇遇选择 | "我选对了路线，赌赢了" | 聪明感 |
| 坊市交易购买 | "我攒的灵石终于派上了用场" | 规划感 |
| 反杀夺宝（正道）/杀人夺宝（魔道） | "风险我冒了，回报是我赢来的" | 险中求存 / 侵略性满足 |
| 炼丹炼器制作 | "我用双手打造了这张卡" | 创造感 |
| 渡劫突破奖励 | "我突破了境界，天地的馈赠" | 仪式感 |

> **身份区分**：「杀人夺宝」的侵略性基调仅适用于魔道身份（如血海殿遗孤）。正道/中立身份中，此获取方式重新框定为「反杀战利品」——击退攻击者后的防御性掠夺，情感基调为「险中求存」。

**避免的基调：**
- 不应该是"抽卡出货"的彩票式兴奋——那是天选之子的幻想
- 不应该是"集齐全图鉴"的收集强迫症——那是博物馆式的冷淡
- 不应该是"所有卡我都要"的贪心——凡人的承载力有限（30-40张的限制就是证明）
- 不应该是"一键免费送"的轻浮——付出才有价值

**正确的基调应该是：**
一位修炼有成的修士坐在洞府中，拂过一件件陪伴他走过漫长岁月的法器，每一件都有来历，每一件都是他活着走下去的原因。在StS模型下——**你的卡组不是你提前策展的结果，而是你在这趟旅途中做出的所有选择的总和。** 每一张卡都在诉说你在这趟旅途中的决策：为什么在战利品中选了青云剑诀而不是枯木逢春诀？为什么在商店卖掉了那张陪你走过炼气期的白卡？这些选择——而非卡牌本身——才是你修仙生涯的编年史。

稀有度的价值传递：

| 稀有度 | 玩家应该感觉 | 反面（不这样做） |
|--------|------------|----------------|
| **白色** 凡器 | "这是最基础的工具，但林渊当年也是从枯木逢春诀开始的" | "垃圾卡，凑数的" |
| **蓝色** 法器 | "可靠的中坚力量，我的卡组少不了它" | "过渡用的，早晚要换" |
| **紫色** 灵器 | "这是我在XX地图拼了命拿到的，值" | "也就那样，不出金" |
| **金色** 法宝/秘术 | "这是我构筑的核心，有了它流派才成型" | "终于出金卡了" |
| **暗金** 通天灵宝/大神通 | "整个BD就是围绕它构建的" | "最强卡，无脑带" |

白色卡不是"废卡"，它们是凡人修行的基石。一个聪明的修士知道如何用最低阶的材料打出最有效的组合——这正是"策略为王"的体现：**限制创造创意，资源匮乏催生智慧。**

### 4. 交互框架：分层情感体验

经过创意评估，卡牌系统的体验设计采用**双层框架**。框架A「修者锦囊」的情感内核（精炼的自信、取舍的踏实感）降级为卡组查看时的情绪底色——你在旅途中偶尔打开行囊，审视你已经拥有的东西，思考还缺什么。但它不再控制交互模型（因为StS模型下无自由编辑）。两个主导框架如下：

---

#### 主导框架1：「秘传薪火」(The Legacy of Secret Techniques)

**一句话定义：** 每张卡牌都是一份修仙传承——你获得的不仅是效果数据，更是一段来自原著世界观的修炼智慧。

**核心比喻：** 想象你是一个行走于修仙界的散修。你从青云剑宗废墟中找到半卷残破的青云剑诀残页，从碎星群岛遗迹中发掘枯骨老祖遗留的功法玉简，从万象真人的残魂那里学到推衍之术。每一张卡都承载着一段故事、一个前辈的遗泽。你不是在"收集卡牌"，你是在"继承传承"。

**情感节奏：**
- 获得卡牌：不只有数值提升，更有剧情小段的浮现——"这是当年林渊从XXX手中夺得的……"
- 绑定本命：不只是为了数值加成，而是"我的角色和这张功法心意相通"的叙事感
- 第一张金色：不是"出货了"的兴奋，而是"我终于得到了这门失传秘术"的肃穆
- 面对熟悉的敌人在关键时刻抽到一张克制的卡：不是概率的幻觉，而是"我当年学这门功法就是为了今天"的叙事闭环

**情感核心词：传承的重量 (Weight of Legacy)**——你手中的每一张牌都有来处、有故事。使用它们是对前人传承的延续，而非单纯的策略工具。

**实施启示：** 每张暗金卡首次获得时，触发一段短叙事描述——不是操作提示，而是一句世界观层面的低语。例如获得"混沌宝瓶（大成）"时浮现："瓶身流转着混沌之气——这才是混沌宝瓶的真正面目，你终于等到这一天。"

**对应支柱：** 支柱4（核心），支柱3（外围）
**对应MDA：** 幻想(3)、叙事(4)、发现(2)
**落地难度：** 中（仅暗金卡需要叙事片段，约11条，每条50-80字）

---

#### 主导框架2：「一局棋，一着子」(One Move, One Piece)

**一句话定义：** 卡牌系统如棋局——棋子（卡牌）数量有限但变化无穷，高手之间的差距不在于棋子贵重，而在于布局的深谋远虑。

**核心比喻：** 围棋只有黑白二子，每一子的价值完全取决于它在棋盘上的位置和与周围子的关系。同理，本游戏的卡牌系统——一张白色低费卡在正确的时机、正确的阵位、正确的绑定的组合下，可能比一张金色高费卡发挥更大的战术价值。胜负不在卡牌贵贱，而在你如何落子。

**情感节奏：**
- 前10局：你在学规则，"原来这张卡和那张卡有联动"
- 10-50局：你在探索流派，"正道续航流这样组更强"
- 50-100局：你在精进，"这个对局我换3张卡就能优化"
- 100局以上：你在炫技，"我用纯白色卡组打过金丹Boss"

**情感核心词：棋手的从容 (Strategist's Composure)**——你面对的每次出牌选择都是一次落子，情绪不是紧张而是专注。你知道自己手中有什么、牌库里还有什么、要通向何方。

**实施启示：** 战斗UI中卡牌以标准弧形手牌呈现，但辅以牌库计数、弃牌堆预览等战术信息。强调"你手中是当前回合的战术选择，你牌库中是你这趟旅途累积的全部资源"的心理模型。

**对应支柱：** 支柱1（核心），支柱2（外围）
**对应MDA：** 挑战(1)、表达(5)
**落地难度：** 低（标准卡牌战斗UI + 战术信息面板）

---

### 框架整合：两个关键时刻

卡牌系统在两个关键时刻服务于不同的MDA审美：

- **获得新卡的瞬间 → 框架「秘传薪火」**：尤其暗金卡，赋予重量感和世界观厚度
- **战斗中的出牌决策 → 框架「一局棋」**：让每一手的选择都感觉像深思熟虑的落子，而非碰运气

两者在Run的弧线中交替：探索阶段（获得→薪火）→战斗阶段（决策→棋局）→商店/事件（取舍→锦囊的底色）。这是一个完整的"这趟修仙旅途"的体验闭环。

---

### 验证标准 (Validation Criteria)

如何在"面向玩家幻想"的前提下验证卡牌系统设计正确：

1. 玩家在论坛上讨论时，是说"我抽到了XX"还是"我在XX地图拿到了XX"？ -> 预期后者
2. 玩家拆解一张白色卡时，是快速点过的操作还是稍有犹豫？ -> 预期稍有犹豫（因为每张卡都代表一段经历）
3. 玩家在一局结束后，能回忆起至少2-3张关键卡的"获得故事"？ -> 预期能
4. 新手玩家初始看到境界渐进式卡组容量（炼气20→化神40）时，是感到"好少"还是"够我发挥了"？ -> 预期"够我发挥了"
5. 玩家讨论流派时，是只讨论金色核心，还是也讨论白色/蓝色卡的战术位置？ -> 预期也讨论低稀有度卡的用途
6. 玩家连续两局用同一个身份开局后，是抱怨"又没抽到核心卡"还是说"这局走的路线不一样，卡组也完全不同"？ -> 预期后者

## 详细设计 (Detailed Design)

### 核心规则 (Core Rules)

#### 1. 卡牌数据模型

**架构：模板与实例分离 (Template/Instance Separation)**

卡牌系统采用两层数据架构，以在 Godot 4.6 中正确支持共享模板 + 运行时可变状态：

- **CardTemplate (Resource)**：不可变卡牌定义。222 个 `.tres` 文件，所有同名卡实例共享。存储在 `assets/cards/templates/`。
- **CardInstance (RefCounted)**：运行时可变卡牌副本。每张获得/生成的卡牌拥有独立的等级、铭刻、绑定状态。通过 `card_instance_id` 唯一标识。

> **Godot 关键约束**：Resource 是共享引用——对 Resource 的写入会污染所有引用者。因此模板必须保持只读，所有可变状态必须在 CardInstance 层管理。参见 `card-effect-engine.md` §4 的效果实例化模式。

**CardTemplate（模板）共有字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `card_id` | StringName | 模板唯一标识符（如 `char_lin_yuan_base_01`） |
| `name` | String | 卡牌名称 |
| `type` | Enum | 角色 / 功法 / 法宝 / 阵法 / 丹药 / 符箓 |
| `rarity` | Enum | 白色·凡器 / 蓝色·法器 / 紫色·灵器 / 金色·法宝 / 暗金·通天灵宝 |
| `cost` | Integer | 费用（灵力消耗） |
| `faction_tags` | StringName[] | 阵营标签，最多3个 |
| `description` | String | 规则描述文本（从效果数据自动生成——参见 card-effect-engine.md） |
| `flavor_text` | String | 背景叙述文本（世界观层） |

**CardInstance（实例）附加字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `card_instance_id` | Integer | 全局唯一单调递增ID，由游戏状态管理器分配 |
| `template_id` | StringName | 指向 CardTemplate.card_id |
| `level` | Integer | 1级起，上限由稀有度决定 |
| `inscriptions` | Array | 0-3条铭刻副属性 |
| `breakthrough_layers` | Integer | 0-3层突破 |
| `binding_target_id` | StringName | 当前绑定的角色 instance_id（功法/法宝卡） |
| `acquired_chapter` | Integer | 获得所在章节ID |
| `acquired_event_id` | StringName | 获得来源事件/战斗ID |
| `acquired_method` | Enum | 获得方式：掉落/商店/事件/制作/渡劫奖励 |

**类型专属字段：**

| 类型 | 专属字段 |
|------|----------|
| **角色卡** | `base_hp`, `base_attack`, `innate_skill`, `technique_slots`(3), `artifact_slots`(3) |
| **功法卡** | `effect_type`, `effect_value`, `binding_bonus_mult`(1.5x), `native_owner`(本命角色ID), `stack_limit`(同名叠加张数上限，3-5)，`stack_multiplier`(每层叠加乘数，默认1.5) |
| **法宝卡** | `equip_effect`, `trigger_condition`, `cooldown`, `native_owner`, `stack_limit`(同名叠加张数上限，3-5)，`stack_multiplier`(每层叠加乘数，默认1.5) |
| **阵法卡** | `faction_requirement`, `required_count`, `aura_effect`, `activation_condition` |
| **丹药卡** | `effect_type`, `effect_value`, `duration_turns`, `stack_limit`(3) |
| **符箓卡** | `effect_type`, `effect_value`, `target_type`, `base_fail_chance` |

#### 2. 卡组构建规则

- 卡组容量由境界决定（炼气20→筑基25→金丹30→元婴35→化神40），不含角色卡。详见 `deck-editing-system.md`（卡组规则权威来源）和 `realm-system.md` §2。
- 角色位：6个角色位，不计入卡组容量
- 单种丹药最多带3张入卡组，单种符箓最多带8张入卡组
- 卡组中最多放2张暗金卡（暗金上限按境界渐进：金丹=1张、元婴=2张、化神=2张——见 realm-system.md）
- 卡组构建采用渐进构筑模型（StS式）：开局固定初始卡组 → 战斗战利品三选一 → 商店购买/删除 → 事件增删 → 无自由编辑（详见 deck-editing-system.md）
- 手牌上限：10张。回合结束时若手牌超过上限，需弃至10张

#### 3. 上场规则

- 上场角色数量上限由玩家当前大境界决定：

| 境界 | 可上场角色数 |
|------|:----------:|
| 炼气期 | 2 人 |
| 筑基期 | 3 人 |
| 金丹期 | 4 人 |
| 元婴期 | 5 人 |
| 化神期 | 6 人（上限） |

- 战斗开始前从6个角色位中选择至多上限数量的角色上场
- 上场角色自动以其绑定的功法/法宝效果进入战斗
- 场上最多同时放置3个阵法
- 单个角色仅能触发1个阵法的效果加成（不可叠加）

#### 4. 角色与绑定卡的生死契约

- 角色阵亡时，该角色绑定的所有功法/法宝随角色洗回牌库（而非进入弃牌堆或永久失去）
- 阵亡角色在战斗结束后转为「不可用」状态（本局剩余时间无法再次上场——详见 deployment-system.md §10）
- 绑定卡洗回牌库的设计意图：角色阵亡的战术惩罚是「本场战斗中失去该角色的战力 + 绑定卡需要重新抽取」，而非永久性的构筑破坏。这保留了角色死亡的紧张感，但遵循支柱2「苟道成长」的惩罚原则——渐进式而非灾难性
- 丹药「涅槃丹」可以复活已阵亡的角色（恢复可用状态），但角色返回时为空载状态（之前绑定的功法/法宝已在牌库中，需重新抽到并打出绑定）
- 此规则的调优旋钮：洗回牌库 vs 进入弃牌堆的选择可通过配置切换；若试玩反馈阵亡惩罚太轻，可将"洗回牌库"改为"进入弃牌堆"（需手动回收）作为更高难度选项

#### 5. 本命绑定加成

- 当功法/法宝的 `native_owner` 匹配绑定角色的 `card_id` 前缀时，效果 ×1.5
- 同一张功法/法宝不可被多个角色同时本命绑定
- 本命加成是乘法叠加（而非加法），防止多协同导致数值失控

#### 6. 卡牌生命周期

```
获得 → 收藏 → 构筑入组 → 战斗使用/绑定 → 拆解或永久失去
```

- **获得**：战斗掉落、Boss奖励、探索事件、坊市购买、炼丹炼器制作、渡劫奖励
- **收藏**：已获得未使用的卡牌进入收藏池，可在主菜单图鉴和战利品/商店界面中浏览
- **构筑**：通过战利品三选一、商店买卖删卡、事件增删逐步调整卡组——无独立的自由编辑阶段（详见 deck-editing-system.md）
- **战斗使用**：抽牌→出牌→绑定/消耗
- **拆解**：在商店界面或事件中出售/拆解不需要的卡牌换取灵石，高品质卡拆解产出更多

### 状态与转换 (States and Transitions)

#### 卡牌位置状态

每张卡牌在同一时刻处于以下6种位置状态之一：

```
牌库 (Deck) ←→ 手牌 (Hand) ←→ 场上 (Field) ←→ 弃牌堆 (Graveyard)
                  ↓                    ↓
             绑定状态 (Bound)     阵法激活 (Formation Active)
```

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **牌库** | 卡组中未抽到的牌 | 构筑完成、洗牌 | 抽牌进入手牌 |
| **手牌** | 玩家当前可用的牌 | 抽牌阶段、卡牌效果 | 打出（场上/绑定/消耗）、弃牌 |
| **场上（角色）** | 角色已上场 | 打出角色卡 | 阵亡→弃牌堆 |
| **绑定（功法/法宝）** | 已绑定到角色 | 打出后选择绑定角色 | 角色阵亡→洗回牌库 |
| **场上（阵法）** | 阵法已部署 | 打出阵法卡 | 效果失效/被破除→弃牌堆 |
| **弃牌堆** | 已消耗/阵亡的牌 | 各种离开方式 | 部分卡牌效果可回收 |
| **消耗** | 永久移除 | 部分卡牌效果指定 | 不可恢复 |

#### 卡牌实例成长状态

每张卡牌实例同时持有以下成长状态：

| 状态 | 说明 | 变化方式 |
|------|------|----------|
| **等级** | 1级起，上限由稀有度决定 | 消耗灵石+同名卡升级 |
| **铭刻条数** | 0-3条副属性 | 炼器铭刻增加 |
| **突破层数** | 0-3层，提升基础属性 | 消耗天材地宝突破 |
| **本命绑定** | 是否已激活本命加成 | 绑定对应角色时激活 |

## 公式 (Formulas)

### 1. 角色卡费用模型

**境界驱动模型**——角色卡的费用严格由境界决定，稀有度不影响费用。

| 境界层级 (L) | 费用 Cost |
|:---:|:---:|
| 炼气 (L=1) | 1 |
| 筑基 (L=2) | 2 |
| 金丹 (L=3) | 3 |
| 元婴 (L=4) | 4 |
| 化神 (L=5) | 5 |

### 2. 角色卡属性基准

**属性总值公式：**

```
stat_total(realm, rarity) = base_total + realm_growth × (L-1) + rarity_bonus(r)
```

**境界-稀有度约束：** 暗金卡（r=4）仅在金丹(L≥3)及以上境界存在。炼气(L=1)和筑基(L=2)的暗金卡 stat_total 无定义——实现时若查询应返回错误。

**变量：**

| 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|
| L | int | [1,5] | 境界层级（炼气=1, 筑基=2, 金丹=3, 元婴=4, 化神=5） |
| r | int | [0,4] | 稀有度索引（白=0, 蓝=1, 紫=2, 金=3, 暗金=4）。r=4 时 L≥3 为必要条件 |
| base_total | int | 10 | 白色炼气角色属性总值基准 |
| realm_growth | int | 4 | 每提升一个大境界增加的属性总值 |
| rarity_bonus | int[] | [0, 1, 2, 4, 6] | 按稀有度[白,蓝,紫,金,暗金]的额外属性总值 |

**属性分配规则：**

角色按 HP 占比 70%~80%、ATK 占比 20%~30% 分配属性值：

```
HP = floor(stat_total × 0.7) + tank_adjustment(L, role_type)
ATK = stat_total - HP
```

**tank_adjustment 派生规则：**

```
tank_adjustment(L, role_type):
  若 role_type == "肉盾": 返回 min(2, floor(L / 2) + 1)  // 炼气=1, 筑基=2, 金丹+=2
  若 role_type == "均衡": 返回 1
  若 role_type == "输出": 返回 max(0, 1 - floor(L / 3))  // 炼气~筑基=1, 金丹~元婴=0, 化神=-1
```

此规则替代原来25个手动挑选的魔法数字——`tank_adjustment` 现在完全由境界(L)和角色定位(role_type)派生。

**各境界/稀有度对照表：**

| 境界 | L | 白(r=0) HP/ATK | 蓝(r=1) | 紫(r=2) | 金(r=3) | 暗金(r=4) |
|:----:|:-:|:---------------:|:--------:|:--------:|:--------:|:---------:|
| 炼气 | 1 | 10 → HP8/ATK2 | 11 → HP8/ATK3 | 12 → HP9/ATK3 | 14 → HP11/ATK3 | — |
| 筑基 | 2 | 14 → HP11/ATK3 | 15 → HP11/ATK4 | 16 → HP12/ATK4 | 18 → HP14/ATK4 | — |
| 金丹 | 3 | 18 → HP14/ATK4 | 19 → HP14/ATK5 | 20 → HP15/ATK5 | 22 → HP17/ATK5 | 24 → HP18/ATK6 |
| 元婴 | 4 | 22 → HP17/ATK5 | 23 → HP17/ATK6 | 24 → HP18/ATK6 | 26 → HP20/ATK6 | 28 → HP21/ATK7 |
| 化神 | 5 | 26 → HP20/ATK6 | 27 → HP20/ATK7 | 28 → HP21/ATK7 | 30 → HP23/ATK7 | 32 → HP24/ATK8 |

> **注**：表中数值按 tank_adjustment 公式推算（肉盾角色在各境界均适用 +2/均衡+1）。实际设计每张角色卡时，`role_type` 决定确切的 tank_adjustment 取值。

**输出范围：**
- 最低：炼气白色角色 HP8/ATK2
- 最高：化神暗金角色 HP24/ATK8
- 同境界内高稀有度的优势在于特殊能力和略高属性，而非数值碾压

### 3. 非角色卡费用范围

功法/法宝/阵法/丹药/符箓的费用由效果强度和稀有度共同决定：

| 稀有度 | 费用下限 | 费用上限 |
|:-----:|:-------:|:-------:|
| 白色 | 0 | 2 |
| 蓝色 | 1 | 3 |
| 紫色 | 2 | 4 |
| 金色 | 3 | 5 |
| 暗金 | 5 | 6 |

**费用选取算法：** 在给定稀有度的范围内，具体费用由效果强度决定：
```
cost = min(上限, 下限 + floor(effect_power / power_per_cost[r]))
```
其中 `effect_power` 由 card-effect-engine.md 的效果评估公式计算，`power_per_cost` 为各稀有度每点费用的效果强度权重（白=2, 蓝=3, 紫=4, 金=5, 暗金=6）。此算法适用于程序化卡牌生成；手工设计的卡牌可在范围内自由赋值。

### 4. 拆解产出公式

> **拆解公式以 `resource-system.md` 为权威来源。** 以下复述用于本 GDD 的完整性。与 resource-system.md 不一致时，以 resource-system.md 为准。

```
disenchant_value(rarity, level) → int:
  base = [10, 30, 100, 400, 2000][rarity_index]    // rarity_index: 白=0, 蓝=1, 紫=2, 金=3, 暗金=4
  bonus = floor(base × level × 0.05)                 // 每级 +5% 基础值
  return base + bonus
```

**各稀有度最大等级与拆解产出：**

| 稀有度 | max_level | 1级拆解 | 满级拆解 |
|:-----:|:---------:|:-------:|:--------:|
| 白色 | 1 | 10灵石 | 10灵石（不可升级） |
| 蓝色 | 5 | 30灵石 | 37灵石 |
| 紫色 | 10 | 100灵石 | 150灵石 |
| 金色 | 15 | 400灵石 | 700灵石 |
| 暗金 | 20 | 2000灵石 | 4000灵石 |

> **经济设计备注**：升级在纯粹经济上从来不合理（投入远超拆解回报）——升级是纯战力投资，非经济决策。此设计是有意为之：玩家应为战力升级，而非为转卖升级。

### 5. 卡牌升级公式

```
upgrade_cost(level, rarity) = base_cost[rarity] × (growth_rate ^ (level - 1))
其中 growth_rate = 1.2（可配置，安全范围 1.1-1.5）
```

| 稀有度 | base_cost (灵石) | max_level | 升到满级总成本 |
|:-----:|:---------------:|:---------:|:-------------:|
| 白色 | — | 1 | 不可升级 |
| 蓝色 | 15 | 5 | ~112灵石 |
| 紫色 | 50 | 10 | ~520灵石 |
| 金色 | 150 | 15 | ~2,600灵石 |
| 暗金 | 500 | 20 | ~19,000灵石 |

升级所需同名卡数量：升到等级N需要消耗1张同模板的任意等级卡牌（不消耗灵石替代方案——具体由经济平衡决定）。

### 6. 卡牌掉落概率

> **⚠ 注意：掉落池权重以 `realm-system.md` 为唯一真相来源。** 实际实现时调用 `get_realm_property(L, "card_pool_tier")` 查询境界对应的稀有度权重。本节保留敌人类型倍率和境界差距调节两项规则（卡牌系统独有），与境界无关。

**敌人类型倍率：** 普通×1.0、精英×1.5、Boss×2.0（且保底稀有度+1——稀有度索引最低+1，如白→蓝）

**境界差距调节：** 玩家高于敌人2个大境界以上，掉落倍率降至0.3

**倍率叠加规则：** 敌人类型倍率和境界差距倍率**乘法叠加**。示例：精英(×1.5) + 境界差距(×0.3) = 最终倍率 ×0.45

**暗金解锁门槛：** 仅当敌人境界 ≥ 元婴(L=4) 时，暗金卡才进入掉落池

**每战掉落数量：**
```
drops_per_battle = base_drops + bonus_from_enemy_type
base_drops = 1（常规敌人）/ 2（精英）/ 3（Boss）
bonus_from_enemy_type = 0（常规）/ 1（精英，随机50%概率）/ 1（Boss，确保）
// 范围：1-3张（常规敌人1张、精英1-2张、Boss 3-4张）
```
境界差距调节也影响掉落数量——当倍率降至0.3时，掉落数量 `floor(drops × 0.3)` 但至少保证1张（Boss保底）。

**稀有度权重查询：** 见 `realm-system.md` §5 卡牌掉落池与境界绑定（`get_realm_property(L, "card_pool_tier")` → 权重数组）。

## 边界情况 (Edge Cases)

- **同名多张卡**：卡组中同名卡（如2张枯木逢春诀）可重复携带。同名功法/法宝可绑定到不同角色（各自独立计算），也可叠加到同一角色——效果乘法叠加，所有同名副本共享一个绑定位，上限由 `cardTemplate.stack_limit` 决定（详见 binding-system.md §10「同名卡叠加规则」）。同名丹药最多3张、符箓最多8张
- **卡组容量**：卡组容量由境界决定（炼气20→筑基25→金丹30→元婴35→化神40）。遵循 StS 式渐进构筑模型——卡组无强制下限，最低保护=5张。详见 `deck-editing-system.md`（卡组规则权威来源）。
- **角色位未满**：玩家可只选部分角色进入战斗，实际上场人数 = min(角色位选择人数, 境界上限)
- **手牌上限**：手牌上限10张，回合结束时若手牌超过10张需弃至10张。此上限防止0费卡无限循环和手牌区域不可用
- **0费卡牌约束**：0费卡牌每回合同名卡仅可打出1次——防止无手牌限制场景下的首回合击杀漏洞（FTK）
- **弃牌堆回收**：弃牌堆中的卡牌可通过特定卡牌效果回收（如某些符箓技能），但无法手动回收
- **同名卡拆解**：0费白色卡（如低级炼气丹、速行符）拆解按白色基准10灵石，不打折
- **升级实例化**：卡牌升级作用于单张实例，同名卡的不同实例独立拥有等级
- **绑定卡阵亡后处理**：角色阵亡则绑定卡洗回牌库（非永久失去），收藏池中其他同名卡实例的等级/铭刻不受影响。阵亡角色在本局剩余时间不可用（见 deployment-system.md §10）。
- **暗金卡限制**：卡组中最多带2张暗金卡，即便化神期6个角色位也遵循此上限
- **天赋与境界压制**：角色卡的天赋效果不受境界压制影响——境界压制仅作用于基础攻防数值和伤害计算
- **阵法冲突**：单个角色仅能触发1个阵法的效果加成，场上多阵法时由玩家手动指定该角色归属哪个阵法
- **卡牌效果嵌套**：当一张卡的效果触发另一张卡的效果时，按触发顺序逐层结算，同层级按卡牌ID优先顺序解决

| **炼丹炼器系统** | 软依赖 | 卡牌合成配方查询、新卡牌实例生成 |
| **境界系统** | 软依赖 | 卡牌解锁条件的境界门槛查询 |
| **存档系统** | 硬依赖 | 卡牌实例状态（等级、铭刻、绑定关系）的序列化/反序列化 |

### 设计依赖顺序

1. 游戏状态管理器 → 2. 卡牌系统（数据模型） → 3. 资源系统 → 4. 卡组编辑系统 → 5. 战斗系统 | 绑定系统 | 阵营系统 | 阵法系统 | 卡牌效果引擎 → 6. AI系统 | 存档系统 → 7. 炼丹炼器系统 | 境界系统

## 调优参数 (Tuning Knobs)

以下为可配置数值，安全范围和影响说明：

| 参数 | 默认值 | 安全范围 | 过低的影响 | 过高的影响 |
|------|:-----:|:--------:|-----------|-----------|
| 卡组容量下限 | 无强制下限（最低保护=5张） | 5-20 | 参见 deck-editing-system.md（卡组规则权威来源） |
| 卡组容量上限 | 40张 | 30-50 | 流派变化空间小 | 卡组过于松散 |
| 角色卡费用基准 | L（境界层级） | 不可调 | — | — |
| base_total（白色炼气总值） | 10 | 8-12 | 角色太脆，绑定体验差 | 角色站场过久 |
| realm_growth（每境界成长） | 4 | 3-5 | 后期角色仍然脆弱 | 境界碾压，前期角色无用 |
| HP占比基准 | 70% | 60%-80% | 角色太脆 | 战斗拖沓 |
| 拆解白色卡产出 | 10灵石 | 5-20 | 拆解无意义 | 刷白色卡变相刷钱 |
| 掉落-白色权重占比（炼气期） | 75/100 | 60-90 | 新玩家难以组成卡组 | 获得蓝色卡概率太低 |
| 掉落实例（卡牌掉落数量/战） | 1-3张 | 1-5张 | 收集进度缓慢 | 收藏饱和太快 |
| 暗金卡上限/卡组 | 2张 | 1-3张 | 玩家无法围绕暗金构筑 | 全金队破坏稀有度生态 |
| 升级指数增长率 | 1.2 | 1.1-1.5 | 升级成本过低，快速满级 | 升级成本过高，玩家放弃养成 |
| 本命绑定倍率 | 1.5x | 1.3x-2.0x | 本命加成感觉不明显 | 非本命绑定失去竞争力 |

## 视觉/音频需求 (Visual/Audio Requirements)

卡牌系统是视觉密集型的，每张卡牌需在UI中传递类型、稀有度、境界、阵营信息。

### 卡牌视觉层级

| 层级 | 内容 | 视觉要求 |
|:----:|------|----------|
| 1. 稀有度 | 白/蓝/紫/金/暗金 | 卡牌边框颜色 + 卡面底部光效色调 + 稀有度图标 |
| 2. 卡牌类型 | 角色/功法/法宝/阵法/丹药/符箓 | 卡牌顶部/底部类型徽章 + 不同类型的卡面布局差异 |
| 3. 境界 | 炼气→化神 | 角色卡头像旁的境界标识（如炼气=1颗星） |
| 4. 阵营 | 剑修/体修/法修/丹修/阵修 | 小图标标记在卡牌角落 |
| 5. 本命激活 | 本命绑定状态 | 特殊边框光效或"契"字标识 |

### 卡牌尺寸参考

- 卡牌在手中：~120×168 px（标准卡牌比例 5:7）
- 卡牌放大查看：~240×336 px
- 收藏库网格：~80×112 px（缩略图）

### 动画需求

- 打出卡牌：从手牌滑向目标区域（0.2s）
- 绑定功法/法宝：卡牌化作流光缠绕目标角色（0.4s）
- 角色上场：从牌位飞入战场（0.3s）
- 阵法部署：在地面展开阵图光效（0.5s）
- 丹药/符箓使用：卡牌燃烧/破碎消散（0.3s）
- 拆解卡牌：卡牌熔炼为光点飞入灵石堆（0.4s）
- 获得新卡：卡牌从战利品界面翻转展示（0.5s）

### 音频需求

> **音频权威来源**：card-system.md 是本项目中卡牌音效的唯一权威来源。其他系统（combat-ui-system.md 等）引用此处定义的音频事件，不得重复定义。

#### 卡牌交互音效

- 抽牌：纸牌滑出的清脆声
- 出牌（角色）：沉稳的落地/登场音
- 出牌（功法）：书卷翻动或能量汇聚声
- 出牌（法宝）：金属碰撞或器物共鸣音
- 出牌（阵法）：低沉的嗡鸣声，伴随持续的环境底噪
- 使用丹药：液体倾注或药瓶碰撞声
- 使用符箓：纸张燃烧声，伴随短促的符咒嗡鸣

#### 稀有度音频梯度

所有出牌音效通过参数化处理（pitch-shift + EQ）区分稀有度——无需额外音频样本：

| 稀有度 | 音高偏移 | EQ调整 | 情感效果 |
|:-----:|:--------:|:------:|----------|
| 白色 | 基准 | 基础 | 朴实无华 |
| 蓝色 | +1半音 | 轻微增强低频 | 可靠有分量 |
| 紫色 | +2半音 | 增强中高频 | 锐利有穿透力 |
| 金色 | +3半音 + 额外和声层 | 全频段增强 | 升华感 |
| 暗金 | +5半音 + 双和声层 | 全频段增强 + 混响尾音 | 圣物降临 |

#### 掉落/获得音效

- 白色卡掉落：轻微的纸牌落地声
- 蓝色卡掉落：较清脆的收集音
- 紫色卡掉落：有分量的确认音
- 金色卡掉落：特殊的升华音高 + 短乐句(1s)
- 暗金卡掉落：低沉的圣物共鸣音 + 叙事揭示短乐句(2-3s，参考 Hades 传奇祝福)
- 首次获得暗金卡：以上 + 框架B「秘传薪火」叙事文本配音(可选，依预算而定)

#### 卡牌成长音效

- 卡牌升级：灵光汇聚声，短促上扬（0.5s）
- 铭刻成功：金石刻画声，清脆敲击（0.3s）
- 突破层数：突破屏障的碎裂声 + 能量涌出（0.8s）
- 本命激活：角色与卡牌共鸣的和声（0.6s）

#### 卡牌状态音效

- 卡牌拆解：熔炼的火焰声 + 灵石碰撞声（0.4s）
- 手牌已满提示：轻柔的阻挡音（0.2s）
- 牌库洗牌：多张纸牌快速交错声
- 阵法激活/失效：阵图展开/收缩的环境音（0.5s）

#### 音频技术规范

- 格式：OGG Vorbis，48kHz，单声道（SFX）/ 立体声（环境音与短乐句）
- 响度标准：-18 LUFS（SFX），-23 LUFS（环境音）
- MVP 预算：约15-18个SFX样本 + 5个稀有度参数配置（在audio-system.md预估的30+ SFX范围内）
- 优先级：出牌音（6种类型） > 掉落音（5种稀有度） > 成长音（4种） > 状态音（4种）

## 验收标准 (Acceptance Criteria)

### 卡组与容量
- **GIVEN** 新游戏开局选择「青云剑宗·外门弟子」身份，**WHEN** 进入初始卡组，**THEN** 卡组包含该身份对应的固定初始卡组（角色+功法+法宝+丹药/符箓），总张数 ≥ 8张
- **GIVEN** 玩家处于炼气期，**WHEN** 卡组张数达到20张（境界上限），**THEN** 战利品三选一时第三个选项替换为「跳过」（不可再添加卡牌）
- **GIVEN** 玩家突破到筑基期，**WHEN** 查看卡组容量，**THEN** 容量上限变为25张
- **GIVEN** 玩家卡组仅剩5张卡牌(最低保护)，**WHEN** 在商店选择「删除卡牌」，**THEN** 阻止删除并提示「已达最低卡组数量」

### 卡牌类型限制
- **GIVEN** 玩家卡组中已有2张同名丹药，**WHEN** 战斗奖励的三选一中出现第3张同名丹药，**THEN** 该选项可选择（允许达到上限3张）
- **GIVEN** 玩家卡组中已有3张同名丹药，**WHEN** 商店随机商品中出现同名丹药，**THEN** 该商品标记为「已达上限」且不可购买
- **GIVEN** 玩家卡组中已有2张暗金卡(化神期)，**WHEN** 战利品中出现第3张暗金卡，**THEN** 该选项不可选择并提示「暗金卡超限（上限2张）」

### 角色上场
- **GIVEN** 玩家有6个已选角色位，**WHEN** 境界为炼气期开始战斗，**THEN** 最多上场2人（实际上场人数 = min(勾选人数, 境界上限)）
- **GIVEN** 玩家仅勾选1个角色（少于炼气上限2人），**WHEN** 开始战斗，**THEN** 允许——仅1人上场

### 卡牌打出与绑定
- **GIVEN** 玩家进入战斗，**WHEN** 打出功法/法宝卡并选择目标角色，**THEN** 卡牌从手牌消失，效果附加到目标角色（持续到角色阵亡或战斗结束）
- **GIVEN** 角色卡 `native_owner` 匹配绑定角色的 `card_id` 前缀，**WHEN** 计算效果数值，**THEN** 效果值 ×1.5（逻辑验证），UI显示本命加成标识（视觉验证——分开测试）

### 角色阵亡与复活
- **GIVEN** 角色阵亡，**WHEN** 检查绑定状态，**THEN** 该角色的所有绑定功法/法宝洗回牌库（非进入弃牌堆、非永久失去）
- **GIVEN** 角色阵亡后用「涅槃丹」复活，**WHEN** 检查绑定状态，**THEN** 该角色空载（无绑定卡，原绑定卡已在牌库中）
- **GIVEN** 角色阵亡，**WHEN** 战斗结束后检查角色位，**THEN** 该角色标记为「不可用」——本局剩余时间无法再次上场

### 手牌限制
- **GIVEN** 玩家手牌已有10张，**WHEN** 抽牌阶段触发，**THEN** 不再抽牌并提示「手牌已满」
- **GIVEN** 玩家回合结束时手牌有12张（超限），**WHEN** 进入弃牌阶段，**THEN** 系统要求弃至10张
- **GIVEN** 玩家本回合已打出1张0费符箓卡，**WHEN** 尝试打出同名0费符箓卡，**THEN** 阻止并提示「本回合同名0费卡已使用」

### 阵法
- **GIVEN** 场上已有3个阵法，**WHEN** 玩家再打出阵法卡，**THEN** 提示「场上阵法已达上限(3个)」，卡牌留在手牌
- **GIVEN** 场上有2个阵法、3个己方角色，**WHEN** 玩家为新上场的第4个角色指定阵法归属，**THEN** 弹出阵法选择界面——该角色仅能从已有阵法中选择1个归属
- **GIVEN** 单个角色已归属阵法A，**WHEN** 尝试将其归属改为阵法B，**THEN** 允许——该角色从阵法A脱离，归属阵法B

### 拆解/出售
- **GIVEN** 玩家在商店出售一张1级白色卡，**WHEN** 结算，**THEN** 获得10灵石
- **GIVEN** 玩家在商店出售一张1级金色卡，**WHEN** 结算，**THEN** 获得400灵石（精确值，不含"+"）
- **GIVEN** 玩家确认拆解一张稀有度为蓝色及以上的卡牌，**WHEN** 拆解完成，**THEN** 界面提供5秒撤销按钮（"撤销拆解"），点击后恢复卡牌并退还灵石

### 卡牌掉落
- **GIVEN** 玩家高于敌人2个大境界以上，**WHEN** 战斗结束掉落结算，**THEN** 掉落倍率为正常值的0.3倍（乘法叠加于敌人类型倍率之上）
- **GIVEN** 敌人为元婴期 Boss，**WHEN** 战斗掉落结算，**THEN** 暗金卡进入掉落池（稀有度权重由 realm-system.md 定义）
- **GIVEN** 敌人为金丹期 Boss，**WHEN** 战斗掉落结算，**THEN** 暗金卡不在掉落池中

### 牌库与弃牌堆
- **GIVEN** 牌库为空，**WHEN** 抽牌阶段触发，**THEN** 弃牌堆自动洗入牌库，然后继续抽牌
- **GIVEN** 同回合中牌库第二次为空，**WHEN** 再次需要抽牌，**THEN** 不触发洗牌——仅当弃牌堆非空时洗牌，若弃牌堆也为空则跳过抽牌

### 本命绑定不可共享
- **GIVEN** 功法卡「青云剑诀」(native_owner=林渊)已绑定到角色A(林渊)，**WHEN** 尝试将同名功法卡「青云剑诀」绑定到角色B(非林渊)，**THEN** 允许绑定但本命加成不激活
- **GIVEN** 功法卡「青云剑诀」已绑定到角色A(林渊)且激活本命加成，**WHEN** 尝试将同名功法卡也绑定到角色A，**THEN** 阻止——同一角色不可绑定同名功法/法宝多次

### 卡牌实例独立性
- **GIVEN** 玩家拥有2张同名「枯木逢春诀」卡牌实例，**WHEN** 将实例A升级到3级，**THEN** 实例B保持1级——两个实例的等级独立

### 卡牌效果嵌套
- **GIVEN** 卡牌A的效果触发卡牌B的效果，**WHEN** 两者在同一层级结算，**THEN** 按 card_id 字母序优先结算（确定性结果，非随机）

### 获得来源追踪
- **GIVEN** 玩家通过战斗掉落获得一张卡牌，**WHEN** 在图鉴中查看该卡，**THEN** 显示「获得于第X章，XX战斗」（acquired_chapter + acquired_event_id 对应文本）
