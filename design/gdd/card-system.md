# 卡牌系统 (Card System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-21
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱1「自由组牌，策略为王」、支柱2「苟道成长，步步为营」、支柱3「机缘巧合，意外之喜」、支柱4「始于凡人，忠于原著」

## 概述 (Overview)

卡牌系统是《凡人修仙传：掌天问道》的核心数据基础设施，定义了游戏中全部卡牌的数据结构、类型体系、稀有度层次和收藏管理逻辑。系统支持6种卡牌类型——角色卡、功法卡、法宝卡、阵法卡、丹药卡、符箓卡——每种类型有独立的属性模板和行为规则。卡牌数据驱动所有下游系统（战斗、绑定、阵法、卡组编辑），是游戏内容的基石。

对玩家来说，卡牌系统意味着一个丰富的收藏世界：他们在探索中收集新卡牌、在营寨中自由构筑30-40张的卡组、通过拆解不需要的卡牌换取灵石、通过本命绑定加成发掘角色+功法+法宝之间的协同效应。222张卡牌覆盖5种流派方向，每局游戏的卡牌收集旅程都独一无二。

## 玩家幻想 (Player Fantasy)

### 创意总监声明 (Creative Director's Vision)

卡牌系统是玩家在这个修仙世界中最直接、最亲密的交互对象。它既是你手中的兵器库，也是你一路走来所经历的一切的见证。卡牌系统必须传递的核心情感是：**我不是在抽奖，我是在经营一个修仙者的传承。** 每一张卡牌都应该让你想起——"这张是我在乱星海遗迹里搏命拿到的"，而不是"这包开出来的"。

我以创意总监的身份确立以下幻想方向，作为所有下游设计（视觉效果、音效、UI、数值平衡）的创意北极星。

---

### 1. 核心情感：凡人藏经阁 (The Mortal's Archive)

**你在收集和使用卡牌时应该感受到的情感：**

这不是英雄收集神器的爽快感，而是一位谨慎的修士在漫长岁月中积累家底的踏实感。

| 情感层 | 触发场景 | 强度 |
|---------|----------|------|
| **踏实感** | 你的卡组构筑完毕，30-40张每一张都有明确作用 | 常在——每次编辑完成 |
| **豁然开朗** | 发现两张卡牌间意想不到的协同（例如青元剑诀+青竹蜂云剑绑定韩立） | 高频——每次新组合 |
| **后怕式庆幸** | 在关键时刻抽到一张刚好能解局的卡 | 中频——每局1-2次 |
| **百感交集** | 拆解一张曾经帮你打过Boss的白卡，换取灵石 | 低频——但每次都有分量 |
| **意难平** | 这局没搜到流派的最后一块拼图，没能成型 | 低频——驱动的重玩欲望 |

卡牌系统不应让玩家感觉"我又抽到了一张没用的垃圾卡"，而应让玩家感觉"这张卡现在用不上，但它的能力我记住了，下次流派可能用得上"。

**关键区别：** 大多数卡牌Roguelike的乐趣来自"每次开箱都有新玩具"的新奇感。本游戏的独特之处在于：**收集的满足感来自积累和优化，而非获得。** 拆解一张卡不是为了把它当垃圾丢掉，而是把它作为材料转化为灵石——就像修仙者将用不上的法器熔炼为精粹。这本身就符合核心幻想：凡人从不浪费任何资源。

### 2. 基调定位：艰辛但踏实 (Arduous but Earned)

**作为"凡人修仙"而非"天选之子"，卡牌收集的基调：**

基调必须是 **艰辛的**，但不是 **绝望的**。艰辛让每一次获得都有分量。

| 收集方式 | 基调 | 情感回报 |
|---------|------|---------|
| 战斗后掉落 | "险胜之后，战利品是应得的" | 胜任感 |
| 洞府奇遇选择 | "我选对了路线，赌赢了" | 聪明感 |
| 坊市交易购买 | "我攒的灵石终于派上了用场" | 规划感 |
| 杀人夺宝 | "风险我冒了，回报是我抢来的" | 侵略性满足 |
| 炼丹炼器制作 | "我用双手打造了这张卡" | 创造感 |
| 渡劫突破奖励 | "我突破了境界，天地的馈赠" | 仪式感 |

**避免的基调：**
- 不应该是"抽卡出货"的彩票式兴奋——那是天选之子的幻想
- 不应该是"集齐全图鉴"的收集强迫症——那是博物馆式的冷淡
- 不应该是"所有卡我都要"的贪心——凡人的承载力有限（30-40张的限制就是证明）
- 不应该是"一键免费送"的轻浮——付出才有价值

**正确的基调应该是：**
一位修炼有成的修士坐在洞府中，拂过一件件陪伴他走过漫长岁月的法器，每一件都有来历，每一件都是他活着走下去的原因。这就是"凡人藏经阁"的幻想——你的卡组就是你修仙生涯的编年史。

稀有度的价值传递：

| 稀有度 | 玩家应该感觉 | 反面（不这样做） |
|--------|------------|----------------|
| **白色** 凡器 | "这是最基础的工具，但韩立当年也是从长春功开始的" | "垃圾卡，凑数的" |
| **蓝色** 法器 | "可靠的中坚力量，我的卡组少不了它" | "过渡用的，早晚要换" |
| **紫色** 灵器 | "这是我在XX地图拼了命拿到的，值" | "也就那样，不出金" |
| **金色** 法宝/秘术 | "这是我构筑的核心，有了它流派才成型" | "终于出金卡了" |
| **暗金** 至宝/通天之术 | "整个BD就是围绕它构建的" | "最强卡，无脑带" |

白色卡不是"废卡"，它们是凡人修行的基石。一个聪明的修士知道如何用最低阶的材料打出最有效的组合——这正是"策略为王"的体现：**限制创造创意，资源匮乏催生智慧。**

### 4. 候选框架：玩家与卡牌系统交互的情感体验

经过创意评估，以下三个框架候选作为卡牌系统的体验设计北极星。选择哪一个取决于最终我们希望玩家在关掉游戏后印象最深的是什么。

---

#### 框架A：「修者锦囊」(The Cultivator's Satchel)

**一句话定义：** 你的卡组是你的百宝囊——每一张卡都是你主动决定携带的工具，不是被动接收的收藏品。

**核心比喻：** 一个经常远行的修士不会把家当全部背上。他会仔细斟酌：这张符箓占一格，但可能救命；这件法器占一格，但我在这个区域用不上；这瓶丹药虽然好，但我有了足够的替代品。锦囊有限，智慧无限。

**情感节奏：**
- 开局：锦囊空空，只有几件可靠的老家伙
- 探索中：不断收获新的"工具"，在营寨中反复权衡留下哪件、拆掉哪件
- 成型：锦囊中的每件物品都是精挑细选，没有浪费的空间
- 渡劫：锦囊的考验——你的选择是否经得起最严苛的检验

**情感核心词：精炼的自信 (Refined Confidence)**——每一次打开锦囊，你都确定里面的每一张卡都有一个位置、一个用途、一个理由。不是"我有什么"，而是"我选择带什么"。

**UI与视觉启示：** 卡组编辑界面不应像仓库，而应像一个打开的锦囊/储物袋。待选卡牌以"散落在桌上"而非"列表"的方式呈现，强化"我在挑选随身物品"的心理模型。

**对应支柱：** 支柱1（核心），支柱2（外围）
**对应MDA：** 挑战(1)、表达(5)

---

#### 框架B：「秘传薪火」(The Legacy of Secret Techniques)

**一句话定义：** 每张卡牌都是一份修仙传承——你获得的不仅是效果数据，更是一段来自原著世界观的修炼智慧。

**核心比喻：** 想象你是一个行走于修仙界的散修。你从七玄门废墟中找到半卷残破的青元剑诀残页，从乱星海遗迹中发掘玄骨上人遗留的功法玉简，从大衍神君的残魂那里学到推衍之术。每一张卡都承载着一段故事、一个前辈的遗泽。你不是在"收集卡牌"，你是在"继承传承"。

**情感节奏：**
- 获得卡牌：不只有数值提升，更有剧情小段的浮现——"这是当年韩立从XXX手中夺得的……"
- 绑定本命：不只是为了数值加成，而是"我的角色和这张功法心意相通"的叙事感
- 第一张金色：不是"出货了"的兴奋，而是"我终于得到了这门失传秘术"的肃穆
- 面对熟悉的敌人在关键时刻抽到一张克制的卡：不是概率的幻觉，而是"我当年学这门功法就是为了今天"的叙事闭环

**情感核心词：传承的重量 (Weight of Legacy)**——你手中的每一张牌都有来处、有故事。使用它们是对前人传承的延续，而非单纯的策略工具。

**实施启示：** 每张金卡或暗金卡首次获得时，触发一段短叙事描述——不是操作提示，而是一句世界观层面的低语。例如获得"掌天瓶（大成）"时浮现："瓶身流转着混沌之气——这才是掌天瓶的真正面目，你终于等到这一天。"

**对应支柱：** 支柱4（核心），支柱3（外围）
**对应MDA：** 幻想(3)、叙事(4)、发现(2)

---

#### 框架C：「一局棋，一着子」(One Move, One Piece)

**一句话定义：** 卡牌系统如棋局——棋子（卡牌）数量有限但变化无穷，高手之间的差距不在于棋子贵重，而在于布局的深谋远虑。

**核心比喻：** 围棋只有黑白二子，每一子的价值完全取决于它在棋盘上的位置和与周围子的关系。同理，本游戏的卡牌系统——一张白色低费卡在正确的时机、正确的阵位、正确的绑定的组合下，可能比一张金色高费卡发挥更大的战术价值。胜负不在卡牌贵贱，而在你如何落子。

**情感节奏：**
- 前10局：你在学规则，"原来这张卡和那张卡有联动"
- 10-50局：你在探索流派，"正道续航流这样组更强"
- 50-100局：你在精进，"这个对局我换3张卡就能优化"
- 100局以上：你在炫技，"我用纯白色卡组打过金丹Boss"

**情感核心词：棋手的从容 (Strategist's Composure)**——你面对的每次出牌选择都是一次落子，情绪不是紧张而是专注。你知道自己的棋子在哪里、为什么在那里、要通向何方。

**实施启示：** 战斗UI中，卡牌不应以"手牌"方式呈现（暗示随机抽取的运气要素），而应以"布局/选项"方式呈现——你手上的是你提前选好的棋子，现在只是落子时机的问题。这强化"策略为王"而非"抽牌看脸"的心理模型。

**对应支柱：** 支柱1（核心），支柱2（外围）
**对应MDA：** 挑战(1)、表达(5)

---

### 框架选择建议

| 维度 | 框架A「修者锦囊」 | 框架B「秘传薪火」 | 框架C「一局棋」 |
|------|----------------|----------------|----------------|
| **情感核心** | 精炼的自信 | 传承的重量 | 棋手的从容 |
| **MDA主力** | 挑战(1), 表达(5) | 幻想(3), 叙事(4), 发现(2) | 挑战(1), 表达(5) |
| **支柱主力** | 支柱1 | 支柱4 > 支柱3 | 支柱1 > 支柱2 |
| **差异化优势** | 解释了"为什么卡有限制反而有趣" | 区分了我们和杀戮尖塔 | 解释了策略深度的来源 |
| **落地难度** | 中（主要是UI和编辑体验） | 高（需要大量叙事植入） | 低（主要是交互框架和UI隐喻） |
| **对222张卡的覆盖** | 所有卡都有战术位置 | 只有金/暗金有故事 | 所有卡都有战略价值 |

**最终的框架整合建议：**

不要选择单一框架，而是按交互场景分层使用：

- **卡组编辑场景 -> 框架A「修者锦囊」**：让编辑卡组感觉像行囊准备，强调整合和取舍的踏实感
- **获得新卡的瞬间 -> 框架B「秘传薪火」**：尤其金卡和暗金卡，赋予重量感和世界观厚度
- **战斗中的出牌决策 -> 框架C「一局棋」**：让每一手的选择都感觉像深思熟虑的落子，而非碰运气

三者在不同时刻服务于不同的MDA审美，共同组成完整的卡牌幻想体系。这是我作为创意总监的最终创意指导方向。

---

### 验证标准 (Validation Criteria)

如何在"面向玩家幻想"的前提下验证卡牌系统设计正确：

1. 玩家在论坛上讨论时，是说"我抽到了XX"还是"我在XX地图拿到了XX"？ -> 预期后者
2. 玩家拆解一张白色卡时，是快速点过的操作还是稍有犹豫？ -> 预期稍有犹豫（因为每张卡都代表一段经历）
3. 玩家在一局结束后，能回忆起至少2-3张关键卡的"获得故事"？ -> 预期能
4. 新手玩家初始看到30-40张的卡组限制时，是感到"好少"还是"够我发挥了"？ -> 预期"够我发挥了"（精炼感）
5. 玩家讨论流派时，是只讨论金色核心，还是也讨论白色/蓝色卡的战术位置？ -> 预期也讨论低稀有度卡的用途

## 详细设计 (Detailed Design)

### 核心规则 (Core Rules)

#### 1. 卡牌数据模型

**所有卡牌共有字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `card_id` | String | 唯一标识符（如 `char_hanli_base_01`） |
| `name` | String | 卡牌名称 |
| `type` | Enum | 角色 / 功法 / 法宝 / 阵法 / 丹药 / 符箓 |
| `rarity` | Enum | 白色·凡器 / 蓝色·法器 / 紫色·灵器 / 金色·法宝 / 暗金·至宝 |
| `cost` | Integer | 费用（灵力消耗） |
| `faction_tags` | String[] | 阵营标签，最多3个 |
| `description` | String | 规则描述文本 |
| `flavor_text` | String | 背景叙述文本（世界观层） |

**类型专属字段：**

| 类型 | 专属字段 |
|------|----------|
| **角色卡** | `base_hp`, `base_attack`, `innate_skill`, `technique_slots`(3), `artifact_slots`(3) |
| **功法卡** | `effect_type`, `effect_value`, `binding_bonus_mult`(1.5x), `native_owner`(本命角色ID) |
| **法宝卡** | `equip_effect`, `trigger_condition`, `cooldown`, `native_owner` |
| **阵法卡** | `faction_requirement`, `required_count`, `aura_effect`, `activation_condition` |
| **丹药卡** | `effect_type`, `effect_value`, `duration_turns`, `stack_limit`(3) |
| **符箓卡** | `effect_type`, `effect_value`, `target_type`, `base_fail_chance` |

#### 2. 卡组构建规则

- 卡组总容量：上限40张（不含角色卡）——详见 deck-editing-system.md
- 角色位：6个角色位，不计入40张卡组上限
- 单种丹药最多带3张入卡组，单种符箓最多带8张入卡组
- 卡组中最多放2张暗金卡
- 卡组构建采用渐进构筑模型（StS式）：开局固定初始卡组 → 战斗战利品三选一 → 商店购买/删除 → 事件增删 → 无自由编辑（详见 deck-editing-system.md）

#### 3. 上场规则

- 上场角色数量上限由玩家当前大境界决定：

| 境界 | 可上场角色数 |
|------|:----------:|
| 炼气期 | 2 人 |
| 筑基期 | 3 人 |
| 金丹期 | 4 人 |
| 元婴期 | 5 人 |
| 化神期 | 6 人（上限） |

- 战斗开始前从6个角色位中选择至多上限数量的角色上场
- 上场角色自动以其绑定的功法/法宝效果进入战斗
- 场上最多同时放置3个阵法
- 单个角色仅能触发1个阵法的效果加成（不可叠加）

#### 4. 角色与绑定卡的生死契约

- 角色阵亡时，该角色绑定的所有功法/法宝随角色进入弃牌堆，永久失去
- 丹药「大还丹」可以复活已阵亡的角色，但不恢复已失去的绑定卡（复活后角色为空载状态）
- 此规则使角色阵亡的代价有真实分量——玩家的构筑投入与生存风险管理紧密挂钩

#### 5. 本命绑定加成

- 当功法/法宝的 `native_owner` 匹配绑定角色的 `card_id` 前缀时，效果 ×1.5
- 同一张功法/法宝不可被多个角色同时本命绑定
- 本命加成是乘法叠加（而非加法），防止多协同导致数值失控

#### 6. 卡牌生命周期

```
获得 → 收藏 → 构筑入组 → 战斗使用/绑定 → 拆解或永久失去
```

- **获得**：战斗掉落、Boss奖励、探索事件、坊市购买、炼丹炼器制作、渡劫奖励
- **收藏**：已获得未使用的卡牌进入收藏池，可自由浏览
- **构筑**：从收藏池中选择卡牌编入卡组，随时在营寨调整
- **战斗使用**：抽牌→出牌→绑定/消耗
- **拆解**：不需要的卡牌可拆解为灵石，高品质卡拆解产出更多

### 状态与转换 (States and Transitions)

#### 卡牌位置状态

每张卡牌在同一时刻处于以下6种位置状态之一：

```
牌库 (Deck) ←→ 手牌 (Hand) ←→ 场上 (Field) ←→ 弃牌堆 (Graveyard)
                  ↓                    ↓
             绑定状态 (Bound)     阵法激活 (Formation Active)
```

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **牌库** | 卡组中未抽到的牌 | 构筑完成、洗牌 | 抽牌进入手牌 |
| **手牌** | 玩家当前可用的牌 | 抽牌阶段、卡牌效果 | 打出（场上/绑定/消耗）、弃牌 |
| **场上（角色）** | 角色已上场 | 打出角色卡 | 阵亡→弃牌堆 |
| **绑定（功法/法宝）** | 已绑定到角色 | 打出后选择绑定角色 | 角色阵亡→弃牌堆 |
| **场上（阵法）** | 阵法已部署 | 打出阵法卡 | 效果失效/被破除→弃牌堆 |
| **弃牌堆** | 已消耗/阵亡的牌 | 各种离开方式 | 部分卡牌效果可回收 |
| **消耗** | 永久移除 | 部分卡牌效果指定 | 不可恢复 |

#### 卡牌实例成长状态

每张卡牌实例同时持有以下成长状态：

| 状态 | 说明 | 变化方式 |
|------|------|----------|
| **等级** | 1级起，上限由稀有度决定 | 消耗灵石+同名卡升级 |
| **铭刻条数** | 0-3条副属性 | 炼器铭刻增加 |
| **突破层数** | 0-3层，提升基础属性 | 消耗天材地宝突破 |
| **本命绑定** | 是否已激活本命加成 | 绑定对应角色时激活 |

## 公式 (Formulas)

### 1. 角色卡费用模型

**境界驱动模型**——角色卡的费用严格由境界决定，稀有度不影响费用。

| 境界层级 (L) | 费用 Cost |
|:---:|:---:|
| 炼气 (L=1) | 1 |
| 筑基 (L=2) | 2 |
| 金丹 (L=3) | 3 |
| 元婴 (L=4) | 4 |
| 化神 (L=5) | 5 |

### 2. 角色卡属性基准

**属性总值公式：**

```
stat_total(realm, rarity) = base_total + realm_growth × (L-1) + rarity_bonus(r)
```

**变量：**

| 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|
| L | int | [1,5] | 境界层级（炼气=1, 筑基=2, 金丹=3, 元婴=4, 化神=5） |
| r | int | [0,4] | 稀有度索引（白=0, 蓝=1, 紫=2, 金=3, 暗金=4） |
| base_total | int | 10 | 白色炼气角色属性总值基准 |
| realm_growth | int | 4 | 每提升一个大境界增加的属性总值 |
| rarity_bonus | int[] | [0, 1, 2, 4, 6] | 按稀有度[白,蓝,紫,金,暗金]的额外属性总值 |

**属性分配规则：**

角色按 HP 占比 70%~80%、ATK 占比 20%~30% 分配属性值：

```
HP = floor(stat_total × 0.7) + tank_adjustment
ATK = stat_total - HP
```

其中 `tank_adjustment` 对肉盾角色 +1~2 HP（相应减少 ATK），对输出角色 -0~1 HP（相应增加 ATK）。

**各境界/稀有度对照表：**

| 境界 | L | 白(r=0) HP/ATK | 蓝(r=1) | 紫(r=2) | 金(r=3) | 暗金(r=4) |
|:----:|:-:|:---------------:|:--------:|:--------:|:--------:|:---------:|
| 炼气 | 1 | 10 → HP8/ATK2 | 11 → HP8/ATK3 | 12 → HP9/ATK3 | 14 → HP11/ATK3 | — |
| 筑基 | 2 | 14 → HP11/ATK3 | 15 → HP11/ATK4 | 16 → HP12/ATK4 | 18 → HP14/ATK4 | — |
| 金丹 | 3 | 18 → HP14/ATK4 | 19 → HP14/ATK5 | 20 → HP15/ATK5 | 22 → HP17/ATK5 | 24 → HP18/ATK6 |
| 元婴 | 4 | 22 → HP17/ATK5 | 23 → HP17/ATK6 | 24 → HP18/ATK6 | 26 → HP20/ATK6 | 28 → HP21/ATK7 |
| 化神 | 5 | 26 → HP20/ATK6 | 27 → HP20/ATK7 | 28 → HP21/ATK7 | 30 → HP23/ATK7 | 32 → HP24/ATK8 |

**输出范围：**
- 最低：炼气白色角色 HP8/ATK2
- 最高：化神暗金角色 HP24/ATK8
- 同境界内高稀有度的优势在于特殊能力和略高属性，而非数值碾压

### 3. 非角色卡费用范围

功法/法宝/阵法/丹药/符箓的费用由效果强度和稀有度共同决定：

| 稀有度 | 费用下限 | 费用上限 |
|:-----:|:-------:|:-------:|
| 白色 | 0 | 2 |
| 蓝色 | 1 | 3 |
| 紫色 | 2 | 4 |
| 金色 | 3 | 5 |
| 暗金 | 5 | 6 |

### 4. 拆解产出公式

```
spirit_stones(card) = base_value(rarity) + level_bonus(rarity, card_level)
base_value(r) = floor(10 × rarity_scale[r])
level_bonus(r, lv) = level_base[r] + floor((lv-1) × level_mult[r])
```

**变量：**

| 符号 | 值/范围 | 描述 |
|------|:-------:|------|
| rarity_scale | [1, 3, 10, 40, 200] | 各稀有度灵石倍率 |
| level_base | [0, 1, 2, 5, 10] | 各稀有度等级基准值 |
| level_mult | [0, 0.5, 0.5, 0.5, 0.5] | 每级等级增益 |

**各稀有度1级拆解产出：**
| 稀有度 | 1级拆解 | 满级拆解 |
|:-----:|:-------:|:--------:|
| 白色 | 10灵石 | 10灵石（不可升级） |
| 蓝色 | 30灵石 | 32灵石 |
| 紫色 | 100灵石 | 109灵石 |
| 金色 | 400灵石 | 435灵石 |
| 暗金 | 2000灵石 | 2095灵石 |

### 5. 卡牌掉落概率

> **⚠ 注意：掉落池权重以 `realm-system.md` 为唯一真相来源。** 以下数值为历史参考，实际实现时调用 `get_realm_property(L, "card_pool_tier")` 查询境界对应的稀有度权重。本节保留敌人类型倍率和境界差距调节两项规则（卡牌系统独有），与境界无关。

**敌人类型倍率：** 普通×1.0、精英×1.5、Boss×2.0（且保底稀有度+1）

**境界差距调节：** 玩家高于敌人2个大境界以上，掉落倍率降至0.3

**暗金解锁门槛：** 仅当敌人境界 ≥ 元婴(L=4) 时，暗金卡才进入掉落池

**稀有度权重查询：** 见 `realm-system.md` §5 卡牌掉落池与境界绑定（`get_realm_property(L, "card_pool_tier")` → 权重数组）。

## 边界情况 (Edge Cases)

- **同名多张卡**：卡组中同名卡（如2张长春功）可重复携带，同名功法/法宝可绑定到不同角色，同名丹药最多3张、符箓最多8张
- **卡组容量**：卡组容量由境界决定（炼气20→筑基25→金丹30→元婴35→化神40）。遵循 StS 式渐进构筑模型——卡组无强制下限，最低保护=5张。详见 `deck-editing-system.md`（卡组规则权威来源）。
- **角色位未满**：玩家可只选部分角色进入战斗，实际上场人数 = min(角色位选择人数, 境界上限)
- **手牌上限**：无手牌上限，回合结束时保留全部手牌进入下一回合
- **弃牌堆回收**：弃牌堆中的卡牌可通过特定卡牌效果回收（如某些符箓技能），但无法手动回收
- **同名卡拆解**：0费白色卡（如低级炼气丹、速行符）拆解按白色基准10灵石，不打折
- **升级实例化**：卡牌升级作用于单张实例，同名卡的不同实例独立拥有等级
- **绑定卡阵亡后继承**：角色阵亡则绑定卡永久失去（进入弃牌堆），但收藏池中其他同名卡实例的等级/铭刻不受影响
- **暗金卡限制**：卡组中最多带2张暗金卡，即便化神期6个角色位也遵循此上限
- **天赋与境界压制**：角色卡的天赋效果不受境界压制影响——境界压制仅作用于基础攻防数值和伤害计算
- **阵法冲突**：单个角色仅能触发1个阵法的效果加成，场上多阵法时由玩家手动指定该角色归属哪个阵法
- **卡牌效果嵌套**：当一张卡的效果触发另一张卡的效果时，按触发顺序逐层结算，同层级按卡牌ID优先顺序解决

## 验收标准 (Acceptance Criteria)

- **GIVEN** 新游戏开局，**WHEN** 查看初始收藏池，**THEN** 至少有3种初始身份对应的预设卡组可预览
- **GIVEN** 玩家在卡组编辑界面，**WHEN** 从收藏池拖入角色卡，**THEN** 角色计入角色位（不计入30-40张计数）
- **GIVEN** 玩家在卡组编辑界面，**WHEN** 卡组总张数不足30或超过40，**THEN** 系统提示不可出发战斗
- **GIVEN** 玩家有6个已选角色，**WHEN** 境界为炼气期开始战斗，**THEN** 最多上场2人
- **GIVEN** 玩家进入战斗，**WHEN** 打出功法/法宝卡并选择目标角色，**THEN** 卡牌从手牌消失，效果永久附加到目标角色
- **GIVEN** 角色阵亡，**WHEN** 检查绑定状态，**THEN** 该角色的所有绑定功法/法宝进入弃牌堆
- **GIVEN** 角色阵亡后用「大还丹」复活，**WHEN** 检查绑定状态，**THEN** 该角色空载（无绑定卡）
- **GIVEN** 角色卡 `native_owner` 匹配绑定角色，**WHEN** 计算效果数值，**THEN** 显示为 ×1.5 的绿色加成标识
- **GIVEN** 玩家拆解一张白色卡，**WHEN** 结算，**THEN** 获得10灵石
- **GIVEN** 玩家拆解一张金色卡，**WHEN** 结算，**THEN** 获得400+灵石
- **GIVEN** 场上已有3个阵法，**WHEN** 玩家再打出阵法卡，**THEN** 提示已达上限
- **GIVEN** 玩家携带3张同名丹药入卡组，**WHEN** 检查卡组，**THEN** 允许
- **GIVEN** 玩家携带4张同名丹药入卡组，**WHEN** 检查卡组，**THEN** 阻止并提示上限为3张
- **GIVEN** 玩家卡组中有3张暗金卡，**WHEN** 保存卡组，**THEN** 提示暗金卡超限（上限2张）
- **GIVEN** 玩家高于敌人2个大境界以上，**WHEN** 战斗结束翻牌，**THEN** 掉落数量显著减少（境界差距调节生效）
- **GIVEN** 敌人为元婴期 Boss，**WHEN** 战斗掉落结算，**THEN** 暗金卡进入掉落池
- **GIVEN** 敌人为金丹期 Boss，**WHEN** 战斗掉落结算，**THEN** 暗金卡不掉落
