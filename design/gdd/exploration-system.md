> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-23
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱3「机缘巧合，意外之喜」、支柱2「苟道成长，步步为营」

## 概述

探索系统管理「从进入地图到挑战Boss」的完整探索阶段——包括地图选择、节点图生成、节点导航、事件/战斗/商店等节点的触发调度，以及地图通关后的奖励结算。探索系统是「地图上的操作者」：它不负责事件内容（那是事件系统的职责）、不负责战斗流程（那是战斗系统的职责），而是负责地图的结构化呈现和玩家在其中的移动与交互调度。

对玩家来说，探索是两次战斗之间的「呼吸节奏」——在节点图上选择路线、在事件中做出决策、在商店中消费资源、在回复点调整状态。探索系统的质量决定了每局游戏的「路径感」：同一张地图，不同路径带来截然不同的体验组合。探索系统也是Roguelike重复可玩性的核心载体——随机节点配置让每局的路都有新鲜感。

## 玩家幻想

探索系统的核心情感是**路径的审慎感**——每一次移动都是一个微决策：走左边的事件还是右边的战斗？这条近路能省行动力但跳过了支路奖励。这种「选择路径」的体验贯穿整个探索阶段。

| 情感 | 触发场景 |
|------|---------|
| **路线的审慎** | 面对分岔路口——三个节点连过来两个方向，选哪个？ |
| **刚好够到的满足** | 行动力刚好到Boss节点——「算得刚刚好，舒服」 |
| **错失的遗憾** | 行动力不够走支路，放弃一个事件点——「那里面可能有好东西」 |
| **奇遇的惊喜** | 到达事件节点发现好选项——「这趟走得值」 |
| **碾压的回味** | 回到旧地图高境界横扫——「这里已经困不住我了」 |
| **Boss前的紧张** | 到达Boss节点前最后一格——「准备够了吗？上了」 |

探索不是让玩家「看风景」，而是让玩家**规划路径并承担规划的结果**。每条路径都代表一个隐性的决策树：选这条路→遇到这些事件/战斗→获得这些资源→以这种状态面对Boss。探索系统的核心乐趣不在「走」本身，而在「决定往哪走」的思考过程。

## 详细设计

### 核心规则

#### 1. 地图选择

玩家在探索阶段开始时选择要进入的地图：

- 当前境界突破后，该境界下**所有地图一次性解锁**（境界系统定义map_unlock）
- 玩家从已解锁的地图列表中选择一张进入，列表中显示：
  - 地图名称 + 难度评级（低/中/高/极高）
  - 最高允许境界
  - 推荐境界 + 压制提示（如果当前境界高于地图允许）
  - 该地图已获得的通关奖励（如未通关则显示「未通关」）
- 炼气期地图全部通关后解锁炼气→筑基的渡劫条件（境界系统管辖）
- 玩家**不可重复进入同一地图的同一张节点图**（地图通关后标记为「已通关」，下次进入同一地图会生成新的节点图）
- 已通关地图**再次进入需要消耗灵石**（传送费），费用根据该地图的已进入次数和地图难度计算

**地图重入消耗规则：**

重入传送费公式见§公式 — 地图重入传送费。此处保留概览：

| 参数 | 默认值 | 说明 |
|:----:|:-----:|------|
| 低难度地图基价 | 30 灵石 | 青云剑宗、苍玄正道盟、碎星外环 |
| 中难度地图基价 | 60 灵石 | 血渊秘境、碎星内海 |
| 高难度地图基价 | 100 灵石 | 云澜城、西域古林、西域边境、慕兰草原、归墟·浮云陆、归墟·万妖谷 |
| 极高难度地图基价 | 150 灵石 | 丹霞谷、万古殿、西域、归墟通道、归墟·古魔战场 |
| 苍玄古战场 | 免费（触发剧情专用，通关后不可重入） | 剧情特殊处理 |

**次数递增规则：**
- `entry_count` 为该地图在本局中的总进入次数（含第1次）
- 第1次进入 — 0灵石（首次免传送费）
- 第2次进入 — 基价×1.0
- 第3次进入 — 基价×1.5
- 第4次进入 — 基价×2.0
- 以此类推，每多进1次+50%
- 详细公式见§公式 — 地图重入传送费

**永久免费地图（经济安全阀）：**
- 每个境界保留**1张永久免费地图**，通关后重入始终免费（不消耗灵石，不递增entry_count）
- 炼气期永久免费：**青云剑宗**
- 筑基期永久免费：**碎星外环**
- 金丹期永久免费：**西域古林**
- 元婴期永久免费：**慕兰草原**
- 化神期永久免费：**归墟·浮云陆**
- 永久免费地图确保玩家始终有至少1个可免费进入的地图，防止「全通+灵石不足」导致无法继续游戏

**交互流程：**
```
① 玩家在地图选择界面点击已通关地图
② 弹出确认窗口：
   - 永久免费地图：「[地图名]为永久免费地图，可无限次重入」
   - 非免费地图：「重入[地图名]需要消耗[XX]灵石，确认传送？」
   - 显示当前灵石余额
   - 显示本次费用 / 累计已进入次数
   - [确认传送] / [取消]
③ 确认后：
   - 非免费地图：调用 spend_resource("ling_shi", cost)
     · 余额充足 → 扣除灵石，进入地图
     · 余额不足 → 提示「灵石不足」，按钮灰色不可选
   - 永久免费地图 → 直接进入（无消耗）
④ 玩家仍可免费选择未通关地图（无消耗）
```

**地图状态：**

| 状态 | 含义 |
|------|------|
| **未解锁** | 境界不足，不可见 |
| **已解锁** | 境界达标，可进入 |
| **已通关** | 已击败该地图Boss，可消耗灵石再次进入（生成新节点图）；永久免费地图除外 |

#### 2. 节点图布局

每张地图是一张**有向无环图（DAG）**，由多层节点组成，玩家从入口层（第0层）逐层推进到Boss层（末层）：

```
示例：中型地图节点图布局

入口(第0层)          [入口]
                    /    \
第1层          [战斗]    [事件]
                /  \    /    \
第2层        [事件] [战斗] [商店]
               \    /    \    /
第3层          [精英]    [事件]
                    \    /
第4层(Boss)        [Boss]
```

**布局规则：**
- 每张地图有4~6层（含入口层和Boss层），具体层数由地图配置决定
- 每层有2~4个节点，节点数量随层数变化（Boss层固定1个）
- 每个节点连接**相邻层的1~2个节点**（确保连通性——不存在不可达节点）
- 同一层内节点互不相连（只能跨层移动）
- 除Boss节点外，至少存在2条不同的路径到达Boss（避免单一线性路径）
- 所有路径最终汇聚到Boss节点

**节点类型一览：**

| 节点类型 | 图标 | 描述 | 消耗行动力 |
|---------|:----:|------|:--------:|
| **入口** | 🏁 | 地图起始点，无交互 | 不消耗 |
| **普通战斗** | ⚔ | 与普通敌人战斗，获得基础奖励 | 1 |
| **精英战斗** | ⚡ | 与精英敌人战斗，获得更高级奖励 | 1 |
| **Boss** | 👑 | 地图最终Boss，击败后通关 | 1 |
| **事件** | ❓ | 触发随机事件（由事件系统处理） | 1 |
| **商店** | 🏪 | 消耗灵石购买卡牌/丹药 | 1 |
| **传送** | ✦ | 不消耗行动力的快速移动 | 0 |
| **行动力泉** | 💧 | 回复全部行动力（每图限1次） | 0 |
| **渡劫台** | ⛈ | 修为已满时可触发渡劫突破 | 1 |
| **回复点** | 💚 | 队伍全体回复50%已损失HP | 1 |

**节点分布规则（按地图难度）：**

| 地图难度 | 总层数 | 每层节点数 | 精英数 | 商店数 | 事件数 |
|:-------:|:-----:|:---------:|:-----:|:-----:|:-----:|
| 低 | 4层（含入口+Boss） | 2~3 | 0~1 | 0~1 | 1~2 |
| 中 | 5层（含入口+Boss） | 2~4 | 1~2 | 1 | 2~3 |
| 高 | 5层（含入口+Boss） | 3~4 | 2 | 1~2 | 2~3 |
| 极高 | 6层（含入口+Boss） | 3~4 | 2~3 | 1~2 | 2~4 |

#### 3. 地图生成流程

```
generate_map(map_id, player_realm):
  ① 读取地图配置模板（层数、节点类型池、每层节点数范围）
  ② 根据配置生成节点图骨架（DAG结构）
  ③ 分配节点类型：
     a. 第0层固定为入口
     b. 末层固定为Boss
     c. 其余层从节点类型池中加权随机分配
     d. 确保精英战数量、商店数量符合地图难度配置
  ④ 连接节点（保证连通性）：
     a. 每层每个节点至少连接上层1个节点
     b. 至少2条独立路径可达Boss
  ⑤ 分配具体事件模板到事件节点（调用事件系统的加权选择）
  ⑥ 分配具体敌人阵容到战斗节点（调用AI系统的地图敌人池）
  ⑦ 分配商店库存到商店节点（调用卡牌系统/资源系统的商店表）
  ⑧ 地图生成完成 → 等待玩家进入
```

**随机性保证：**
- 同一张地图每次进入生成不同的节点布局（节点类型位置和连接关系变化）
- 事件节点的具体事件内容在到达该节点时才确定（避免SL刷事件）
- 商店库存的内容在所有商店节点生成时确定（但玩家看到前不展示具体内容）
- 战斗节点的敌人阵容在节点生成时确定

#### 4. 节点导航

**可见性规则：**

| 条件 | 节点显示状态 |
|------|-------------|
| 当前所在层及所有已暴露的节点 | 完全可见（类型图标 + 名称） |
| 下方2层的所有相邻节点 | 可见类型图标 + 名称 |
| 其他未暴露层（当前层+3及更深） | 显示为迷雾节点（类型隐藏） |

玩家到达新节点后，该节点所在层及下方相邻层的节点变为可见。此2层可见度使玩家能提前规划2步路线——在「规划路径」的策略深度与「探索未知」的发现惊喜之间取得平衡。

**移动规则：**
- 从当前节点移动到**下一层的可达节点**（有边连接），消耗1行动力
- 不可在同一层内横向移动（无跨层连接）
- 不可回退到上一层的已访问节点
- 不可跳过未探索的中间层（必须逐层推进）
- 传送节点允许跨层移动到指定节点（消耗0行动力，但需已解锁传送路径）

**已访问节点标记：**
- 入口 + 已访问节点 → 标记为绿色（已探索）
- 不可达节点 → 灰色（路径未连接或未解锁）
- 行动力不足 → 节点图标变灰，悬停显示"行动力不足"

#### 5. 行动力限制与探索结束

行动力系统提供 `current_ap`，探索系统消费：

- 每次移动到非传送节点 → 消耗1行动力
- 玩家停留在当前节点不消耗行动力（可查看信息、整理卡组等）
- 行动力归零但仍然在Boss节点 → 照常挑战Boss（不消耗额外行动力）
- 行动力归零但Boss节点未到达 → 提前终结探索阶段（进入Boss战或结算）

**探索阶段结束条件：**

| 条件 | 处理方式 |
|:----:|---------|
| 击败Boss | 通关→地图通关奖励结算→返回地图选择界面 |
| 行动力归零且无法到达Boss | 立即结算（获得已收集的资源，无通关奖励） |
| 行动力归零但已在Boss节点 | 可选择立即挑战Boss（不消耗行动力） |
| 战斗失败 | 按战败结算（保留50%本局探索中累积获得的修为，灵石/卡牌/物品全额保留） |
| 战斗中撤退 | 按战败处理，保留已获得的非绑定物品 |

**地图通关奖励：**

击败Boss后结算：
```
map_clear_reward():
  ① 固定奖励：根据地图难度获得灵石（低=50、中=100、高=200、极高=300）
  ② 修为奖励：根据地图等级获得修为（低=50、中=80、高=120、极高=150）
  ③ 首次通关额外奖励：根据地图配置（可能是卡牌/丹药/天赋）
  ④ 地图标记为「已通关」
  ⑤ 检查是否满足下一境界的突破条件
  ⑥ 返回地图选择界面
```

#### 6. Boss节点与地图通关

- 每张地图Boss节点固定为末层（最后一层）
- Boss战击败后地图通关，不可重复挑战同一次生成的Boss
- Boss战失败：按标准战败处理，角色阵亡的绑定卡永久失去，角色标记不可用
- Boss战失败后玩家回到地图选择界面（该地图未通关，可再次进入但需重新生成节点图）
- Boss的难度由地图难度 + 当前境界共同决定（确保回低境界地图时Boss难度不碾压）

#### 7. 回旧地图的特殊规则

当玩家回到已通关的低境界地图时：

- 地图等级使用该地图的默认配置（敌人强度按地图本身境界设定）
- 玩家的境界属性被**柔性压制**到该地图的 `max_allowed_realm`——而非完全压制：
  - 费用上限和基础速度：压制到 `map_max_L`（进攻属性受限于地图——高境界玩家不能秒杀低境界敌人）
  - 上场人数和行动力上限：**不压制**，使用玩家当前境界的实际值（防御/战术属性保留——玩家因成长而在从容度上获得可感的优势）
  - 此柔性压制设计确保「碾压的回味」幻想得以实现：玩家不会秒杀敌人，但能用更多角色、更充裕的行动力来从容应对——成长是真实可感的
- 卡牌掉落按当前 card_pool_tier 的下一档计算
- **灵石奖励减少**「境界差额惩罚」——境界差越大，能捞到的灵石越少（高阶修士去低级地图无利可图）。惩罚应用于通关奖励和战斗战利品中的灵石
- Boss奖励中的修为按地图基准值计算（不是按当前境界）
- 回旧地图仍消耗行动力，且地图生成新的节点图（非重复使用旧图）
- 永久免费地图（见§1）不受境界差额惩罚影响——即使gap≥2，免费地图始终可重入，灵石收益按实际公式计算

#### 8. 地图初始解锁顺序

初始状态仅解锁**青云剑宗**（炼气·低难度）。玩家依次推进：

```
青云剑宗 → 血渊秘境 → 云澜城 → 丹霞谷 → [突破筑基]
→ 苍玄正道盟 → 苍玄古战场 → [境界跌落→碎星外环] → [重新筑基] → 碎星内海 → [突破金丹]
→ 万古殿 → 西域古林 → 西域边境 → [突破元婴]
→ 西域 → 慕兰草原 → 归墟通道 → [突破化神]
→ 归墟·浮云陆 → 归墟·万妖谷 → 归墟·古魔战场（通关）
```

- 每个境界突破后，该境界下的所有地图一次性解锁
- 同境界内的地图**不需要按顺序通关**——所有地图同时开放，玩家自由选择
- 苍玄古战场（筑基·中）是唯一序列锁定的地图——必须在此触发剧情才能解锁碎星群岛

#### 9. 地图间过渡

- 地图通关后自动返回**地图选择界面**（不强制进入下一地图）
- 玩家可从地图选择界面自由选择已解锁的任何地图进入
- 以下情况强制跳转到特定地图：
  - 苍玄古战场剧情触发后→自动解锁并导航到碎星外环
  - 渡劫突破后→弹出提示「新地图已解锁」，不强制跳转

#### 10. 事件节点与事件系统的协作

探索系统与事件系统的边界划分：

| 职责 | 所属系统 |
|------|---------|
| 地图上分配事件节点 | 探索系统 |
| 决策事件的具体事件模板 | 事件系统（加权随机选择） |
| 事件触发时弹出事件面板 | 事件系统（调用UI） |
| 事件结果结算 | 事件系统（通过GSM） |
| 连锁事件流程 | 事件系统 |
| 事件触发后回到探索地图 | 探索系统（回调） |

**事件触发流程（信号架构）：**
```
① 玩家移动到事件节点（探索系统处理移动）
② 探索系统识别 node_type == EVENT → 发射 signal: event_node_reached(map_pool, player_realm)
③ 事件系统监听此信号 → select_event(map_pool, player_realm) → 返回事件实例
④ 事件系统发射 signal: event_panel_requested(event_instance) → UI系统展示面板
⑤ 玩家选择选项 → 事件系统结算
⑥ 结算完成 → 事件系统发射 signal: event_resolved(result) 
⑦ 探索系统监听此信号 → 恢复节点交互状态（玩家可继续移动）
```

信号定义（Godot 4.6）：
| 信号名称 | 发射方 | 载荷 | 监听方 |
|---------|--------|------|--------|
| `event_node_reached` | 探索系统 | `(map_pool: String, player_realm: int)` | 事件系统 |
| `event_resolved` | 事件系统 | `(result: Dictionary)` | 探索系统 |

### 状态与转换

**地图生命周期状态：**

```
[地图选择界面]
    ↓ 选择地图 → 检查境界条件
[地图加载中]
    ↓ 地图生成完毕
[探索进行中] ←── 节点移动循环
    ├── 到达事件节点 → 事件触发 → 结算 → 回到探索
    ├── 到达战斗节点 → 加载战斗 → 战斗结束 → 回到探索
    ├── 到达商店节点 → 打开商店 → 离开 → 回到探索
    ├── 到达行动力泉 → 回复 → 回到探索
    ├── 到达渡劫台 → 触发渡劫 → 渡劫结束 → 回到探索
    └── 到达Boss节点 → 加载Boss战 → 战斗结束
        ├── Boss战胜利 → [地图通关] → 奖励结算 → 地图选择界面
        └── Boss战失败 → [战败结算] → 回到地图选择界面
    ↑ 行动力归零 → [探索结束] → 回到地图选择界面
```

**探索阶段状态定义：**

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **地图选择** | 玩家在已解锁地图列表中选择 | 进入探索阶段/上一地图通关 | 选择地图→地图加载中 |
| **地图加载中** | 生成节点图 | 选择地图后 | 生成完成→探索进行中 |

| **事件交互中** | 事件面板打开（阻塞移动） | 到达事件节点 | 事件结算完成 |
| **战斗中** | 战斗场景加载（阻塞移动） | 到达战斗/精英/Boss节点 | 战斗结束 |
| **商店中** | 商店界面打开（阻塞移动） | 到达商店节点 | 离开商店 |
| **地图通关** | Boss战胜利，结算奖励 | Boss战胜利 | 奖励结算完成→地图选择 |
| **探索结束** | 行动力归零或玩家选择离开 | 无可移动节点/主动离开 | 回到地图选择 |

## 公式

### 1. 地图解锁判定

```
is_map_unlocked(map, player_realm) → bool:
  return player_realm >= map.required_realm
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| player_realm | int | [1, 5] | 玩家当前境界层级 |
| map.required_realm | int | [1, 5] | 地图要求的境界层级 |

### 2. 地图有效境界（柔性压制公式）

```
map_effective_realm(player_L, map_max_L) → {offensive_L, defensive_L}:
  # 回旧地图时的柔性压制：进攻属性受限于地图，防御/战术属性保留玩家实际境界
  if player_L <= map_max_L:
    return {offensive_L: player_L, defensive_L: player_L}  # 无压制
  return {
    offensive_L: map_max_L,   # 费用上限、基础速度 → 压制到地图上限
    defensive_L: player_L,    # 上场人数、行动力上限 → 保留玩家实际境界
  }
```

回旧地图时：
- **进攻属性**（费用上限、基础速度）：按 `offensive_L`（= map_max_L）计算——高阶玩家不能秒杀低境界敌人
- **防御/战术属性**（上场人数、行动力上限）：按 `defensive_L`（= player_L）计算——玩家因成长获得更多战术选择，真实可感
- 同境界或越阶挑战时（player_L ≤ map_max_L）：所有属性按玩家实际境界计算

此柔性压制实现了「碾压的回味」幻想——玩家不能秒杀，但能用更多角色、更充裕的行动力从容应对。

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| player_L | int | [1, 5] | 玩家实际境界 |
| map_max_L | int | [1, 5] | 地图最高允许境界 |
| offensive_L | int | [1, 5] | 进攻属性的有效境界层级 |
| defensive_L | int | [1, 5] | 防御/战术属性的有效境界层级 |

### 3. 可移动判定

```
can_move_to(from_node, to_node, current_ap, visited_nodes) → bool:
  # 行动力泉节点：AP=0 时仍可移入（救命节点的特殊豁免）
  if to_node.is_ap_spring: return true
  # 传送节点：AP=0 时仍可移入
  if current_ap <= 0 and not (to_node.is_teleport or to_node.is_ap_spring):
    return false
  if to_node in visited_nodes: return false       # 不可回退
  if to_node not in from_node.connections: return false  # 必须在相邻层
  if to_node.is_locked: return false               # 条件锁定的节点
  return true
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| current_ap | int | [0, max_ap] | 当前行动力 |
| visited_nodes | Set | — | 已访问节点ID集合 |
| to_node.is_ap_spring | bool | — | 目标节点是否为行动力泉 |
| to_node.is_teleport | bool | — | 目标节点是否为传送节点 |

### 4. 节点类型权重分配

```
assign_node_type(layer_index, map_config, rng) → NodeType:
  weights = map_config.layer_weights[layer_index]  # {战斗:40, 事件:30, 商店:15, 回复:10, 精英:5}
  total = sum(weights.values())
  if total <= 0: return fallback_type             # 守卫：所有权重为零时回退
  roll = rng.randi_range(0, total - 1)
  cumulative = 0
  for type, weight in weights:
    cumulative += weight
    if roll < cumulative: return type
  return fallback_type
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| layer_index | int | [0, total_layers-1] | 当前层索引 |
| weights | Dict | — | 节点类型→权重映射，总和=100（含回复:10） |
| rng | RandomNumberGenerator | — | Godot 4.6 RNG实例 |
| fallback_type | NodeType | — | 权重分配失败时的默认类型（建议=战斗） |

权重配置（可通过调优参数调整）：
- 战斗: 40 | 事件: 30 | 商店: 15 | 回复: 10 | 精英: 5 = 100
- 精英/商店数量约束在权重分配后通过后处理强制执行（见§3 地图生成流程 步骤③d）

### 5. 地图通关奖励（基础）

```

|------|------|------|------|
| map.difficulty | String | 低/中/高/极高 | 地图难度等级 |
| is_first_clear | bool | — | 是否为本局首次通关该地图 |

### 6. 境界差额惩罚与完整奖励计算

```
realm_gap_penalty(player_L, map_max_L) → float:
  gap = player_L - map_max_L
  if gap <= 0: return 1.0
  return max(0.1, 1.0 - (gap × 0.3))  # 每多1级-30%，最低保留10%
```

差额惩罚应用于**通关奖励**和**战斗战利品**中的所有灵石来源：
- 通关奖励灵石 = `floor(base_ling_shi × realm_gap_penalty(player_L, map_max_L))`
- 战斗战利品灵石 = `floor(battle_loot_ling_shi × realm_gap_penalty(player_L, map_max_L))`
- 修为不受差额惩罚（保持基础值）

**完整通关奖励计算（组合公式5+6）：**
```
map_clear_reward_full(map, player_L, is_first_clear) → {ling_shi, cultivation, extra}:
  base = {见公式5}
  reward = base[map.difficulty]
  penalty = realm_gap_penalty(player_L, map.max_allowed_realm)
  reward.ling_shi = floor(reward.ling_shi × penalty)  # 灵石受差额惩罚
  # reward.cultivation 保持不变 —— 修为不受惩罚
  if is_first_clear:
    reward.extra = map.first_clear_reward
  return reward
```

**示例：** 金丹期（L=3）回青云剑宗（map_max_L=1）→ gap=2 → penalty=0.4 → 基础灵石×0.4
**示例：** 元婴期（L=4）回青云剑宗 → gap=3 → penalty=0.1 → 基础灵石×10%（保底）
**示例：** 金丹期（L=3）回丹霞谷（map_max_L=1）→ gap=2 → penalty=0.4 → 基础灵石×40%，战斗战利品灵石也×0.4

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| player_L | int | [1, 5] | 玩家实际境界 |
| map_max_L | int | [1, 5] | 地图最高允许境界 |
| gap | int | [0, 4] | 境界差距 |
| 返回值 | float | [0.1, 1.0] | 灵石惩罚系数 |

heal_at_rest_node(lost_hp_total) → int:

### 8. 节点可达路径数

```
reachable_boss_paths(graph, start_node, boss_node) → int:
  # 计算从入口到Boss节点的顶点不相交路径数
  # "独立路径" = 除起点和终点外不共享任何节点的路径（vertex-disjoint）

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| graph | DAG | — | 地图节点有向无环图 |
| start_node | Node | — | 入口节点（第0层唯一节点） |
| boss_node | Node | — | Boss节点（末层唯一节点） |

### 9. 地图生成节点间距

```
nodes_per_layer(map_config) → [int]:
  # 每层节点数在配置的[min, max]范围内随机
  # 确保总层数在4~6之间
  layers = rng.randi_range(4, 6) if not map_config.fixed_layers else map_config.fixed_layers
  result = [rng.randi_range(map_config.min_per_layer, map_config.max_per_layer) for _ in range(layers)]
  # 入口层和Boss层固定为1个节点
  result[0] = 1       # 第0层 = 入口
  result[layers-1] = 1  # 末层 = Boss
  return result
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| map_config.fixed_layers | int or null | 4~6 或 null | 固定层数（null=随机） |
| map_config.min_per_layer | int | 2~3 | 每层最少节点数（中间层） |
| map_config.max_per_layer | int | 3~4 | 每层最多节点数（中间层） |
| result[0] | int | 1 | 入口层固定1节点 |
| result[layers-1] | int | 1 | Boss层固定1节点 |

### 10. 地图重入传送费

```
map_reentry_cost(map_difficulty, entry_count) → int:
  base = {"低": 30, "中": 60, "高": 100, "极高": 150}
  # 第1次进入（首次进入）免费
  if entry_count <= 1: return 0
  multiplier = 1.0 + (entry_count - 2) × 0.5  # 第2次=1.0, 第3次=1.5, ...
  multiplier = min(multiplier, 3.0)            # 硬上限：最多3倍基价（防止无界增长）
  return floor(base[map_difficulty] × multiplier)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| map_difficulty | String | 低/中/高/极高 | 地图难度等级 |
| entry_count | int | [1, ∞) | 本局该地图总进入次数 |
| multiplier | float | [0.0, 3.0] | 费用倍率，有3.0硬上限 |
| 返回值 | int | [0, 450] | 本次进入所需灵石（极大值=150×3.0） |

**示例：** 高难度地图，第3次进入 → base=100, multiplier=1.5 → 150灵石
**示例：** 低难度地图，第2次进入 → base=30, multiplier=1.0 → 30灵石
**示例：** 中难度地图，第4次进入 → base=60, multiplier=2.0 → 120灵石
**示例：** 极高难度地图，第7次及以后 → base=150, multiplier=3.0(已达上限) → 450灵石

> **注意**：§1 概述中的重入费用伪代码已被此处的正式公式取代。如两个版本存在差异，以此公式为准。

### 11. 探索阶段结束的资源保留

```
exploration_end_resources(boss_defeated, battle_lost, collected_resources) → Resources:
  if boss_defeated:
    return collected_resources  # 全额保留
  if battle_lost:
    # 标准战败：保留50%本局探索中累积获得的修为（非仅本次战斗）
    # 灵石和卡牌全额保留（已获得的不扣除）
    retained = {
      ling_shi: collected_resources.ling_shi,
      cultivation: floor(collected_resources.cultivation × 0.5),
      cards: collected_resources.cards,  # 已获得的卡牌全额保留
      items: collected_resources.items,  # 丹药/材料等全额保留
    }
    return retained
  # 行动力归零结束 / 主动退出
  return collected_resources  # 全额保留已收集的资源
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| boss_defeated | bool | — | Boss是否被击败 |
| battle_lost | bool | — | 是否在战斗中失败 |
| collected_resources | Resources | — | 本局探索中累积收集的所有资源 |
| collected_resources.cultivation | int | [0, ∞) | 本局探索累计获得的修为 |
| collected_resources.ling_shi | int | [0, ∞) | 本局探索累计获得的灵石 |
| collected_resources.cards | Array | — | 本局探索中获得的所有卡牌 |
| collected_resources.items | Array | — | 本局探索中获得的所有物品（丹药/材料等） |

> **注意**：行动力归零结束与主动退出探索均全额保留资源——两者结算相同，区别仅在于触发方式（系统强制 vs 玩家主动）。主动退出按钮在探索HUD中始终可用。

## 边界情况

- **所有可用路径都被战斗/事件节点覆盖**：标准情况——每条路径的体验不同，玩家根据偏好选择路线
- **某层只有一个可达节点**：允许——如果该层只有1个节点，所有上一层的连接都指向它（汇聚点），该节点成为必经之路
- **玩家到达节点时行动力正好用完**：该节点仍可交互（进入事件/战斗/商店），交互本身不消耗行动力；交互结束后不能再移动
- **商店节点中灵石不足购买**：高亮价格 + 玩家灵石数，灵石不足的选项灰色显示
- **精英战斗失败后重新进入地图**：地图重新生成新节点图，之前的战斗结果不继承（除角色阵亡标记不可用外）
- **回旧地图时境界被压制到炼气，且灵石获取大幅减少**：高阶修士回低阶地图收益递减——金丹期回青云剑宗灵石收益只剩40%（gap=2），但卡牌仍按当前pool_tier的下一档计算
- **Boss战前最后一个节点是事件/商店**：标准情况——玩家可在Boss前做最后一次准备（购买丹药、触发回复事件等）
- **全部角色不可用时进入商店**：允许——玩家仍可进入商店购买角色卡或复活丹药；如果买了角色卡，回到地图后有可用角色即可继续
- **地图首次通关奖励是一张卡牌但卡组已满**：奖励直接发放（超限触发标准弃牌流程）
- **渡劫台节点出现在非满修为玩家面前**：可进入但不可触发渡劫，显示「修为尚未圆满」
- **苍玄古战场受伤跌落后回到地图选择**：自动解锁碎星外环并提示；玩家可在此选择进入碎星外环或回到已通关的炼气地图刷资源
- **灵石不足无法进入已通关地图**：在地图选择界面点击已通关地图时，费用 > 当前灵石余额 → 弹出「灵石不足」提示，按钮灰色不可选；玩家可选择其他未通关（免费）地图或先去其他地图赚灵石
- **同一地图多次重入费用递增**：设计意图——防止玩家反复刷同一地图快速积累资源，鼓励探索不同地图；第2次进入费用合理（低成本回访），第5+次费用较高（应已足够完成该地图的目标）
- **重入地图后行动力重置**：标准规则——每次进入地图行动力重置为当前境界上限，不受重入费用影响
- **苍玄古战场通关后不可重入**：剧情地图，通关触发境界跌落，后续流程不再需要此地图；始终显示为「已通关·不可重入」
- **灵石为0时无法进入任何已通关地图**：玩家只能选择未通关地图（免费）或先在其他地图收集灵石后再回访
- **玩家在探索中打开菜单保存游戏**：存档保存完整地图状态（DAG节点图结构 + 节点类型 + 连接关系 + 已访问节点 + 当前节点位置 + 剩余行动力）。读档后恢复到离开时的地图状态——玩家期望在中途读档后从原位置继续。事件内容在到达时确定（见§3），故不会鼓励SL
- **地图生成后玩家退出游戏（未开始探索）**：如果地图已生成但玩家尚未进入（尚在「地图选择界面」），退出后节点图不保留——下次进入同一地图时重新生成
- **玩家主动退出探索阶段（未通关）**：探索HUD中始终显示「退出探索」按钮。主动退出 = 保留已获得资源（全额），不获得通关奖励和首次通关奖励，地图标记为「未通关」。结算与行动力归零结束相同——区别仅在于触发方式（玩家主动 vs 系统强制）
- **所有地图均已通关但境界不足突破**：玩家可回永久免费地图刷修为/资源（每境界1张永久免费地图确保始终有可进入的地图），也可消耗灵石进入非免费地图。这属于P2「苟道成长」的设计范畴——鼓励稳健发育
- **同一地图多次进入的节点图差异**：同一地图每次进入随机生成的节点布局不同（层数可能变化、节点类型位置变化），但地图难度和事件池不变
- **战斗节点触发特殊剧情战斗**：某些战斗节点被标记为剧情战斗（如苍玄古战场·正魔大战），击败后触发连锁剧情——探索系统不做特殊处理，由事件系统/剧情系统接管后续流程
- **传送节点连通到未解锁层**：传送节点的目标层必须已解锁（如本局尚未到达过），否则传送节点不可激活
- **到达Boss节点但行动力为0**：允许进入Boss战（Boss战不消耗行动力）——玩家不会被卡在Boss门前

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 地图总层数 | 4~6层 | 3~8层 | 地图太短、探索感不足 | 地图太长、行动力不够 |
| 每层节点数 | 2~4个 | 2~5个 | 路径单一无选择余地 | 选择过多决策疲劳 |
| 节点池权重（战斗/事件/商店/回复/精英） | 40/30/15/10/5 | 可调 | 某类节点过少或过多 | 同左 |
| 回复点回复量 | 50%已损HP | 30%~70% | 回复效果不明显，玩家不会为其规划路线 | 回复过多削弱事件/商店价值 |
| 精英战出现率 | 每图1~2个 | 0~3个 | 缺乏挑战节点 | 精英太难影响地图节奏 |
| 商店每图数量 | 1~2个 | 1~3个 | 消耗灵石机会少 | 商店过多削弱战斗奖励 |
| 首次通关额外奖励卡牌 | 1张（蓝色） | 白~紫 | 首次通关缺乏吸引力 | 奖励过高降低重玩需求 |
| 地图通关灵石（低→极高） | 50/100/200/300 | 25~500 | 经济来源不足 | 经济溢出 |
| 地图通关修为（低→极高） | 50/80/120/150 | 25~300 | 修为获取太少卡关 | 修为获取太快跳过内容 |
| 境界差额惩罚（每级） | -30%（最低10%） | -20%~-50% | 回旧地图收益太高成最优解 | 回旧地图完全没收益 |
| 重入传送费基价（低/中/高/极高） | 30/60/100/150 | 20~200 | 重入门槛低，可反复刷同一地图 | 重入太贵，失去回旧地图意义 |
| 重入传送费递增 | +50%/次，上限3.0×基价 | +25%~+100%，上限可调 | 多次重入无约束 | 几次后费用高到无法重入 |
| 重入传送费硬上限 | 3.0×基价 | 2.0~5.0× | 低上限失去递增惩罚意义 | 高上限导致无界费用增长 |
| 节点图生成保底独立路径 | 2条 | 1~3条 | 路径单一无选择 | 路径太多开发成本高 |
| 战斗节点中精英占比 | 10%~20% | 5%~30% | 精英太少地图缺乏高低落差 | 精英太多普通战无存在感 |
| 节点可见层数 | 当前+下方2层 | 1~3层 | 路径无法规划（1层=盲走） | 迷雾失去探索感（3+=全透） |

## 验收标准

### 地图选择与解锁
- **GIVEN** 玩家炼气期，**WHEN** 进入探索阶段，**THEN** 地图选择界面显示青云剑宗为唯一可选地图
- **GIVEN** 玩家筑基期，**WHEN** 进入探索阶段，**THEN** 地图选择界面显示炼气期全部地图 + 筑基期地图
- **GIVEN** 玩家筑基期且有3张筑基期地图已解锁，**WHEN** 进入地图选择界面，**THEN** 3张筑基期地图全部显示为可选状态（无顺序锁定标记）
- **GIVEN** 地图从未进入过(entry_count=0)，**WHEN** 选择该地图，**THEN** 不弹出灵石确认窗口，直接进入地图
- **GIVEN** 突破到新境界，**WHEN** 回到地图选择界面，**THEN** 新境界地图全部解锁，弹出「新地图已解锁」提示，但不强制跳转至特定地图

### 节点图生成
- **GIVEN** 玩家选择地图进入，**WHEN** 地图加载完成，**THEN** 节点图主界面显示入口节点（高亮）+ Boss节点 + 连接线；入口节点固定在第0层，Boss节点固定在末层
- **GIVEN** 节点图生成，**WHEN** 从入口节点执行图遍历到Boss节点，**THEN** 存在至少2条节点不重叠的独立路径（vertex-disjoint）可达Boss
- **GIVEN** 同一地图已进入过至少1次，**WHEN** 再次进入同一地图，**THEN** 节点类型位置或连接关系与上一次不同（非完全相同的节点图）
- **GIVEN** 节点图生成完成，**WHEN** 玩家到达事件节点，**THEN** 事件内容在到达时才确定（非地图生成时固定）
- **GIVEN** 节点图生成完成，**WHEN** 玩家首次到达任意商店节点，**THEN** 商店库存已在生成时确定（不因到达顺序而改变库存内容）
- **GIVEN** 节点图生成完成，**WHEN** 玩家到达战斗节点，**THEN** 敌人阵容与节点生成时确定的阵容一致

### 节点导航与可见性
- **GIVEN** 玩家在当前节点，**WHEN** 点击下一层的可达节点，**THEN** 移动成功，行动力-1
- **GIVEN** 玩家在当前节点，**WHEN** 点击同一层其他节点，**THEN** 不可移动
- **GIVEN** 玩家已访问过某节点，**WHEN** 点击回退，**THEN** 不可移动
- **GIVEN** 玩家在入口节点(第0层)，**WHEN** 检查第1层和第2层节点，**THEN** 与入口有边连接的第1、2层节点显示其类型图标和名称
- **GIVEN** 玩家在入口节点(第0层)，**WHEN** 检查第3层及更远层，**THEN** 所有第3+层节点显示为迷雾节点（类型图标和名称均隐藏）
- **GIVEN** 玩家到达第1层节点，**WHEN** 检查第2层和第3层，**THEN** 第2、3层节点类型和名称变为可见
- **GIVEN** 玩家已从入口移动到第1层的战斗节点，**WHEN** 检查入口节点和第1层战斗节点，**THEN** 两节点均显示绿色标记/边框
- **GIVEN** 当前节点有不可达的相邻层节点，**WHEN** 查看节点，**THEN** 不可达节点显示为灰色（路径未连接）
- **GIVEN** 行动力=0/7且当前在第2层，**WHEN** 悬停在第3层可达节点上，**THEN** 节点变灰且显示tooltip「行动力不足」

### 行动力与移动
- **GIVEN** 行动力=1/7，**WHEN** 移动到下一节点，**THEN** 行动力=0/7
- **GIVEN** 行动力=0/7且当前在非Boss非传送非行动力泉节点，**WHEN** 点击下一层节点，**THEN** 移动被拒绝
- **GIVEN** 行动力=0/7且在行动力泉的相邻节点，**WHEN** 点击行动力泉，**THEN** 允许进入（0 AP豁免）
- **GIVEN** 行动力=0/7且在Boss节点，**WHEN** 点击Boss，**THEN** 允许进入Boss战

### 节点交互
- **GIVEN** 玩家到达事件节点，**WHEN** 进入，**THEN** 事件面板弹出（无确认弹窗）
- **GIVEN** 玩家到达商店节点，**WHEN** 进入，**THEN** 商店界面打开（无确认弹窗）
- **GIVEN** 玩家到达行动力泉节点，**WHEN** 进入，**THEN** 行动力回满且不消耗AP
- **GIVEN** 玩家到达回复点节点，**WHEN** 进入，**THEN** 队伍回复已损失HP的50%
- **GIVEN** 玩家到达Boss节点，**WHEN** 点击进入，**THEN** 弹出确认弹窗显示Boss名称和能力描述 + 进入/再准备一下按钮
- **GIVEN** 渡劫台节点且修为未满，**WHEN** 进入，**THEN** 显示「修为尚未圆满」提示，节点不消耗

### 通关与奖励
- **GIVEN** 玩家击败低难度地图Boss，**WHEN** 结算，**THEN** 获得50灵石 + 50修为 + 地图标记为「已通关」
- **GIVEN** 玩家击败中难度地图Boss，**WHEN** 结算，**THEN** 获得100灵石 + 80修为 + 地图标记为「已通关」
- **GIVEN** 玩家击败高难度地图Boss，**WHEN** 结算，**THEN** 获得200灵石 + 120修为 + 地图标记为「已通关」
- **GIVEN** 玩家击败极高难度地图Boss，**WHEN** 结算，**THEN** 获得300灵石 + 150修为 + 地图标记为「已通关」
- **GIVEN** 地图首次通关，**WHEN** 结算，**THEN** 奖励列表包含map.first_clear_reward字段（卡牌ID/丹药ID/天赋ID之一）且首通奖励高亮显示
- **GIVEN** Boss战失败，**WHEN** 结算，**THEN** 回到地图选择界面，地图未通关

### 回旧地图
- **GIVEN** 金丹期(L=3)回到青云剑宗(map_max_L=1)，**WHEN** 进入战斗，**THEN** 费用上限和基础速度按炼气期(L=1)计算，上场人数和行动力上限按金丹期(L=3)计算
- **GIVEN** 金丹期(L=3)回到青云剑宗，**WHEN** 通关Boss，**THEN** 灵石奖励 = floor(50 × 0.4) = 20灵石（gap=2→40%）
- **GIVEN** 金丹期(L=3)回到青云剑宗，**WHEN** 战斗中获得灵石战利品，**THEN** 战利品灵石也受realm_gap_penalty=0.4影响
- **GIVEN** 金丹期(L=3, card_pool_tier=3)回青云剑宗(炼气地图)，**WHEN** 战斗节点掉落卡牌，**THEN** 卡牌池按card_pool_tier=4的配置抽取
- **GIVEN** 青云剑宗已通关，**WHEN** 再次进入该地图，**THEN** 行动力重置为当前境界上限且节点图布局与上一次进入不同

### 重入费用
- **GIVEN** 地图首次通关后回到地图选择界面，**WHEN** 点击已通关地图，**THEN** 弹出确认窗口显示重入费用（低难度第2次=30灵石）
- **GIVEN** 低难度地图第2次重入，**WHEN** 计算费用，**THEN** 需30灵石（30×1.0）
- **GIVEN** 高难度地图第3次重入，**WHEN** 计算费用，**THEN** 需150灵石（100×1.5）
- **GIVEN** 低难度地图第4次重入，**WHEN** 计算费用，**THEN** 需30 × 2.0 = 60灵石
- **GIVEN** 极高难度地图第7次重入，**WHEN** 计算费用，**THEN** 需150 × 3.0 = 450灵石（已达硬上限）
- **GIVEN** 永久免费地图（青云剑宗/碎星外环/西域古林/慕兰草原/归墟·浮云陆），**WHEN** 第N次重入，**THEN** 费用始终为0灵石

- **GIVEN** 灵石=50且重入费用=30，**WHEN** 点击确认重入，**THEN** 扣除30灵石，进入地图
- **GIVEN** 灵石重入消耗后进入地图，**WHEN** 加载完成，**THEN** 行动力重置为当前境界上限

### 探索结束与存储
- **GIVEN** 行动力归零且Boss未到达，**WHEN** 结算，**THEN** 探索提前终结→展示已获得资源→返回地图选择界面（全额保留资源，无通关奖励）
- **GIVEN** 玩家在战斗中选择撤退，**WHEN** 结算，**THEN** 保留100%已获得的灵石+卡牌+物品，保留50%本局累计修为，回到地图选择界面
- **GIVEN** 玩家在探索中（位于第2层，AP=3/7），**WHEN** 保存游戏→退出→读档，**THEN** 恢复到第2层同一节点位置，AP=3/7，已访问节点列表完整
- **GIVEN** 玩家在探索中点击「退出探索」按钮，**WHEN** 确认退出，**THEN** 保留已获资源，地图保持「未通关」，回到地图选择界面

### 边缘情况
- **GIVEN** 所有角色阵亡不可用，**WHEN** 玩家到达商店节点，**THEN** 商店界面正常打开，玩家可购买角色卡或复活丹药
- **GIVEN** 地图首次通关奖励是一张卡牌但卡组已满，**WHEN** 发放奖励，**THEN** 奖励直接发放（超限触发标准弃牌流程）
- **GIVEN** 所有已解锁地图均已通关且修为未满突破条件，**WHEN** 检查地图选择界面，**THEN** 玩家仍可选择永久免费地图（0灵石）或消耗灵石选择其他已通关地图
- **GIVEN** 传送节点的目标层在当前局尚未被玩家到达过，**WHEN** 检查传送节点，**THEN** 传送节点显示为不可激活状态（灰色）
- **GIVEN** 玩家主动退出探索阶段（HUD按钮），**WHEN** 确认退出，**THEN** 保留已获资源，地图保持「未通关」
