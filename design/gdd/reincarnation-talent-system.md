## 概述

轮回天赋系统是游戏的 Roguelike 元成长层——玩家每局通关或死亡后，根据本局表现获得「轮回点」，用于在天赋树中解锁永久性被动加成。这些天赋跨局保留，让每一次失败都不是终点而是积累，每一次通关都开启新的可能性。

对玩家来说，轮回天赋系统解决了 Roguelike 最核心的情感矛盾——「我打了 2 小时的局死了，什么都没留下」。天道轮回，修为不灭——死后的积累让下一次开局更强，解锁的新身份、新卡牌、新机制让重复游玩始终有新鲜感。这正是「仙途问道」的主题映射：修仙本就是一世又一世的积累，轮回是天道法则而非游戏惩罚。

## 玩家幻想

| 情感 | 触发场景 |
|------|---------|
| **失败有价值** | 死亡结算时看到轮回点+12——「至少解锁了一个天赋，下局更强」 |
| **规划的期待** | 在天赋树界面斟酌哪个分支先解锁——「先点满修为线，还是先开阵道双杰？」 |
| **积累的骄傲** | 看到天赋树点亮了一大片——「我已经轮回 30 次了」 |
| **新身份的惊喜** | 攒够轮回点解锁「阵道双杰」——「终于能玩阵法流了！」 |
| **策略深度** | 根据已解锁天赋调整开局策略——「这局我有灵石加成+修为加速，走百艺流」 |
| **天道轮回的沉浸感** | 死亡不叫 Game Over，叫「轮回转世」——贴合修仙世界观 |

轮回天赋的核心情感不是「变强了所以简单了」，而是**「每一世都在为下一世铺路」**——这是修仙者的时间观，也是 Roguelike 玩家最珍视的积累感。

## 详细设计

### 核心规则

#### 1. 轮回触发时机

轮回结算在以下两种情况下触发：

| 触发条件 | 结算内容 | 说明 |
|---------|---------|------|
| **角色死亡**（战斗中全员阵亡且无法复活） | 轮回点结算 + 解锁检测 | 游戏结束→进入轮回结算界面 |
| **通关**（击败化神期最终 Boss） | 轮回点结算 + 解锁检测 + 通关标记 | 通关后的轮回结算奖励更丰厚 |

轮回结算流程：
```
① 展示本局结算面板：
   - 最高境界 reached
   - 击杀敌人数
   - 收集卡牌种类数
   - 炼丹/炼器次数
   - 解锁的新卡牌
② 计算轮回点获得量
③ 检查是否解锁新天赋节点（满足解锁条件）
④ 更新 progression.dat
⑤ 进入天赋树界面（玩家可消费轮回点）
⑥ 玩家确认后返回主菜单
```

#### 2. 轮回点获取公式

```
reincarnation_points(run_result) → int:
  base = 0
  
  # 境界奖励（主要来源）
  base += (realm_reached - 1) × 5   # 炼气=0, 筑基=5, 金丹=10, 元婴=15, 化神=20
  
  # 通关奖励
  if run_result == "victory":
    base += 10   # 通关额外奖励
  
  # 击杀奖励
  base += min(elites_killed, 10) × 1    # 每击杀1个精英+1（上限10）
  base += min(bosses_killed, 5) × 2     # 每击杀1个Boss+2（上限5）
  
  # 收集奖励
  unique_cards = count_unique_cards_collected_this_run()
  base += floor(unique_cards / 10) × 1  # 每收集10种不同卡牌+1（上限5）
  
  # 炼丹/炼器奖励
  craft_count = total_alchemy + total_forge
  base += floor(craft_count / 5) × 1    # 每5次炼制+1（上限5）
  
  return base
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|:----:|------|
| realm_reached | int | [1, 5] | 本局达到的最高境界 |
| elites_killed | int | [0, ∞) | 本局击杀精英敌人数 |
| bosses_killed | int | [0, ∞) | 本局击杀Boss数 |
| unique_cards | int | [0, 222] | 本局收集的不同卡牌种类数 |
| craft_count | int | [0, ∞) | 本局炼丹+炼器总次数 |

**典型轮回点收益：**
| 本局表现 | 境界 | 轮回点 |
|---------|:----:|:-----:|
| 新手第一局，炼气期死亡 | 炼气 | 2~5 |
| 筑基期中段死亡 | 筑基 | 8~15 |
| 金丹期Boss战死 | 金丹 | 15~25 |
| 元婴期通关 | 元婴 | 25~40 |
| 化神期完美通关 | 化神 | 35~50 |

#### 3. 天赋树结构

天赋树分为 **5 条分支**，每条 4 层，共 20 个天赋节点。每条分支有独立的解锁条件和逐层递进的消耗：

```
天赋树层级结构（按分支）：

【修为之道】——— 加速修为积累
  L1: 勤修苦练    cost=3  修为获取+10%
  L2: 厚积薄发    cost=5  突破所需修为-10%（修为上限×0.9）
  L3: 道心通明    cost=8  溢出修为转化率+50%（150修为=1属性丹）
  L4: 天道酬勤    cost=12 每局开始赠送培元丹×1（+50修为）

【资源之道】——— 加速资源获取
  L1: 灵脉感应    cost=3  探索灵石掉落+10%
  L2: 坊市老客    cost=5  商店卡牌价格-15%
  L3: 炼器余料    cost=8  炼丹/炼器灵材消耗-1（最少1）
  L4: 苍玄行者    cost=15 解锁隐藏身份「阵道双杰」

【战意之道】——— 战斗属性加成
  L1: 锋芒初显    cost=3  开局首回合费用+1
  L2: 血战不退    cost=5  角色最大生命+2
  L3: 会心一击    cost=8  暴击率+5%
  L4: 破釜沉舟    cost=12 己方角色低于50%HP时攻击+2

【卡牌之道】——— 卡组构筑优势
  L1: 博闻强记    cost=3  开局额外多抽1张牌（首回合抽牌+1）
  L2: 精挑细选    cost=5  战利品三选一变为四选一
  L3: 暗金天命    cost=8  暗金卡上限+1（突破境界限制）
  L4: 万法归宗    cost=12 卡组上限+5（45张→受此天赋影响）

【轮回之道】——— 特殊机制解锁
  L1: 轮回印记    cost=5  每局开始随机获得1张蓝卡
  L2: 因果不灭    cost=8  死亡后可保留1张卡牌带入下一局（从本局卡组中选择1张）
  L3: 万界穿梭    cost=12 开局可从2个随机身份中选择（而非固定选择）
  L4: 超脱轮回    cost=20 开局境界从炼气中期开始（修为从200/1000开始）+额外轮回点获取+20%
```

**分支解锁规则：**
- 每条分支的 L1 天赋从开局就可见并可解锁
- L2 需要 L1 已解锁（同分支前置）
- L3 需要 L2 已解锁 + 玩家至少通关 1 次（或死亡 5 次以上）——「道心考验」
- L4 需要 L3 已解锁 + 最高境界达到金丹以上——「大道之基」

**天赋效果叠加规则：**
- 同一分支内的天赋效果可叠加（L1+L2+L3+L4 全部生效）
- 不同分支间的天赋效果可叠加（修为之道 + 资源之道同时生效）
- 数值叠加为加法（如修为获取+10%+卡牌效果=独立修正叠加）

#### 4. 轮回点存储与消费

轮回点存储在 `progression.reincarnation_points` 中（持久化到 `progression.dat`）：

```
# progression.dat 中的轮回天赋数据
{
  "reincarnation_points": 25,          # 当前可用轮回点
  "total_points_earned": 120,          # 累计获得（用于统计）
  "unlocked_talents": [                # 已解锁的天赋节点ID列表
    "cultivation_1", "cultivation_2",
    "resource_1", "combat_1", "combat_2",
    "card_1"
  ],
  "talent_tree_progress": {            # 每条分支当前解锁到第几层
    "cultivation": 2,
    "resource": 1,
    "combat": 2,
    "card": 1,
    "reincarnation": 0
  },
  "total_reincarnations": 8,           # 累计轮回次数
  "highest_realm_ever": "金丹",         # 历史最高境界
  "victories": 0                       # 通关次数
}
```

**消费规则：**
- 轮回点在天赋树界面消费，不可退款（选择即锁定）
- 已解锁的天赋永久生效，不可关闭
- 轮回点不足时对应天赋节点灰色显示
- 轮回点无上限，可以攒够一次性解锁多条分支

#### 5. 天赋生效时机

天赋在每局开始时自动应用：

```
apply_all_talents():
  for talent_id in progression.unlocked_talents:
    talent_def = TALENT_TREE[talent_id]
    match talent_def.effect_type:
      "cultivation_boost":    GSM.player.cultivation_modifier += talent_def.value
      "resource_boost":       GSM.player.resource_modifier += talent_def.value
      "combat_stat":          GSM.player.combat_modifiers[talent_def.stat] += talent_def.value
      "card_advantage":       GSM.player.deck_modifiers.append(talent_def.effect)
      "special_unlock":       GSM.player.special_unlocks.append(talent_def.effect)
      "identity_unlock":      GSM.progression.available_identities.append(talent_def.identity_id)
  
  # 特殊天赋：暗金天命（突破暗金卡上限）
  if "darkgold_limit_plus" in unlocked:
    card_system.darkgold_limit_modifier += 1
  
  # 特殊天赋：超脱轮回（开局修为提升）
  if "headstart" in unlocked:
    GSM.player.cultivation = 200  # 而非 0
```

天赋在**开局身份选择之前**生效——这意味着身份选择时可以看到天赋带来的额外选项（如万界穿梭提供2个随机身份）。

#### 6. 天赋解锁前置条件（软解锁）

除了轮回点消耗外，部分天赋需要满足游戏内条件才能出现在天赋树中（可见但不可解锁）：

| 天赋 | 可见条件 | 解锁条件 |
|------|---------|---------|
| 天道酬勤 (修为L4) | 修为L3已解锁 | 历史最高境界≥金丹 |
| 苍玄行者 (资源L4) | 资源L3已解锁 | 历史累计收集≥50种不同卡牌 |
| 破釜沉舟 (战意L4) | 战意L3已解锁 | 累计击杀≥30个精英敌人 |
| 万法归宗 (卡牌L4) | 卡牌L3已解锁 | 本局曾达到卡组40张上限 |
| 超脱轮回 (轮回L4) | 轮回L3已解锁 | 至少通关1次 |

未满足解锁条件的天赋节点显示为灰色+条件描述（如「需通关1次」）。

#### 7. 天赋与身份专属天赋的关系

- 身份专属天赋（如青云剑宗的+15%灵石掉落）与轮回天赋的效果**独立叠加**
- 相同类型效果以**加法**叠加：身份灵石+15% + 灵脉感应+10% = +25%灵石掉落
- 轮回天赋不影响身份选择的初始卡组内容——天赋在身份选择之前生效，但初始卡组由身份定义

#### 8. 轮回结算中的新卡牌解锁

除了天赋节点，轮回结算还检测「新卡牌解锁」——某些卡牌需要特定成就才能出现在全局卡池中：

```
check_card_unlocks(run_result):
  unlocks = []
  
  # 境界里程碑解锁
  if run_result.highest_realm >= 金丹 and "card_golden_core" not in unlocked:
    unlocks.append("card_golden_core")  # 解锁金丹期限定金卡
  
  # 特殊成就解锁
  if run_result.bosses_killed >= 10 and "card_boss_slayer" not in unlocked:
    unlocks.append("card_boss_slayer")
  
  # 身份通关解锁
  if run_result == "victory" and run_result.identity == "丹霞谷弟子":
    unlocks.append("card_alchemy_master")  # 解锁万象真人角色卡
  
  return unlocks
```

> **注**：新卡牌解锁是轻量机制——不是大量卡牌需要解锁，而是 5~10 张特殊卡牌作为「轮回惊喜」分布在不同成就条件中。绝大多数卡牌（200+/222）通过正常游戏获取即可。

### 状态与转换

```
天赋生命周期：

[初始状态] —— progression 为空或不存在
    ↓ 第1局游戏结束
[轮回结算] → 获得轮回点
    ↓ 在天赋树中消费
[天赋解锁] → 写入 progression.dat
    ↓ 下一局开始
[天赋生效] → 自动应用所有已解锁天赋
    ↓ 本局游戏...
[游戏结束] → 再次轮回结算
    ↓
循环 — 每次轮回积累更多天赋

同一节点状态：
  未可见 → 满足可见条件 → 可见·未解锁 → 消费轮回点 → 已解锁·生效中
```

**天赋不可重置**——解锁即永久。这是设计上的意图：轮回天赋代表「天道的积累」，不可逆转。玩家在解锁前需要认真考虑优先级。

## 公式

### 1. 轮回点结算

```
calculate_reincarnation_points(run_data) → int:
  points = 0
  
  # 境界
  points += max(0, (run_data.realm_reached - 1)) × 5
  
  # 通关
  if run_data.result == "victory":
    points += 10
  
  # 击杀
  points += min(run_data.elites_killed, 10) × 1
  points += min(run_data.bosses_killed, 5) × 2
  
  # 收集
  unique_cards = run_data.unique_cards_collected
  points += min(floor(unique_cards / 10), 5) × 1
  
  # 炼制
  craft_total = run_data.alchemy_count + run_data.forge_count
  points += min(floor(craft_total / 5), 5) × 1
  
  # 超脱轮回加成
  if "reincarnation_4" in unlocked_talents:
    points = floor(points × 1.2)
  
  return points
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|:----:|------|
| realm_reached | int | [1, 5] | 本局达到的最高境界 |
| result | enum | victory/death | 本局结束方式 |
| elites_killed | int | [0, ∞) | 击杀精英数 |
| bosses_killed | int | [0, ∞) | 击杀Boss数 |
| unique_cards_collected | int | [0, ∞) | 收集的不同卡牌种类 |
| alchemy_count | int | [0, ∞) | 炼丹操作次数 |
| forge_count | int | [0, ∞) | 炼器操作次数 |

### 2. 天赋效果叠加

```
apply_talent_modifier(base_value, modifier_list) → float:
  total_mod = 1.0
  for mod in modifier_list:
    total_mod += mod / 100.0  # 百分比转换为小数
  return base_value × total_mod
```

**示例：** 灵石掉落基准100，身份+15%，灵脉感应+10% → 100 × (1 + 0.15 + 0.10) = 125

### 3. 天赋树解锁验证

```
can_unlock(talent_id, progression) → {allowed: bool, reason: String}:
  talent = TALENT_TREE[talent_id]
  
  # 轮回点充足
  if progression.reincarnation_points < talent.cost:
    return {false, "轮回点不足（需要{talent.cost}，当前{progression.reincarnation_points}）"}
  
  # 同分支前置
  if talent.prerequisite and talent.prerequisite not in progression.unlocked_talents:
    return {false, "需要先解锁前置天赋"}
  
  # 游戏内条件
  if talent.unlock_condition:
    if not talent.unlock_condition(progression):
      return {false, talent.unlock_description}
  
  return {true, ""}
```

## 边界情况

- **首次游戏无轮回天赋**：第一次游玩时 progression.dat 不存在，所有天赋未解锁。轮回系统使用默认值（无加成），这是预期的新手体验——「从凡人开始」
- **轮回点攒着不用**：允许——轮回点无过期时间，玩家可以攒够一次性解锁多条分支。天赋树界面显示「可用轮回点：N」
- **已解锁全部天赋（20/20）**：玩家成为「超脱轮回者」——轮回点仍然累计但无新天赋可解锁。未来版本可扩展新分支
- **progression.dat 损坏或丢失**：重置为初始值——所有天赋解锁和轮回点丢失（损失已解锁的天赋）。这是一个风险点——需要在存档系统中通过完整性校验缓解
- **死亡结算 vs 通关结算**：两种结算都获取轮回点但通关有+10额外奖励。死亡同样可解锁天赋（不需要通关）
- **阵道双杰解锁后，新游戏仍选择其他身份**：允许——苍玄行者解锁的是身份的「可选性」，不是强制使用。玩家可以在身份选择界面自由选择
- **因果不灭（保留1张卡牌到下一局）的选卡时机**：在死亡结算界面，展示本局卡组中所有卡牌，玩家选择1张保留。新开局时该卡自动出现在初始卡组中（不计入身份初始卡牌数量）
- **因果不灭留的卡与身份初始卡组冲突**：不冲突——身份初始卡组照常发放，保留卡额外加入。保留卡受卡组上限约束（一局开始时的初始卡组通常很小，不会触发上限）
- **精挑细选（四选一）与三选一UI**：战利品面板扩展为4个选项，其他规则不变（达上限后卡牌不可选等）
- **暗金天命（暗金卡上限+1）**：金丹期暗金上限原为1→变为2；元婴期2→变为3。此效果对炼气/筑基无效（原有上限=0，+1=1，但炼气/筑基的 card_pool_tier 本身就不出暗金——所以此天赋实际在金丹后生效）
- **超脱轮回（开局200修为）**：开局修为从200/1000开始（而非0/1000），玩家接近突破但未突破。配合勤修苦练（+10%修为获取），开局积累速度明显更快
- **万界穿梭（2个随机身份选择）**：开局随机从6个身份（含已解锁的隐藏身份）中抽2个，玩家选1个。这2个身份不会重复
- **天赋效果在战斗中动态计算**：破釜沉舟（低于50%HP+2攻击）在每回合开始时检查HP百分比，动态开关。其他天赋在局开始时固定应用
- **多条天赋数值叠加的极端情况**：灵石掉落理论上限25%（身份15%+灵脉感应10%），修为获取上限10%（勤修苦练），均为温和加成，不会导致数值失控
- **声望/成就系统（未来）与轮回天赋的交互**：成就系统设计时，可将「解锁N个轮回天赋」作为成就条件。当前代码中 achievement 表与 unlocked_talents 独立

## 依赖关系

| 依赖系统 | 性质 | 说明 |
|----------|:----:|------|
| **存档系统** | 硬依赖 | progression.dat 的读写——天赋数据、轮回点、解锁状态的持久化 |
| **游戏状态管理器** | 硬依赖 | 天赋效果在 player 域的注册和读取；本局运行数据的统计和提供 |
| **开局身份选择系统** | 软依赖 | 阵道双杰（隐藏身份）的解锁标记由本系统提供；万界穿梭影响身份选择流程 |
| **修为养成系统** | 软依赖 | 修为获取加成、溢出转化率加成 |
| **资源系统** | 软依赖 | 灵石掉落加成、灵材消耗减免 |
| **卡牌系统** | 软依赖 | 暗金卡上限修正、卡组上限修正、新卡牌解锁 |
| **战斗系统** | 软依赖 | 首回合费用加成、HP加成、暴击加成、低血量攻击加成 |
| **卡组编辑系统** | 软依赖 | 战利品选项数修正（精挑细选） |

### 上游依赖设计顺序

存档系统（已设计）→ 游戏状态管理器（已设计）→ 身份选择（已设计）→ **轮回天赋系统**

轮回天赋系统处于「元」层——它不直接参与局内玩法循环，而是在局与局之间的结算界面运行。所有软依赖系统在设计时不需要等待轮回天赋完成，只需预留天赋效果查询接口即可。

## 验收标准

- **GIVEN** 玩家第1次游戏（无 progression.dat），**WHEN** 检查天赋树，**THEN** 所有节点为未解锁，轮回点=0
- **GIVEN** 玩家本局达到筑基期后死亡，**WHEN** 轮回结算，**THEN** 获得至少5轮回点（境界=5）
- **GIVEN** 玩家本局通关（化神期 victory），**WHEN** 轮回结算，**THEN** 获得境界20+通关10=30基础点+击杀/收集/炼制额外点
- **GIVEN** 玩家有3轮回点，**WHEN** 尝试解锁消耗5的天赋，**THEN** 提示轮回点不足
- **GIVEN** 玩家已解锁「勤修苦练」（修为+10%），**WHEN** 新局开始，**THEN** 修为获取值=基准×1.1
- **GIVEN** 玩家已解锁「灵脉感应」（灵石+10%）且选择青云剑宗（灵石+15%），**WHEN** 获取灵石，**THEN** 总加成+25%
- **GIVEN** 玩家已解锁「苍玄行者」，**WHEN** 新局进入身份选择，**THEN** 「阵道双杰」显示为可选（无🔒遮罩）
- **GIVEN** 玩家未解锁「苍玄行者」，**WHEN** 身份选择界面查看阵道双杰，**THEN** 显示🔒遮罩+"需要轮回天赋 '苍玄行者' 解锁"
- **GIVEN** 玩家已解锁「精挑细选」，**WHEN** 战斗胜利进入战利品选择，**THEN** 展示4个选项（而非3个）
- **GIVEN** 玩家已解锁「暗金天命」且在金丹期，**WHEN** 检查暗金卡上限，**THEN** 上限=2（原1+1）
- **GIVEN** 玩家已解锁「博闻强记」，**WHEN** 新局首回合抽牌，**THEN** 额外多抽1张
- **GIVEN** 玩家已解锁「血战不退」，**WHEN** 检查所有角色HP，**THEN** 每个角色HP+2
- **GIVEN** 玩家已解锁「锋芒初显」，**WHEN** 新局首回合，**THEN** 费用+1
- **GIVEN** 玩家已解锁「因果不灭」且本局死亡，**WHEN** 结算界面选卡保留，**THEN** 下一局初始卡组包含该保留卡
- **GIVEN** 玩家已解锁「万界穿梭」，**WHEN** 新局开始，**THEN** 随机展示2个身份供选择（而非展示全部可选身份）
- **GIVEN** 玩家已解锁「超脱轮回」，**WHEN** 新局开始，**THEN** 修为从200/1000开始；轮回结算时点数额外+20%
- **GIVEN** 玩家解锁天赋节点，**WHEN** 操作完成，**THEN** progression.dat 立即更新
- **GIVEN** 玩家在天赋树界面，**WHEN** 可用轮回点=0，**THEN** 所有未解锁节点显示为灰色（不可点击）
- **GIVEN** progression.dat 损坏，**WHEN** 游戏启动，**THEN** 轮回天赋重置为初始状态，提示"轮回数据已重置"
