## 概述

轮回天赋系统是游戏的 Roguelike 元成长层——玩家每局通关或死亡后，根据本局表现获得「轮回点」，用于在天赋树中解锁永久性被动加成。这些天赋跨局保留，让每一次失败都不是终点而是积累，每一次通关都开启新的可能性。

对玩家来说，轮回天赋系统解决了 Roguelike 最核心的情感矛盾——「我打了 2 小时的局死了，什么都没留下」。天道轮回，修为不灭——死后的积累让下一次开局更强，解锁的新身份、新卡牌、新机制让重复游玩始终有新鲜感。这正是「仙途问道」的主题映射：修仙本就是一世又一世的积累，轮回是天道法则而非游戏惩罚。

**v2.0 核心变更（2026-07-24 全面修订）：**
- 主动天赋槽位机制——解锁全部 20 个天赋，每局装备 N = 5 + floor(已解锁数/4) 个
- 境界奖励曲线从线性改为平方增长（消除速度自杀最优）
- 全解锁后溢出点兑换「天机抽取」系统
- 天赋总成本从 160 上调至 330（目标 25-35 局完成）
- 因果不灭暗金卡限制 + 公式统一化 + AC 从 19 条扩展至 38 条

## 玩家幻想

| 情感 | 触发场景 |
|------|---------|
| **失败有价值** | 死亡结算时看到轮回点+3——「至少解锁了一个天赋，下局更强」（保证最低 3 点） |
| **规划的期待** | 在天赋树界面斟酌哪些天赋装备到槽位——「这次走修为+战意，还是资源+卡牌？」 |
| **积累的骄傲** | 看到天赋树点亮了一大片——「我已经轮回 30 次了」 |
| **新身份的惊喜** | 攒够轮回点解锁「阵道双杰」——「终于能玩阵法流了！」 |
| **策略深度** | 根据已解锁天赋和本局流派调整槽位配置——「这局我有灵石加成+修为加速，走百艺流」 |

轮回天赋的核心情感不是「变强了所以简单了」，而是**「每一世都在为下一世铺路」**——这是修仙者的时间观，也是 Roguelike 玩家最珍视的积累感。

## 详细设计

### 核心规则

#### 1. 轮回触发时机

轮回结算在以下两种情况下触发：

| 触发条件 | 结算内容 | 说明 |
|---------|---------|------|
| **角色死亡**（战斗中全员阵亡且无法复活） | 轮回点结算 + 解锁检测 | 游戏结束→进入轮回结算界面 |
| **通关**（击败化神期最终 Boss） | 轮回点结算 + 解锁检测 + 通关标记 | 通关后的轮回结算奖励更丰厚 |

轮回结算流程：
```
① 展示本局结算面板：
   - 最高境界 reached
   - 击杀敌人数
   - 收集卡牌种类数
   - 炼丹/炼器次数
   - 解锁的新卡牌
② 计算轮回点获得量
③ 检查是否解锁新天赋节点（满足解锁条件）
④ 更新 progression.dat
⑤ 进入天赋树界面（玩家可消费轮回点或调整槽位装备）
⑥ 玩家确认后返回主菜单
```

#### 2. 轮回点获取公式（v2 修订版——唯一权威版本）

```
calculate_reincarnation_points(run_data) → int:
  points = 0
  
  # 境界奖励（主要来源——平方增长消除自杀最优）
  # 炼气=2, 筑基=8, 金丹=18, 元婴=32, 化神=50
  points += run_data.realm_reached² × 2
  
  # 通关奖励
  if run_data.result == "victory":
    points += 10
  
  # 击杀奖励（按境界梯度解锁上限）
  elites_cap = [3, 6, 10, 15, 20][run_data.realm_reached - 1]
  bosses_cap = [1, 3, 5, 7, 10][run_data.realm_reached - 1]
  points += min(run_data.elites_killed, elites_cap) × 1
  points += min(run_data.bosses_killed, bosses_cap) × 2
  
  # 收集奖励
  unique_cards = run_data.unique_cards_collected
  points += min(floor(unique_cards / 10), 5) × 1
  
  # 炼制奖励
  craft_total = run_data.alchemy_count + run_data.forge_count
  points += min(floor(craft_total / 5), 5) × 1
  
  # 超脱轮回加成（round 替代 floor，保证低分也有收益）
  if "reincarnation_4" in progression.unlocked_talents:
    points = round(points × 1.2)
  
  # 保底机制（每次死亡至少 3 点）
  if run_data.result == "death":
    points = max(3, points)
  
  return points
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|:----:|------|
| realm_reached | int | [1, 5] | 本局达到的最高境界（炼气=1, 筑基=2, 金丹=3, 元婴=4, 化神=5） |
| result | enum | "victory" / "death" | 本局结束方式 |
| elites_killed | int | [0, ∞) | 本局击杀精英数（上限按境界：3/6/10/15/20） |
| bosses_killed | int | [0, ∞) | 本局击杀Boss数（上限按境界：1/3/5/7/10） |
| unique_cards_collected | int | [0, 222] | 本局收集的不同卡牌种类数 |
| alchemy_count | int | [0, ∞) | 炼丹操作次数 |
| forge_count | int | [0, ∞) | 炼器操作次数 |

**典型轮回点收益（v2 修订版）：**
| 本局表现 | 境界 | 轮回点（范围） |
|---------|:----:|:-----:|
| 新手第一局，炼气期死亡 | 炼气 | 3~8 |
| 筑基期中段死亡 | 筑基 | 11~22 |
| 金丹期 Boss 战死 | 金丹 | 22~40 |
| 元婴期通关 | 元婴 | 36~55 |
| 化神期完美通关 | 化神 | 55~80 |

#### 3. 天赋树结构

天赋树分为 **5 条分支**，每条 4 层，共 20 个天赋节点。每条分支有独立的解锁条件和逐层递进的消耗：

```
天赋树层级结构（按分支）：

【修为之道】——— 加速修为积累
  L1: 勤修苦练    cost=8  修为获取+10%
  L2: 厚积薄发    cost=12 突破所需修为-10%（修为上限×0.9）
  L3: 道心通明    cost=18 溢出修为转化率+50%（150修为=1属性丹）
  L4: 天道酬勤    cost=28 每局开始赠送培元丹×1（+50修为）

【资源之道】——— 加速资源获取
  L1: 灵脉感应    cost=8  探索灵石掉落+10%
  L2: 坊市老客    cost=12 商店卡牌价格-15%
  L3: 炼器余料    cost=18 炼丹/炼器灵材消耗-1（最少1）
  L4: 苍玄行者    cost=30 解锁隐藏身份「阵道双杰」

【战意之道】——— 战斗属性加成
  L1: 锋芒初显    cost=8  开局首回合费用+1
  L2: 血战不退    cost=12 角色最大生命+2
  L3: 会心一击    cost=18 暴击率+5%
  L4: 破釜沉舟    cost=25 己方角色低于50%HP时攻击+2

【卡牌之道】——— 卡组构筑优势
  L1: 博闻强记    cost=8  开局额外多抽1张牌（首回合抽牌+1）
  L2: 精挑细选    cost=12 战利品三选一变为四选一
  L3: 暗金天命    cost=18 暗金卡上限+1（突破境界限制）
  L4: 万法归宗    cost=25 卡组上限+5（45张→受此天赋影响）

【轮回之道】——— 特殊机制解锁
  L1: 轮回印记    cost=8  每局开始随机获得1张蓝卡
  L2: 因果不灭    cost=14 死亡后可保留1张卡牌带入下一局（稀有度上限≤史诗/紫色）
  L3: 万界穿梭    cost=20 开局可从2个随机身份中选择（而非固定选择）
  L4: 超脱轮回    cost=35 开局境界从炼气中期开始（修为从200/1000开始）+额外轮回点获取+20%
```

**天赋总成本（v2 修订版）：** 5×L1(8) + 5×L2(12) + 5×L3(18) + 5×L4(25/28/30/35) = 40 + 60 + 90 + 143 = **333 点**

**分支解锁规则：**
- 每条分支的 L1 天赋从开局就可见并可解锁
- L2 需要 L1 已解锁（同分支前置）
- L3 需要 L2 已解锁 + 玩家至少通关 1 次（或死亡 5 次以上）——「道心考验」
- L4 需要 L3 已解锁 + 最高境界达到金丹以上——「大道之基」

**天赋槽位机制（v2 新增）：**
- 每局开始前，玩家从已解锁的全部天赋中选择 N 个装备到槽位
- N = 5 + floor(已解锁天赋总数 / 4)，向下取整
- 初始（0 已解锁）可装备 5 个，20 个全解锁时可装备 10 个
- 槽位装备在开局身份选择之前进行——身份选择可看到天赋带来的额外选项
- 已解锁但未装备的天赋不生效——玩家必须在槽位限制内做出取舍
- 战斗中和局间不可更换装备的天赋，仅在每局开始前可调整

**天赋效果叠加规则：**
- 同一分支内的天赋效果可叠加（L1+L2+L3+L4 全部生效——若均装备）
- 不同分支间的天赋效果可叠加（修为之道 + 资源之道同时生效——若均装备）
- 数值叠加为加法（如修为获取 +10% + 卡牌效果 = 独立修正叠加）

#### 4. 轮回点存储与消费

轮回点存储在 `progression.reincarnation_points` 中（持久化到 `progression.dat`）：

```
# progression.dat 中的轮回天赋数据
{
  "reincarnation_points": 25,          # 当前可用轮回点
  "total_points_earned": 120,          # 累计获得（用于统计）
  "unlocked_talents": [                # 已解锁的天赋节点ID列表
    "cultivation_1", "cultivation_2",
    "resource_1", "combat_1", "combat_2",
    "card_1"
  ],
  "equipped_talents": [                # 当前装备的槽位（v2 新增）
    "cultivation_1", "resource_1",
    "combat_1", "combat_2"
  ],
  "active_slots": 5,                   # 当前可用槽位数
  "talent_tree_progress": {            # 每条分支当前解锁到第几层
    "cultivation": 2,
    "resource": 1,
    "combat": 2,
    "card": 1,
    "reincarnation": 0
  },
  "total_reincarnations": 8,           # 累计轮回次数
  "highest_realm_ever": "金丹",         # 历史最高境界
  "victories": 0                       # 通关次数
}
```

**消费规则：**
- 轮回点在天赋树界面消费，不可退款（选择即锁定）
- 已解锁的天赋永久保留，但可通过槽位系统灵活装备/卸除
- 轮回点不足时对应天赋节点灰色显示
- 轮回点无上限，可以攒够一次性解锁多条分支

**溢出兑换（v2 新增）：**
- 全 20 节点解锁后，每 5 轮回点可兑换 1 次「天机抽取」
- 天机抽取随机产出：灵石×50 (40%) / 培元丹×1 (25%) / 随机蓝卡 (20%) / 随机紫卡 (10%) / 随机暗金卡 (5%)
- 每局最多兑换 3 次，让轮回点在后期保持价值

#### 5. 天赋生效时机

天赋在每局开始时通过槽位选择自动应用：

```
apply_equipped_talents():
  for talent_id in progression.equipped_talents:
    talent_def = TALENT_TREE[talent_id]
    match talent_def.effect_type:
      "cultivation_boost":    GSM.player.cultivation_modifier += talent_def.value
      "resource_boost":       GSM.player.resource_modifier += talent_def.value
      "combat_stat":          GSM.player.combat_modifiers[talent_def.stat] += talent_def.value
      "card_advantage":       GSM.player.deck_modifiers.append(talent_def.effect)
      "special_unlock":       GSM.player.special_unlocks.append(talent_def.effect)
      "identity_unlock":      GSM.progression.available_identities.append(talent_def.identity_id)
  
  # 特殊天赋：暗金天命（突破暗金卡上限）
  if "darkgold_limit_plus" in equipped:
    card_system.darkgold_limit_modifier += 1
  
  # 特殊天赋：超脱轮回（开局修为提升）
  if "headstart" in equipped:
    GSM.player.cultivation = 200  # 而非 0
```

天赋槽位在**开局身份选择之前**分配——这意味着身份选择时可以看到天赋带来的额外选项（如万界穿梭提供 2 个随机身份）。

#### 6. 天赋解锁前置条件（软解锁）

除了轮回点消耗外，部分天赋需要满足游戏内条件才能出现在天赋树中（可见但不可解锁）：

| 天赋 | 可见条件 | 解锁条件 |
|------|---------|---------|
| 天道酬勤 (修为L4) | 修为L3已解锁 | 历史最高境界≥金丹 |
| 苍玄行者 (资源L4) | 资源L3已解锁 | 历史累计收集≥50种不同卡牌 |
| 破釜沉舟 (战意L4) | 战意L3已解锁 | 累计击杀≥30个精英敌人 |
| 万法归宗 (卡牌L4) | 卡牌L3已解锁 | 本局曾达到卡组40张上限 |
| 超脱轮回 (轮回L4) | 轮回L3已解锁 | 至少通关1次 |

未满足解锁条件的天赋节点显示为灰色+条件描述（如「需通关1次」）。

#### 7. 天赋与身份专属天赋的关系

- 身份专属天赋（如青云剑宗的+15%灵石掉落）与轮回天赋的效果**独立叠加**
- 相同类型效果以**加法**叠加：身份灵石+15% + 灵脉感应+10% = +25%灵石掉落
- 轮回天赋不影响身份选择的初始卡组内容——天赋在身份选择之前分配槽位，但初始卡组由身份定义

#### 8. 轮回结算中的新卡牌解锁

除了天赋节点，轮回结算还检测「新卡牌解锁」——某些卡牌需要特定成就才能出现在全局卡池中：

```
check_card_unlocks(run_data, progression):
  unlocks = []
  
  # 境界里程碑解锁
  if run_data.highest_realm >= 金丹 and "card_golden_core" not in unlocked:
    unlocks.append("card_golden_core")  # 解锁金丹期限定金卡
  
  # 特殊成就解锁（累计击杀而非单局）
  if progression.total_bosses_killed >= 5 and "card_boss_slayer" not in unlocked:
    unlocks.append("card_boss_slayer")
  
  # 身份通关解锁
  if run_data.result == "victory" and run_data.identity_id == "dan_xiagu" and "card_alchemy_master" not in unlocked:
    unlocks.append("card_alchemy_master")  # 解锁万象真人角色卡
  
  return unlocks
```

> **注**：新卡牌解锁是轻量机制——不是大量卡牌需要解锁，而是 5~10 张特殊卡牌作为「轮回惊喜」分布在不同成就条件中。Boss 击杀条件已改为累计 lifetime 值，确保实际可达。身份判定改用 ID 而非裸字符串。

### 状态与转换

```
天赋生命周期：

[初始状态] —— progression 为空或不存在
    ↓ 第1局游戏结束
[轮回结算] → 获得轮回点
    ↓ 在天赋树中消费
[天赋解锁] → 写入 progression.dat
    ↓ 下一局开始→进入槽位选择
[天赋装备] → 选择 N 个已解锁天赋装备（N = 5 + floor(unlocked/4)）
    ↓ 本局游戏...
[游戏结束] → 再次轮回结算
    ↓
循环 — 每次轮回积累更多天赋 → 更多槽位 → 更多策略组合

同一节点状态：
  未可见 → 满足可见条件 → 可见·未解锁 → 消费轮回点 → 已解锁·待装备 → 装备后生效中
```

**天赋不可重置**——解锁即永久。这是设计上的意图：轮回天赋代表「天道的积累」，不可逆转。**槽位系统替代了重置机制**——玩家在每局开始前灵活调整装备的天赋组合，而非永久关闭某个天赋。

```
  points += min(run_data.elites_killed, elites_cap) × 1
  points += min(run_data.bosses_killed, bosses_cap) × 2
  
  # 收集奖励
  unique_cards = run_data.unique_cards_collected
  points += min(floor(unique_cards / 10), 5) × 1
  
  # 炼制奖励
  craft_total = run_data.alchemy_count + run_data.forge_count
  points += min(floor(craft_total / 5), 5) × 1
  
  # 超脱轮回加成（round 替代 floor，保证低分也有收益）
  if "reincarnation_4" in progression.unlocked_talents:
    points = round(points × 1.2)
  
  # 保底机制（每次死亡至少 3 点）
  if run_data.result == "death":
    points = max(3, points)
  
  return points
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|:----:|------|
| realm_reached | int | [1, 5] | 本局达到的最高境界（炼气=1, 筑基=2, 金丹=3, 元婴=4, 化神=5） |
| result | enum | "victory" / "death" | 本局结束方式 |
| elites_killed | int | [0, ∞) | 本局击杀精英数 |
| bosses_killed | int | [0, ∞) | 本局击杀Boss数 |
| unique_cards_collected | int | [0, 222] | 本局收集的不同卡牌种类数 |
| alchemy_count | int | [0, ∞) | 炼丹操作次数 |
| forge_count | int | [0, ∞) | 炼器操作次数 |

### 2. 主动槽位数量计算

```
active_slots = 5 + floor(total_unlocked_talents / 4)
```

槽位数量随解锁进度增长：
- 0 已解锁：5 个
- 4 已解锁：6 个
- 8 已解锁：7 个
- 12 已解锁：8 个
- 16 已解锁：9 个
- 20 已解锁：10 个

### 3. 天赋效果叠加

```
apply_talent_modifier(base_value, modifier_list) → float:
  total_mod = 1.0
  for mod in modifier_list:
    total_mod += mod / 100.0  # 百分比转换为小数
  return base_value × total_mod
```

**示例：** 灵石掉落基准 100，身份 +15%，灵脉感应 +10% → 100 × (1 + 0.15 + 0.10) = 125

### 4. 天赋树解锁验证

```
can_unlock(talent_id, progression) → {allowed: bool, reason: String}:
  talent = TALENT_TREE[talent_id]
  
  if talent_id not in TALENT_TREE:
    return {false, "天赋不存在"}
  
  if progression.reincarnation_points < talent.cost:
    return {false, "轮回点不足（需要{talent.cost}，当前{progression.reincarnation_points}）"}
  if talent.prerequisite and talent.prerequisite not in progression.unlocked_talents:
    return {false, "需要先解锁前置天赋"}
  if talent.unlock_condition:
    if not talent.unlock_condition(progression):
      return {false, talent.unlock_description}
  return {true, ""}
```

## 边界情况

- **首次游戏无轮回天赋**：第一次游玩时 progression.dat 不存在，所有天赋未解锁。槽位 = 5，玩家从零开始选择。这是预期的新手体验——「从凡人开始」
- **轮回点攒着不用**：允许——轮回点无过期时间，玩家可以攒够一次性解锁多条分支
- **已解锁全部天赋（20/20）**：成为「超脱轮回者」——轮回点仍然累计，但可通过天机抽取兑换随机初始资源包（灵石/丹药/卡牌），保持结算界面价值
- **progression.dat 损坏或丢失**：重置为初始值——所有天赋解锁和轮回点丢失（损失已解锁的天赋）。通过存档系统的完整性校验缓解此风险
- **死亡结算 vs 通关结算**：两种结算都获取轮回点但通关有+10额外奖励。死亡同样可解锁天赋（不需要通关）
- **阵道双杰解锁后，新游戏仍选择其他身份**：允许——苍玄行者解锁的是身份的「可选性」，不是强制使用
- **因果不灭（保留1张卡牌到下一局）**：保留卡稀有度上限 ≤ 史诗（紫色），不可保留暗金/橙色卡。保留卡受卡组上限约束，若已达上限则需先弃 1 张
- **精挑细选（四选一）与三选一UI**：战利品面板扩展为4个选项，已达持有上限的卡牌仍显示灰色不可选
- **暗金天命（暗金卡上限+1）**：金丹期暗金上限原为 1→变为 2；元婴期 2→变为 3。此效果对炼气/筑基无效（card_pool_tier 不产出暗金），但天赋节点注明"金丹期起效"
- **超脱轮回（开局200修为）**：开局修为从 200/1000 开始（而非 0/1000），配合勤修苦练（+10%修为获取），开局积累速度明显更快。与厚积薄发（突破门槛×0.9）叠加后首局突破速度需监测
- **破釜沉舟（低于50%HP+2攻击）**：每回合开始时检查 HP 百分比，动态开关。HP 恰好 50% 时判定为不触发（低于 50% 的含义为严格小于）
- **多条天赋数值叠加**：灵石掉落理论上限 25%（身份 15% + 灵脉感应 10%），修为获取上限 10%（勤修苦练），均为温和加成
- **全解锁后槽位上限 10**：玩家必须在 20 个已解锁天赋中选择 10 个装备，创造每局不同的策略组合
- **天机抽取（溢出兑换）**：每 5 溢出点兑换 1 次，随机获得灵石/丹药/卡牌。每局限 3 次

轮回天赋系统处于「元」层——它不直接参与局内玩法循环，而是在局与局之间的结算界面运行。所有软依赖系统在设计时不需要等待轮回天赋完成，只需预留天赋效果查询接口和槽位装备接口即可。

## 验收标准

### 基础功能（17 条）
- **GIVEN** 玩家第1次游戏（无 progression.dat），**WHEN** 检查天赋树，**THEN** 所有节点为未解锁，轮回点=0，槽位=5
- **GIVEN** 玩家本局达到筑基期后死亡，击杀 0 精英、0 Boss，收集 0 卡牌，炼制 0 次，**WHEN** 轮回结算，**THEN** 获得 8 轮回点
- **GIVEN** 玩家本局达到炼气期后死亡，击杀 0 精英、0 Boss，收集 0 卡牌，炼制 0 次，**WHEN** 轮回结算，**THEN** 获得 3 轮回点（保底触发）
- **GIVEN** 玩家本局通关（化神期 victory），击杀 0 精英、0 Boss，收集 0 卡牌，炼制 0 次，**WHEN** 轮回结算，**THEN** 获得 60 轮回点（境界 50 + 通关 10）
- **GIVEN** 玩家有 7 轮回点，尝试解锁消耗 8 的天赋，**WHEN** 操作，**THEN** 提示轮回点不足
- **GIVEN** 玩家有 8 轮回点，尝试解锁消耗 8 的天赋，**WHEN** 操作，**THEN** 解锁成功，轮回点变为 0
- **GIVEN** 玩家已解锁「勤修苦练」并装备到槽位，**WHEN** 新局开始，**THEN** 修为获取值 = 基准 × 1.1
- **GIVEN** 玩家已装备「灵脉感应」且选择青云剑宗，**WHEN** 获取灵石，**THEN** 总加成 +25%
- **GIVEN** 玩家已解锁「苍玄行者」，**WHEN** 新局进入身份选择，**THEN** 「阵道双杰」显示为可选
- **GIVEN** 玩家已装备「精挑细选」，**WHEN** 战斗胜利进入战利品选择，**THEN** 展示 4 个选项（而非 3 个）
- **GIVEN** 玩家已装备「暗金天命」且在金丹期，**WHEN** 检查暗金卡上限，**THEN** 上限 = 2（原 1 + 1）
- **GIVEN** 玩家已装备「博闻强记」，**WHEN** 新局首回合抽牌，**THEN** 抽牌数量 = 默认值 + 1
- **GIVEN** 玩家已装备「血战不退」，**WHEN** 新局开始且角色加入队伍，**THEN** 每个角色最大 HP + 2
- **GIVEN** 玩家已装备「锋芒初显」，**WHEN** 新局首回合，**THEN** 费用 + 1
- **GIVEN** 玩家已装备「超脱轮回」，**WHEN** 新局开始，**THEN** 修为从 200/1000 开始
- **GIVEN** 玩家已装备「因果不灭」且本局死亡，**WHEN** 结算界面选卡保留，**THEN** 保留卡稀有度 ≤ 史诗（仅蓝/紫可选）
- **GIVEN** 玩家已装备「万界穿梭」，**WHEN** 新局开始，**THEN** 随机展示 2 个互不相同的身份供选择

### 槽位机制（5 条，v2 新增）
- **GIVEN** 玩家已解锁 4 个天赋，**WHEN** 进入槽位装备界面，**THEN** 可装备槽位 = 6
- **GIVEN** 玩家已解锁 8 个天赋，**WHEN** 进入槽位装备界面，**THEN** 可装备槽位 = 7
- **GIVEN** 玩家已解锁 12 个天赋，**WHEN** 进入槽位装备界面，**THEN** 可装备槽位 = 8
- **GIVEN** 玩家已解锁 16 个天赋，**WHEN** 进入槽位装备界面，**THEN** 可装备槽位 = 9
- **GIVEN** 玩家已解锁 20 个天赋，**WHEN** 进入槽位装备界面，**THEN** 可装备槽位 = 10

### 溢出兑换（2 条，v2 新增）
- **GIVEN** 玩家已全解锁 20 个天赋且轮回点 ≥ 5，**WHEN** 进行天机抽取，**THEN** 消耗 5 轮回点获得随机资源之一，剩余轮回点减少 5
- **GIVEN** 玩家已全解锁 20 个天赋且轮回点 ≥ 15，**WHEN** 连续抽取 3 次，**THEN** 消耗 15 轮回点获得 3 份随机资源，提示已达本局上限

### 边界情况（5 条）
- **GIVEN** progression.dat 损坏，**WHEN** 游戏启动，**THEN** 轮回天赋重置为初始状态，提示"轮回数据已重置"
- **GIVEN** 玩家已装备「破釜沉舟」，角色 HP = 50%，**WHEN** 回合开始，**THEN** 攻击力无加成（低于 50% 为严格小于）
- **GIVEN** 玩家已装备「因果不灭」，保留暗金卡，**WHEN** 尝试保留，**THEN** 暗金卡不可选（仅蓝/紫可选）
- **GIVEN** 玩家在槽位装备界面，**WHEN** 尝试装备超出槽位数量的天赋，**THEN** 提示需先卸除已装备天赋
- **GIVEN** 玩家在天赋树界面，**WHEN** 可用轮回点 = 0，**THEN** 所有未解锁节点显示为灰色（不可点击）

### 再生效规则（4 条）
- **GIVEN** 玩家已解锁天赋且将其装备到槽位，**WHEN** 新局开始，**THEN** 装备的天赋效果生效
- **GIVEN** 玩家已解锁天赋但未装备，**WHEN** 战斗中或局间，**THEN** 该天赋不生效
- **GIVEN** 玩家在槽位装备界面卸除已装备天赋，**WHEN** 确认卸除，**THEN** 该天赋不生效，槽位释放
- **GIVEN** 玩家装备天赋超过槽位上限，**WHEN** 检查，**THEN** 槽位不足提示

### 公式验证（5 条）
- **GIVEN** 玩家本局炼气期死亡，击杀 0 精英、0 Boss、0 卡牌、0 炼制，**WHEN** 结算，**THEN** 轮回点 = 3（保底验证）
- **GIVEN** 玩家本局通关化神，击杀 10 精英、5 Boss、50 卡牌、10 炼制，超脱未解锁，**WHEN** 结算，**THEN** 轮回点 = 50 + 10 + 10 + 10 + 5 + 2 = 87
- **GIVEN** 玩家已解锁超脱轮回，本局通关化神，击杀 10 精英、5 Boss、50 卡牌、10 炼制，**WHEN** 结算，**THEN** 轮回点 = round(87 × 1.2) = 104
- **GIVEN** 玩家未解锁超脱轮回，本局炼气期 death，全 0，**WHEN** 结算，**THEN** 轮回点 = max(3, 2) = 3（保底覆盖 0 点）
- **GIVEN** 玩家已解锁超脱轮回，本局炼气期 death，全 0，**WHEN** 结算，**THEN** 轮回点 = max(3, round(2×1.2)) = 3（保底优先于加成）
