## 概述

卡组编辑UI系统负责所有与卡牌管理相关的视觉交互界面——包括战利品三选一面板、商店购买/出售/删卡界面、卡组浏览界面以及超限弃牌界面。卡组编辑UI不执行卡组变更逻辑（那是卡组编辑系统的职责），而是将卡组数据呈现为可操作的界面，并将玩家的选择转化为卡组变更指令。

对玩家来说，卡组编辑UI是「修仙路上的每一次选择」——战后三选一的期待、商店中精打细算的权衡、删卡时的心痛——UI需要让每一次选择都有分量和清晰的反馈。

## 玩家幻想

卡组编辑UI的核心情感是**选择的清晰度**——玩家在做每个卡牌相关决策时，都能清楚地看到选项的细节和对自己构筑的影响。

| 情感 | 触发场景 |
|------|---------|
| **选择的审慎** | 战利品三选一——悬停查看每张卡牌效果，「这张能进我的构筑吗？」 |
| **购买的期待** | 商店中看到一张关键流派卡——「终于刷出来了！」 |
| **删卡的心痛** | 确认删除一张陪伴很久的卡牌——「是时候放手了」 |
| **出售的算计** | 出售多余卡牌换灵石——「这张用不上，卖掉凑删卡费」 |
| **构筑的审视** | 浏览当前卡组全貌——「这套构筑成型了，流畅！」 |

## 详细设计

### 核心规则

#### 1. 战利品三选一面板

战斗胜利后弹出：

```
┌──────────────────────────────────────────┐
│  ⚔ 战斗胜利！                              │
│                                           │
│  选择一项奖励：                              │
│                                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│  │ 青云剑诀   │  │  灵石×25  │  │  回血丹   │ │
│  │ ⬜蓝色·功法│  │  🪙      │  │  💊消耗品 │ │
│  │ 费用：3   │  │  灵石堆   │  │  15%HP  │ │
│  │ 效果：[..] │  │  图标     │  │  回复    │ │
│  │          │  │          │  │          │ │
│  │  🔄新卡   │  │          │  │          │ │
│  └──────────┘  └──────────┘  └──────────┘ │
│                                           │
│  卡组：28/30张       [跳过]                │
└──────────────────────────────────────────┘
```

**界面规则：**
- 三个选项以卡片形式等宽排列展示
- 卡牌选项：正面展示（卡名、稀有度、费用、效果简述、类型标签）
  - 卡牌左上角显示稀有度颜色条
  - 首次获得的卡牌显示「🔄新卡」标签
  - 鼠标悬停显示完整卡牌详情（放大效果）
- 灵石选项：灵石图标+数量（如「灵石×25」），卡片背景金色
- 消耗品选项：对应图标+效果简述（如「回血15%HP」）
- 卡组已满（≥上限）时：卡牌选项灰色+「卡组已满」标记，不可选；灵石/消耗品可选
- 右上角显示当前卡组数量（如28/30）
- 「跳过」按钮：放弃所有奖励（触发确认「放弃所有奖励？」）
- 选中任一选项后，面板关闭，其他选项消失

**选中动画：**
```
① 悬停→选项轻微放大（1.05×）+外发光
② 点击确认→被选中的选项飞向HUD（卡牌→卡组，灵石→资源栏）
③ 其余选项淡出消失
④ 面板关闭
```


**界面规则：**
- 商品以卡片网格排列（4~6列）
- 每张商品卡正面展示：
  - 卡牌图标/缩略图
  - 卡名 + 稀有度颜色条
  - 类型标签（功法/丹药/符箓等）
  - 价格（灵石数字+图标）
- 价格 ≤ 当前灵石 → 正常显示，可点击购买
- 价格 > 当前灵石 → 灰色+「灵石不足」标记，不可购买
- 卡组已满 → 卡牌商品灰色+「卡组已满」标记
- 已售罄的商品 → ❌标记，不可再买
- 顶部显示当前灵石余额
- 底部显示当前卡组数量
- 「刷新商店」按钮：消耗灵石刷新商品列表
- 「离开」按钮：关闭商店返回探索

**购买交互：**
```
① 点击可购买的商品卡
② 弹出确认窗口：「确认购买 [卡名]（[价格]灵石）？」
③ 确认→灵石扣除+卡牌加入卡组+商品标记已售
④ 取消→返回浏览
```

**界面规则：**
- 当前卡组以列表/网格展示，所有卡牌可选
- 点击卡牌→选中（✓标记），再次点击取消选中
- 每次只能选择1张卡牌删除
- 底部显示本次费用（根据已删次数计算）
- 不可用角色卡牌不显示在删除列表中（已在阵亡时自动移除）
- 选中卡牌后→显示「确认删除」按钮
- 灵石不足删卡→按钮不可用+提示「灵石不足」

**删卡交互：**
```
① 点击一张卡牌选中
② 底部显示费用+「确认删除」按钮
③ 点击确认→弹出确认窗口：「确认删除 [卡名]？删除后不可恢复」
④ 确认→灵石扣除+卡牌从卡组移除
⑤ 取消→返回删卡界面
```

## 公式

卡组编辑UI不执行数值计算——所有费用/价值计算由卡组编辑系统和资源系统完成。

### 1. 卡牌稀有度颜色

```
rarity_color(rarity) → Color:
  match rarity:
    1(白): return "#CCCCCC"
    2(蓝): return "#4488FF"
    3(紫): return "#AA44FF"
    4(金): return "#FFAA00"
    5(暗金): return "#FF4400"
```

## 边界情况

- **战利品三选一中所有选项都是卡牌且卡组已满**：三个选项都灰色+「卡组已满」，玩家被迫跳过（无损失）
- **商店商品列表为空（全部售罄）**：显示「本店商品已售罄」提示，仅显示刷新和离开按钮
- **删卡界面中所有卡牌都是重要的（玩家不想删任何一张）**：允许取消返回，不强制删卡
- **出售卡牌时误操作**：确认窗口二次确认，「出售后不可恢复」
- **超限弃牌时玩家先选了3张又取消2张**：已选计数实时更新，确认时以最终选择为准
- **卡组浏览中筛选结果为0**：显示「没有符合条件的卡牌」提示
- **商店刷新后商品列表与刷新前有重复**：允许——商店刷新是随机重置，重复商品概率存在
- **多次连续删卡的费用递增显示**：UI显示当前次数的费用，并标注本次是第几次删卡

## 验收标准

- **GIVEN** 战斗胜利，**WHEN** 结算，**THEN** 弹出三选一面板，3个选项清晰展示
- **GIVEN** 战利品三选一，**WHEN** 悬停卡牌选项，**THEN** 卡牌放大显示详情
- **GIVEN** 战利品三选一，**WHEN** 点击一个选项，**THEN** 面板关闭，奖励生效
- **GIVEN** 卡组已满且战利品包含卡牌，**WHEN** 显示面板，**THEN** 卡牌选项灰色+「卡组已满」
- **GIVEN** 进入商店，**WHEN** 加载完成，**THEN** 显示商品网格+灵石余额
- **GIVEN** 商店中商品价格>灵石，**WHEN** 展示，**THEN** 该商品灰色+「灵石不足」
- **GIVEN** 商店中点击可购买商品，**WHEN** 确认，**THEN** 灵石扣除+卡牌加入卡组
- **GIVEN** 进入删卡界面，**WHEN** 选中一张卡牌，**THEN** 显示本次费用+确认按钮
- **GIVEN** 删卡确认，**WHEN** 执行，**THEN** 卡牌移除+灵石扣除
- **GIVEN** 出售卡牌，**WHEN** 选中2张+确认，**THEN** 卡牌移除+灵石增加
- **GIVEN** 卡组超限，**WHEN** 触发，**THEN** 超限弃牌面板弹出，玩家需选择超限张数弃掉
- **GIVEN** 卡组浏览界面，**WHEN** 点击筛选标签，**THEN** 仅显示对应类型的卡牌
- **GIVEN** 卡牌详情浮窗，**WHEN** 点击卡牌，**THEN** 显示卡名、效果、稀有度、来源
