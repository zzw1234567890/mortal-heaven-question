# 渡劫突破系统 (Tribulation Breakthrough System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-22
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱2「苟道成长，步步为营」

## 概述

渡劫突破系统管理玩家从一个大境界晋升到下一个大境界的完整流程——从修为满值后触发渡劫、到特殊Boss战、再到突破成功后的全局状态更新。渡劫本质上是一场**特殊规则下的Boss战斗**，与传统战斗的关键区别在于：渡劫战有独立的天劫Boss机制（雷伤累积）、渡劫丹可降低难度、失败只有修为损失而无永久性惩罚。

对玩家来说，渡劫是每局游戏中最重要的「期中考试」——它验证你当前的构筑是否足够强大，也从结果上决定你能否进入下一阶段的内容。渡劫成功带来的快感是质变的（新地图、新卡池、新上限），而渡劫失败的损失可控（仅10%修为），鼓励玩家在觉得「差不多够了」的时候就去尝试——哪怕失败，损失也不致命。

## 玩家幻想

渡劫系统的核心情感是**渡劫前的紧张与突破后的豁然开朗**——每次渡劫都是一次「赌上所有」的仪式感时刻。

| 情感 | 触发场景 |
|------|---------|
| **渡劫前的紧张** | 修为满后决定去渡劫——「这套构筑能打过天劫吗？」 |
| **准备的笃定** | 使用渡劫丹、调整卡组——「能做的都做了，剩下的看实战了」 |
| **天劫的压迫** | 雷伤debuff不断叠加——「得加快节奏，拖久了扛不住」 |
| **突破的狂喜** | 击败天劫Boss那一刻——「我突破了！」 |
| **越阶的骄傲** | 筑基期就敢渡金丹劫——「劳资就是天选之人」 |
| **失败的坦然** | 渡劫失败后——「差一点，回去再攒攒，下次一定过」 |

渡劫是「苟道成长」支柱的最高体现：**你不需要一次就过**，失败了可以重来。重要的是你从每次失败中学到了什么——是构筑不够强？还是准备不够充分？下次再来就是了。

## 详细设计

### 核心规则

#### 1. 渡劫触发条件

修为达到当前境界的 max_cultivation（修为条满100%）后自动满足突破条件，玩家在探索地图时可通过**事件点**主动触发渡劫。

**触发方式：**

| 方式 | 条件 | 说明 |
|------|------|------|
| 地图渡劫点 | 修为已满 + 在地图上 | 探索地图中出现「渡劫台」事件格，移动到此格触发 |
| 自动提示跳转 | 修为已满 + 任意时刻 | 修为满值时弹出提示「修为已达瓶颈」，点击「前往渡劫」自动导航到渡劫台 |
| 越阶渡劫 | 修为已满 + 任意境界 | 不进入渡劫台，直接在当前地图触发高一级的天劫（见§6） |

**修为未满时不可触发渡劫**——修为条刻度和提示系统会在接近满值时提醒，但不满值不能开始。

#### 2. 渡劫准备阶段

触发渡劫后，进入**准备阶段**：

```
tribulation_preparation():
  ① 弹出确认窗口：「你即将面对天劫——此战无法撤退，准备好再开始」
     - [开始渡劫] / [再准备一下（返回探索）]
  ② 如果玩家选择「开始」：
     a. 允许调整上场角色（同战前备战阶段）
     b. 允许使用渡劫丹（从背包选择并立即生效）
     c. 锁定当前卡组（不可编辑卡组）
  ③ 确认 → 进入渡劫战斗
```

**渡劫丹**（特殊消耗品，在炼丹系统或商店中获得）：
- 使用后本场渡劫战中生效，一次渡劫可使用多枚
- 效果叠加，同种渡劫丹效果不叠加（取最高值）

| 渡劫丹 | 稀有度 | 效果 | 获取方式 |
|--------|:-----:|------|---------|
| 避雷符丹 | 蓝色 | 首回合获得「雷免」-前2次雷伤无效 | 商店/炼丹 |
| 凝神丹 | 蓝色 | 渡劫战中角色DEF+2 | 商店/炼丹 |
| 天劫护体丹 | 紫色 | 渡劫战中全场友方「雷伤」叠加速度减半（每2回合叠1层而非每回合） | 炼丹 |
| 破劫丹 | 金色 | 渡劫战中天劫Boss HP-20%（显示为「已削弱」） | 炼丹（需丹方） |

#### 3. 渡劫战斗规则

渡劫战继承标准战斗系统，但有独立规则覆盖：

**天劫Boss：雷灵**

| 境界 | 天劫Boss | Boss境界 | Boss特殊机制 |
|:----:|:--------:|:--------:|:-----------:|
| 炼气→筑基 | 炼气雷灵 | 炼气期 | 每回合末给全场玩家角色叠1层「雷伤」 |
| 筑基→金丹 | 筑基雷灵 | 筑基期 | 每回合末叠1层「雷伤」+ 每3回合一次范围攻击 |
| 金丹→元婴 | 金丹雷灵 | 金丹期 | 每回合末叠1层「雷伤」+ 每2回合一次范围攻击 + Boss有护盾 |
| 元婴→化神 | 元婴雷灵 | 元婴期 | 每回合末叠2层「雷伤」+ 每2回合范围攻击 + 护盾 + 召唤雷球小怪 |
| 化神→通关 | 化神雷灵 | 化神期 | （最终Boss战替代渡劫——不由本系统处理） |

> 注：化神期后的「通关」由最终Boss战处理，不是渡劫系统的一部分。

**雷伤Debuff机制：**

```
雷伤（每层）：
  每回合结束时，角色受到 层数 点伤害（不可减免）
  层数无上限，战斗中不会自动清除
  例：第5回合有5层雷伤 → 回合结束时受到5点伤害
```

- 雷伤是渡劫战特有的强制机制——它创造了「软时限」：拖得越久，雷伤层数越高，伤害越大
- 雷伤不可被驱散、不可减免（渡劫丹提供的「雷免」除外）
- 雷伤伤害在所有其他伤害结算后应用（与境界压制无关）
- 渡劫成功后雷伤消失（不跨战斗保留）

**渡劫战特殊规则：**

| 规则 | 内容 |
|------|------|
| **不可撤退** | 渡劫战中「撤退」按钮不可用——必须打完 |
| **胜败判定** | 标准规则：敌方全灭=胜，己方全灭=败 |
| **击败天劫Boss后战斗立即结束** | 不需要清空小怪——Boss阵亡即胜利 |
| **战斗中无法使用特定丹药** | 仅禁用品：修为丹（渡劫战中没有修为奖励）、回城符（不可离开） |
| **阵亡角色跨战斗规则不变** | 渡劫战中阵亡的角色战后标记为「不可用」，走标准死亡规则 |
| **无战利品** | 渡劫战胜利后不进入三选一战利品流程——奖励固定为突破本身 |

#### 4. 渡劫成功

天劫Boss被击败后：

```
tribulation_success():
  ① 战斗胜利动画/提示
  ② 调用 realm_up() 触发境界升级：
     - realm_level += 1
     - 新境界 max_cultivation 更新
     - cultivation 保留（如800/1000→800/1500）
     - 修为溢出系统结算属性丹（如有溢出池）
     - 所有境界绑定数值更新（费用、上场人数、卡组上限等）
     - 行动力回满
  ③ 发放固定奖励：
     - 金卡×1（从新境界卡池随机选取）
     - （如果是越阶渡劫→额外金卡×1）
  ④ 解锁新地图入口
  ⑤ 广播 realm_upgraded 信号
  ⑥ UI展示突破动画 + 新境界信息
```

**固定金卡奖励：**
- 渡劫成功后必定获得1张金色稀有度卡牌
- 该金卡从新境界的卡牌池中选取（如渡劫到金丹→从天南卡池的金卡中随机）
- 越阶渡劫额外+1张金卡（共2张）
- 金卡不经过稀有度权重判定——100%出金
- 金卡奖励直接发放到卡组（不受40张上限约束——如超限触发标准弃牌流程）

#### 5. 渡劫失败

玩家在渡劫战中被天劫击败：

```
tribulation_failure():
  ① 战斗失败动画/提示
  ② 修为损失：
     - current_cultivation -= floor(max_cultivation × 0.1)
     - 最少损失50修为（防止数值太小无感）
     - 修为不会降至0以下（跌破0时取0）
     - 溢出池不受影响
  ③ 不重置任何其他状态：
     - 卡组不变
     - 角色状态按标准战斗结算（阵亡角色标记不可用）
     - 卡牌不掉落、不销毁
  ④ 回到探索地图
  ⑤ 显示提示：「渡劫失败，修为损失10%。调整构筑后再试一次」
```

**失败连续保护：**
- 连续2次渡劫失败后，第3次触发渡劫时自动获得「天劫庇护」buff：全队HP+20%（仅本次渡劫有效）
- 此保护机制是为了防止「卡死在某个境界无法推进」——Roguelike不应出现死锁
- 连续失败计数器在成功渡劫后重置

#### 6. 越阶渡劫

玩家可在**当前修为已满**时，选择触发下一大境界的天劫（如炼气期满修为→直接渡筑基劫）：

```
cross_realm_tribulation():
  条件：
    - 当前境界修为已满
    - 下一个大境界的天劫Boss已配置
    - 玩家主动选择「越阶渡劫」选项（非默认）
  
  流程：
    - 天劫Boss = 当前境界+1的雷灵（如炼气期的越阶渡劫Boss = 筑基雷灵）
    - Boss境界 = 玩家当前境界+1 → 触发境界压制（越阶战受压制惩罚）
    - 渡劫成功 → 正常突破到玩家当前境界+1
      （如炼气→筑基后→再正常突破到金丹——不跳过中间境界）
    - 额外奖励：额外获得1张金卡（越阶奖励）+ 成就「越阶突破」标记
  
  注意：
    - 越阶渡劫不能跳阶——炼气期最多渡筑基劫，不能跳过筑基直接渡金丹劫
    - 越阶渡劫后玩家境界正常升1级，不是「跨级」而是「在当前级提前触发下一级」
    - 越阶渡劫成功后，玩家境界从L变为L+1，不是L+2
```

> **设计意图**：越阶渡劫不是「速通」手段，而是「提前挑战更强Boss以获得额外奖励」的高风险高回报选项。天劫Boss高1级意味着受压制惩罚（伤害×0.8），让战斗更具挑战性。额外金卡是给敢于冒险的玩家的奖励。

#### 7. 渡劫的整局次数

一局游戏中，玩家最多可完成4次渡劫（炼气→筑基→金丹→元婴→化神）：
- 每次渡劫对应一个大境界突破
- 化神期→通关由最终Boss战处理（不是渡劫系统范围）
- 无「小境界突破」设计（每个大境界内没有小境界划分）

### 状态与转换

```
渡劫生命周期（每局×4次）：

[[探索地图]]
    ↓ 修为满 + 到达渡劫台
[准备阶段] ← 可选使用渡劫丹 + 调整上场角色
    ↓ 确认开始
[渡劫战斗中]
    ├── 胜利 → [渡劫成功] → 境界升级 + 金卡奖励 → 进入新地图
    └── 失败 → [渡劫失败] → 修为-10% → 回到探索地图
              ↑ 可重试（修为再次攒满后）
```

**渡劫状态机：**

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **未就绪** | 修为未满，不可渡劫 | 突破后/初始 | 修为达标→可渡劫 |
| **可渡劫** | 修为已满，可触发 | 修为满值提示 | 触发渡劫→准备阶段 |
| **准备中** | 渡劫准备阶段 | 触发渡劫事件 | 确认→渡劫战斗；取消→返回可渡劫 |
| **渡劫战中** | 特殊Boss战斗进行中 | 确认开始 | 胜利/失败判定 |
| **突破成功** | 境界提升完成 | 击败天劫Boss | 动画播放后→新境界地图 |
| **渡劫失败** | 被天劫击败 | 己方全灭 | 修为结算后→回到探索（修为不满） |

## 公式

### 1. 渡劫失败修为损失

```
tribulation_failure_penalty(current, max_cult) → int:
  loss = floor(max_cult × 0.1)
  loss = max(loss, 50)
  return max(current - loss, 0)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| current | int | [0, max_cult] | 当前修为值 |
| max_cult | int | [1000, 5063] | 修为上限 |
| loss | int | [50, 507] | 实际损失（最少50） |

**示例：** 筑基期 max_cult=1500，当前1200 → loss=max(150,50)=150 → 剩余1050

### 2. 雷伤伤害

```
lightning_damage(turn, layers_per_turn=1) → int:
  layers = turn × layers_per_turn
  return layers
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| turn | int | [1, max] | 当前回合数 |
| layers_per_turn | int | 1 或 2 | 每回合叠层数 |
| 返回值 | int | [1, max] | 本回合雷伤 |

### 3. 连续失败保护

```
auto_protection(consecutive_failures) → bool:
  return consecutive_failures >= 2
```

连续2次失败后第3次自动获得「天劫庇护」：全队HP+20%（仅本次渡劫）。

## 边界情况

- **渡劫前卡组不满40张但特定卡牌在别的角色手上**：备战阶段允许调整上场角色，但不能编辑卡组——你在渡劫前不能临时往卡组加卡，但可以调整谁上场
- **渡劫成功后超限金卡导致卡组>40张**：金卡直接发放→超限→触发标准弃牌流程。金卡不会因为超限消失
- **渡劫失败时修为已低于max_cultivation × 0.1**：修为不会降至0以下，最少50修为的损失下限兜底
- **渡劫战中阵亡的角色对后续影响**：标准规则——阵亡角色战斗后标记「不可用」。这意味着渡劫失败不仅损失修为，还可能损失角色。这是设计意图：渡劫有风险，角色阵亡的代价须考虑
- **连续失败后已经使用过渡劫丹**：渡劫丹是消耗品——每次渡劫消耗已使用的丹药。连续失败可以重新获取渡劫丹（无需重置，但需要额外的资源）
- **玩家在渡劫准备阶段不调整任何配置直接开始**：允许——渡劫准备的默认配置是当前上场角色，直接开始即可
- **修为正好满时触发渡劫与溢出转化顺序**：修为满后先触发「可渡劫」状态，玩家选择渡劫或继续探索。如果继续探索获得更多修为→走溢出转化；如果去渡劫→渡劫成功后结算溢出
- **越阶渡劫时境界压制叠加天劫Boss难度**：越阶渡劫的Boss比玩家高1级→玩家伤害×0.8（压制）。这意味着越阶渡劫的Boss既有更高的基础属性又有压制惩罚——确实是高难度挑战
- **渡劫丹的「雷免」效果与雷伤结算顺序**：雷免优先级高于雷伤——「前2次雷伤无效」意味着前2次雷伤结算时伤害=0
- **连续失败保护在成功渡劫后重置**：重置为0——不会跨境界继承。但这个保护不存档（因为是防止死锁的辅助机制，不应成为依赖项）

## 验收标准

- **GIVEN** 修为未满，**WHEN** 尝试触发渡劫，**THEN** 不可触发
- **GIVEN** 修为已满，**WHEN** 在地图渡劫台事件格，**THEN** 可触发渡劫
- **GIVEN** 触发渡劫进入准备阶段，**WHEN** 选择「再准备一下」，**THEN** 返回探索，渡劫状态保持可触发
- **GIVEN** 渡劫准备阶段，**WHEN** 选择渡劫丹并开始，**THEN** 渡劫丹效果在战斗中生效
- **GIVEN** 渡劫战开始，**WHEN** 检查战斗规则，**THEN** 不可撤退、无战利品
- **GIVEN** 渡劫战第3回合结束，**WHEN** 结算雷伤，**THEN** 每个上场角色受到3点雷伤（3层×1伤害/层）
- **GIVEN** 渡劫战第5回合结束且为元婴劫，**WHEN** 结算雷伤，**THEN** 受到10点雷伤（5回合×2层/回合）
- **GIVEN** 渡劫战中击败天劫Boss，**WHEN** 结算，**THEN** 战斗立即结束（不清小怪也判胜）
- **GIVEN** 渡劫成功，**WHEN** 检查境界，**THEN** realm_level+=1，cultivation保留，max_cultivation更新
- **GIVEN** 渡劫成功，**WHEN** 检查奖励，**THEN** 获得1张金卡（从新境界卡池）
- **GIVEN** 越阶渡劫成功，**WHEN** 检查奖励，**THEN** 获得2张金卡
- **GIVEN** 筑基期修为1200/1500，**WHEN** 渡劫失败，**THEN** 修为变为1050/1500（-150=10%×1500）
- **GIVEN** 渡劫失败，**WHEN** 检查角色状态，**THEN** 阵亡角色标记为不可用
- **GIVEN** 连续2次渡劫失败，**WHEN** 第3次开始渡劫，**THEN** 自动获得「天劫庇护」buff
- **GIVEN** 天劫庇护激活后渡劫成功，**WHEN** 检查连续失败计数器，**THEN** 重置为0
- **GIVEN** 渡劫成功，**WHEN** 检查行动力，**THEN** 行动力回满
- **GIVEN** 渡劫成功，**WHEN** 检查地图，**THEN** 新地图解锁
