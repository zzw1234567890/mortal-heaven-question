## 概述

主菜单是游戏启动时首先展示的界面，提供新游戏、继续游戏、设置、退出等功能入口。设置界面管理游戏的所有可配置选项（音量、画面、按键绑定、语言等）。主菜单和设置是玩家对游戏的「第一印象」和「随时调整的后台」。

## 详细设计

### 核心规则

#### 1. 主菜单界面

```
┌──────────────────────────────────────────┐
│                                          │
│         🜁 仙途问道            │
│                                          │
│        ┌──────────────────┐              │
│        │   [新游戏]        │              │
│        │   [继续游戏]      │ ← 有存档时亮起 │
│        │   [设置]          │              │
│        │   [制作人员]      │              │
│        │   [退出]          │              │
│        └──────────────────┘              │
│                                          │
│  版本 v0.1.0                   Steam      │
└──────────────────────────────────────────┘
```

- 新游戏：进入身份选择界面
- 继续游戏：读取最近的存档（无存档时灰色不可用）
- 设置：打开设置界面
- 制作人员：显示开发团队信息
- 退出：退出游戏（Steam平台）
- 背景为动态场景（如修仙世界云海/山脉缓缓移动）

### 状态与转换

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **主菜单** | 游戏启动界面 | 启动/退出到主菜单 | 选择新游戏/设置/退出 |
| **身份选择** | 新游戏开局 | 点击新游戏 | 确认身份→游戏开始 |
| **设置** | 设置界面 | 点击设置 | 关闭设置→返回 |
| **按键绑定** | 按键重映射 | 设置中点击按键绑定 | 返回→设置 |
| **制作人员** | 制作人员列表 | 主菜单点击 | 返回→主菜单 |

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **游戏状态管理器** | 新游戏/读档指令 | 存档数据（继续游戏检测是否有存档） |
| **存档系统** | 读档请求（点击继续游戏） | 存档是否存在 |
| **身份选择系统** | 新游戏→进入身份选择 | — |

## 验收标准

- **GIVEN** 游戏启动，**WHEN** 加载完成，**THEN** 显示主菜单界面
- **GIVEN** 没有存档，**WHEN** 查看继续游戏，**THEN** 按钮灰色不可用
- **GIVEN** 有存档，**WHEN** 点击继续游戏，**THEN** 加载最近存档进入游戏
- **GIVEN** 主菜单，**WHEN** 点击新游戏，**THEN** 进入身份选择界面
- **GIVEN** 主菜单，**WHEN** 点击设置，**THEN** 打开设置界面
- **GIVEN** 设置中修改音量，**WHEN** 调整滑动条，**THEN** 音量实时变化
- **GIVEN** 设置中修改分辨率，**WHEN** 应用，**THEN** 画面分辨率改变
- **GIVEN** 按键绑定界面，**WHEN** 点击一个按键+按下新键，**THEN** 绑定更新
- **GIVEN** 按键绑定时新按键已使用，**WHEN** 确认，**THEN** 提示冲突+覆盖选项
