## 概述

战斗UI系统负责战斗场景中所有面向玩家的视觉交互界面——包括手牌区、场上角色区、费用栏、阵法区、敌方信息区、回合阶段指示器、攻击目标选择以及战斗结算面板。战斗UI系统不执行战斗逻辑（那是战斗系统的职责），而是将战斗系统的状态数据转化为玩家可理解、可操作的视觉界面，并将玩家的操作（出牌、选目标）转化为战斗系统的指令。

对玩家来说，战斗UI就是他们与游戏核心交互的直接界面。每一局战斗的所有操作——从出牌到攻击、从布阵到撤退——都通过战斗UI完成。UI的响应速度、信息清晰度和操作流畅度直接决定了玩家对「策略为王」支柱的感知质量。

> **⚠ 重大设计同步（2026-07-24）**：战斗系统于 2026-07-23 更新为 **7 阶段模型（阶段 0-6）**，移除了独立的"布阵阶段"（阵位调整现为备战阶段一次性操作）。本 GDD 已全面同步至 7 阶段模型。详见 `combat-system.md` 第 114-126 行。

## 玩家幻想

战斗UI的核心情感是**清晰的掌控感**——玩家在战斗中每时每刻都能清楚地知道：我有多少费用、我的手牌有什么、我的角色状态如何、敌人下一步可能做什么。UI不应让玩家「找信息」，而应让信息主动呈现给玩家。

| 情感 | 触发场景 |
|------|---------|
| **掌控的清晰** | 一眼看到费用、手牌、场上状态——「我知道现在是什么局面」 |
| **操作的流畅** | 拖拽出牌到目标的无缝体验——「指哪打哪」 |
| **反馈的及时** | 打出卡牌后的即时效果展示——「我知道这张牌生效了」 |
| **战局的透明** | 看到敌方状态和buff/debuff——「我知道敌人在干什么」 |
| **收获的满足** | 战斗胜利时亲手选择战利品——「这仗打得值」 |

战斗UI的核心原则是：**不干扰决策，只辅助决策**。UI应让玩家的注意力集中在策略思考上，而非解读界面元素上。

## 详细设计

### 核心规则

#### 1. 战斗HUD整体布局

```
┌──────────────────────────────────────────────────────┐
│ [回合阶段]                    [阵法区]    [撤退]     │
│                                                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ 敌方前1   │  │ 敌方前2   │  │ 敌方前3   │ ←敌方侧  │
│  └──────────┘  └──────────┘  └──────────┘          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ 敌方后1   │  │ 敌方后2   │  │ 敌方后3   │          │
│  └──────────┘  └──────────┘  └──────────┘          │
│                                                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ 己方前1   │  │ 己方前2   │  │ 己方前3   │ ←己方侧  │
│  └──────────┘  └──────────┘  └──────────┘          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ 己方后1   │  │ 己方后2   │  │ 己方后3   │          │
│  └──────────┘  └──────────┘  └──────────┘          │
│                                                      │
│  ┌──────────────────────────────┐                    │
│  │ [费用栏] 5/5                 │                    │
│  ├──────────────────────────────┤                    │
│  │ [手牌区] 弧形排列7张卡牌      │ ←底部              │
│  └──────────────────────────────┘                    │
└──────────────────────────────────────────────────────┘
```

屏幕从上到下依次为：回合阶段信息区 + 阵法区 → 敌方角色区 → 己方角色区 → 费用栏 → 手牌区

> **📌 待UX设计验证**：当前布局以空间位置为优先（敌方→己方→费用→手牌），而玩家的决策流程是"费用→手牌→场上→敌方"。这两种层级存在结构性矛盾——是否应将费用栏左对齐到阶段指示器下方形成"状态信息列"，需在 `/ux-design` 中验证并解决。

#### 2. 手牌区

- 位于屏幕底部，以**弧形**排列当前手牌
- 手牌区域高度约屏幕的25%，宽度100%
- 每张卡牌以**缩小版正面**展示（卡名、费用图标（左上角）、稀有度颜色边框、类型标签）
- 手牌超过7张时堆叠排列（部分重叠），玩家可左右滑动/滚轮浏览
- 费用不足的卡牌**灰显**（降低饱和度+费用图标变灰），不可拖拽
- 可拖动的卡牌在鼠标悬停时放大120%，显示卡牌详情（效果全文）
- 拖拽出牌：鼠标按住卡牌向目标区域拖拽释放，拖拽过程中卡牌跟随鼠标

**手牌区状态：**

| 状态 | 视觉表现 | 说明 |
|------|---------|------|
| **默认** | 弧形排列，卡牌彩色，费用正常显示 | 所有手牌均可查看 |
| **灵能预览**（阶段2外） | 卡牌降低饱和度至60%，叠加🔒小锁标记，**不可拖拽但可悬停查看详情** | 玩家可在非出牌阶段规划下回合操作 |
| **费用不足** | 卡牌灰显，费用数字红色 | 费用<卡牌费用，不可出牌 |
| **目标选择中** | 可用的卡牌高亮（金色边框），不可用的深灰 | 已选中一张卡牌，正在选目标 |
| **放大预览** | 鼠标悬停的卡牌放大+详细信息弹出 | 全阶段可用（即使在阶段2外） |
| **拖拽中** | 卡牌跟随鼠标，半透明效果 | — |

**设计原理**：阶段2外的手牌保留60%饱和度+悬停预览可用（"灵能预览"模式），而非完全灰显。这遵循"不干扰决策，只辅助决策"的原则——玩家在攻击声明/敌方行动阶段可以预览卡牌效果来规划下回合，减少"等待无事可做"的挫败感。

#### 3. 场上角色区

己方和敌方角色各占屏幕的一部分：

**角色位展示信息：**
```
┌────────────────────────┐
│ [角色头像]  [境界标记]  │
│ [角色名称]  [HP条]     │ ← HP条显示当前/最大HP
│ [状态图标区]            │ ← buff/debuff图标排列
│ [绑定功法图标] [绑定法宝] │ ← 已绑定的卡牌小图标
└────────────────────────┘
```

- 角色头像：圆形裁切，背景色代表阵营/稀有度
- HP条：绿色→黄色→红色（HP<30%时），数字显示当前/最大HP
- 境界标记：仅高于玩家境界时显示↑标记
- 状态图标区：buff 图标在左（绿色边框），debuff 图标在右（红色边框）
- 绑定功法/法宝：显示绑定卡牌的小型图标（空位显示灰色槽位）
- 待命角色：角色头像外圈显示沙漏标记，HP条上方显示「待命」文字
- 已行动角色：角色头像外圈显示灰色遮罩，「已行动」标记
- 阵亡角色：角色位显示骷髅+消散动画，移除后变为空位（虚线框）

**前后排视觉区分：**
- 前排角色位边框为实线（金色/黄色），后排为虚线（蓝色）
- 前后排之间有明显间距分隔线
- 空位以浅色虚线框表示

#### 4. 费用栏

位于手牌区上方、角色区下方：

```
┌─────────────────────────────────────┐
│  ⚡ 5 / 5     [+2临时]   ≡牌库:12  │
└─────────────────────────────────────┘
```

- 费用图标（闪电/灵力标志）+ **实际可用费用 / 基础上限**（含临时费用）
- 费用颜色基于 `实际可用 / (基础上限+临时费用)` 比率：
  - 比率 > 0.5 → 白色（充裕）
  - 比率 > 0.0 → 黄色（紧张）
  - 比率 = 0.0 → 红色（枯竭）
- 临时费用（丹药增加）显示为额外绿色+数字
- 右侧显示牌库剩余张数（点击可展开牌库详情）
- 弃牌堆显示在旁边（灰色+数字），点击可展开弃牌堆详情

> **设计决策**：费用颜色基于"实际可用费用"而非"基础费用"。若玩家基础费用 0/5 但有 +3 临时（共3可用），颜色应为黄色而非红色——玩家仍有3费可用，红色会误导。

#### 5. 阵法区

位于屏幕顶部中间：

```
┌──────┐ ┌──────┐ ┌──────┐
│阵·苍  │ │阵·魔  │ │ 空   │  ← 最多3个
│玄盟阵 │ │域血阵 │ │      │
└──────┘ └──────┘ └──────┘
```

- 已部署阵法以卡片形式展示（阵法名+阵营标签）
- 空阵法位以灰色虚线框显示
- 阵法激活效果（光环）以阵法卡片外发光表示（颜色对应阵营）
- 鼠标悬停阵法卡→显示阵法效果详情

#### 6. 回合阶段指示器

位于屏幕左上角：

```
阶段 2/7 · 出牌阶段
████████░░░░░░░ 60%
```

- 显示当前阶段名称（7阶段模型：准备/抽牌/出牌/攻击声明/攻击结算/敌方行动/结束）
- 阶段进度条（当前阶段剩余时间或简单进度指示）
- 回合编号显示（第X回合）
- 阶段切换时：阶段名短暂放大动画+轻微音效

**7阶段快速参考：**

| 编号 | 阶段名称 | 玩家交互 |
|:----:|---------|---------|
| 0 | 准备 | 不可操作（仅观看） |
| 1 | 抽牌 | 不可操作（仅观看） |
| 2 | 出牌 | 手牌可拖拽、费用可见 |
| 3 | 攻击声明 | 角色可选目标（点击式或拖拽式，见§7） |
| 4 | 攻击结算 | 不可操作（仅观看） |
| 5 | 敌方行动 | 不可操作（仅观看，手牌灵能预览可用） |
| 6 | 结束 | 不可操作，回合结束效果触发 |

#### 7. 攻击目标选择

阶段3（攻击声明）的交互流程：

**模式A — 点击式（默认，推荐首次MVP）：**
```
① 当前可攻击的己方角色头像外圈显示蓝色脉冲光效
② 玩家点击一个己方角色（或已预设了攻击目标）
③ 该角色可攻击的目标（敌方角色）高亮：红色边框+脉冲
④ 不可攻击的目标（前排保护下的后排）灰色 + 🔒标记
⑤ 点击一个敌方目标 → 确认攻击声明
⑥ 所有己方角色都声明完攻击目标后 → 自动进入阶段4
⑦ 玩家也可跳过某个角色的攻击（角色上显示「跳过」按钮）
```

**模式B — 拖拽式（备选，原型阶段对比测试）：**
```
① 玩家从己方角色按下鼠标 → 拖出瞄准线 → 有效目标高亮红色 → 释放在敌方目标上确认
② 不可攻击目标灰色+🔒标记，释放在无效目标上不触发
③ 已声明的目标显示角色→目标的瞄准箭头连线
```

**当前设计为点击式**（模式A），原因为精确性——点击避免误触。拖拽式与出牌操作一致但存在误触风险。最终选择待原型阶段 A/B 测试数据决定。

**已声明的目标显示：**
- 角色→目标之间有箭头连线
- 阶段切换进入阶段3时，模式指示器显示"点击选择攻击目标"提示文字（简洁，动画出现后淡出）
- 如果原型测试后选择拖拽式，提示文字同步更新

- 一回合内多个角色的攻击声明可任意顺序进行
- 已经声明目标的角色显示目标箭头指向
- 玩家可在阶段3结束前修改已声明的目标（点击角色→清除已有目标→重新选择）
- 阶段4（攻击结算）自动播放，不可交互

**备战界面规则：**
- 角色卡以缩略卡片形式展示（角色头像、HP、ATK、绑定概览）
- 已选角色打✓标记，绑定卡图标缩略显示在角色卡右下角
- **角色选择状态（4种）：**
  | 状态 | 视觉标记 | 交互 |
  |------|---------|------|
  | **已选** | ✓ + 金色高亮边框 | 点击可取消选择 |
  | **可选未选** | ☐ + 正常颜色 | 点击选中（若名额未满） |
  | **可替换** | ⊕ 图标 + 正常颜色 | 名额已满时点击→弹出"用[角色名]替换[已选角色名]？" |
  | **不可用（阵亡）** | 灰色+💀骷髅标记 | 不可选择，悬停提示阵亡状态 |
- 阵亡角色不可选择，鼠标悬停提示阵亡状态
- 已选人数不满上限时显示提示「还可以选择X人」
- 已选满4人时未选角色显示⊕标记，点击弹出替换确认
- 阵位布局预览实时更新（拖拽角色到头像区自动排位，或直接在预览区拖拽调整前后排）
- 角色悬停时显示"此角色与已选阵容的协同信息"（绑定关联、阵营加成）
- 确认按钮在选择了至少1个可用角色后可点击

#### 9. 战斗结算面板

**胜利面板：**
```
┌──────────────────────────┐
│         🎉 胜利！          │
│                           │
│  战利品选择（3选1）：      │
│  ┌────────┐ ┌────────┐   │
│  │  青云剑诀 │ │  灵石   │   │
│  │  (蓝·功法) │ │  ×25   │   │
│  │  [已选中]  │ │         │   │
│  └────────┘ └────────┘   │
│  ┌────────┐              │
│  │  回血丹  │              │
│  │  (消耗品)│              │
│  └────────┘              │
│                           │
│  [确认选择]  ← 点击战利品高亮选中，点击确认生效  │
└──────────────────────────┘
```

**二次确认规则：** 点击战利品只高亮选中（不立即确认），需再点击[确认选择]按钮。防止胜利情绪高峰期的误触。战利品展示后 0.5s 延迟才变为可点击状态（"先看清，再选择"）。

**失败面板：**
```
┌──────────────────────────┐
│         💀 战败            │
│                           │
│  损失50%本次战斗修为       │
│  阵亡角色：林渊（不可用）   │
│                           │
│  [返回探索]                │
└──────────────────────────┘
```

- 胜利时弹出三选一战利品面板（标准卡牌正面展示 / 灵石图标+数值 / 消耗品）
- 战败时弹出损失摘要（修为损失、阵亡角色列表、绑定卡失去提示）
- 结算面板在展示完成后需玩家点击确认才关闭（不自动关闭）

#### 10. 撤退确认

```
┌──────────────────────────┐
│  ⚠ 确认撤退？             │
│                           │
│  撤退视为战败处理：         │
│  · 保留已获得的非绑定物品   │
│  · 已阵亡角色的绑定卡将失去 │
│                           │
│  [确认撤退]  [继续战斗]    │
└──────────────────────────┘
```

- 撤退按钮始终在战斗中可点击（位于右上角，符合杀戮尖塔/暗黑地牢等卡牌游戏习惯）
- 点击后弹出确认窗口

**渡劫战撤退（2026-07-24 修订）：**
- 渡劫战允许撤退，但代价显著放大——符合"苟道成长，步步为营"支柱：
  - 修为损失：80%（而非普通战败的50%）
  - 失去本次渡劫机会（需重新积累修为再次尝试）
  - 已阵亡角色的绑定卡仍永久失去
- 渡劫撤退确认面板使用特殊警示UI（红色边框+特殊警告文字"畏劫而退，修为散尽"）
- **进入渡劫战前弹出战前警告**："天劫在前，退路将断。确认进战？"——给予玩家最后一次取舍抉择

> **设计原理**：原设计"天劫不可退"与支柱P2"苟道成长"矛盾。苟道的核心是"每一步都精打细算，每一次都能活着回来"——禁止撤退剥夺了玩家的取舍代理权。改为高代价撤退保留了"天劫威严"（通过严重的修为/机会成本），同时保留了"苟道"的代理权。

#### 11. 战斗日志

- 可收起的滚动文本日志（默认折叠为小标签）
- 点击展开为半屏覆盖，覆盖在手牌区上方
- **阶段2（出牌）进入时自动收起**——保证玩家出牌时不被日志遮挡

- 日志条目格式：`[回合·阶段] 行动者 → 动作 → 目标 → 数值`（显示具体数值，如"造成5点伤害"）
- 日志保留整场战斗的所有记录（退出战斗后清空）

## 公式

战斗UI系统不执行计算逻辑——所有数值计算由战斗系统和相关系统完成。UI只负责展示战斗系统提供的数据。

### 1. 手牌排列弧度

```
hand_card_position(index, total_cards, screen_width) → (x, y, rotation):
  if total_cards == 0: return []  # 守卫：空手牌不计算位置
  arc_angle = min(30°, total_cards × 3°)  # 最多30°弧度
  base_y = screen_bottom × 0.86  # 使用百分比而非固定像素，支持响应式
  spacing = min(card_width + 10px, screen_width / total_cards)
  x = screen_center + (index - (total_cards - 1) / 2) × spacing
  y = base_y - abs(x - screen_center) × tan(arc_angle / 2)
  rotation = (index - (total_cards - 1) / 2) × (arc_angle / total_cards)
  return (x, y, rotation)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| index | int | [0, total_cards-1] | 卡牌在手牌中的索引 |
| total_cards | int | [0, 10] | 当前手牌数量（0时早返回空数组） |
| 返回值 | (float, float, float) | — | 卡牌位置(x, y)和旋转角度 |

**实现建议：** 使用 Godot 的 `Control` 节点（非 HBoxContainer）作为手牌父容器，配合自定义脚本计算弧形位置。通过 `position` 和 `rotation` 设置子节点位置，而非依赖 Container 自动布局。`base_y` 使用屏幕高度的百分比（86%）以支持响应式分辨率。

### 2. 手牌堆叠重叠量（>7张时）

```
card_overlap_offset(total_cards, card_width) → float:
  if total_cards <= 7: return card_width + 10  # 无重叠，标准间距
  excess = total_cards - 7
  overlap = (card_width × 0.3) × excess  # 每多一张重叠30%卡宽
  return card_width - overlap / total_cards
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| total_cards | int | [8, 10] | 堆叠触发阈值以上 |
| overlap | float | — | 相邻卡牌重叠的像素量 |

**Z层序：** 中心卡牌优先（index 接近中心者 z_index 最高），防止堆叠时左/右边缘卡遮挡中心卡。

### 3. HP条颜色阈值

```
hp_bar_color(hp_ratio) → Color:
  if hp_ratio > 0.6: return GREEN
  if hp_ratio > 0.3: return YELLOW
  return RED
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| hp_ratio | float | [0.0, 1.0] | 当前HP / 最大HP |

**阈值理由**：60% 以上为安全区（大多数一击无法斩杀），30% 以下为一击必杀危险区。HP条动画使用 `Tween.create_tween()` 平滑过渡 0.3s。批量伤害时使用排队机制——同一目标连续受伤时，直接计算最终HP值并执行一次0.3s过渡，避免"HP条先降一半再降一半"的视觉抖动。若性能分析显示12+同时Tween掉帧，改用共享 lerp 管理器（`_physics_process()` 中批量插值）。

### 4. 费用显示颜色

```
cost_color(available, base_max, temporary) → Color:
  total_max = base_max + temporary
  ratio = available / total_max
  if ratio > 0.5: return WHITE
  if ratio > 0.0: return YELLOW
  return RED
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| available | int | [0, base_max+temporary] | 实际可用费用 |
| base_max | int | ≥0 | 基础费用上限 |
| temporary | int | ≥0 | 临时费用加值 |

**与HP颜色阈值的差异**：费用使用50%阈值（而非HP的60%），因为费用是消费能力而非生存指标——低于50%时大多数高费卡牌不可用，决策空间明显收缩。

### 5. 前后排视觉缩放

```
formation_row_scale(is_front_row) → float:
  return 1.0 if is_front_row else 0.85
```

后排角色头像整体缩放至85%，提供透视深度暗示。边框样式独立于缩放（前排实线金色、后排虚线蓝色）。

### 6. 低分辨率文字缩放

```
font_size_responsive(base_font_size, screen_height) → int:
  if screen_height >= 1080: return base_font_size
  if screen_height >= 720: return max(base_font_size × 0.85, 12)  # 中文最小可读字号12pt
  return 12  # 保底
```

### 7. 悬停预览延迟

```
hover_preview_delay_ms = 300  # 统一调优参数，所有悬停目标共用
```

所有悬停目标（手牌、角色、阵法、状态图标、费用栏）使用统一的 300ms 延迟。此参数在调优参数表中可调。

## 边界情况

- **手牌超过屏幕宽度容纳**：手牌堆叠显示（每张卡牌部分重叠），玩家可拖拽滑动查看；最左侧和最右侧的卡牌露出手牌边缘指示可滚动
- **手牌为0**：手牌区显示「无手牌」提示文字，费用栏正常显示，不可出牌
- **手牌满10张抽牌**：第11张卡牌直接进入弃牌堆（非手牌溢出）——UI展示"牌库→弃牌堆"飞入动画，提示文字"手牌已满，卡牌进入弃牌堆"短暂显示（2s后淡出）
- **场上角色全灭（战斗结束）**：立刻切入战败结算面板，不展示阵亡动画逐个播放
- **费用为0但手牌有0费卡**：0费卡正常彩色可出牌；费用>0的卡灰显
- **长时间悬浮卡牌**：悬浮卡牌预览保持打开直到鼠标移出卡牌区域，不自动关闭
- **同时声明多个角色攻击目标**：玩家逐一声明（点击角色→点击目标），已完成声明的角色有标记提示
- **攻击结算中玩家退出到桌面**：结算中断→下次读档回到战斗开始（战斗进度不保存）
- **宽屏/超宽屏适配**：手牌区自适应宽度，角色区居中不拉伸；两侧以游戏背景填充（非纯黑边），使用 `anchor_*` 预设实现百分比布局
- **低分辨率（1280×720）**：文字按 `font_size_responsive()` 公式缩小至85%，最小12pt中文。角色HP条和卡牌详情使用响应式布局适配。低于1280×720的分辨率不予支持
- **战斗中点击角色详情和手牌同时交互**：悬停面板互斥策略——同一时刻仅一个面板展开。手牌放大预览区域设置 `mouse_filter = MOUSE_FILTER_IGNORE` 防止二次触发角色悬停
- **敌方角色过多超出屏幕**：敌方角色位自动缩小比例（最多6个角色，设计上可控）
- **临时费用用完消失**：费用栏中临时费用部分动画消失（绿色数字淡出），基础费用不变
- **结算面板中卡牌三选一与灵石选项**：均匀网格排列（`(panel_width - margins) / 3` 每格等宽），不同宽高比的物品垂直居中，等宽约束
- **拖拽中阶段切换**：玩家在阶段2拖拽卡牌时阶段计时到期→拖拽释放但卡牌不入场（无效释放），卡牌飞回手牌原位置（动画0.2s），阶段正常切换
- **渡劫战撤退**：允许撤退，但弹窗显示特殊警示（红色边框 + "畏劫而退，修为散尽 —— 损失 80% 修为 + 失去本次渡劫机会"）。进入渡劫战前弹出"天劫在前，退路将断。确认进战？"确认

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 手牌弧形最大角度 | 30° | 20°~45° | 弧形扁平缺乏空间层次 | 太弯曲遮挡视野 |
| 手牌间距 | 卡宽+10px | 0~+30px | 太挤无法分辨多张卡 | 太宽超出屏幕 |
| 手牌悬停放大倍率 | ×1.2 | ×1.0~×1.5 | 放大不够看不清 | 放大太大挡视野 |
| 拖拽启动阈值（像素） | 8px | 4~16px | 轻轻碰到就触发拖拽（误触） | 拖拽感觉黏滞不灵敏 |
| 拖拽中卡牌透明度 | 0.7 | 0.5~0.9 | 太透明看不清 | 与正常卡牌区分不够 |
| HP条颜色阈值（绿→黄） | 60% | 40%~80% | 绿色区过大弱化危机感 | 黄色区过大过度警示 |
| HP条颜色阈值（黄→红） | 30% | 15%~50% | 红色区过小危机感不足 | 红色区过大麻木 |
| HP条动画时长 | 0.3s | 0.2s~0.5s | 过渡太快无反馈感 | 拖慢战斗节奏 |
| 悬停预览延迟 | 300ms | 150ms~500ms | 过早弹出造成干扰 | 等待太久感觉迟钝 |
| 战斗日志保留战斗 | 整场 | 最后50条~整场 | 回溯历史不足 | 内存占用（可优化） |
| 结算面板自动关闭延迟 | 无穷（需手动点） | 3~5s自动 | 手动操作多一步 | 玩家没看清就关了 |
| 备战选择动画时长 | 0.3s/角色 | 0.1s~0.5s | 太快没有确认感 | 太慢不耐烦 |
| 攻击结算动画最短时长 | 0.5s | 0.3s~1.0s | 太快看不清伤害 | 太慢拖节奏 |
| 灵能预览饱和度 | 60% | 30%~80% | 与完全灰显区分不够 | 与可用卡牌太接近 |
| 卡牌重叠比例（>7张） | 30% | 20%~50% | 堆叠太浅溢出屏幕 | 堆叠太深边卡无法辨认 |

> **调优参数说明变更（2026-07-24）**："卡牌拖拽灵敏度"已替换为更具体的 `拖拽启动阈值（像素）` 和 `拖拽中卡牌透明度`。原"0.8（归一化）"定义模糊不可调优。

### Draw Call 预算（Godot 4.6）

当前设计预估 draw items：
- 16 角色位 × ~10 items/位 = ~160 items
- 7 手牌 × ~7 items/张 = ~50 items
- 阵法区 + 费用栏 + 指示器 + 日志 ≈ 25 items
- **合计 ≈ 235 items**

超出 200 draw call 预算。缓解策略：
- 使用单一 TextureAtlas 合并所有状态图标、阵营指示器、卡牌边框
- HP 条改用 `draw_rect()` 纯色绘制（非纹理 ProgressBar）
- 考虑 `SubViewport` 渲染战斗日志（独立合成）
- 最终绘制的 draw call 数量以 Godot 内置调试器实际测量为准

### D3D12 渲染器风险（Godot 4.6）

Godot 4.6 将 Windows 默认渲染器从 Vulkan 改为 D3D12。以下 UI 效果需烟雾测试：
- **费用消散粒子**：`CPUParticles2D` + `modulate.a` 动画在 D3D12 下的混合模式可能不同
- **伤害/治疗数字淡出**：Label 节点 `modulate.a` 在 shader 背景上的 alpha 混合
- **阵法光环**：4.6 glow 重做主要影响 3D 渲染，2D `CanvasModulate` + `Light2D` 不受影响，但自定义 shader glow 需测试
- **拖拽半透明**：`modulate.a < 1.0` 的 Control 节点应在 D3D12 下正常——但不同 alpha 值（0.3/0.5/0.7）都需验证

## 验收标准

### 战斗HUD初始化
- **GIVEN** 进入战斗，**WHEN** 加载完成，**THEN** 手牌区、场上角色区、费用栏、回合阶段指示器均渲染可见
- **GIVEN** 进入战斗，**WHEN** HUD加载完成，**THEN** 手牌区显示正确数量的卡牌，费用栏显示当前费用数据，回合指示器显示阶段0

### 备战阶段
- **GIVEN** 战斗开始（备战阶段），**WHEN** 查看界面，**THEN** 显示角色选择面板，可选角色高亮+☐标记，可用但名额已满角色显示⊕标记，不可用角色灰色+💀骷髅
- **GIVEN** 备战界面已加载，**WHEN** 名额已满（4人）且玩家点击⊕标记角色，**THEN** 弹出替换确认"用[新角色名]替换[已选角色名]？"
- **GIVEN** 备战界面，**WHEN** 未选任何角色，**THEN** [确认出战]按钮灰色不可点击
- **GIVEN** 备战界面，**WHEN** 玩家拖拽阵位布局预览中的角色头像，**THEN** 角色在前后排之间移动，布局预览实时更新

### 手牌交互
- **GIVEN** 阶段2（出牌阶段），**WHEN** 查看手牌，**THEN** 费用充足的卡牌彩色，费用不足的灰显
- **GIVEN** 阶段2，**WHEN** 拖拽一张可出牌的卡牌到目标区域，**THEN** 卡牌跟随鼠标（8px阈值后启动拖拽），释放后执行出牌
- **GIVEN** 阶段2之外（0/1/3/4/5/6），**WHEN** 查看手牌，**THEN** 手牌进入灵能预览模式（饱和度60%+🔒标记），可悬停查看详情但不可拖拽
- **GIVEN** 阶段2，**WHEN** 拖拽进行中阶段计时到期，**THEN** 卡牌飞回手牌原位置（0.2s），阶段正常切换，卡牌不出场

### HP条与视觉反馈
- **GIVEN** 场上角色HP变化，**WHEN** 结算，**THEN** HP条在 0.3s 内从旧值线性过渡到新值（使用 Tween.create_tween()），HP数字即时更新
- **GIVEN** 同一目标连续受到多次伤害，**WHEN** 批量结算，**THEN** HP条直接过渡到最终值一次（不串行播放多次过渡）
- **GIVEN** 场上角色HP<30%，**WHEN** 结算后，**THEN** HP条变为红色
- **GIVEN** 角色阵亡，**WHEN** HP归0，**THEN** 播放阵亡动画（碎裂+消散），角色移除，阵位变为空（虚线框）

### 费用表现
- **GIVEN** 费用消耗，**WHEN** 出牌后，**THEN** 费用栏数字减少，播放消耗动画（跳动+光粒消散0.2s）
- **GIVEN** 临时费用加入，**WHEN** 丹药/效果生效，**THEN** 费用栏显示绿色+数字，实际可用费用颜色基于 `(available / (base+temp))` 比率
- **GIVEN** 基础费用为0但有+3临时，**WHEN** 查看费用栏，**THEN** 费用数字黄色（非红色），显示"0/0 [+3临时]"

### 攻击目标选择
- **GIVEN** 阶段3（攻击声明），**WHEN** 进入阶段，**THEN** 模式指示器显示"点击选择攻击目标"提示文字（动画出现→淡出）
- **GIVEN** 阶段3，**WHEN** 点击己方角色，**THEN** 可选敌方目标高亮红色
- **GIVEN** 阶段3，**WHEN** 点击被前排保护的敌方后排，**THEN** 不可选，显示🔒标记
- **GIVEN** 阶段3，**WHEN** 已声明角色攻击目标，**THEN** 角色→目标之间显示箭头连线

### 战斗日志
- **GIVEN** 战斗中全程，**WHEN** 查看日志标签，**THEN** 日志默认折叠为小标签
- **GIVEN** 日志折叠，**WHEN** 点击标签，**THEN** 日志展开为半屏覆盖，显示完整行动记录（含具体数值）
- **GIVEN** 日志展开，**WHEN** 阶段切换到阶段2（出牌），**THEN** 日志自动收起
- **GIVEN** 日志展开，**WHEN** 鼠标滚轮滚动，**THEN** 日志内容上下滚动，最新条目在最底部自动滚入视图
- **GIVEN** 战斗结束后，**WHEN** 返回探索，**THEN** 日志内容清空

### 阵法区
- **GIVEN** 阵法已部署，**WHEN** 查看阵法区，**THEN** 阵法卡显示阵名+阵营标签，激活阵法显示发光效果
- **GIVEN** 阵法区有空位，**WHEN** 查看，**THEN** 空位显示灰色虚线框
- **GIVEN** 鼠标悬停阵法卡，**WHEN** 超过300ms，**THEN** 显示阵法效果详情

### 牌库与弃牌堆
- **GIVEN** 战斗中，**WHEN** 查看费用栏，**THEN** 牌库剩余张数数字显示正确
- **GIVEN** 战斗中，**WHEN** 点击牌库计数，**THEN** 展开牌库详情（可滚轮浏览，卡背展示）
- **GIVEN** 战斗中，**WHEN** 点击弃牌堆计数，**THEN** 展开弃牌堆详情（可滚轮浏览，卡面展示）
- **GIVEN** 手牌满10张时抽牌，**WHEN** 牌库有卡，**THEN** 卡牌从牌库直飞弃牌堆（不出现在手牌区）+ 文字提示"手牌已满，卡牌进入弃牌堆"（2s淡出）

### 结算面板
- **GIVEN** 战斗胜利，**WHEN** 结算，**THEN** 弹出战利品面板，战利品展示 0.5s 后才可变点击
- **GIVEN** 战利品面板，**WHEN** 点击一个战利品选项，**THEN** 高亮选中（非立即确认）
- **GIVEN** 战利品面板，**WHEN** 点击[确认选择]按钮（未选中任何项），**THEN** 按钮灰色不可点击
- **GIVEN** 战利品面板，**WHEN** 选中项且点击[确认选择]，**THEN** 战利品确认生效，面板关闭
- **GIVEN** 战斗战败，**WHEN** 结算，**THEN** 弹出损失摘要面板（修为损失+阵亡角色+绑定卡损失）

### 撤退
- **GIVEN** 战斗中点击撤退，**WHEN** 确认，**THEN** 弹出撤退确认窗口
- **GIVEN** 非渡劫战撤退，**WHEN** 确认撤退，**THEN** 弹出标准撤退面板（"视为战败处理，保留非绑定物品"）
- **GIVEN** 渡劫战撤退，**WHEN** 确认撤退，**THEN** 弹出特殊警示面板（红色边框 + "损失80%修为+失去渡劫机会" + "畏劫而退，修为散尽"）
- **GIVEN** 渡劫战进入前，**WHEN** 备战界面，**THEN** 弹出"天劫在前，退路将断。确认进战？"确认弹窗

### 悬停详情
- **GIVEN** 鼠标悬停手牌卡牌，**WHEN** 超过300ms，**THEN** 卡牌放大120%+显示详情（全阶段可用，包括灵能预览模式）
- **GIVEN** 鼠标悬停场上角色，**WHEN** 超过300ms，**THEN** 显示角色详情面板
- **GIVEN** 鼠标悬停状态图标，**WHEN** 超过300ms，**THEN** 显示状态名称+剩余回合
- **GIVEN** 手牌详情面板已展开，**WHEN** 鼠标同时移到角色区，**THEN** 角色详情面板不弹出（互斥策略——同时最多一个详情面板）

### 阶段指示器
- **GIVEN** 阶段切换，**WHEN** 切换完成，**THEN** 阶段指示器更新名称和进度条（7阶段模型0-6）
- **GIVEN** 阶段0-6全程，**WHEN** 回合编号变化，**THEN** 回合编号递增更新
