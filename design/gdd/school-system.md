## 概述

流派系统是游戏的元玩法层——它不定义新机制，而是通过识别玩家卡组中的卡牌构成模式（阵营分布、角色组合、阵法选择、卡牌类型比例），判定玩家当前构筑偏向哪种流派风格，并在满足条件时给予策略指引和战术增益奖励。系统支持5种预设流派（正道发育流、魔道快攻流、正邪混合流、灵界真灵流、百艺炼丹流），每种流派有独立的激活条件、效果和弱点。

对玩家来说，流派系统是「策略的确认与奖励」——不是告诉你「你必须走这条路」，而是在你已经在走某条路时说「你的选择是对的，这是你的回报」。流派系统让有方向性的构筑收到正反馈，同时为新手玩家提供构筑路线参考（在卡组查看界面显示「当前流派倾向：正道发育流（78%匹配）」）。

## 玩家幻想

流派系统的核心情感是**路线的确认感**——「我走的路是对的，而且不是每个人都能走这条路」。

| 情感 | 触发场景 |
|------|---------|
| **流派成型** | 看到流派激活提示——「正道发育流已激活」——构筑有了名字 |
| **专属增益** | 流派奖励在战斗中生效——「这就是我选这条路的原因」 |
| **构筑对照** | 在卡组查看界面看到流派匹配度百分比——像个构筑评分 |
| **独特体验** | 不同流派打同一个Boss完全不同的体验——「这局玩魔道快攻，上局走正道，感觉像两个游戏」 |
| **探索发现** | 凑齐一套非主流流派（如百艺炼丹流）后的成就感——「这个流派被我玩明白了」 |

流派系统的设计哲学是：**不惩罚混合构筑，奖励纯粹构筑**。你完全可以走混合路线（正邪混合流就是为此设计的），但如果你集中投入某个方向，流派系统会说「干得漂亮」。

## 详细设计

### 核心规则

#### 1. 流派定义结构

每个流派由一个模板定义，包含激活条件、核心增益和弱点：

```
SchoolTemplate {
  id: String                         # 流派ID（如 "righteous_dev"）
  name: String                       # 显示名称（如 "正道发育流"）
  tagline: String                    # 一句话口号
  description: String                # 流派描述（玩法风格说明）
  detection: {                       # 检测条件——全部满足则激活
    min_major_faction_count: int     # 大阵营最低上场人数（如正道≥3）
    required_characters: [String]    # 必需角色卡ID（可选）
    required_formations: [String]    # 必需阵法卡ID（可选）
    card_type_ratio: {type, min_pct} # 卡牌类型占比（如丹药卡≥15%）
    excluded_cards: [String]         # 禁止携带的卡牌（可选）
  }
  effects: [                         # 激活后的增益效果列表
    {type, target, value, trigger}
  ]
  weakness: String                   # 流派的克制弱点描述
  visual_theme: String               # 流派视觉主题色/光效
}
```

| 激活条件 | 卡组中丹药卡占比 ≥20%；角色「大衍神君」在场或已收藏；本局已进行过至少3次炼丹/炼器操作 |
| 增益效果 | 丹药卡效果 +20%（如回复量、属性增量）；炼丹/炼器时消耗的灵材 -1（最少消耗1）；每使用3张丹药卡，回复1点行动力（每局最多触发3次）；丹药卡可突破稀有度上限炼制的概率 +10% |

#### 3. 流派检测机制

流派检测在以下时机触发和更新：

| 触发时机 | 检测内容 | 结果 |
|---------|---------|------|
| 开局初始卡组写入后 | 检测是否符合任一流派条件 | 初始流派状态（可能无流派） |
| 每次卡组变更（获得/删除卡牌） | 重新检测所有流派 | 流派状态更新（激活/失效/切换） |
| 每次上场角色变更（战斗开始前） | 重新检测场上阵营条件 | 流派增益准备就绪 |
| 每次战斗回合开始 | 检测战中动态条件 | 流派增益持续生效/失效 |
| 每次炼丹/炼器完成（百艺流派） | 检测操作次数条件 | 百艺流派可能在此刻激活 |

```
school_detection_flow():
  ① 获取当前状态的完整数据：
     - 上场角色列表及其阵营标签
     - 卡组中全部卡牌的列表
     - 本局资源操作记录（炼丹/炼器次数）
     - 当前玩家境界
  ② 遍历5种流派模板，按优先级顺序检测：
     优先级：灵界真灵流 > 正道发育流 > 魔道快攻流 > 正邪混合流 > 百艺炼丹流
  ③ 对每个流派：
     a. 检查所有必要条件——任一不满足则跳过
     b. 所有条件满足 → 标记为 "可激活"
  ④ 如果有多个流派可激活 → 选择优先级最高的
  ⑤ 如果无流派可激活 → 流派状态为 "无"
  ⑥ 如果当前激活流派改变 → 广播 school_changed 信号
```

**优先级规则说明**：灵界真灵流的条件最苛刻、收益最高，所以优先级也最高（如果满足灵界条件，走灵界流而非正道）。同一时刻只能激活**一个流派**。

**流派切换**：游戏中可能发生流派切换（如前期走正道发育流，后期拿到关键卡转换为灵界真灵流）。切换时旧流派增益立即失效，新流派增益立即生效，中间无重叠期。

#### 4. 流派匹配度（软提示）

除了「已激活」的硬状态外，系统还计算每个流派的**匹配度百分比**——即玩家当前构筑距离激活该流派还有多远：

```
match_score(school, state):
  对各条件打分：
  - 阵营人数：当前数 / 要求数 × 权重
  - 必备角色：已拥有数 / 要求数 × 权重
  - 卡牌类型比例：当前比例 / 要求比例 × 权重
  - 境界要求：满足=100%，不满足=按距离打折
  总分 = 加权平均（各条件权重在流派模板中定义）
  返回 {score: 0~100, missing: [缺失的关键条件列表]}
```

匹配度在卡组查看界面展示——不激活流派也能看到「流派倾向」：

```
当前流派倾向：
  正道发育流    78%  ← 差1个正道角色可激活 ✓
  正邪混合流    45%  ← 缺1个正道+1个魔道角色
  魔道快攻流    30%  ← 低费卡占比不足
```

这不仅帮助新玩家理解流派方向，也为老玩家提供「我还差什么」的明确指引。

#### 5. 流派增益的生效范围

流派增益分为两类：

**静态增益（全局生效）：** 满足条件即持续生效，不依赖战中操作
- 正道角色减伤（战斗中每击计算时应用）
- 魔道角色攻击加成（战斗中属性计算时应用）
- 正邪混合的ATK/DEF加成

**动态增益（战中触发）：** 需要在战斗中满足特定行为
- 魔道击杀抽牌（每击杀触发）
- 正邪共鸣概率减费（每回合判定）
- 百艺丹药效果加成（使用丹药时应用）
- 灵界光环叠加（每上场一个灵界角色时应用）

静态增益在战斗开始时一次性计算，动态增益在战斗中实时响应。

#### 6. 流派与身份/轮回天赋的交互

| 交互维度 | 说明 |
|---------|------|
| **身份流派倾向** | 七玄门弟子更容易走正道发育流；魔修遗孤更容易走魔道快攻流 |
| **天赋流派强化** | 轮回天赋可以强化特定流派（如「正道发育流减伤+1」「魔道快攻首回合费用再+1」） |
| **天赋解锁流派变体** | 未来可解锁流派变体（如「正道·意志流」——以护盾为核心的正道亚种） |
| **身份流派解锁** | 某些流派只有特定身份才容易激活（如「百艺炼丹流」对大衍神君的要求，大衍神君角色卡需要事件获得） |

#### 7. 流派UI反馈

流派系统在以下位置提供UI反馈：

| 位置 | 内容 | 触发条件 |
|------|------|---------|
| **卡组查看界面** | 流派匹配度条（前3高流派的名称+百分比） | 打开卡组查看时 |
| **战斗开始前** | 上场角色确认后→流派状态提示（「正道发育流·已激活」） | 上场完毕进入战斗时 |
| **战斗HUD** | 流派名称缩略标记在界面角落（纯标识，不占空间） | 流派激活的整场战斗中 |
| **流派激活/切换** | 屏幕中央浮动文字 + 对应主题色光效 | 流派状态变更时 |
| **流派增益触发** | 对应效果的战斗日志文本（如「正道发育流·回合回复 HP +2」） | 增益触发时 |

#### 8. 流派平衡设计原则

| 原则 | 说明 |
|------|------|
| **无绝对最强流派** | 每种流派有明确的克制关系和弱点，没有通解 |
| **流派≈构筑路线，≠难度选择** | 正道发育流不是「简单模式」，魔道快攻流不是「高手模式」——它们是平行选择 |
| **混合不惩罚** | 不满足任何流派时，玩家不受到惩罚（纯中立，无负面效果） |
| **流派可切换** | 一局中流派可随构筑变化切换，不锁定 |
| **流派≠胜负** | 流派增益是锦上添花，不是胜负的决定性因素——构筑质量仍是最重要的 |

### 状态与转换

```
流派状态生命周期：

[无流派] —— 初始状态（开局）
    ↓ 满足流派激活条件
[流派A·激活] —— 增益生效
    ↓ ↓
    | ↕ 构筑变化→流派A失效+流派B满足条件
    | [流派B·激活] —— 增益切换
    | ↕ 战斗开始
    | [流派B·激活·战中] —— 动态增益响应
    |    ↓ 战斗结束
    | [流派B·激活] —— 回到非战状态
    ↓ ↓
[流派A·激活] —— 如果构筑回归流派A
    ↓
[无流派] —— 构筑偏离所有流派条件
    ↓ 再次满足条件
重新激活
```

**状态变更不频繁**——流派通常在一局中稳定在1~2次切换（除非玩家大量调整构筑）。

## 公式

### 1. 流派检测——匹配度计算

```
calculate_match(school, state) → {score: float, missing: String[]}
  score = 0
  total_weight = 0
  missing = []
  
  conditions = school.detection
  
  # 阵营人数条件（如正道≥3）
  if conditions.min_major_faction_count:
    weight = 40
    count = count_faction_on_field(state, conditions.faction)
    ratio = min(count / conditions.min_major_faction_count, 1.0)
    score += weight × ratio
    total_weight += weight
    if count < conditions.min_major_faction_count:
      missing.append("还需{count}个{conditions.faction}角色（当前{count}/{conditions.min_major_faction_count}）")
  
  # 必备角色条件（如需要韩立）
  if conditions.required_characters:
    weight = 30
    owned = count_owned(state, conditions.required_characters)
    required = len(conditions.required_characters)
    ratio = owned / required
    score += weight × ratio
    total_weight += weight
    for each character not owned:
      missing.append("缺少核心角色：{character}")
  
  # 卡牌类型占比（如丹药≥20%）
  if conditions.card_type_ratio:
    weight = 20
    actual_ratio = calc_type_ratio(state.deck, conditions.card_type_ratio.type)
    required_ratio = conditions.card_type_ratio.min_pct
    ratio = min(actual_ratio / required_ratio, 1.0)
    score += weight × ratio
    total_weight += weight
    if actual_ratio < required_ratio:
      missing.append("{type}卡占比不足（{actual_ratio:.0f}%/{required_ratio:.0f}%）")
  
  # 境界条件（如灵界流需要金丹）
  if conditions.min_realm:
    weight = 10
    ratio = state.realm_level >= conditions.min_realm ? 1.0 : 0.0
    score += weight × ratio
    total_weight += weight
    if not ratio:
      missing.append("境界不足（需{conditions.min_realm}级）")
  
  total_score = total_weight > 0 ? round(score / total_weight × 100) : 0
  return {score: total_score, missing}
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| score | float | [0, 100] | 加权总分 |
| total_weight | int | [0, 100] | 总权重 |
| ratio | float | [0.0, 1.0] | 各条件满足比例 |

```
apply_school_effects(school, battle_state):
  if not school: return  # 无流派
  
  for effect in school.effects:
    match effect.type:
      "regen":          # 每回合回复HP

      "damage_reduce":  # 减伤

      "attack_boost":   # 攻击加成

      "draw_on_kill":   # 击杀抽牌

      "cost_discount":  # 费用折扣（概率）
```

## 边界情况

- **同时满足多个流派条件**：按优先级选择最高的。如正道+正邪同时满足→走正道（优先级更高）
- **流派条件在战斗中变化**：增益在战斗开始时锁定，战中不变。战后重新检测（避免战中反复切换造成混乱）
- **流派增益被敌人驱散**：流派增益是系统级效果，不可被敌方驱散——不占用常规buff位
- **百艺炼丹流激活后失去条件（丹药被弃置导致占比不足）**：流派一旦激活，在本局内不退位（防止频繁闪烁）。只有切换到另一个流派或被更高优先级流派覆盖时才失效
- **灵界真灵流激活后境界下降**：境界不会下降（只升不降），所以不会发生
- **流派切换时增益叠加**：不会叠加——切换时旧增益立即清空，新增益应用。同一时刻唯一流派
- **流派匹配度显示但无流派激活**：正常——匹配度是软提示，激活是硬状态。匹配度≥50%就有显示价值
- **百艺炼丹流的操作计数跨战斗保存**：计数保存在GSM的 `player.school_data.alchemy_count` 中，随存档持久化，不因战斗重置
- **魔道快攻流的「首回合+1费」与魔修遗孤天赋叠加**：允许叠加——魔修遗孤选择魔道快攻流时，首回合费用 = 基础 + 1（天赋） + 1（流派）= 战术极限爆发
- **流派检测在开局时「卡组中某些卡暂时不在场上」**：流派激活条件中的阵营人数以**场上角色**为准，不是以卡组为准。这意味着流派可能在战斗开始前才最终确认激活
- **开局初始卡组不满足任何流派条件**：预期行为——开局通常5~10张卡，不会激活任何流派。流派在收集了足够卡牌后才逐步成型

## 验收标准

- **GIVEN** 场上正道角色≥3、无魔道角色，**WHEN** 检测流派，**THEN** 激活「正道发育流」
- **GIVEN** 场上魔道角色≥3、低费卡占比≥50%，**WHEN** 检测流派，**THEN** 激活「魔道快攻流」
- **GIVEN** 场上正道≥2且魔道≥2，**WHEN** 检测流派，**THEN** 激活「正邪混合流」
- **GIVEN** 玩家金丹期以上、场上灵界角色≥2，**WHEN** 检测流派，**THEN** 激活「灵界真灵流」
- **GIVEN** 丹药卡≥20%、大衍神君在场、已炼丹3次，**WHEN** 检测流派，**THEN** 激活「百艺炼丹流」
- **GIVEN** 同时满足正道发育流和正邪混合流条件，**WHEN** 检测流派，**THEN** 激活优先级更高的「正道发育流」
- **GIVEN** 正道发育流激活，**WHEN** 正道角色在战斗中受伤害，**THEN** 伤害 -1
- **GIVEN** 正道发育流激活，**WHEN** 每回合结束时，**THEN** 正道角色回复2HP
- **GIVEN** 魔道快攻流激活，**WHEN** 战斗前3回合，**THEN** 魔道角色 ATK +2
- **GIVEN** 魔道快攻流激活，**WHEN** 魔道角色击杀敌方，**THEN** 抽1张牌
- **GIVEN** 正邪混合流激活，**WHEN** 正道和魔道角色同时在场，**THEN** 双方获得 +1 ATK / +1 DEF
- **GIVEN** 灵界真灵流激活，**WHEN** 检查角色状态，**THEN** 灵界角色 HP +3、ATK +1
- **GIVEN** 灵界真灵流激活且3个灵界角色在场，**WHEN** 计算光环，**THEN** 全体友方 HP +3（每人+1×3）
- **GIVEN** 百艺炼丹流激活，**WHEN** 使用一张回复100HP的丹药，**THEN** 实际回复120HP（+20%）
- **GIVEN** 流派激活后构筑变化导致条件不再满足，**WHEN** 战后检测，**THEN** 更高优先级的流派激活或无流派
- **GIVEN** 无流派激活，**WHEN** 查看卡组界面，**THEN** 显示流派匹配度Top3
- **GIVEN** 魔道快攻流激活，**WHEN** 战斗进入第4回合，**THEN** ATK加成消失（前3回合后失效）
- **GIVEN** 流派激活提示出现，**WHEN** 玩家未操作，**THEN** 2秒后自动消失
