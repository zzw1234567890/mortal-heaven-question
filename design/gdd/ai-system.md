# AI系统 (Enemy AI System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-22
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱1「自由组牌，策略为王」、支柱2「苟道成长，步步为营」

## 概述

AI系统负责敌方在战斗中的所有决策行为——包括战前阵容配置、每回合的卡牌使用、攻击目标选择、功法/法宝绑定、阵法部署与覆盖等。系统根据敌方角色模板生成敌方卡组（非随机抽卡——敌方卡组是预定义的技能池），使用加权优先级决策树驱动每回合行动。系统区分普通敌人、精英敌人和Boss三种智能层级，Boss拥有独特的行为模式和阶段转换机制。

对玩家来说，AI系统决定了战斗的「对手感」——敌人不是静止的木桩，它们会治疗、会集火、会部署阵法。好的AI让玩家觉得「这个对手有脑子」，差的AI让战斗变成单向输出。AI的设计目标不是「最大化胜率」，而是「创造有意义的决策空间」——玩家的每一个操作都应当是对AI可能行动的反制，而非机械执行最优解。

暂无相关ADR。

## 玩家幻想

AI系统的核心情感是**对手的逼真感**——不是拟人化的聪明，而是可预测但有变化的战术行为。

当玩家看到敌人治疗残血角色时，不会说「这AI在作弊」，而会说「果然，他知道我会优先打那个」。

当Boss在半血转入第二阶段、改变行为模式时，玩家感觉「这才像真正的Boss战——不只是一个血多的普通敌人」。

当精英敌人部署阵法时，玩家意识到「这个战斗不是单纯拼数值——他也有策略」。

AI不需要完美（完美AI在回合制游戏里会变成无解难题），它需要**可信**——玩家能理解敌人为什么那样做，即使自己因此吃亏。

## 详细设计

### 核心规则

#### 1. 敌方智能层级

| 层级 | 适用敌人 | 决策复杂度 | 行为特点 |
|:----:|---------|:---------:|---------|
| **普通** | 杂兵、小型妖兽 | 低 | 简单优先级：攻击最残血、随机使用技能 |
| **精英** | 精英修士、中级妖兽、守关弟子 | 中 | 有基本的战术考量：集火、保持前排、低血量时撤退或治疗 |
| **Boss** | 每章关底Boss、渡劫天劫 | 高 | 多阶段转换、独特技能组、自适应策略、狂暴机制 |

对同一地图的敌人，普通占70%、精英占25%、Boss占5%（地图末固定出现）。

#### 2. 敌方卡组构成

敌方不使用玩家的30-40张卡牌系统。敌方卡组是**预定义的技能池**，每回合从中选择可用技能：

```
EnemyTemplate {
  id: String                    # 敌人模板ID
  name: String                  # 显示名称
  realm: int                    # 境界等级 1~4
  is_elite: bool                # 是否为精英
  is_boss: bool                 # 是否为Boss
  max_hp: int                   # 最大生命
  attack: int                   # 基础攻击
  defense: int                  # 基础防御
  faction_tags: String[]        # 阵营标签（部分敌人有）
  formation_limit: int          # 可用阵法位（普通0、精英1、Boss最多2）
  skill_pool: [                 # 技能池（每回合选1~2个执行）
    Skill { id, name, type, cost, target_type, effect, weight, cooldown }
  ]
  behavior_profile: {           # 行为配置
    aggression: float           # 攻击性 0.0~1.0（高=优先攻击而非治疗）
    focus_fire: float           # 集火倾向 0.0~1.0
    front_priority: float       # 前排维护倾向 0.0~1.0
    retreat_threshold: float    # 撤退血量阈值（低于此值尝试撤退）
  }
  phase_transitions: [          # Boss专属-阶段转换
    Phase { trigger: {hp_below, turn_after}, behavior_override, skill_unlock }
  ]
  rewards: {                    # 战斗获胜奖励
    ling_shi_min, ling_shi_max,
    card_drops: [{card_id, chance}],
    cultivation_reward
  }
}
```

**技能池示例：**

```
普通敌人「散修」技能池：
  ① 基础攻击（weight=50, cooldown=0）—— 造成 ATK×1.0 伤害
  ② 防御姿态（weight=20, cooldown=3）—— 本回合 DEF+2
  ③ 回春术（weight=15, cooldown=4）—— 回复自身20%HP

精英敌人「掩月宗长老」技能池：
  ① 月华斩（weight=40, cooldown=0）—— 造成 ATK×1.2 伤害
  ② 月影阵（weight=25, cooldown=3）—— 全体敌方减攻10% 2回合
  ③ 灵月护盾（weight=20, cooldown=4）—— 自身 DEF+3 2回合
  ④ 月华普照（weight=15, cooldown=5）—— 全体友方回复15%HP

Boss「墨居仁（夺舍）」技能池（阶段1）：
  ① 夺命爪（weight=35, cooldown=0）—— 造成 ATK×1.5 伤害
  ② 噬魂术（weight=25, cooldown=2）—— 吸血，伤害的50%转为自身回复
  ③ 鬼影步（weight=20, cooldown=3）—— 闪避下次攻击
  ④ 夺舍预备（weight=20, cooldown=0）—— 叠加夺舍层数

Boss「墨居仁（夺舍）」技能池（阶段2, HP<50%）：
  ① 夺命爪·强化（weight=30, cooldown=0）—— 造成 ATK×2.0 伤害
  ② 噬魂术·群体（weight=25, cooldown=2）—— 全体吸血
  ③ 夺舍降临（weight=25, cooldown=5）—— 夺取玩家一个角色（控制一回合）
  ④ 暗影爆发（weight=20, cooldown=4）—— 全体 AOE 伤害
```

#### 3. 敌方上阵与阵位分配

敌方上阵人数由战斗配置定义（不是动态计算的），与玩家境界匹配：

| 玩家境界 | 普通敌人上场 | 精英敌人上场 | Boss上场 |
|:-------:|:----------:|:----------:|:-------:|
| 炼气 | 2人（前2后0） | 2人（前1后1） | 2人（前1后1） |
| 筑基 | 3人（前2后1） | 3人（前2后1） | 3人（前2后1） |
| 金丹 | 4人（前2后2） | 4人（前2后2） | 5人（前3后2） |
| 元婴 | — | 5人（前3后2） | 6人（前3后3） |

**前排/后排分配规则：**
- 防御较高的敌人自动分配前排
- 攻击较高的敌人自动分配后排（受到前排保护）
- 如果敌方只有2人或1人，全部分配前排（无后排保护）
- 精英和Boss可能手动配置阵位以应对特定战术

#### 4. 每回合AI决策流程

战斗阶段6「敌方行动阶段」的决策流程：

```
① 获取当前场上所有己方（敌方）角色状态
② 获取当前手牌（技能池中可用技能，考虑冷却和费用限制）
③ 计算各技能的优先级分数：
   for each 可用技能:
     score = skill.weight × 修正系数
     修正系数根据当前战场状态动态计算：
       - 如果己方残血角色存在 → 治疗类技能 score × 1.5
       - 如果己方前排减员 → 召唤/防御类 score × 1.3
       - 如果玩家某个角色高威胁 → 集火类 score × 1.4
       - 如果己方有阵法位空 → 阵法部署 score + 20
④ 按分数从高到低排序
⑤ 选择最高分的技能执行（如果费用足够）
⑥ 如果有剩余费用 → 从剩余技能中选择次高分执行
⑦ 所有敌方角色执行攻击声明（每个角色选择目标攻击）
```

#### 5. 目标选择逻辑

```
select_target(attacker, player_field):
  可用目标 = 玩家前排存活角色（如果存在）否则玩家后排存活角色
  
  if attacker.behavior_profile.focus_fire > 0.5:
    # 集火模式：优先攻击已受伤的角色
    目标 = 可用目标中 HP% 最低的角色
    if 有多个同HP%目标 → 选防御最低的
  else:
    # 分散模式：随机选择目标，但残血角色权重×2
    权重列表 = 每个角色基础权重1.0，残血角色加权重
    目标 = 加权随机选择
  
  特殊情况：
  - 如果玩家角色有「嘲讽」效果 → 强制攻击嘲讽角色
  - 如果玩家后排有高威胁角色（如阵法阵眼）→ AI可能使用穿透攻击绕过前排
  - Boss在特定阶段可能有固定目标优先级
```

#### 6. 敌方绑定与阵法

**敌方绑定：**
- 精英和Boss在战前预配置绑定（不需要在战斗中打出功法/法宝卡）
- 绑定效果在战斗开始时自动生效（等同于已在绑定位上）
- 普通敌人无绑定（仅用基础属性和技能池）
- 敌方绑定不消耗费用、不占用出牌机会
- 敌方角色阵亡时绑定自动消失（不走绑定系统的"进入弃牌堆"流程——简化为直接移除）
- 玩家**不可见**敌方绑定的具体效果（但可见绑定槽位占用情况）

**敌方阵法：**
- 精英敌人可部署1个阵法、Boss最多2个（占用阵法位与玩家共享规则）
- 敌方阵法的激活条件与玩家相同（条件判定规则一致）
- 敌方阵法的部署由AI决策流程控制（当有空位且有可用阵法卡时触发）
- 敌方阵法的覆盖规则与玩家一致（满了3个需覆盖）
- 敌方阵法的角色归属由AI自动分配（不显示归属选择界面）
- 敌方仅使用预设阵法（不会动态生成阵法）

#### 7. Boss阶段转换

Boss战斗特有机制——当Boss生命值低于特定阈值时触发阶段转换：

```
BossPhaseTransition {
  trigger: {
    hp_below: float    # 生命值低于此比例时触发（如 0.5 = 50%HP）
    turn_after: int    # 或在此回合数后触发（如 turn >= 5）
  }
  effects: {
    behavior_override: BehaviorProfile  # 替换当前行为配置
    skill_unlock: [String]              # 解锁新技能加入技能池
    skill_remove: [String]              # 锁定旧技能从技能池移除
    reset_cooldowns: bool               # 是否重置所有技能冷却
    heal_percent: float                 # 转换时回复HP比例（如 0.1 = 回复10%）
    animation: String                   # 转换动画资源
  }
}
```

转换规则：
- 每个Boss最多3个阶段（起始阶段 + 2个转换）
- 转换触发瞬间Boss免疫所有伤害（动画播放期间无敌）
- 转换完成后，新的行为配置立即生效
- 转换动画是强制播放的（不可跳过），给玩家一个节奏缓冲
- 如果触发条件在Boss被击杀的同一帧达到 → 击杀优先（不触发转换）

#### 8. 敌方撤退逻辑

部分敌人（非Boss）在战况不利时可能撤退：

```
check_retreat(enemy):
  if not enemy.template.behavior_profile.retreat_threshold:
    return false  # 该敌人不会撤退
  ally_hp_ratio = sum(hp) / sum(max_hp)
  if ally_hp_ratio < retreat_threshold:
    roll = randf()
    if roll < 0.5:  # 50%概率撤退
      return true
  return false
```

撤退后：
- 战斗判定为玩家胜利（但奖励减半——撤退的敌人不会掉落全部战利品）
- 已阵亡的敌方角色不会随撤退复活
- 撤退不会出现在Boss战中

#### 9. 难度缩放

敌人属性根据玩家当前境界动态缩放，确保有挑战性但不至于碾压：

```
scale_enemy(template, player_realm):
  base_level = template.realm
  if player_realm > base_level:
    # 玩家的境界超过敌人基准→按比例提升敌人属性
    scale = 1.0 + (player_realm - base_level) × 0.3
    template.max_hp = round(template.max_hp × scale)
    template.attack = round(template.attack × scale)
    template.defense = round(template.defense × scale)
  # 如果玩家境界低于敌人基准→境界压制规则已在战斗系统中处理
```

#### 10. AI决策的可视化与玩家学习

- 敌方出牌时，技能名称和效果在战斗日志中显示
- 敌方部署阵法时，阵法名称+激活状态在阵法区显示
- Boss阶段转换时，有独立的转换动画+文本提示
- 玩家无法看到敌方的内部决策分数（即不可见AI的"思考过程"）
- 战斗结束后，玩家无法复盘敌方手牌（不可见敌方未使用的技能）——这保持了神秘感

### 状态与转换

```
敌方战斗状态（由战斗系统管理）：
  战斗开始 → 敌方阵容部署 → 战斗循环（回合制）→ 战斗结束
                                           ↓
                                   全部阵亡 → 胜利
                                   玩家撤退 → 胜利（奖励减半）
                                   玩家全灭 → 战败
                                   撤退成功 → 战败（敌人消失）
```

Boss阶段转换状态机：

```
阶段1（起始） → HP<50% （或回合条件） → 阶段转换动画 → 阶段2（强化）
阶段2 → HP<20% → 阶段转换动画 → 阶段3（狂暴）
阶段3 → HP=0 → 阵亡
```

## 公式

### 1. 技能选择分数

```
skill_score(skill, field_state) = skill.base_weight × modifier(field_state)

modifier(field_state) = 1.0
  + (ally_low_hp_count > 0 AND skill.is_heal ? 0.5 : 0)
  + (ally_front_dead AND skill.is_defense ? 0.3 : 0)
  + (player_high_threat AND skill.is_attack ? 0.4 : 0)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| base_weight | int | [0, 100] | 技能基础权重（来自模板） |
| modifier | float | [1.0, 2.5] | 战场状态修正系数 |
| ally_low_hp_count | int | [0, 6] | 血量低于30%的友方角色数 |
| ally_front_dead | bool | true/false | 是否有前排友方阵亡 |
| player_high_threat | bool | true/false | 玩家是否有高威胁角色 |

### 2. 目标选择

```
target_score(character, attacker) = base(1.0)
  + (character.hp_pct < 0.3 ? 0.5 : 0)          # 残血加分
  + (character.is_taunting ? 999 : 0)             # 嘲讽强制目标
  - (character.defense × 0.1)                     # 高防御减分

selected = max_weighted_random(candidates, target_score)
```

### 3. 撤退判定

```
retreat_probability(ally_hp_ratio, threshold) = 
  ally_hp_ratio < threshold ? 0.5 : 0.0
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| ally_hp_ratio | float | [0.0, 1.0] | 敌方总血量剩余比例 |
| threshold | float | [0.0, 1.0] | 撤退阈值（来自模板配置） |

### 4. Boss阶段触发

```
should_transition(boss, turn, hp_pct) → int:
  for each phase in boss.phase_transitions:
    if hp_pct <= phase.hp_below and turn >= phase.turn_after:
      if not phase.triggered:
        return phase.index
  return -1  # 不触发
```

### 5. 难度缩放

```
scaled_stat(base, player_realm, enemy_realm):
  if player_realm <= enemy_realm: return base  # 不变（境界压制已处理）
  scale = 1.0 + (player_realm - enemy_realm) × 0.3
  return round(base × scale)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| player_realm | int | [1,5] | 玩家当前境界 |
| enemy_realm | int | [1,5] | 敌人模板基准境界 |
| scale | float | [1.0, 1.9] | 缩系数（玩家高2级时最大×1.6，高3级×1.9） |

## 边界情况

- **敌方所有技能都在冷却中**：AI使用普通攻击（基础攻击，无技能效果，不消耗费用）
- **敌方费用不足**：跳过高费技能，选择费用最低的可用技能；如果全都不够，仅使用普通攻击
- **敌方无存活角色时AI决策**：不可能发生——阶段6的前提是敌方至少有一个角色存活
- **Boss转换触发瞬间被击杀**：转换不触发，Boss正常阵亡
- **Boss被多次压到转换线以下（如血量波动）**：每个转换条件只触发一次，触发后锁定
- **敌方撤退后部分角色阵亡**：撤退只影响存活角色，已阵亡角色不复活
- **玩家使用嘲讽后AI集火**：嘲讽强制AI所有可攻击角色攻击嘲讽目标——但法术效果（非攻击的debuff）仍可作用于其他角色
- **敌方精英死亡后绑定如何处理**：绑定随角色阵亡直接移除（不走玩家的绑定链销毁流程）
- **敌方角色被送到后排后前排空缺**：敌方AI在阶段6优先判断前排状态——如果前排无人且后排有人，AI会将部分后排角色移动到前排（布阵调整由AI在阶段6自动执行，不占用出牌机会）
- **敌方使用阵法覆盖时**：走标准覆盖流程（弹窗由AI自动确认，无玩家操作）
- **超高难度下Boss三阶段全部触发**：设计上Boss最多3个阶段（含起始），在最高难度下Boss可能在回合数到达时自动进入下一阶段（即使血量未到阈值），防止玩家拖回合磨死Boss
- **玩家境界远高于地图敌人基准**：敌人按难度缩放提升属性，但技能池和智能层级不变——打低级图不会变成无脑碾压，但优势明显可接受

## 依赖关系

| 依赖系统 | 性质 | 说明 |
|----------|------|------|
| **战斗系统** | 硬依赖 | 接收阶段6驱动信号；提供场上状态用于决策；执行AI行动指令 |
| **卡牌系统** | 硬依赖 | 敌人技能效果需要卡牌效果引擎解析（技能效果复用卡牌效果机制） |
| **上场与阵位系统** | 硬依赖 | 敌方阵位由AI系统定义，开战前需要部署到阵位系统 |
| **阵法系统** | 软依赖 | 精英/Boss可能部署阵法，需要调用阵法系统的部署接口 |
| **绑定系统** | 软依赖 | 精英/Boss的预配置绑定需要绑定系统的绑定注册接口 |
| **游戏状态管理器** | 软依赖 | 读取玩家境界用于难度缩放 |

### 上游依赖设计顺序

战斗系统 → AI系统
卡牌系统效果引擎 → AI系统（敌方技能效果解析复用）

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 集火阈值（focus_fire） | 0.6 | 0.0-1.0 | AI不集火战斗太简单 | AI完美集火玩家体验差 |
| 攻击性（aggression） | 0.7 | 0.0-1.0 | AI保守治疗太多战斗拖沓 | AI只会攻击无战术感 |
| 撤退触发概率 | 0.5 | 0.3-0.8 | 敌人死战不退 | 敌人频繁撤退玩家烦躁 |
| Boss阶段数上限 | 3 | 2-4 | Boss战缺少变化 | 阶段太多战斗拖沓 |
| 难度缩放系数/级 | ×1.3 | ×1.1-×1.5 | 高境界打低图太简单 | 高境界打低图仍有难度（不合理） |
| 敌方普通攻击伤害系数 | ×1.0 | ×0.8-×1.2 | 普攻太弱AI技能依赖 | 普攻太强技能无存在感 |
| 敌方技能池大小 | 3-5 | 2-8 | 行为可预测无聊 | AI行为太随机无战术感 |
| Boss转换无敌帧 | 1回合 | 0-2回合 | 转换中被打断 | 无敌太久玩家挫败感 |
| 精英撤退阈值 | 0.2 | 0.1-0.4 | 精英死战不退 | 精英频繁撤退 |

## 验收标准

- **GIVEN** 战斗阶段6开始，**WHEN** AI决策，**THEN** 敌方角色执行至少一个可用技能或攻击
- **GIVEN** 敌方技能池中有治疗技能且友方残血，**WHEN** AI决策，**THEN** 治疗技能权重高于攻击技能
- **GIVEN** 敌方前排阵亡而后排有角色，**WHEN** AI决策，**THEN** 后排角色自动补位到前排
- **GIVEN** 普通敌人，**WHEN** 战前配置，**THEN** 无绑定、无法阵
- **GIVEN** 精英敌人，**WHEN** 战前配置，**THEN** 有预配置绑定、可部署1个阵法
- **GIVEN** Boss敌人，**WHEN** 战前配置，**THEN** 有预配置绑定、可部署最多2个阵法、有阶段转换定义
- **GIVEN** Boss血量降到50%以下，**WHEN** 检查阶段转换，**THEN** 触发转换动画，新技能解锁
- **GIVEN** Boss在阶段转换期间受到致命伤害，**WHEN** 结算，**THEN** Boss阵亡，转换不触发
- **GIVEN** 敌方总血量低于撤退阈值（0.2），**WHEN** AI决策，**THEN** 有50%概率撤退
- **GIVEN** 敌方撤退，**WHEN** 战斗结束，**THEN** 判定为玩家胜利，奖励减半
- **GIVEN** 玩家境界高于敌人基准，**WHEN** 敌人属性计算，**THEN** 按公式缩放
- **GIVEN** 玩家境界低于敌人基准，**WHEN** 敌人属性计算，**THEN** 不缩放（境界压制由战斗系统处理）
- **GIVEN** 敌方所有技能都在冷却，**WHEN** AI决策，**THEN** 使用普通攻击
- **GIVEN** 玩家角色激活嘲讽，**WHEN** 敌方选择攻击目标，**THEN** 所有可攻击敌方强制攻击嘲讽目标
- **GIVEN** Boss所有阶段触发完毕，**WHEN** 继续战斗，**THEN** Boss保持最终阶段行为模式
