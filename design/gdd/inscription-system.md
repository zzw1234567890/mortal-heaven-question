## 概述

凡人的成长哲学是「物尽其用」——一件法宝到手只是开始，通过铭刻附加副属性，让同一件法宝在玩家手中产生差异化的战术价值。铭刻系统的核心是**在机缘中做出策略性抉择**：每次铭刻从随机候选池中选出最优解，玩家还可支付额外灵材进行**定向铭刻**——指定一个属性方向（攻击/防御/战术），该方向属性的出现权重获得加成。这是「我亲手打造的法宝」而非「系统掉落的标准品」。

## 玩家幻想

「我的法宝和你的不一样」——铭刻系统服务的核心情感是**养成专属感**。同一种法宝，在你手中通过一次次铭刻选择长出了不同的副属性组合——每一条副属性都是你亲手选的，记录了你的构筑思路。这是一种「养孩子」式的投入感——你不只是在收集法宝，你是在培养它们。

| 情感 | 触发场景 |
|------|----------|
| **专属感** | 「我这个上品法器是+2攻击+吸血的——你的呢？而且这三条属性是我亲手选的」 |
| **策略的满足** | 三择中出现攻击+1、暴击+3%、虚弱——「我构筑缺暴击，选暴击」 |
| **精炼的成就** | 三条铭刻全是自己选出来的有用属性——「完美了，这是我自己养出来的」 |
| **成长的期待** | 刚炼出一件好法宝，还有三个铭刻空位可以慢慢投入灵材培养 |
| **投入的权衡** | 已经铭刻了两次（花了1+2=3灵材），第三次要3灵材——「值不值得继续投？」 |

## 详细设计

### 核心规则

#### 1. 铭刻基本规则

- 每件法宝初始拥有 **0 条副属性**，最多可铭刻 **3 条**
- 每次铭刻时，玩家确认消耗灵材（立即扣除）→ 系统从副属性池中 **随机 roll 出 3 个互不相同的候选副属性**，玩家从中**选择 1 个**附加到法宝上
- 铭刻次数不受限制——已满 3 条后可继续铭刻，玩家**手动选择 1 条已有副属性**替换，然后从 3 个候选中选择新属性
- 铭刻操作不可逆——选定即生效，无法撤销
- **确认即扣灵材**：点击铭刻确认后灵材立即扣除。候选展示后不可取消（灵材已消耗），但玩家可在确认前取消（此时灵材未扣）。确认对话框必须完整披露：「确认后立即消耗 N 中级灵材。铭刻候选展示后即使取消，灵材也不会退还。是否继续？」
- **递增成本**：同一件法宝的第 N 次铭刻消耗 **N × 中级灵材×1**（第1次=1灵材，第2次=2灵材，第3次=3灵材，以此类推）。**软上限**：第 6 次及以后固定消耗 5 中级灵材/次（避免后期成本失控）
- **境界门槛**：T4（稀有）副属性仅在 **筑基期（境界L≥2）及以后**才进入候选池。炼气期（L=1）铭刻只能获得 T1-T3 属性
- **定向铭刻（策略输入层）**：玩家可在确认铭刻前选择一个**属性方向**——「攻击向」（攻击/暴击/暴伤权重×1.5）、「防御向」（防御/生命/回血权重×1.5）、「战术向」（吸血/虚弱/破甲/灵力萃取权重×1.5）——**不额外消耗灵材**。方向选择后，该方向的属性在候选生成时获得 1.5 倍权重加成。玩家也可选择「无偏向」保持原始权重——给追求特定构筑的玩家更多策略代理权，同时保持机缘的随机本质（支柱3）

#### 2. 副属性池

| 副属性 | 效果 | 初始权重 | 品质梯级 | 境界门槛 |
|--------|------|:--------:|:--------:|:--------:|
| 攻击+1 | 角色攻击力+1 | 22 | T1 | 无 |
| 防御+1 | 角色防御力+1 | 18 | T1 | 无 |
| 暴击+3% | 暴击概率+3%（加法叠加） | 15 | T2 | 无 |
| 暴击伤害+5% | 暴击时额外+5%伤害（加法叠加） | 12 | T2 | 无 |
| 生命+2 | 角色最大生命+2 | 10 | T3 | 无 |
| 吸血+2% | 造成伤害时回复伤害值2%的HP（每层加法叠加，向下取整）。例：1层=2%，2层=4% | 8 | T3 | 无 |
| 「虚弱」 | 攻击时使目标本回合伤害-1（可叠加层数） | 6 | T3 | 无 |
| 费用-1 | 该法宝卡费用-1（最低为1，不叠加） | 4 | T4（稀有） | 筑基期+（L≥2） |
| 回合开始回HP+1 | 装备角色每回合开始回复1HP（可叠加） | 3 | T4（稀有） | 筑基期+（L≥2） |
| 「破甲」 | 攻击时使目标本回合防御-2（可叠加层数） | 3 | T4（稀有） | 筑基期+（L≥2） |
| 「灵力萃取」 | 装备角色每回合开始获得+1临时灵力（仅本回合可用，可叠加） | 2 | T4（稀有） | 筑基期+（L≥2） |

#### 3. 品质梯级与权重规则

副属性按稀有度分为 T1~T4 四个梯级：
- **T1**（攻击/防御）：最常见的基础数值
- **T2**（暴击/暴伤）：中频概率属性
- **T3**（生命/吸血/虚弱）：较稀有，偏战术效果
- **T4**（费用-1/回合回血/破甲/灵力萃取）：极其稀有，改变战斗节奏。仅境界L≥2（筑基期+）可获取

**候选生成规则：**
```
generate_candidates(existing_inscriptions, realm_level, to_replace_idx, direction=None):
  # Step 1: 计算有效已有属性列表（排除即将被替换的属性）
  effective_existing = existing_inscriptions 复制
  if to_replace_idx is not None:
    effective_existing 移除 to_replace_idx 位置的属性
  
  # Step 2: 计算基础权重
  weights = {
    攻击+1:22, 防御+1:18,
    暴击+3%:15, 暴伤+5%:12,
    生命+2:10, 吸血+2%:8, 虚弱:6,
    费用-1:4, 回HP+1:3, 破甲:3, 灵力萃取:2
  }
  
  # Step 2.5: 定向铭刻方向加权（如有选择，在境界加成前应用）
  if direction == "攻击向":
    weights["攻击+1"] = floor(weights["攻击+1"] × 1.5)
    weights["暴击+3%"] = floor(weights["暴击+3%"] × 1.5)
    weights["暴伤+5%"] = floor(weights["暴伤+5%"] × 1.5)
  elif direction == "防御向":
    weights["防御+1"] = floor(weights["防御+1"] × 1.5)
    weights["生命+2"] = floor(weights["生命+2"] × 1.5)
    if realm_level >= 2:
      weights["回HP+1"] = floor(weights["回HP+1"] × 1.5)
  elif direction == "战术向":
    weights["吸血+2%"] = floor(weights["吸血+2%"] × 1.5)
    weights["虚弱"] = floor(weights["虚弱"] × 1.5)
    if realm_level >= 2:
      weights["破甲"] = floor(weights["破甲"] × 1.5)
      weights["灵力萃取"] = floor(weights["灵力萃取"] × 1.5)
  # "无偏向" 或不传 direction 则跳过此步
  
  # Step 3: 境界加成先应用（优先于惩罚，确保T4受罚时与T1-T3公平）
  if realm_level >= 2:
    bonus = floor(realm_level × 2)
    weights["费用-1"] += bonus
    weights["回HP+1"] += bonus
    weights["破甲"] += bonus
    weights["灵力萃取"] += bonus
  else:
    # 炼气期：T4 属性从池中移除
    weights 移除 "费用-1", "回HP+1", "破甲", "灵力萃取"
  
  # Step 4: 已有相同副属性时权重减半（基于有效已有列表，含已加成的T4）
  # 最少保留权重1，防止三叠同属性时归零
  for each stat in effective_existing:
    if stat in weights and weights[stat] > 0:
      weights[stat] = max(1, floor(weights[stat] × 0.5))
  
  # Step 5: 不放回抽取 3 个互不相同的候选
  candidates = weighted_sample_without_replacement(weights, 3)
  return candidates
```

**权重叠加规则：**
- 暴击+3%：**可叠加**——3条暴击+3% = +9%暴击率
- 暴击伤害+5%：**可叠加**——3条暴伤+5% = +15%暴击额外伤害
- 费用-1：**不叠加**——即使有2条费用-1也只生效1条，法宝最低1费
- 回合回HP+1：**可叠加**——2条=每回合回2HP
- 吸血+2%：**可叠加**——2条=造成伤害时回复伤害值4%的HP（加法叠加）
- 虚弱：**可叠加**——攻击时叠加层数。2条虚弱=每次攻击使目标本回合伤害-2
- 破甲：**可叠加**——攻击时叠加层数。2条破甲=每次攻击使目标本回合防御-4
- 灵力萃取：**可叠加**——2条=每回合开始获得+2临时灵力

#### 4. 铭刻对已满法宝的处理

当法宝已有 3 条副属性时再次铭刻：
1. 玩家点击铭刻 → 确认消耗 N × 中级灵材×1（N=该法宝累计铭刻次数+1），灵材立即扣除
2. 玩家**手动选择 1 条已有副属性**作为被替换目标
3. 系统按权重 roll 出 3 个候选副属性（排除被替换的那条，但可能 roll 到其他已有属性），所有候选互不相同
4. 玩家从 3 个候选中选择 1 个，替换生效，不可撤销
5. 步骤 2 中玩家可以取消操作——但**灵材已在步骤1扣除，不可退还**。取消仅回退到法宝详情，不消耗额外资源

> **设计理由：** 让玩家选择替换哪条属性而非随机，同时让玩家从多个候选中选择新属性——每一步都是策略决策。递增成本创造自然的投入上限：铭刻越多越贵，玩家需要权衡"继续投入还是接受现状"。

#### 5. 法宝升级与铭刻的关系

- 法宝卡牌升级（消耗灵石+同名卡）不改变已有铭刻
- 铭刻属性在法宝升级后保留——副属性与等级独立成长
- 卡牌拆解时，铭刻副属性对拆解价值有影响：**返还铭刻总消耗灵材的 50%（向下取整）**。例如：铭刻了 3 次（累计消耗 1+2+3=6 灵材）→ 拆解额外返还 3 灵材

### 状态与转换

```
法宝状态: [无铭刻] → 铭刻第1次(1灵材) → [1条副属性] → 铭刻第2次(2灵材) → [2条副属性] → 铭刻第3次(3灵材) → [3条副属性]
                                                                                                ↓
                                                                                          [3条满] → 铭刻第N次(N灵材)→玩家选择替换哪条→系统roll 3候选→玩家选1→替换生效
```

## 公式

### 1. 候选属性生成

```
generate_candidates(existing_inscriptions, realm_level, to_replace_idx, direction=None) → Substat[3]:
  base_weights = {
    "atk+1":22, "def+1":18,
    "crit+3":15, "crit_dmg+5":12,
    "hp+2":10, "lifesteal+2%":8, "weakness":6,
    "cost-1":4, "regen+1":3, "armor_break":3, "mana_extract":2
  }
  
  # 构建有效已有列表（排除被替换属性）
  effective = []
  for i in [0..len(existing_inscriptions)-1]:
    if i != to_replace_idx:
      effective.append(existing_inscriptions[i])
  
  # 定向铭刻方向加权（如有选择，在境界加成前应用）
  if direction == "attack":
    base_weights["atk+1"] = floor(base_weights["atk+1"] × 1.5)
    base_weights["crit+3"] = floor(base_weights["crit+3"] × 1.5)
    base_weights["crit_dmg+5"] = floor(base_weights["crit_dmg+5"] × 1.5)
  elif direction == "defense":
    base_weights["def+1"] = floor(base_weights["def+1"] × 1.5)
    base_weights["hp+2"] = floor(base_weights["hp+2"] × 1.5)
    if realm_level >= 2:
      base_weights["regen+1"] = floor(base_weights["regen+1"] × 1.5)
  elif direction == "tactical":
    base_weights["lifesteal+2%"] = floor(base_weights["lifesteal+2%"] × 1.5)
    base_weights["weakness"] = floor(base_weights["weakness"] × 1.5)
    if realm_level >= 2:
      base_weights["armor_break"] = floor(base_weights["armor_break"] × 1.5)
      base_weights["mana_extract"] = floor(base_weights["mana_extract"] × 1.5)
  # direction=None 或不传 → 跳过，保持原始权重
  
  # 境界加成先应用（优先于惩罚，确保T4受罚时与T1-T3公平）
  if realm_level >= 2:
    bonus = floor(realm_level × 2)
    base_weights["cost-1"] += bonus
    base_weights["regen+1"] += bonus
    base_weights["armor_break"] += bonus
    base_weights["mana_extract"] += bonus
  else:
    # 炼气期：移除 T4
    移除 "cost-1", "regen+1", "armor_break", "mana_extract" from base_weights
  
  # 费用-1 已存在时从池中完全移除（不叠加 → 死抽候选，不应出现）
  if "cost-1" in effective:
    移除 "cost-1" from base_weights
  
  # 已有相同属性权重减半（作用于已含加成后的权重）
  # max(1, ...) 防止三叠同属性归零
  for stat in effective:
    if stat in base_weights:
      base_weights[stat] = max(1, floor(base_weights[stat] × 0.5))
  
  # 不放回抽取 3 个互不相同的候选
  candidates = weighted_sample_without_replacement(base_weights, 3)
  return candidates
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|:----:|------|
| existing_inscriptions | string[] | [0, 3] | 当前法宝已有副属性列表（card-system.md 中定义为 `inscriptions`） |
| realm_level | int | [1, 5] | 当前境界层级（1=炼气, 2=筑基, 3=金丹, 4=元婴, 5=化神） |
| to_replace_idx | int or null | [0, 2] or null | 玩家选中的被替换属性索引；无替换时为 null |
| direction | string or null | "attack" / "defense" / "tactical" / null | 定向铭刻方向；null=无偏向 |
| bonus | int | [2, 10] | 稀有属性权重加成（仅 L≥2） |

**示例 1：** 金丹期（L=3）玩家铭刻空法宝，选择「攻击向」→ 攻击/暴击/暴伤权重×1.5 后 bonus=6，roll 3 个不同候选（含 T4），玩家选 1 个

**示例 2：** 炼气期（L=1）玩家铭刻 → T4 从池中移除，仅 T1-T3 进入候选池。玩家选择「无偏向」→ 原始权重

**示例 3：** 法宝已有 [atk+1, def+1]，玩家选择替换 def+1（idx=1），to_replace_idx=1 → effective=["atk+1"]，atk+1 权重 22→max(1,11)=11，def+1 被排除不参与惩罚

**示例 4：** 法宝已有 [atk+1, cost-1, regen+1]（L=5），to_replace_idx=null → effective=["atk+1","cost-1","regen+1"]，费用-1 从池中完全移除（已存在且不叠加），atk+1→11，regen+1→max(1,floor(13×0.5))=6

### 3. 铭刻费用（递增）

```
inscribe_cost(artifact_inscription_count) → (item, qty):
  N = min(artifact_inscription_count + 1, 5)  # 该法宝累计铭刻次数+1，最高5灵材/次
  return (中级灵材, N)
```

| 铭刻次数 | 消耗灵材 | 累计消耗 |
|:--------:|:--------:|:--------:|
| 第 1 次 | 1 | 1 |
| 第 2 次 | 2 | 3 |
| 第 3 次 | 3 | 6 |
| 第 4 次 | 4 | 10 |
| 第 5 次 | 5 | 15 |
| 第 6 次及以后 | 5（软上限） | 15+5×(N-5) |

> **灵材收入参考**（交叉引用 resource-system.md）：炼气期单图预期中级灵材收入约 3-5，筑基期约 7-10。前 3 次铭刻填满一件法宝需 6 灵材——约等于炼气期 1-2 张地图的全部中级灵材产出。第 6 次起的 5 灵材/次软上限确保成本始终可承受但不低廉。

### 4. 拆解返还

```
disenchant_inscription_refund(total_materials_spent_on_inscriptions) → int:
  if total_materials_spent_on_inscriptions == 0:
    return 0  # 从未铭刻过的法宝拆解不返还额外灵材（防止零投入空手套灵材）
  refund = floor(total_materials_spent_on_inscriptions × 0.5)
  return max(1, refund)  # 至少返还1灵材（即使只铭刻1次）
```

## 边界情况

- **首次铭刻**：空法宝首次铭刻——确认消耗灵材后 roll 3 个不同候选（炼气期无 T4），玩家选 1 个加入 inscriptions[]。灵材已扣，不可取消
- **炼气期铭刻**：T4 属性不可用——候选池仅含 T1-T3（攻击/防御/暴击/暴伤/生命/吸血/虚弱，共 7 种）
- **铭刻成本递增**：同一法宝的第 4 次铭刻需 4 灵材——若灵材不足，铭刻按钮灰显并提示所需数量
- **铭刻确认即扣灵材**：点击铭刻确认后灵材立即扣除，不退还。即使后续在替换选择界面或候选选择界面取消，灵材已消耗——此设计确保每次铭刻都是经过慎重考虑的投入（支柱2）
- **3条满后铭刻**：弹出替换选择界面，灵材在确认铭刻时已扣。玩家选 1 条要替换的旧属性 → 系统 roll 3 个不同候选（排除被替换属性）→ 玩家选 1 个新属性 → 替换生效
- **玩家反悔（替换界面/候选界面）**：玩家可取消操作退回法宝详情——但灵材已在确认铭刻时扣除，不可退还。法宝状态不变
- **所有副属性完全相同（3条攻击+1）**：允许——概率极低但理论上可行。这是玩家可以自豪的「极品法宝」
- **费用-1叠加**：费用-1不叠加——即使有2条费用-1也只生效1条，法宝最低1费
- **回合回HP叠加**：可叠加——2条「回合开始回HP+1」=每回合回2HP
- **吸血叠加**：可叠加——2条「吸血+2%」=造成伤害时回复伤害值4%的HP（加法叠加）
- **虚弱叠加**：可叠加——攻击时叠加层数。2条虚弱=每次攻击使目标本回合伤害-2
- **破甲叠加**：可叠加——攻击时叠加层数。2条破甲=每次攻击使目标本回合防御-4
- **灵力萃取叠加**：可叠加——2条灵力萃取=每回合开始获得+2临时灵力（仅本回合可用）
- **暴击叠加**：可叠加——3条「暴击+3%」=+9%暴击率（加法叠加）
- **暴击伤害叠加**：可叠加——3条「暴击伤害+5%」=+15%暴击额外伤害（加法叠加）
- **虚弱与破甲持续回合**：debuff效果仅持续到敌人本回合结束（与「本回合」绑定），不跨回合累积
- **灵力萃取的临时灵力**：获得的灵力仅限本回合使用，回合结束时清空。不影响下回合基础灵力
- **铭刻时灵材不足**：提示「需要中级灵材×N」（N为本次铭刻所需数量），按钮灰显不可操作
- **铭刻后立即生效**：铭刻完成后不重新抽牌——当前战斗中已抽到的法宝不受新铭刻影响，下一场战斗生效
- **铭刻不消耗行动力**：在营寨中或探索界面铭刻时不消耗任何行动力
- **拆解铭刻法宝**：返还铭刻总消耗灵材的 50%（向下取整，至少返1）。例如铭刻 1 次（消耗 1 灵材）→ 返还 max(1, floor(1×0.5)) = 1 灵材；铭刻 2 次（1+2=3 灵材）→ 返还 floor(3×0.5)=1 灵材；铭刻 3 次（1+2+3=6 灵材）→ 返还 floor(6×0.5)=3 灵材
- **候选不够 3 个**：如果可用属性池少于 3 种（例如炼气期仅 7 种 T1-T3，且 3 种已有且权重减半后可能仍在池中），始终 roll 出所有可用属性作为候选（至少 1 个）
- **多次铭刻后累计次数重置**：每件法宝独立计数。同一件法宝的铭刻次数从 0 开始，拆解后计数消失

## 验收标准

### 核心铭刻流程

- **AC-1**：GIVEN 玩家境界=炼气(L=1)，背包有中级灵材≥1，装备空铭刻法宝(副属性=0条)，WHEN 点击铭刻按钮确认消耗 1 灵材，THEN 展示 3 个互不相同的候选副属性（均为 T1-T3，不含 T4），玩家选择 1 个后灵材-1，法宝获得选中的那条副属性
- **AC-2**：GIVEN 玩家境界=炼气，背包中级灵材=0，装备可铭刻法宝，WHEN 打开铭刻界面，THEN 铭刻按钮灰显(不可点击)，按钮上显示"灵材不足，需要中级灵材×1"
- **AC-3**：GIVEN 玩家点击铭刻按钮，系统弹出确认对话框显示「确认后立即消耗 1 中级灵材。铭刻候选展示后即使取消，灵材也不会退还。是否继续？」，WHEN 点击"取消"（不确认铭刻），THEN 灵材不消耗，返回法宝详情，法宝副属性不变

### 递增成本

- **AC-4**：GIVEN 玩家法宝已铭刻 1 次(副属性=1条)，背包中级灵材≥2，WHEN 点击铭刻，THEN 铭刻确认显示消耗为 2 中级灵材，确认后灵材-2，法宝获得第 2 条副属性
- **AC-5**：GIVEN 玩家法宝已铭刻 2 次(副属性=2条)，背包中级灵材≥3，WHEN 点击铭刻，THEN 铭刻确认显示消耗为 3 中级灵材，确认后灵材-3，法宝获得第 3 条副属性

### T4 境界门槛（单元测试 — 直接验证 generate_candidates() 函数）

- **AC-6**：GIVEN 调用 `generate_candidates(existing_inscriptions=[], realm_level=1, to_replace_idx=null)`，THEN 返回的 weights 字典中不含 T4 属性键（"cost-1", "regen+1", "armor_break", "mana_extract"）
- **AC-7**：GIVEN 调用 `generate_candidates(existing_inscriptions=[], realm_level=2, to_replace_idx=null)`，THEN 返回的 weights 字典中含全部 4 个 T4 属性键，且 bonus = 4
- **AC-7a**：GIVEN 调用 `generate_candidates(existing_inscriptions=[], realm_level=2, to_replace_idx=null, direction="attack")`，THEN weights["atk+1"] = floor(22 × 1.5) = 33，weights["crit+3"] = floor(15 × 1.5) = 22，weights["crit_dmg+5"] = floor(12 × 1.5) = 18

### 重复属性权重减半（单元测试 — 直接验证权重计算）

- **AC-8**：GIVEN 调用 `generate_candidates(existing_inscriptions=["atk+1"], realm_level=2, to_replace_idx=null)`，THEN weights["atk+1"] = floor(22 x 0.5) = 11，weights["def+1"] = 18（未受惩罚）
- **AC-8a**：GIVEN 调用 `generate_candidates(existing_inscriptions=["atk+1", "atk+1"], realm_level=2, to_replace_idx=0)`（替换一条 atk+1），THEN effective = ["atk+1"]（被替换的 idx=0 不计入），weights["atk+1"] = floor(22 x 0.5) = 11（仅一条受罚）
- **AC-8b**：GIVEN 调用 `generate_candidates(existing_inscriptions=["cost-1"], realm_level=5, to_replace_idx=null)`，THEN weights["cost-1"] = max(1, floor((4 + 10) x 0.5)) = 7（境界加成优先于惩罚，T4 属性与 T1 公平受罚）
- **AC-8c**：GIVEN 调用 `generate_candidates(existing_inscriptions=["weakness", "weakness", "weakness"], realm_level=1, to_replace_idx=null)`（三叠同属性），THEN weights["weakness"] = max(1, floor(6 x 0.5 x 0.5 x 0.5)) = 1（权重归零保护，不会从池中消失）

### 3条满替换流程

- **AC-9**：GIVEN 玩家法宝已有 3 条副属性[攻击+1, 防御+1, 暴击+3%]，背包中级灵材=4(第 4 次铭刻需 4 灵材)，WHEN 点击铭刻确认（灵材-4），THEN 进入替换选择界面，列出 3 条副属性且每条旁有"替换"按钮
- **AC-10**：GIVEN 当前在替换选择界面，玩家选中"防御+1"，WHEN 展示 3 个候选后玩家选 1 个确认，THEN 法宝副属性变为[攻击+1, 新属性, 暴击+3%]，防御+1 消失
- **AC-11**：GIVEN 当前在替换选择界面（灵材已在确认时扣除），WHEN 点击取消，THEN 返回法宝详情，法宝 3 条副属性不变，已消耗的灵材不退还

### 战斗中副属性生效

- **AC-12**：GIVEN 玩家装备法宝有「攻击+1」，角色基础攻击力=10，进入战斗，WHEN 查看角色属性面板，THEN 攻击力显示为 11
- **AC-13**：GIVEN 玩家装备法宝有「吸血+2%」，角色当前 HP=50/100，进入战斗，WHEN 角色对敌人造成 10 点伤害，THEN 角色 HP 从 50 变为 50（10×2%=0.2→floor=0，不回复）；WHEN 角色对敌人造成 100 点伤害，THEN 角色 HP 从 50 变为 52（100×2%=2）
- **AC-14**：GIVEN 玩家装备法宝有「虚弱」，敌人攻击力=5，WHEN 玩家角色使用普通攻击命中该敌人，THEN 该敌人在其本回合内造成的伤害从 5 降低至 4
- **AC-15**：GIVEN 玩家装备法宝有「破甲」，敌人防御力=10，WHEN 玩家角色使用普通攻击命中该敌人，THEN 该敌人在其本回合内的防御力从 10 降至 8
- **AC-16**：GIVEN 玩家装备法宝有「灵力萃取」，当前灵力=5，WHEN 玩家回合开始，THEN 灵力显示为 6；WHEN 玩家点击"结束回合"且下一回合开始，THEN 灵力显示为 6（上回合临时灵力已清空，新回合萃取+1）
- **AC-17**：GIVEN 玩家装备法宝有「回合开始回HP+1」，角色当前 HP=50/100，WHEN 玩家回合开始，THEN HP 从 50 变为 51
- **AC-18**：GIVEN 玩家装备法宝有 2 条「灵力萃取」，当前灵力=5，WHEN 玩家回合开始，THEN 灵力显示为 7（+2 临时灵力）
- **AC-19**：GIVEN 玩家装备法宝有「费用-1」，法宝原费用=3，WHEN 查看该法宝卡牌，THEN 卡牌显示费用为 2
- **AC-19a**：GIVEN 玩家装备法宝有「费用-1」，法宝原费用=1，WHEN 查看该法宝卡牌，THEN 卡牌显示费用为 1（最低1规则——不会降为0）
- **AC-20**：GIVEN 玩家装备法宝有「暴击+3%」2 条，WHEN 查看角色属性面板，THEN 暴击率显示为基础暴击率 +6%

### 拆解返还

- **AC-21**：GIVEN 玩家法宝累计铭刻 1 次（消耗 1 中级灵材），WHEN 拆解该法宝，THEN 玩家获得基础拆解灵石 + 额外 1 中级灵材（至少返1，max(1, floor(1x0.5))=1）
- **AC-21a**：GIVEN 玩家法宝累计铭刻 2 次（消耗 1+2=3 中级灵材），WHEN 拆解该法宝，THEN 玩家获得基础拆解灵石 + 额外 1 中级灵材（floor(3x0.5)=1）

### 集成测试

- **AC-22**：GIVEN 玩家装备 2 件法宝各有「攻击+1」，WHEN 查看角色属性面板，THEN 攻击力 = 基础攻击力 +2（两件法宝均生效）
- **AC-22a**：GIVEN 玩家装备 2 件法宝各有「防御+1」，WHEN 查看角色属性面板，THEN 防御力 = 基础防御力 +2
- **AC-22b**：GIVEN 玩家装备法宝A有「攻击+1」、法宝B有「暴击+3%」，WHEN 查看角色属性面板，THEN 攻击力+1 且 暴击率+3%（不同属性独立加成）
