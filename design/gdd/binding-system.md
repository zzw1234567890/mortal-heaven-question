# 功法/法宝绑定系统 (Gongfa/Artifact Binding System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户

## 概述

功法/法宝绑定系统是连接卡牌系统与战斗系统的核心桥梁，负责管理功法卡和法宝卡与场上角色之间的绑定关系、本命加成判定、绑定生命周期以及解绑逻辑。系统记录每个角色的绑定位状态（最多3功法+3法宝）、绑定效果引用、本命绑定标识，并在角色阵亡时触发绑定卡回收（进入弃牌堆）。

对玩家来说，绑定系统是构筑策略从"卡组编辑界面"延伸到"战斗执行层面"的关键节点。每次打出功法/法宝卡时选择目标角色的决策，决定了流派能否成型、本命加成能否激活。绑定不是一次性的"装备"——它是一场赌博：绑定越强，角色阵亡时损失越大。这种"投入产出比"的计算是绑定系统的核心策略深度。

暂无相关ADR。

## 玩家幻想

### 创意总监声明 (Creative Director's Vision)

绑定系统是玩家在战斗中最具"分量"的决策点之一。在这个系统中，卡牌不再是一次性消耗品——它们是你在角色身上的"投资"。玩家需要权衡：把这张金色功法绑定到这个主力角色上，能让他变得极强，但万一他阵亡了呢？绑定系统的核心情感张力正是**"不舍得，但必须用"**——最强的效果往往绑定最深的投入，而投入越深，失去的风险越大。

我确立以下幻想方向，作为所有下游设计的创意北极星。

### 1. 核心情感：修士的契约 (The Cultivator's Bond)

| 情感层 | 触发场景 | 强度 |
|--------|----------|------|
| **托付的慎重** | 决定将哪张功法/法宝绑定到哪个角色 | 每次出牌都是一次托付 |
| **羁绊的满足** | 本命加成激活（×1.5金色标识亮起） | 中频——构筑成型的回报 |
| **未雨绸缪的焦虑** | 主力角色残血时——"他不能死，他身上绑着我的核心卡" | 高频——每场战斗的潜在压力 |
| **战损的清醒** | 角色阵亡，绑定卡进弃牌堆——"我失去了它" | 低频但沉重 |
| **重生的空荡** | 大还丹复活角色，但绑定位空空如也 | 低频——"重新开始"的感觉 |

**关键区别：** 大多数卡牌游戏中，"装备"是随时可更换的。本游戏的绑定系统让"装备"变成了"契约"——你把一张功法/法宝绑定到角色上，就是与这个角色立下了不可逆的约定。角色的安危直接影响你的构筑完整性。

绑定系统的稀缺性随境界动态变化：

| 境界 | 上场角色 | 每角色功法+法宝 | 总功法+法宝容量 |
|:----:|:-------:|:--------------:|:--------------:|
| 炼气 | 2 | 1 + 1 | 2 + 2 = **4** |
| 筑基 | 3 | 2 + 2 | 6 + 6 = **12** |
| 金丹 | 4 | 2 + 2 | 8 + 8 = **16** |
| 元婴 | 5 | 3 + 3 | 15 + 15 = **30** |
| 化神 | 6 | 3 + 3 | 18 + 18 = **36** |

炼气期精打细算——你只能带2张功法+2张法宝（共4个绑定位），每一张的选择都至关重要。到化神期则有36个绑定位，构筑空间大幅释放。这种成长本身就告诉玩家：**"你变强了——你体内能容纳的功法越来越多了。"**

## 详细设计

### 核心规则

#### 1. 绑定数据结构

```
BindingRecord = {
  binding_id: String            # 唯一标识
  card_instance_id: String      # 卡牌实例ID（区分同名卡的不同副本）
  card_template_id: String      # 卡牌模板ID（用于同名判定）
  card_name: String             # 卡牌显示名称
  card_rarity: Enum             # 稀有度（用于销毁特效和拆解产出）
  slot_type: Enum [功法, 法宝]  # 绑定位类型
  slot_index: int               # 在绑定位数组中的索引
  bound_character_id: String    # 绑定的角色ID
  is_native: bool               # 是否激活本命加成
  native_multiplier: float     # 1.0 或 1.5（绑定时预计算，不运行时重查）
  activated_turn: int          # 绑定的回合数（用于效果排序）
  accumulated_bonus: Dictionary # 已积累的数值加成（覆盖时保留）
  is_suspended: bool           # 角色离场时为true，重新上场恢复
}
```

**全局绑定注册表（BindingManager Autoload）：**

| 查询 | 数据结构 | 更新时机 |
|------|---------|---------|
| "角色X绑了哪些卡？" | 角色持有 `BindingSlots` Resource | 绑定/解绑时 |
| "某功法/法宝在谁身上？" | `Dictionary[card_instance_id → character_id]` | 绑定/解绑时 |

#### 2. 绑定位成长

| 境界 | 上场角色 | 每角色功法 | 每角色法宝 | 总功法+法宝 | 本命功法位 | 本命法宝位 |
|:----:|:-------:|:---------:|:---------:|:----------:|:---------:|:---------:|
| 炼气 | 2 | 1 | 1 | 2+2=4 | 1 | 0 |
| 筑基 | 3 | 2 | 2 | 6+6=12 | 1 | 1 |
| 金丹 | 4 | 2 | 2 | 8+8=16 | 1 | 1 |
| 元婴 | 5 | 3 | 3 | 15+15=30 | 1 | 1 |
| 化神 | 6 | 3 | 3 | 18+18=36 | 1 | 1 |

**关键规定**：
- 每个角色最多拥有1个本命功法和1个本命法宝
- 炼气期无法宝位 → 无法宝本命
- 绑定位数量随大境界突破自动成长（无需额外解锁）

#### 3. 正常绑定流程（战斗阶段2：出牌阶段）

```
① 玩家打出功法/法宝卡 → 扣除费用
② 检查目标选择范围：场上己方角色（阵亡/不可选角色灰色）
③ 检查同名卡冲突：
   - 目标角色已绑定同名卡 → 卡牌变为不可用（灰色+提示"已绑定同名"）
   - 无冲突 → 可正常选择
④ 弹出角色选择面板，角色按状态分层：
   - 有空位 → 蓝色高亮 ← 可选中
   - 已满位（可覆盖）→ 橙色高亮 ← 可选中，提示"替换"
   - 已满位且仅剩同名冲突 → 灰色 ← 不可选中
   - 阵亡/限制 → 灰色遮罩
⑤ 玩家选择目标角色
⑥ 系统检查绑定位有空或满，进入对应分支：
   a. 有空位 → 分配空位索引 → 创建BindingRecord
   b. 无空位 → 进入覆盖流程（见规则4）
⑦ 本命判定（native_owner固定，非玩家可选）：
   a. 卡牌 native_owner 前缀匹配角色card_id？
      - 不匹配 → is_native=false, native_multiplier=1.0（无本命资格）
      - 匹配 → 进入步骤b
   b. 该角色同类型的本命位（功法位/法宝位）是否已被占用？
      - 未占用 → 自动设为本命：is_native=true, native_multiplier=1.5
      - 已占用 → is_native=false, native_multiplier=1.0（本命位已满）
   c. 本命绑定一旦确认（无弹窗确认——由卡牌 fixed native_owner 自动决定），在该角色的本次生命周期内不可变更
⑧ 注册到BindingManager（O(1)查询）
⑨ 通知效果引擎：绑定生效
⑩ 卡牌从手牌移除，动画：卡牌飞向角色头像区域
```

#### 4. 覆盖绑定流程

```
① 玩家在已满位角色上确认覆盖
② 预览信息：
   - 旧卡从槽位浮出半透明显示（卡名+核心效果）
   - 新卡叠在上方对比
   - 提示"旧卡将进入弃牌堆"
③ 效果移除序列（严格有序，原子操作）：
   a. BindingManager查询旧卡的所有持续效果
   b. 通知效果引擎：remove_effects_by_source(旧卡instance_id)
      - 持续触发效果停止
      - 积累的数值加成（accumulated_bonus）保留在角色上
   c. 旧卡进入弃牌堆（从"场_已绑定"状态变为"弃牌堆"）
   d. 从BindingManager.card_to_character移除
   e. 从BindingSlots数组移除旧BindingRecord
④ 新卡占用空出的槽位索引
⑤ 创建新BindingRecord，本命判定（规则3步骤⑦）
⑥ 注册到BindingManager
⑦ 通知效果引擎：新绑定生效
⑧ 0.8s反悔窗口（点击卡牌可取消→新卡回手牌，费用不返还）
```

**关键规则**：旧效果移除 → 新效果注册，严格顺序进行，无重叠帧。

#### 5. 绑定生效时机

- 打出后**立即生效**（同一出牌阶段内）
- 效果引擎收到"绑定生效"事件 → 注册持续效果
- 本命加成在绑定瞬间判定并锁定（角色存活期间不变更）

#### 6. 角色离场与重新上场

```
角色下场（阵型调整/人数减少）：
  a. 所有BindingRecord → is_suspended = true
  b. 效果引擎：suspend_effects_by_source(所有绑定卡)
  c. 绑定卡不进弃牌堆
  d. 阵法系统：角色数变更 → 重查阵法激活条件

角色重新上场：
  a. 遍历所有is_suspended=true的BindingRecord
  b. 验证card_instance_id是否仍存在于收藏池中
     - 存在 → is_suspended = false，恢复效果
     - 已不存在（被覆盖/已丢弃）→ 删除BindingRecord，槽位为空
  c. 效果引擎：restore_effects_by_source(恢复的绑定卡)
  d. 阵法系统：角色数变更 → 重查阵法激活条件
```

#### 7. 角色阵亡与大还丹复活

```
角色阵亡：
  a. 每张绑定卡独立处理（非批量移除）
  b. 本命卡使用金色火焰销毁特效，普通卡用蓝色
  c. 旧卡：绑定状态 → 弃牌堆（永久失去）
  d. BindingManager：清除该角色所有条目
  e. 效果引擎：remove_effects_by_source(所有绑定卡)
  f. 阵法系统：角色阵亡 → 重查阵法激活条件

大还丹复活：
  a. 角色复活，状态 = 空载
  b. BindingSlots：完全清空
  c. 原绑定卡已在弃牌堆（不可恢复）
  d. 玩家可在后续回合重新绑定新卡
```

#### 8. 本命绑定规则

**核心原则**：本命宿主是**功法/法宝卡牌固定定义**的（数据配置），不因操作改变。角色只能占用自己的本命位，不能抢其他角色的本命。

**一个角色可能对应多张本命卡（同一native_owner覆盖多张卡），但只能激活一个本命位：**
- 韩立的 `native_owner` 覆盖的卡池可能包含：青元剑诀、大衍诀、梵圣真魔功等多张功法卡
- 韩立可以绑定所有这些功法（只要绑定位够），但**最多只能有1个本命功法生效（×1.5）**
- 第一张绑定到韩立且native_owner匹配的功法卡自动占用了本命功法位
- 后续绑定到韩立的匹配卡（如大衍诀）仍然匹配native_owner，但本命功法位已满→自动普通绑定×1.0

**本命绑定锁定规则：**
- 本命绑定由native_owner固定匹配**自动判定**，无玩家弹窗选择
- 一旦某张匹配卡占用了角色的本命位，在该角色的本次生命周期内**不可解除、不可变更**
- 如果玩家想换本命，只有覆盖（旧卡进弃牌堆，新卡可重新占用本命位）或角色阵亡后重新绑定
- 角色阵亡后，该角色的本命资格消失，但native_owner仍然固定在卡牌定义中——其他角色即使绑定了这些卡也**永远无法获得×1.5**

**常见场景：**

| 场景 | 结果 |
|------|------|
| 韩立绑定"青元剑诀"（native_owner=hanli） | ✅ 本命 ×1.5（匹配+本命位空） |
| 韩立已有本命，再绑定"大衍诀"（native_owner=hanli） | ✅ 普通绑定 ×1.0（本命位已被占） |
| 韩立绑定"长春功"（native_owner=none） | ✅ 普通绑定 ×1.0（不匹配） |
| 韩立阵亡→紫灵绑定"青元剑诀#001"（native_owner=hanli） | ✅ 普通绑定 ×1.0（紫灵不匹配native_owner） |
| 紫灵绑定"紫凝冰诀"（native_owner=ziling） | ✅ 本命 ×1.5（匹配+本命位空） |

#### 9. 覆盖时积累数值保留

- 被覆盖的旧卡**已积累的数值加成**（如"每击杀一人攻击+1"已叠加的层数）保留在角色上
- 旧卡的**持续触发效果**停止（不再响应触发器）
- 新卡的数值和效果替换旧卡的剩余部分
- 这种半保留机制让玩家在覆盖时不会失去前期投入，但战略上仍希望谨慎选择

#### 10. 同名卡绑定限制

- 同一角色不可绑定两张同名功法/法宝
- 打出同名卡时 → 目标角色该卡灰色不可用，提示"已绑定同名"
- 不同角色可各自绑定同名卡（独立计算）

#### 11. 敌方绑定信息可见性

| 信息层级 | 可见性 |
|---------|--------|
| 敌方角色的绑定槽位数 | 公开可见 |
| 敌方绑定的卡名/图标 | 战斗开始时可查看一次，之后从卡背推测 |
| 敌方绑定卡的具体效果 | 不可见（玩家通过游戏知识推测） |
| 本命标记 | 如果玩家之前已见过该卡则可见 |

### 状态与转换

#### 绑定位状态

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **空位** | 可用 | 角色上场、角色复活（空载） | 绑定新卡→已占用 |
| **已占用（普通）** | 已绑定，×1.0 | 绑定成功（非本命） | 被覆盖→弃牌堆；阵亡→弃牌堆 |
| **已占用（本命）** | 已绑定，×1.5 | 本命绑定成功 | 被覆盖（需额外确认）→弃牌堆；阵亡→弃牌堆 |
| **离场保留** | 角色离场但绑定保留 | 角色下场 | 角色重新上场→恢复为已占用 |
| **空载** | 复活后无绑定 | 大还丹复活 | 重新绑定→已占用 |

#### 卡牌生命周期（绑定相关）

```
手牌 → 打出 → 已绑定（场上） → 角色阵亡 → 弃牌堆（永久失去）
                                    ↕
                              角色离场（保留）
                                    ↓
                              被覆盖 → 弃牌堆（旧卡失去）
```

## 公式

### 1. 本命加成

```
effective_value = base_value × native_multiplier
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| base_value | int | 按卡牌定义 | 效果基准数值（来自卡牌模板） |
| native_multiplier | float | 1.0 或 1.5 | 绑定时预计算并锁定，不运行时重查 |

本命加成公式引用自 **卡牌效果引擎** `card-effect-engine.md`。绑定系统负责在绑定时计算并存储 `native_multiplier`，效果引擎负责运行时使用此值计算最终效果。

### 2. 覆盖时积累保留规则

```
overwrite_result(character, old_card, new_card):
  trigger(old_card) ← false        # 旧卡的持续触发效果停止
  keep(accumulated_from_old_card)  # 旧卡的已积累数值加成保留在角色上
  register(new_card)               # 新卡效果注册
```

此规则是逻辑约定，非数值公式。积累的数值（如"每击杀一人攻击+1"已叠加的层数）在覆盖后仍对角色的基础数值生效。

### 3. 绑定位成长

绑定位数量严格按境界固定（见核心规则表格《绑定位成长》），无计算公式。境界突破时自动扩展，无需额外解锁。

## 边界情况

- **同名功法绑定同一角色**：角色已绑定青云功时，再打一张青云功→该角色在目标选择面板中灰色不可选，提示"已绑定同名功法"
- **同名功法绑定不同角色**：允许，每个角色独立计算效果（可叠加）
- **多角色争抢同一功法的本命位**：不存在"争抢"——native_owner固定在卡牌定义中。如果青元剑诀的native_owner=hanli，只有韩立绑定它能获得×1.5，紫灵绑定永远是×1.0
- **覆盖时有积累数值**：覆盖旧卡时，旧卡已积累的数值加成保留在角色上，但持续触发效果停止——"赚了历史，亏了未来"
- **角色离场时绑定卡被覆盖**：角色在离场期间（is_suspended=true），其绑定位可能被其他角色使用吗？——不会，绑定卡在离场时保留，只有角色阵亡或玩家主动覆盖才会失去
- **角色重新上场时绑定卡已不存在**：离场期间被某些特殊效果丢弃的卡，重新上场时验证失败→该绑定位变为空位，不报错
- **角色上场时空位不足**：如果新上场的角色在离场前有3个绑定卡，但当前境界只允许2个→如果角色上场时恢复绑定，超出部分自动释放到弃牌堆
- **本命绑定者阵亡后本命资格**：角色阵亡后，native_owner仍然是固定在卡牌定义中的。其他角色即使绑定了青元剑诀，因为不匹配native_owner，永远无法获得×1.5。本命资格不"释放"给其他角色
- **覆盖本命位的额外确认**：覆盖本命绑定位时，需额外确认弹窗（普通覆盖只需一次确认），旧卡使用更强烈的销毁特效
- **0费功法/法宝卡的绑定**：费用为0的功法/法宝（极罕见）也走完全相同的绑定流程，只是不扣除费用
- **卡牌升级后覆盖**：同名卡升级后覆盖旧卡→旧卡进弃牌堆，新卡以升级后的属性生效
- **绑定中的卡不可拆解**：在收藏界面中，绑定的卡（is_bind=true）拆解按钮灰色不可用，提示"该卡已在战斗中绑定，请先解绑"
- **阵法阵眼角色阵亡与绑定**：阵眼角色若身兼绑定+阵法阵眼，阵亡时绑定卡进入弃牌堆，阵法失效——两者独立处理，互不干扰

## 依赖关系

| 依赖系统 | 性质 | 说明 |
|----------|------|------|
| **卡牌系统** | 硬依赖 | 读取功法/法宝卡牌模板（native_owner、effect_type、base_value、rarity）；获取绑定位成长配置 |
| **战斗系统** | 硬依赖 | 接收出牌阶段绑定请求信号、角色阵亡/离场/上场事件、回合阶段转换信号 |
| **卡牌效果引擎** | 硬依赖 | register_persistent_effect、remove_effects_by_source、suspend_effects_by_source、restore_effects_by_source 接口 |
| **阵法系统** | 软依赖 | 角色绑定状态变更时通知阵法系统重查激活条件（阵眼角色的绑定/阵亡都影响阵法） |
| **UI系统** | 硬依赖 | 提供角色选择面板、覆盖确认UI、绑定状态展示、阵亡动画触发 |
| **资源系统** | 软依赖 | 绑定中的卡不可拆解查询接口 |
| **游戏状态管理器** | 软依赖 | 读取当前境界以确定绑定位上限 |

### 上游依赖设计顺序

卡牌系统 → 战斗系统 → 绑定系统 → 效果引擎

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 本命加成倍率 | ×1.5 | ×1.3 ~ ×2.0 | 本命加成感觉不明显，原著组合无吸引力 | 非本命绑定完全失去竞争力，流派多样性下降 |
| 炼气期功法位/角色 | 1 | 1-2 | 炼气期太短无法体验绑定 | 炼气期绑定位太多，失去稀缺感 |

| 元婴期功法位/角色 | 3 | 2-4 | 高境界绑定位紧张 | 高境界绑定位太充裕 |
| 覆盖反悔窗口 | 0.8s | 0.5-1.5s | 玩家来不及反悔 | 战斗节奏拖沓 |

| 覆盖积累保留 | 积累数值保留 | 全留/全清/半留 | 覆盖太亏，玩家不爱换卡 | 覆盖太赚，无换卡成本 |

## 视觉/音频需求

### 绑定动画

| 事件 | 动画 | 时长 |
|------|------|:---:|
| 功法绑定 | 卡牌化作流光缠绕目标角色 | 0.4s |
| 法宝绑定 | 卡牌化作金属碎片融合到角色 | 0.4s |
| 覆盖旧绑定 | 旧卡浮出→半透明→燃烧消散，新卡落位 | 0.6s |
| 覆盖旧绑定（本命位） | 旧卡金色火焰燃烧消散，新卡金色落位 | 0.8s |
| 角色阵亡→绑定卡失去 | 绑定槽位锁链断开，每张卡独立碎裂动画（本命卡金色火焰，普通卡蓝色） | 1.5s |
| 角色离场保留 | 绑定卡变灰半透明沉入角色轮廓 | 0.3s |
| 角色重新上场 | 绑定卡从角色轮廓升起恢复色彩 | 0.3s |
| 本命激活标识 | 角色头像旁 ★ 金色闪烁亮起 | 0.2s |

### 音效需求

- 绑定功法：书卷翻动/能量汇聚声
- 绑定法宝：金属碰撞或器物共鸣音
- 本命激活：提升音调的金色共鸣音
- 覆盖旧绑定：纸张撕裂声（旧卡）+ 新卡落位声
- 覆盖本命位：玻璃碎裂声（强调损失）
- 角色阵亡→卡牌失去：低沉的消音/消散效果
- 大还丹复活（空载）：空洞的回音

## 验收标准

- **GIVEN** 玩家打出功法卡，**WHEN** 选择目标角色，**THEN** 角色获得功法效果（持续生效）
- **GIVEN** 玩家打出法宝卡，**WHEN** 选择目标角色，**THEN** 角色获得法宝效果（持续生效）
- **GIVEN** 功法/法宝的 native_owner 匹配目标角色，**WHEN** 绑定完成，**THEN** native_multiplier=1.5，效果数值显示×1.5
- **GIVEN** 功法/法宝的 native_owner 不匹配目标角色，**WHEN** 绑定完成，**THEN** native_multiplier=1.0
- **GIVEN** 目标角色已绑定位未满，**WHEN** 打出功法/法宝，**THEN** 绑定到空位
- **GIVEN** 目标角色已绑定位已满，**WHEN** 打出功法/法宝，**THEN** 进入覆盖流程
- **GIVEN** 覆盖流程中玩家确认覆盖，**WHEN** 结算，**THEN** 旧卡进入弃牌堆，新卡落位生效
- **GIVEN** 覆盖流程中玩家反悔（0.8s内取消），**WHEN** 结算，**THEN** 新卡回手牌，旧卡保留不动
- **GIVEN** 覆盖时旧卡有积累数值，**WHEN** 覆盖完成，**THEN** 积累数值保留在角色上
- **GIVEN** 角色已绑定同名功法，**WHEN** 再打出同名功法并选择该角色，**THEN** 卡牌灰色不可选中
- **GIVEN** 炼气期角色，**WHEN** 检查绑定位，**THEN** 功法位=1，法宝位=1
- **GIVEN** 化神期角色，**WHEN** 检查绑定位，**THEN** 功法位=3，法宝位=3
- **GIVEN** 角色被移出战场（阵型调整），**WHEN** 检查绑定状态，**THEN** 绑定卡标记为is_suspended，不进弃牌堆
- **GIVEN** 角色重新上场，**WHEN** 检查绑定卡，**THEN** 之前绑定的卡自动恢复效果
- **GIVEN** 角色阵亡，**WHEN** 检查绑定卡，**THEN** 全部绑定卡进入弃牌堆
- **GIVEN** 角色阵亡后使用大还丹复活，**WHEN** 检查绑定状态，**THEN** 角色为空载（无绑定卡）
- **GIVEN** 角色A已本命绑定青元剑诀，**WHEN** 角色B再绑定同一实例，**THEN** 角色B的is_native=false（B的card_id不匹配native_owner=hanli）
- **GIVEN** 韩立阵亡后紫灵绑定青元剑诀（native_owner=hanli），**WHEN** 检查本命状态，**THEN** is_native=false（native_owner固定不转移）
