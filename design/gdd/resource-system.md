## 概述

资源系统管理玩家在本局游戏中拥有的两种核心经济资源——**灵石**（通用货币）和**灵材**（消耗型制造材料），定义它们的获取渠道、消耗途径、存储规则和经济循环模型。资源系统不直接执行「花钱买东西」的交互（那是商店系统和卡组编辑系统的职责），而是作为**资源存管和数值校验层**——通过GSM提供读写接口，供所有下游系统在结算时调用。

对玩家来说，灵石是「修仙世界的钱」——用来买卡、删卡、买丹药；灵材是「修仙世界的材料」——用来炼丹、炼器、做装备。灵石让你在现有体系中优化，灵材让你创造新东西。两者共同构成玩家在探索之外的经济循环：通过战斗和事件获取资源→在商店/炼丹炉中消费资源→强化自己的构筑。

## 玩家幻想

资源系统的核心情感是**「勤俭持家的安心感」和「资源积累的踏实感」**——你不是亿万富翁，但你清楚每一块灵石和每一份灵材的来处和去处。

| 情感 | 触发场景 |
|------|---------|
| **勤俭的算计** | 在商店看到一张想要的卡但灵石不够——「先不删卡了，省下来买这张」 |
| **积累的踏实** | 看着灵石数字稳定增长——「够用了，这局的底子稳了」 |
| **收获的满足** | 一场Boss战拿到大量灵石——「发财了，可以去消费了」 |
| **挥霍的清醒** | 花大价灵石删掉一张废牌——「贵但值得，卡组流畅多了」 |
| **材料富余的安心** | 背包里有足够灵材炼丹——「随时可以做一批丹药以备不时之需」 |

这两种资源的区别在于：**灵石是「流通」的，灵材是「制造」的。** 灵石决定你能买什么（消费力），灵材决定你能做什么（生产力）。两者的平衡关系——灵石不够就去打战斗/拆卡，灵材不够就去探事件/开混沌宝瓶——构成了经济循环的发动机。

## 详细设计

### 核心规则

#### 1. 资源类型定义

| 资源 | 用途 | 获取方式 | 消耗方式 | 存储上限 |
|:----:|------|---------|---------|:--------:|
| **灵石** | 商店购买卡牌/丹药/消耗品；删卡费用；部分事件消耗 | 战斗战利品、地图通关、拆解卡牌、探索事件、身份天赋加成 | 商店消费、删卡付费、事件支出 | 无硬上限（软上限：一局内合理范围 0~9999） |
| **灵材** | 炼丹消耗、炼器消耗、法宝铭刻消耗 | 探索事件（灵脉采掘）、混沌宝瓶产出、特殊卡牌效果、Boss掉落 | 炼丹/炼器配方消耗 | 无硬上限（通常每局限 0~99，超过不浪费） |

**灵石不可逆向转化为灵材**（不能拿钱买材料——至少在基础规则中不行）。**灵材可出售为灵石**（在商店中出售，转化率由灵材品质决定）。

#### 2. 灵石经济模型

**灵石收入主要来源：**

| 来源 | 基础值 | 受影响因素 | 频次 |
|:----:|:-----:|-----------|:----:|
| 战斗战利品（普通） | 10~25 | 地图难度、是否精英 | 高频（每场战斗） |
| 战斗战利品（精英） | 40~80 | 地图难度 | 中频（每图1~2次） |
| 战斗战利品（Boss） | 100~200 | 地图难度、回旧地图时受境界差额惩罚 | 低频（每图1次） |
| 地图通关奖励 | 50/100/200/300（低→极高） | 地图难度、回旧地图时受境界差额惩罚 | 低频（每图1次） |
| 拆解卡牌 | 10/30/100/400/2000（白→暗金） | 卡牌稀有度、等级 | 中频（按需） |
| 探索事件（灵脉采掘稳妥） | 50 | 无 | 中频 |
| 探索事件（灵脉采掘深挖） | 100~300 | 随机 | 中频（有概率失败） |
| 探索事件（坊市出售灵材） | 按灵材品质 | 灵材品质 | 中频 |
| 身份天赋加成（青云剑宗） | +15% | 身份选择 | 全局持续 |

**灵石支出主要途径：**

| 途径 | 基础值 | 受影响因素 | 频次 |
|:----:|:-----:|-----------|:----:|
| 商店购买卡牌 | 50/150/400/1000/3000（白→暗金） | 卡牌稀有度、商店打折事件 | 中频 |
| 商店购买丹药 | 20/60/150（蓝/紫/金） | 丹药类型 | 中频 |
| 商店删卡 | 50 + 25×(N-1)（N=当前局已删次数） | 删卡次数递增 | 中频 |
| 探索事件（坊市消费） | 按事件配置 | 事件类型 | 中频 |
| 旅店休息（可选设计） | 30（暂定） | — | 低频 |

**灵石经济节奏（预期）：**
- 炼气期：探索一张地图总收入约 200~400 灵石，主要支出为删卡（50/75）和买1~2张关键蓝卡（50~150）
- 筑基期：地图收入 400~800 灵石，可购买紫卡（400）、多次删卡
- 金丹期：地图收入 800~1500 灵石，开始考虑金卡（1000）
- 元婴期及以上：地图收入 1500+ 灵石，经济和构筑相对自由

#### 3. 灵材经济模型

**灵材品质分级：**

| 品质 | 名称 | 出售价（灵石） | 主要用途 |
|:----:|:----:|:------------:|---------|
| 1 | 低级灵材 | 10 | 基础丹药/低级法宝炼制 |
| 2 | 中级灵材 | 30 | 中级丹药/法宝炼制 |
| 3 | 高级灵材 | 80 | 高级丹药/金色法宝炼制 |
| 4 | 顶级灵材 | 200 | 暗金丹药/顶级法宝炼制 |

**灵材收入主要来源：**

| 来源 | 产出 | 受影响因素 | 频次 |
|:----:|:----:|-----------|:----:|
| 探索事件（灵脉采掘稳妥） | 低级灵材×1 | 无 | 中频 |
| 探索事件（灵脉采掘深挖） | 低级~中级灵材×1~3 | 随机 | 中频 |
| 混沌宝瓶（初级） | 低级灵材×1（每3回合） | 装备混沌宝瓶角色 | 持续 |
| 混沌宝瓶（小成） | 随机灵材×1（每回合） | 品质随境界提升 | 持续 |
| 混沌宝瓶（大成） | 灵材×2（每回合必出1高品质） | 装备混沌宝瓶角色 | 持续 |
| Boss掉落 | 中级~高级灵材×1~2 | 地图难度 | 低频（每图1次） |
| 特殊事件 | 按事件配置（如洞府奇遇） | 事件类型 | 低频 |
| 特定卡牌效果（万象罗盘） | 高品质灵材概率+20% | 绑定角色 | 持续 |

**灵材支出主要途径：**

| 途径 | 消耗 | 说明 |
|:----:|:----:|------|
| **炼丹**（基础丹药） | 低级灵材×2 | 产出蓝色丹药卡 |
| **炼丹**（中级丹药） | 中级灵材×2 + 低级灵材×1 | 产出紫色丹药卡 |
| **炼丹**（高级丹药） | 高级灵材×2 + 中级灵材×1 | 产出金色丹药卡 |
| **炼器**（基础法宝） | 低级灵材×3 | 产出蓝色法宝卡 |
| **炼器**（中级法宝） | 中级灵材×3 | 产出紫色法宝卡 |
| **炼器**（高级法宝） | 高级灵材×3 | 产出金色法宝卡 |
| **法宝铭刻** | 中级灵材×1 | 附加1条随机副属性 |
| **特殊事件消耗** | 按事件配置 | — |

**灵材经济节奏（预期）：**
- 炼气期：灵材产出有限（主要靠灵脉采掘），每次获得灵材都要谨慎使用
- 筑基期：混沌宝瓶开始稳定产出，灵材来源多样化，可尝试炼中级丹/器
- 金丹期：灵材产出丰富，可批量炼制高级丹药和法宝
- 元婴期+：灵材富余，可大量铭刻和炼制顶级物品

#### 4. 资源存储位置

所有资源数据由GSM统一管理：

```
# GSM路径
player.resources.ling_shi: int          # 当前灵石数量
player.resources.ling_cai: {            # 灵材库存（按品质分）
  low: int,       # 低级灵材
  medium: int,    # 中级灵材
  high: int,      # 高级灵材
  top: int,       # 顶级灵材
}
```

#### 5. 资源变更接口

所有资源变更通过GSM的标准 `add_resource` / `spend_resource` 接口进行：

```
add_resource(type, amount, quality?) → void:
  # type: "ling_shi" | "ling_cai"
  # amount: 正数（增加）
  # quality: 灵材品质[1~4]，仅ling_cai类型需要

spend_resource(type, amount, quality?) → bool:
  # 返回false表示余额不足（消费失败）
  # 返回true表示扣除成功

get_resource(type, quality?) → int:
  # 查询当前数量
```

灵石变更广播 `resource_changed` 信号，灵材变更同理。

#### 6. 资源上限

- **灵石**：无硬上限。一局内合理范围 0~9999，超过 9999 仍可正常使用但UI显示截断为「9999+」
- **灵材**：每种品质无硬上限，但背包中同类灵材超过 99 时在UI中显示为「99+」（合理预期：炼气期<30，元婴期<99）

#### 7. 跨局资源继承

所有资源**不跨局继承**（Roguelike标准规则——每一局从零开始）。与轮回天赋系统的互动：

- 轮回天赋可能提供「开局额外灵石」（如+50开局灵石）
- 轮回天赋不提供灵材继承（灵材是制造消耗品，不适合作为开局奖励）

#### 8. 拆解卡牌 → 灵石（引用卡牌系统）

卡牌系统中定义的拆解规则：

```
dismantle_value(card):
  # 按稀有度 + 等级计算
  base = [10, 30, 100, 400, 2000][card.rarity]  # 白→暗金
  bonus = base × card.level × 0.05               # 每级+5%
  return floor(base + bonus)
```

拆解操作由卡组编辑系统触发，资源系统负责增加灵石。

#### 9. 出售灵材 → 灵石

```
sell_ling_cai(quality, quantity) → int:
  # 品质[1~4] → 单价[10, 30, 80, 200]
  unit_price = [10, 30, 80, 200][quality - 1]
  total = unit_price × quantity
  # 扣减灵材 → 增加灵石
  return total
```

#### 10. 资源相关的身份天赋

| 身份 | 资源相关天赋 | 效果 |
|------|------------|------|
| 青云剑宗外门弟子 | 财源广进 | 探索灵石掉落量+15% |
| 阵道双杰 | 金钱镖 | 战斗中使用金钱镖获得1灵石（累计经济来源） |

#### 11. 资源与其他系统的数据耦合规则

- 资源系统**不感知**其他系统的经济细节（如「商店中某某卡卖多少钱」）
- 各系统在上层完成自己的数值计算后，调用资源系统的 `spend_resource` / `add_resource` 执行实际数值变更
- 资源系统只保证两个核心约束：**余额充足才能消费**、**消费后正确扣减**

### 状态与转换

资源系统自身无状态机。资源的变更遵循简单的「查询→校验→执行」流程：

```
[请求 spend_resource(ling_shi, 50)]
    ↓
[资源系统查询 player.resources.ling_shi]
    ↓
余额 ≥ 50? ───否──→ 返回 false（消费失败），调用方处理失败逻辑
    ↓ 是
[player.resources.ling_shi -= 50]
    ↓
[广播 resource_changed 信号]
    ↓
[返回 true]

---

[请求 add_resource(ling_cai, medium=1)]
    ↓
[player.resources.ling_cai.medium += 1]
    ↓
[广播 resource_changed 信号]
```

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **游戏状态管理器** | 资源读写接口（add_resource/spend_resource/get_resource） | 存档/读档时资源数据的序列化/反序列化；resource_changed 信号 |
| **卡组编辑系统** | 拆解卡牌→增加灵石；删卡→消费灵石；出售卡牌→增加灵石 | 灵石余额查询；消费请求 |
| **商店系统** | 购买→消费灵石；出售灵材→增加灵石 | 商品价格（本系统不感知） |
| **探索系统** | 地图通关灵石奖励 | 地图通关时调用 add_resource |
| **事件系统** | 事件结果中的资源/灵材增删 | 事件结算时调用 add_resource/spend_resource |
| **战斗系统** | 战利品灵石奖励 | 战斗结算时调用 add_resource |
| **炼丹炼器系统** | 炼丹/炼器消耗灵材→产出卡牌 | 灵材库存查询；消耗请求 |
| **法宝铭刻系统** | 铭刻消耗灵材 | 灵材库存查询；消耗请求 |
| **UI系统** | 当前资源数量（显示在HUD） | 订阅 resource_changed 信号更新显示 |
| **身份选择系统** | 初始灵石设置 | 开局时设置 player.resources.ling_shi |
| **轮回天赋系统** | 开局额外灵石（如天赋提供） | — |

## 公式

### 1. 拆解卡牌价值

```
dismantle_value(rarity, level) → int:
  base = [10, 30, 100, 400, 2000][rarity - 1]  # rarity: 1=白, 2=蓝, 3=紫, 4=金, 5=暗金
  bonus = floor(base × level × 0.05)
  return base + bonus
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| rarity | int | [1, 5] | 卡牌稀有度 |
| level | int | [0, max_level] | 卡牌等级 |
| 返回值 | int | [10, 2095] | 拆解获得的灵石 |

**示例：** 紫色卡（rarity=3, level=0）→ 100灵石；金色卡（rarity=4, level=2）→ 400 + floor(400×2×0.05) = 440灵石

### 2. 出售卡牌价值（商店出售）

```
sell_value(rarity, level) → int:
  return floor(dismantle_value(rarity, level) × 0.8)
```

出售价格为拆解价值的80%（商店抽成）。

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| dismantle_value | int | [10, 2095] | 拆解价值 |
| 返回值 | int | [8, 1676] | 出售获得的灵石 |

### 3. 删卡费用

```
delete_card_cost(delete_count) → int:
  return 50 + 25 × (delete_count - 1)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| delete_count | int | [1, ∞) | 本局中已删卡次数（首次=1） |
| 返回值 | int | [50, ∞) | 本次删卡所需灵石 |

### 4. 出售灵材价值

```
sell_ling_cai_value(quality, quantity) → int:
  unit_price = [10, 30, 80, 200][quality - 1]
  return unit_price × quantity
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| quality | int | [1, 4] | 灵材品质（1=低级, 4=顶级） |
| quantity | int | [1, ∞) | 出售数量 |
| 返回值 | int | [10, ∞) | 获得的灵石 |

### 5. 资源余额校验

```
can_spend(type, amount, quality?) → bool:
  balance = get_resource(type, quality)
  return balance >= amount
```

所有消费操作必须先调用 `can_spend()` 校验，余额不足时操作不可执行（由请求方处理灰显/提示）。

### 6. 灵石收入修正（身份天赋）

```
apply_ling_shi_bonus(base_amount, identity) → int:
  if identity has "ling_shi_boost":
    return floor(base_amount × 1.15)
  return base_amount
```

### 7. 境界差额灵石惩罚（回旧地图）

```
realm_gap_penalty(player_L, map_max_L) → float:
  gap = player_L - map_max_L
  if gap <= 0: return 1.0
  return max(0.1, 1.0 - (gap × 0.3))  # 每多1级-30%，最低保留10%
```

**示例：** 金丹期（L=3）回青云剑宗（map_max_L=1）→ gap=2 → penalty=0.4 → 基础灵石×40%
**示例：** 元婴期回青云剑宗 → gap=3 → penalty=0.1 → 基础灵石×10%（保底）

## 边界情况

- **灵石不足时尝试消费**：`can_spend()` 返回 false，调用方将相关选项灰显或提示「灵石不足」。消费操作永不耗尽灵石后进入负数
- **灵石为0时拆解卡**：拆解是增加灵石而非消耗，0灵石状态下可正常拆解
- **灵石为0时卡组已满无法删卡**：玩家只能通过出售卡牌赚灵石后再删卡，或者接受卡组超限打Boss
- **多次拆解同稀有度卡牌**：拆解价值不衰减（不因同稀有度拆太多了就贬值），白色卡永远是10灵石
- **灵材满99后在UI显示「99+」**：灵材数量超过99时，背包界面显示「99+」（减少UI空间占用），实际数值在详情中可查
- **灵材数量在炼丹时不足**：`can_spend()` 返回 false，按键灰色显示所需灵材数量
- **使用灵石购买卡牌后余额为0**：允许——0灵石是正常状态，不影响其他操作
- **跨地图的灵石继承**：灵石跨地图**完全继承**——进入新地图不清零灵石
- **突破境界后灵石变化**：灵石不变（境界突破只影响境界属性，不影响资源）
- **身份天赋的灵石加成与地图通关奖励**：加成适用——青云剑宗的+15%灵石对所有灵石产出有效（战斗战利品、地图通关、拆解、事件奖励均受加成）
- **出售灵材时灵材数量不足**：`can_spend()` 返回 false，禁止出售超过持有数量的灵材
- **删卡费用持续增长的经济压力**：删卡费用随次数递增（50→75→100→…），目的是防止玩家随意删卡。灵石不足删卡时，玩家可选择出售多余的卡牌赚钱，或接受当前卡组
- **战斗中消耗的丹药涉及灵材**：丹药卡牌在战斗中使用不消耗灵材（灵材仅在制造阶段消耗）。制造好的丹药卡牌在使用时是「已完成的物品」，不引用灵材
- **暗金卡拆解价值过高（2000灵石）**：设计上接受——暗金卡是极其稀有（炼气期不可获得、金丹期+才有1~2张上限），2000灵石的拆解价值是给玩家的「但如果你真舍得拆暗金…」的选择。拆解暗金卡意味着永久失去流派的终极核心

## 验收标准

- **GIVEN** 玩家初始选择青云剑宗外门弟子，**WHEN** 游戏开始，**THEN** 初始化灵石=15
- **GIVEN** 玩家初始灵石=10，**WHEN** 调用 add_resource("ling_shi", 25)，**THEN** 灵石变为35
- **GIVEN** 玩家灵石=50，**WHEN** 调用 spend_resource("ling_shi", 30)，**THEN** 返回true，灵石变为20
- **GIVEN** 玩家灵石=20，**WHEN** 调用 spend_resource("ling_shi", 30)，**THEN** 返回false，灵石仍为20
- **GIVEN** 玩家拆解一张白色卡，**WHEN** 结算，**THEN** 灵石增加10
- **GIVEN** 玩家拆解一张金色卡（等级0），**WHEN** 结算，**THEN** 灵石增加400
- **GIVEN** 玩家在商店出售一张白色卡，**WHEN** 确认，**THEN** 灵石增加8（10×0.8）
- **GIVEN** 玩家首次删卡，**WHEN** 支付，**THEN** 灵石减少50
- **GIVEN** 玩家第5次删卡，**WHEN** 计算费用，**THEN** 费用=50+25×4=150灵石
- **GIVEN** 玩家在商店出售低级灵材×2，**WHEN** 结算，**THEN** 灵石增加20（10×2）
- **GIVEN** 玩家在商店出售高级灵材×1，**WHEN** 结算，**THEN** 灵石增加80
- **GIVEN** 玩家灵石=60且青云剑宗天赋生效，**WHEN** 获得25灵石，**THEN** 实际增加28（floor(25×1.15)）=28，灵石=88
- **GIVEN** 地图通关灵石奖励100，**WHEN** 结算，**THEN** 灵石增加100
- **GIVEN** 玩家背包中有低级灵材×5，**WHEN** 炼丹消耗低级灵材×2，**THEN** 低级灵材变为3
- **GIVEN** 玩家灵石超过9999，**WHEN** 查看HUD，**THEN** 显示「9999+」
- **GIVEN** 玩家灵材同类超过99，**WHEN** 查看背包，**THEN** 显示「99+」
- **GIVEN** 灵石变更，**WHEN** 监听 resource_changed 信号，**THEN** 收到 {type:"ling_shi", delta, balance}
