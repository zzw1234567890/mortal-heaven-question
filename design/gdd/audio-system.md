## 概述

音频管理系统负责游戏中所有音频资源的加载、播放、混音和场景切换时的音频过渡——包括背景音乐（BGM）、音效（SFX）、环境音、UI音效以及语音/对话音频。音频管理系统不直接决定播放什么音频（那是各系统的职责），而是提供**播放接口和混音控制**，确保音频在场景切换时平滑过渡，不出现爆音、重叠或丢失。

对玩家来说，音频是游戏沉浸感的重要组成部分——战斗的紧张音乐、探索的宁静环境音、突破时的恢弘音效，共同构建了修仙世界的听觉体验。

## 详细设计

### 核心规则

#### 1. 音频分类与优先级

| 分类 | 优先级 | 示例 | 同时播放上限 |
|------|:-----:|------|:----------:|
| **BGM**（背景音乐） | 低 | 探索地图BGM、战斗BGM、Boss战BGM | 1首（淡入淡出切换） |
| **环境音** | 低 | 风声、水声、鸟鸣、洞府回声 | 2~3层（叠加） |
| **SFX**（音效） | 中 | 卡牌出牌声、伤害命中、阵亡音 | 8~12（动态混音） |
| **UI音效** | 中 | 按钮点击、菜单打开、通知提示 | 2~3（不重叠） |
| **语音/对话** | 高 | 角色台词、剧情对白 | 1条（独占） |

#### 2. BGM管理

**BGM曲目（按场景）：**

| 场景 | BGM风格 | 过渡方式 |
|------|---------|---------|
| 主菜单 | 空灵、悠远的修仙主题 | 启动时播放 |
| 地图选择 | 轻快的探索前奏 | 从主菜单过渡 |
| 探索（炼气期） | 宁静的山水意境 | 淡入 |
| 探索（筑基期） | 略带紧张的冒险感 | 淡入 |
| 探索（金丹期+） | 壮阔的仙侠风格 | 淡入 |
| 普通战斗 | 节奏适中的战斗曲 | 从探索BGM快速过渡 |
| 精英战斗 | 较紧张的战斗变奏 | 从普通战斗BGM变调 |
| Boss战斗 | 恢弘、紧迫的主题曲 | 独立BGM直接播放 |
| 渡劫战 | 特殊的渡劫主题（紧张+神圣感） | 独立BGM |
| 商店/事件 | 轻松的市井/奇遇风 | 暂停探索BGM |
| 通关结算 | 胜利凯旋曲 | Boss战后过渡 |
| 战败 | 低沉的失败曲 | 战斗BGM渐弱 |

**BGM播放规则：**
- 同一时间仅播放1首BGM（唯一性）
- 场景切换时：旧BGM **淡出**（1s），新BGM **淡入**（1.5s）
- 战斗开始时：探索BGM快速淡出（0.5s）→ 战斗BGM淡入（0.5s）
- Boss战时：战斗BGM直接切为Boss BGM（淡入0.3s，更强冲击力）
- BGM循环播放，起始点无空白间隙
- 设置中BGM音量默认80%（留有余量）

#### 3. SFX管理

**SFX分类索引（例）：**

| 子系统 | 音效示例 |
|--------|---------|
| 战斗 | 攻击命中、技能释放、阵法激活、角色阵亡、暴击额外音 |
| 卡牌 | 抽牌、出牌（分类型：功法/法宝/丹药/符箓）、卡牌升级 |
| 探索 | 移动脚步声、传送、节点到达、宝箱打开、灵泉涌动 |
| 事件 | 事件弹出、选项选中、获得奖励、失败惩罚 |
| UI | 按钮点击、页面切换、HUD提示、确认/取消 |
| 突破 | 渡劫雷声、突破成功金光音、失败低沉音 |

**SFX播放规则：**
- 最多同时播放8~12个SFX（超过时按优先级丢弃最旧的）
- 同类型SFX有**冷却时间**（同一音效最短间隔50ms，防止连击爆音）
- SFX音量默认100%
- 3D空间化：非必需（游戏为2D卡牌，SFX在全局空间播放）
- 战斗中每次攻击音的播放间隔不短于100ms（防止多角色同时攻击时音效堆叠）

#### 4. 环境音管理

环境音作为BGM的补充层：

- 每张地图有对应的环境音预设（风声、水声、鸟鸣等）
- 环境音与BGM叠加播放（互不干扰）
- 环境音音量随场景状态微调（战斗中环境音降低50%，突出战斗音效）
- 探索中环境音持续播放，进入事件/商店时降低但不停止

#### 5. UI音效

- 所有UI交互都有对应的短暂音效（按钮点击、面板打开/关闭等）
- UI音效不叠加（新音效打断旧音效）
- 音量UI系统默认70%（比SFX低，避免太吵）

#### 6. 音频总线架构

```
Master Bus
├── BGM Bus（总线的60%）
│   └── 场景BGM
├── SFX Bus（总线的80%）
│   ├── 战斗SFX
│   ├── 卡牌SFX
│   └── 探索SFX
├── UI Bus（总线的70%）
│   └── UI音效
├── Ambient Bus（总线的50%）
│   └── 环境音
└── Voice Bus（总线的90%）
    └── 语音/对话
```

各总线独立控制音量，设置中的总音量/音乐音量/音效音量分别映射到不同的总线。

#### 7. 音频加载策略

| 资源类型 | 加载方式 | 说明 |
|---------|---------|------|
| BGM | 按需流式加载 | 场景切换时加载下一首，卸载上一首 |
| SFX | 预加载（启动时） | 常用SFX在游戏启动时加载到内存 |
| UI音效 | 常驻内存 | 始终可用 |
| 环境音 | 按场景加载 | 进入地图时加载当前地图的环境音 |
| 语音 | 按需加载 | 触发对话时按需加载 |

#### 8. 音频事件接口

各系统通过统一的音频事件接口触发播放：

```
play_sfx(sfx_id: String, options?: {volume?, pitch?, delay?}) → void
play_bgm(bgm_id: String, options?: {fade_in?, loop?}) → void
stop_bgm(options?: {fade_out?}) → void
set_bus_volume(bus: String, volume: float) → void
play_ambient(ambient_id: String) → void
```

### 状态与转换

| 状态 | 含义 | 音频表现 |
|------|------|---------|
| **主菜单** | 游戏启动 | 主菜单BGM（循环） |
| **身份选择** | 开局选择身份 | 主菜单BGM继续（或轻快的选择音） |
| **探索中** | 玩家在地图上移动 | 地图BGM + 环境音 + 探索SFX |
| **战斗中** | 回合制对战 | 战斗BGM + 战斗SFX（环境音降低） |
| **渡劫战中** | 特殊Boss战 | 渡劫BGM + 渡劫SFX |
| **事件中** | 事件面板打开 | BGM保持 + UI音效（环境音降低） |
| **商店中** | 商店界面 | BGM降低音量 + UI音效（环境音暂停） |
| **地图通关** | Boss击败 | 胜利BGM + 通关SFX |
| **战败** | 战斗失败 | 战败BGM + 低沉的失败音 |
| **暂停** | 游戏暂停 | BGM暂停/音量降低 |

**场景切换音频过渡矩阵：**

| 从→到 | 当前音频动作 | 新音频动作 |
|--------|-------------|-----------|
| 主菜单→探索 | 主菜单BGM淡出(1s) | 地图BGM淡入(1.5s) + 环境音 |
| 探索→战斗 | 探索BGM快速淡出(0.5s) | 战斗BGM淡入(0.5s)，环境音降低 |
| 战斗→探索 | 战斗BGM淡出(1s) | 探索BGM淡入(1s)，环境音恢复 |
| 探索→商店 | 探索BGM音量降低至30%（不断） | 商店环境音淡入 |
| 商店→探索 | 商店环境音淡出 | 探索BGM音量恢复至100% |
| 战斗→通关结算 | 战斗BGM淡出(0.3s) | 胜利BGM淡入(0.5s) |
| 战斗→战败 | 战斗BGM快速渐弱(0.5s) | 战败BGM淡入(0.3s) |
| 探索→渡劫 | 探索BGM淡出(0.3s) | 渡劫BGM淡入(0.3s) |

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **游戏状态管理器** | — | 场景切换信号（触发音频过渡） |
| **事件系统** | SFX播放请求 | 事件触发信号 |
| **战斗系统** | 战斗SFX/BGM播放请求 | 战斗回合/阶段信号 |
| **探索系统** | 地图BGM切换请求 | 地图切换信号 |
| **UI系统** | UI音效播放请求 | UI交互信号 |
| **渡劫突破系统** | 渡劫BGM/SFX播放请求 | 渡劫触发/成功/失败信号 |
| **设置系统** | 音量/静音设置 | 设置变更信号 |

## 调优参数

| 参数 | 默认值 | 安全范围 | 说明 |
|------|:-----:|:--------:|------|
| BGM淡入时长 | 1.5s | 0.5~3.0s | 场景切换时新BGM渐入时长 |
| BGM淡出时长 | 1.0s | 0.3~2.0s | 场景切换时旧BGM渐出时长 |
| 战斗BGM切换速度 | 0.5s | 0.3~1.0s | 探索→战斗的快速过渡 |
| 同时播放SFX上限 | 12 | 8~16 | 超过时丢弃优先级最低的音效 |
| 同SFX最小间隔 | 50ms | 30~100ms | 防止同一音效连击爆音 |
| BGM默认音量 | 80% | 50~100% | 留有余量避免爆音 |
| SFX默认音量 | 100% | 70~100% | 全音量保证冲击力 |
| UI音效默认音量 | 70% | 50~90% | 比SFX稍低不干扰 |
| 环境音默认音量 | 50% | 30~70% | 作为背景不宜突出 |
| 战斗中环境音降低 | -50% | -30%~-70% | 突出战斗音效 |

| 界面 | 核心功能 |
|------|----------|
| **设置-音量** | 总音量/音乐/音效/语音滑动条 |

## 验收标准

- **GIVEN** 游戏启动，**WHEN** 主菜单加载，**THEN** 主菜单BGM播放
- **GIVEN** 进入探索，**WHEN** 地图加载，**THEN** 探索BGM淡入，环境音开始
- **GIVEN** 进入战斗，**WHEN** 场景切换，**THEN** 探索BGM淡出(0.5s)，战斗BGM淡入(0.5s)
- **GIVEN** 战斗结束，**WHEN** 结算，**THEN** 战斗BGM淡出，胜利/战败BGM播放
- **GIVEN** 战斗中打出卡牌，**WHEN** 出牌，**THEN** 对应类型卡牌SFX播放
- **GIVEN** 战斗中有多个音效同时触发，**WHEN** 播放，**THEN** 不超过12个同时播放
- **GIVEN** 同一SFX在50ms内多次触发，**WHEN** 播放，**THEN** 第二个被丢弃
- **GIVEN** 进入商店，**WHEN** 场景切换，**THEN** BGM降低至30%，商店环境音淡入
- **GIVEN** 离开商店，**WHEN** 回到探索，**THEN** BGM恢复至100%，商店环境音淡出
- **GIVEN** 设置中调节BGM音量，**WHEN** 滑动条改变，**THEN** BGM音量实时变化
- **GIVEN** 按F1，**WHEN** 触发热键，**THEN** 所有音频静音/恢复
- **GIVEN** 渡劫触发，**WHEN** 战斗开始，**THEN** 渡劫BGM播放（独立BGM）
