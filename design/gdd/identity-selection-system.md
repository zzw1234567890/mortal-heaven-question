# 开局身份选择系统 (Starting Identity Selection System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-23
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱4「始于凡人，忠于原著」

## 概述

开局身份选择系统管理每局游戏开始前的身份选择流程——玩家从6种预设身份（5默认+1轮回解锁）中选择其一，每种身份绑定一套**完全固定的初始卡组**（无随机成分）、两名初始角色卡、专属天赋和初始灵石。身份选择是每局游戏的「第一个决策点」，决定了玩家本局的基础构筑方向和战术风格。选择写入 GSM 的 `player.identity_id`，后续系统（探索、战斗、事件、商店）均可能引用身份进行差异化处理。

对玩家来说，这是「这局我打算怎么玩」的宣言——6种身份对应6种完全不同的开局体验和战术路线，每一个都可围绕原著角色展开独特的剧情互动。这个选择不可更改（本局内固定），但轮回天赋系统允许跨局解锁隐藏身份或增强已有身份。

|:-:|:----|:--------|:--------|:--------|

## 玩家幻想

开局身份选择的情感核心是**出征前的定调**——类似角色扮演游戏中选择出身和职业时的期待感。

| 情感 | 触发场景 |
|------|---------|
| **选择的重量** | 「这决定了我接下来几小时的打法」 |
| **探索欲** | 「上次走鬼灵门被Boss教育了，这次试试掩月宗」 |
| **角色共鸣** | 原著读者看到「韩立+厉飞雨」——「七玄门开局，爷青回」 |
| **身份认同** | 「我是坚持正道稳扎稳打的修士」vs「我就是为快攻而生的魔修」 |
| **新鲜感** | 解锁「阵道双杰」后——「齐云霄+辛如音，阵法流走起！」 |

> **设计原则**：六个身份不是「简单/普通/困难」的难度梯度，而是「续航/快攻/灵活/控制/辅助/运营」的六种平行方向。每个身份应有足够差异化的初始卡组和专属天赋，确保玩家重复游玩时在开局阶段就有新鲜感。

## 详细设计

### 核心规则

#### 1. 身份定义结构

每个开局身份包含以下数据：

```
IdentityTemplate {
  id: String
  name: String
  description: String
  flavor_text: String
  style_tag: String                    # 风格标签（如「续航」「快攻」）
  initial_deck: {
    cards: [{card_id, count}]
    character_slots: [{card_id, slot_index}]
  }
  initial_resources: {
    ling_shi: int
  }
  talent: {
    id: String, name: String
    description: String
    effect: {type, value, trigger}
  }
  character_details: [{
    card_id: String
    display_name: String
    level: int
  }]
  unlock_condition: {
    default_unlocked: bool
    require_talent: String | null
  }
}
```

#### 7. 身份影响范围

身份不仅仅是开局配置——它在整局游戏中通过以下方式产生持续影响：

| 影响维度 | 说明 | 实现方式 |
|---------|------|---------|
| **初始卡组** | 决定本局构筑的起点，影响前期战斗风格 | 开局写入deck，之后按战利品/事件正常演变 |
| **专属天赋** | 整局持续的被动效果 | 在GSM talent系统注册，各系统检查 |
| **叙事差异化** | 部分事件文本因身份不同而变化 | 事件系统通过 `player.identity_id` 做分支 |
| **初始角色** | 两名初始角色可参与整局，决定前期阵位组合 | 角色位写入，之后可被替换 |
| **流派倾向** | 不同身份天然适配不同流派 | 流派系统检测时参考身份倾向权重 |

## 公式

### 0. 变量定义与取整规则

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| `talent_map` | `Dictionary<String, int>` | 键=天赋ID，值=强度（≥1） | 键值映射天赋注册表 |
| `heal_base` | `int` | 1~999 | 丹药卡基础回复量 |
| `pct_bonuses` | `float[]` | 每个 0.0~1.0 | 百分比加成列表（天赋+角色+装备） |
| `flat_bonuses` | `int[]` | 每个 0~99 | 固定值加成列表 |

**取整规则**：所有百分比加成计算后，结果使用 `floor()` 向下取整。示例：`floor(3.6) = 3`。

**叠加顺序**：百分比加成先做加法叠加，再乘以基础值，最后加固定值加成：
```
result = floor(base x (1 + sum(pct_bonuses))) + sum(flat_bonuses)
```

### 1. 天赋效果注册（键值映射制）

```
# 身份选择时写入 GSM
apply_identity_talent(identity_id):
  talent_def = identity_template[identity_id].talent
  GSM.player.talent_map[talent_def.id] = talent_def.magnitude

# 示例：七玄门灵石加成
if "ling_shi_boost" in GSM.player.talent_map:
  magnitude = GSM.player.talent_map["ling_shi_boost"]  # = 15
  ling_shi_reward = floor(ling_shi_reward * (1 + magnitude / 100.0))

if "first_strike_extra_cost" in GSM.player.talent_map and turn == 1:
  extra = GSM.player.talent_map["first_strike_extra_cost"]  # 身份=1+流派=1→总计2

if "moon_veil_shield" in GSM.player.talent_map and battle_start:

  apply_shield(all_allies, shield_val)

# 示例：黄枫谷丹药加成
if "alchemy_affinity" in GSM.player.talent_map and card.type == 丹药:
  # shop_bonus=20表示商店概率相对加成：基础概率×1.2
  # 治疗效果+10%在丹药叠加公式中处理（见公式2）

# 示例：乱星海重投
if "re_forge_opportunity" in GSM.player.talent_map and first_event_per_map:
  次数 = GSM.player.talent_map["re_forge_opportunity"]  # = 1
  可消耗此天赋重新随机事件结果（限次数/图）

# 示例：阵道双杰阵法减免

```

### 2. 丹药治疗叠加公式

```
heal_total = floor(heal_base x (1 + sum(all_pct_bonuses))) + sum(all_flat_bonuses)

# 示例：黄枫谷弟子 + 陈巧倩在场 + 三转重元功绑定 + 张铁在场，使用小培元丹（3HP）
# pct_bonuses = [0.1（天赋）, 0.1（陈巧倩特性）] → sum = 0.2
# flat_bonuses = [2（三转重元功）, 1（张铁）] → sum = 3
# heal_total = floor(3 x 1.2) + 3 = floor(3.6) + 3 = 3 + 3 = 6HP
```

## 边界情况

- **已解锁的身份在轮回天赋系统中被锁定**：不应发生——轮回天赋只增加可解锁身份，不锁定已解锁的
- **玩家在确认前退出游戏**：身份选择未提交，不写入GSM。下次新游戏仍然进入身份选择界面
- **读档后身份恢复**：存档中包含 `identity_id`，读档后直接恢复
- **身份模板引用的卡牌ID不存在**：系统启动时验证所有身份模板的卡牌ID——如果某个ID无效，阻止该身份可选并记录错误日志
- **初始两名角色之一在流程早期阵亡**：正常——角色阵亡后按战斗系统的阵亡规则处理（绑定卡进入弃牌堆，可用丹药复活）
- **阵道双杰身份未解锁时在选择界面的显示**：卡片正常展示（可见名称和风格），但覆盖半透明「🔒」遮罩，点击时显示「轮回天赋解锁」
- **所有5个默认身份玩家都不喜欢**：可接受——5个已提供足够的多样性，不喜欢全部的情况极少
- **专属天赋与轮回天赋冲突**：轮回天赋系统确保不覆盖专属天赋。同类效果可叠加（加法）
- **多个身份都有相同卡牌（如基础攻击）**：允许——基础攻击是通用卡，所有身份都可能有
- **齐云霄和辛如音作为初始角色的设定**：齐云霄和辛如音在原著中是阵法天才道侣，修为不高但对阵法的理解远超同阶。初始HP/ATK不高（9/3 + 8/2）反映了他们修为尚浅，阵法减费天赋弥补了战力不足

- **GIVEN** 选择掩月宗弟子并确认，**WHEN** 检查卡组，**THEN** 卡组含素女轮回诀×1、冰魄剑诀×1、冰锥术×2、玉清护盾×1、基础攻击×2
- **GIVEN** 选择掩月宗弟子，**WHEN** 检查角色位，**THEN** 位1为聂盈、位2为姜琳
- **GIVEN** 选择黄枫谷弟子并确认，**WHEN** 检查卡组，**THEN** 卡组含三转重元功×1、黄枫剑气×1、固本丹×1、基础攻击×2、小培元丹×2

- **GIVEN** 选择鬼灵门遗孤，**WHEN** 检查 GSM 已注册天赋，**THEN** `player.talents` 包含 `first_strike_extra_cost`（值为 1）
- **GIVEN** 选择掩月宗弟子，**WHEN** 检查 GSM 已注册天赋，**THEN** `player.talents` 包含 `moon_veil_shield`（值为 2）
- **GIVEN** 选择七玄门外门弟子，**WHEN** 检查 GSM 已注册天赋，**THEN** `player.talents` 包含 `ling_shi_boost`（值为 15）
- **GIVEN** 选择黄枫谷弟子，**WHEN** 检查 GSM 已注册天赋，**THEN** `player.talents` 包含 `alchemy_affinity`（商店概率值为 20，效果值为 10）
