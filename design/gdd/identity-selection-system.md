# 开局身份选择系统 (Starting Identity Selection System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-22
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱4「始于凡人，忠于原著」

## 概述

开局身份选择系统管理每局游戏开始前的身份选择流程——玩家从3种预设身份中选择其一，每种身份绑定固定的初始卡组、初始角色卡、专属天赋和初始资源。身份选择是每局游戏的「第一个决策点」，决定了玩家本局的基础构筑方向和战术风格。选择写入 GSM 的 `player.identity_id`，后续系统（探索、战斗、事件、商店）均可能引用身份进行差异化处理。

对玩家来说，这是「这局我打算怎么玩」的宣言——选七玄门外门弟子意味着稳扎稳打、攒资源发育；选魔修遗孤意味着前期抢节奏、高风险高回报；选散修路人意味着拥抱随机性、每局体验不同。这个选择不可更改（本局内固定），但轮回天赋系统允许跨局解锁新的身份或增强已有身份。

## 玩家幻想

开局身份选择的情感核心是**出征前的定调**——类似选择角色职业时的审慎感和期待感。

| 情感 | 触发场景 |
|------|---------|
| **选择的重量** | "这决定了我接下来2小时的打法" |
| **探索欲** | "上次走魔道线被Boss教育了，这次试试散修——万一出好卡呢" |
| **归属感** | 选择七玄门弟子时看到熟悉的初始卡组——「还是长春功顺手」 |
| **新鲜感** | 选择散修路人看到随机卡牌——「不知道这局会给什么，有意思」 |

身份选择最重要的情感是**身份认同**——它不应该只是一个数值选择（哪个最强选哪个），而应该是一种风格宣言。三个身份不是「简单/普通/困难」的难度分级，而是「苟道发育/激进快攻/惊喜随机」的三种平行方向。

## 详细设计

### 核心规则

#### 1. 身份定义结构

每个开局身份包含以下数据：

```
IdentityTemplate {
  id: String                       # 身份ID（如 "seven_peaks_disciple"）
  name: String                     # 显示名称（如 "七玄门外门弟子"）
  description: String              # 简短描述（1-2句）
  flavor_text: String              # 叙事文本（开局时显示的背景介绍）
  initial_deck: {                  # 初始卡组
    cards: [{card_id, count}]      # 卡牌ID + 数量
    character_slots: [{card_id, slot_index}]  # 角色卡 + 放置位置
  }
  initial_resources: {             # 初始资源
    ling_shi: int                  # 开局灵石
  }
  talent: {                        # 专属天赋
    id: String                     # 天赋ID
    name: String
    description: String
    effect: {type, value, trigger} # 效果定义
  }
  character_details: [{            # 初始角色详情
    card_id: String
    display_name: String           # 显示名（如「韩立·少年」）
  }]
  unlock_condition: {              # 解锁条件（轮回天赋系统关联）
    default_unlocked: bool         # 默认可选
    require_talent: String|null     # 需要轮回天赋解锁
  }
}
```

#### 3. 身份选择流程

```
identity_selection_flow():
  ① 新游戏 → 检查是否已选择身份
     - 如果 GSM.player.identity_id 已存在 → 跳过（读档或已选择）
     - 如果为空 → 进入选择界面
  ② UI 展示3个身份卡片（并列布局）：
     每个身份卡片包含：
     - 身份名称 + 简短描述
     - 专属天赋名称+简述
     - 初始卡组概览（展示几张代表性卡牌）
     - 初始灵石数量
     - 初始角色预览
     - 解锁状态（已解锁/未解锁）
  ③ 玩家选择其中一个身份
  ④ 确认弹窗：「确定选择【身份名】？本局不可更改。」
  ⑤ 确认后 → GSM.set("player.identity_id", selected_id)
      → 应用初始状态：
        - 写入初始卡组到 deck.current_deck
        - 写入初始角色卡到 deck.slots
        - 设置初始灵石
        - 激活专属天赋（效果注册到对应系统）
        - 写入叙事文本到 narrative 供开局展示
      → 发射 identity_selected 信号
      → 进入游戏（探索地图或教程）
```

**选择后可返回吗？** 在确认弹窗前可返回重选。确认后不可更改。

#### 4. 身份影响范围

身份不仅仅是开局配置——它在整局游戏中通过以下方式产生持续影响：

| 影响维度 | 说明 | 实现方式 |
|---------|------|---------|
| **初始卡组** | 决定本局构筑的起点，影响前期战斗风格 | 开局写入deck，之后按战利品/事件正常演变 |
| **专属天赋** | 整局持续的被动效果（如灵石加成） | 在GSM talent系统注册，各系统检查 |
| **叙事差异化** | 部分事件文本因身份不同而变化 | 事件系统通过 `player.identity_id` 做分支 |
| **商店折扣** | 未来可扩展：七玄门弟子在正道商店有折扣 | 预留接口，当前不实现 |
| **剧情分支** | 部分剧情选项仅特定身份可用 | 剧情系统通过身份ID判定 |

#### 5. 身份与轮回天赋的交互

轮回天赋系统（系统#26）可扩展身份的维度：

| 扩展类型 | 效果 | 解锁方式 |
|---------|------|---------|
| **新身份** | 解锁第4个隐藏身份（如「掩月宗弟子」） | 轮回天赋解锁 |
| **身份强化** | 七玄门弟子初始灵石+10 / 魔修遗孤首回合费用+2 | 轮回天赋强化 |
| **移除身份限制** | 某个身份变为可用（原需解锁的） | 轮回天赋解锁 |

当前三种身份均为默认解锁。隐藏身份的设计留给轮回天赋系统。

#### 8. 身份重选（新游戏/轮回）

每局新游戏开始时强制进入身份选择。轮回天赋解锁的内容（如新身份）在选择界面中展示，但标记为「已解锁」。

如果玩家读档：跳过身份选择，直接从存档中的 `identity_id` 恢复。存档不包含身份选择界面的状态。

### 状态与转换

```
[[新游戏启动]]
    ↓
[身份选择界面]
    ↓ 选择身份
[确认弹窗]
    ↓ 确认
[应用初始状态] → GSM写入identity_id → 写入初始卡组 → 写入角色 → 设置灵石 → 注册天赋
    ↓
[开局叙事展示]（显示身份的 flavor_text）
    ↓
[进入游戏]（探索地图）
    ↓
[游戏中] — 身份持续影响（天赋生效、事件分支判断）
    ↓
[本局结束]（通关/死亡）
    ↓
[返回主菜单] → 下次新游戏→回到身份选择
```

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **游戏状态管理器** | 写入 `player.identity_id`、`deck.current_deck`、`deck.slots`、`player.resources.ling_shi` | 读取 `player.identity_id`（身份已选跳过选择） |
| **卡牌系统** | 身份模板中的卡牌ID列表（初始卡组） | 卡牌模板数据（验证身份模板中的卡牌ID有效性） |
| **卡组编辑系统** | 初始卡组写入 `deck.current_deck`（卡组编辑的起点） | 后续卡组变更（身份不影响编辑规则） |
| **事件系统** | 事件文本通过 `player.identity_id` 做分支 | 身份条件检查（部分事件需要特定身份） |
| **剧情/叙事系统** | 身份相关的开局叙事文本 | 章节推进中的身份判定 |
| **轮回天赋系统** | 身份解锁条件、身份强化数据 | 已解锁的身份列表、已激活的强化 |
| **战斗系统** | 天赋效果（如魔修首回合+1费） | — |
| **资源系统** | 初始灵石设置 | — |

## 公式

### 1. 随机散修角色生成

```
random_wandering_character() → Card
  pool = all 炼气期蓝色稀有度 角色卡
  if pool is empty: return null  # 不应发生——至少有基础角色卡
  return random_choice(pool)
```

### 2. 天赋效果注册

身份的天赋效果通过全局事件注册，在身份确认时写入GSM的 `player.talents`：

```
apply_identity_talent(identity_id):
  talent = identity_template[identity_id].talent
  GSM.player.talents.append(talent.id)
  # 各系统在相关操作时检查 talent.id 并应用效果
  
  示例检查（灵石获取时）：
  if "ling_shi_boost" in GSM.player.talents:
    amount *= 1.15  # 七玄门弟子灵石 +15%
  
  示例检查（战斗首回合）：
  if "first_strike_extra_cost" in GSM.player.talents and turn == 1:
    extra_cost += 1  # 魔修遗孤首回合 +1 费
```

### 3. 初始资源设置

```
apply_initial_resources(identity_id):
  resources = identity_template[identity_id].initial_resources
  GSM.set("player.resources.ling_shi", resources.ling_shi)
  # 后续资源系统如果扩展其他资源类型，从此处扩展
```

## 边界情况

- **所有身份都未解锁**：不应发生——3种基础身份默认解锁。如果轮回天赋系统设置了「需解锁」，至少有1个身份永远可用
- **玩家在身份确认前退出游戏**：身份选择未提交，不写入GSM。下次新游戏仍然进入身份选择界面
- **读档后的身份恢复**：存档中包含 `identity_id`，读档后直接恢复，不经过身份选择界面
- **身份模板中引用的卡牌ID不存在**：系统启动时验证所有身份模板的卡牌 ID——如果某个ID无效，阻止该身份可选并记录错误日志。所有身份在发布前必须通过卡牌ID完整性测试
- **随机散修角色生成时池为空**：如果全角色卡池为空（极低概率），使用默认的「韩立·少年」作为保底
- **专属天赋与轮回天赋冲突**：轮回天赋系统设计时确保不覆盖专属天赋。专属天赋 + 轮回天赋的同类效果可叠加（加法）
- **身份选择后玩家想反悔**：仅在确认弹窗前可返回。确认后→不可反悔。存档后读档恢复原身份
- **同一个身份重复选择多局**：允许——每次新游戏重新选择，不受上一局选择影响
- **身份选择时卡牌系统未初始化**：身份选择界面应在GSM和卡牌系统就绪后打开（在 game_initialized 信号之后）

## 依赖关系

| 依赖系统 | 性质 | 说明 |
|----------|------|------|
| **游戏状态管理器** | 硬依赖 | 身份ID写入 `player.identity_id`；初始状态应用（卡组、角色、灵石、天赋） |
| **卡牌系统** | 硬依赖 | 身份模板中的卡牌ID有效性验证；随机散修角色从卡牌池提取 |
| **卡组编辑系统** | 硬依赖 | 初始卡组写入 `deck.current_deck`（卡组编辑系统的起点配置） |
| **轮回天赋系统** | 软依赖 | 身份解锁条件、身份强化的扩展接口（当前未实现，预留） |

### 上游依赖设计顺序

游戏状态管理器（已设计） → 卡牌系统（已设计） → 开局身份选择系统 → 卡组编辑系统（已设计）

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 基础身份数量 | 3种 | 3-6 | 选择太少 | 分析瘫痪 |
| 七玄门初始卡牌张数 | 7张 | 5-12 | 开局太弱 | 开局太强 |

| 七玄门灵石加成 | +15% | +10%~+30% | 天赋感觉不明显 | 经济碾压 |

### 音效需求

- 身份卡片入场：低沉而庄严的号角/编钟
- 卡片选中：上升音阶+确认重音
- 确认选择：厚重的确认音+轻微混响
- 开局叙事：柔和的背景弦乐

## 验收标准

- **GIVEN** 新游戏开始，**WHEN** 进入身份选择界面，**THEN** 展示3种身份卡片（七玄门外门弟子/魔修遗孤/散修路人）
- **GIVEN** 身份选择界面，**WHEN** 点击身份卡片，**THEN** 卡片展开显示完整详情
- **GIVEN** 身份选择界面，**WHEN** 未选择任何身份尝试确认，**THEN** 确认按钮不可用
- **GIVEN** 玩家选择七玄门外门弟子并确认，**WHEN** 检查卡组，**THEN** 卡组包含长春功×2、基础攻击×3、简易护盾×2，角色位1为韩立·少年
- **GIVEN** 玩家选择七玄门外门弟子，**WHEN** 检查初始灵石，**THEN** 灵石为15
- **GIVEN** 玩家选择魔修遗孤并确认，**WHEN** 检查卡组，**THEN** 卡组包含玄阴经×2、魔气斩×3、血祭术×1、基础攻击×2，角色位1为魔修护卫
- **GIVEN** 玩家选择魔修遗孤，**WHEN** 检查初始灵石，**THEN** 灵石为10
- **GIVEN** 玩家选择散修路人并确认，**WHEN** 检查卡组，**THEN** 卡组包含基础攻击×4 + 3张随机蓝色卡 + 1张随机紫色卡，角色位1为随机散修角色
- **GIVEN** 玩家选择散修路人，**WHEN** 检查初始灵石，**THEN** 灵石为20
- **GIVEN** 玩家选择七玄门外门弟子，**WHEN** 进入战斗首回合，**THEN** 无额外费用加成（非魔修）
- **GIVEN** 玩家选择魔修遗孤，**WHEN** 战斗首回合，**THEN** 可用费用+1（天赋生效）
- **GIVEN** 玩家选择魔修遗孤，**WHEN** 战斗第2回合，**THEN** 额外费用消失
- **GIVEN** 玩家选择七玄门外门弟子，**WHEN** 探索获得灵石，**THEN** 额外+15%（天赋生效）
- **GIVEN** 玩家选择散修路人，**WHEN** 第一次战斗胜利的战利品展示，**THEN** 至少有一个金色卡牌选项
- **GIVEN** 身份确认弹窗，**WHEN** 点击「返回重选」，**THEN** 回到选择界面，可重新选择
- **GIVEN** 身份确认后，**WHEN** 读档，**THEN** 跳过身份选择，直接从存档中的 identity_id 恢复
- **GIVEN** 身份模板中的某个卡牌ID不存在，**WHEN** 系统启动验证，**THEN** 该身份标记为不可选并记录错误
- **GIVEN** 散修路人随机池为空（极端情况），**WHEN** 生成角色，**THEN** 使用韩立·少年作为保底
