## 概述

结局分支系统是叙事层的「终局裁决者」——它不定义每章的局部选择（那是剧情系统的职责），而是**跨章节聚合所有 story_flags，在游戏通关时判定最终结局走向**。系统将玩家在5章中累积的 30+ 个 story_flag 输入，映射到 3 条全局结局线（含每条线的变体），并在通关后呈现对应的尾声叙事、结局CG和通关奖励。

对玩家来说，结局分支系统是「我这一路的选择最终通往哪里」的答案——第1章拒绝墨渊、第3章坚守正道、第4章带银翎飞升……每一个决策不是孤立的，它们在结局时汇聚成一个完整的故事弧线。这种「累积的选择终有回响」的感觉，是叙事重玩价值的核心驱动力。

结局系统的设计信条：**结局不是随机抽奖——每一步选择都在铺路，结局只是路的终点**。玩家在结局前不知道具体是哪条线，但能看到「你的选择正在导向……」的模糊提示。

## 玩家幻想

| 情感 | 触发场景 |
|------|---------|
| **终局的回响** | 尾声文本中提及「因为你第2章摧毁了枯骨洞府，东域正道得以……」——「我的选择被记住了」 |
| **路线的主人感** | 通关后看到结局路线图——「我走的是『正道守护者·银翎相伴』线，你的呢？」 |
| **重玩的动力** | 发现结局图鉴中有3条主线+6种变体——「下次走魔道路线看看」 |
| **遗憾的美学** | 因为某个 story_flag 条件不满足，最想要的结局灰了——「下一局，我一定……！」 |
| **凡人的重量** | 第5章最终选择：「飞升仙界 / 留在归墟之境 / 归隐东域」——每个选择背后是不同的价值观，不是简单的「好/坏」结局 |
| **二周目的期待** | 通关后轮回天赋+结局图鉴双解锁——「这次有了暗金天命，试试能不能走到『超脱轮回』结局」 |

结局系统的核心情感是**收束**——5章的漂泊在结局时凝聚成一个有意义的句号。这个句号因玩家的每一个选择而不同。

## 详细设计

### 核心规则

#### 1. 全局结局判定模型

结局系统在**第5章章末BOSS击败后+第5章章末结局选择完成时**触发最终判定。判定流程：

```
determine_global_ending():
  ① 收集输入数据：
     a. 所有5章的 ending_chosen（每章末选择的结局分支ID）
     b. 所有 story_flags（~30个布尔标记）
     c. 当前境界（realm_level）
     d. 本局身份（identity_id）
     e. 累计通关次数（来自 progression）
  
  ② 计算3条结局线各自的「符合度得分」：
     - 飞升仙界线：score = sum(飞升线相关flag权重)
     - 留在归墟之境线：score = sum(守护线相关flag权重)
     - 归隐东域线：score = sum(凡人线相关flag权重)
  
  ③ 取最高分 → 作为「主结局线」
  
  ④ 在主结局线内部，根据特定条件判定「变体」：
     - 飞升仙界线 → 变体：独自飞升 vs 携伴飞升
     - 留在归墟之境线 → 变体：成为归墟之境守护者 vs 建立新秩序
     - 归隐东域线 → 变体：隐居凡间 vs 开宗立派
  
  ⑤ 返回最终结局ID（如 "ascension_solo"、"guardian_order" 等）
```

#### 2. 三条结局主线

##### A. 飞升仙界 —— 天道终点

> **主题**：修仙者的终极追求——飞升成仙，超脱轮回。
> **情感基调**：宏大、释然、新的开始

| 条件 | 权重 | 说明 |
|------|:----:|------|
| 第5章选择「飞升仙界」 | +30 | 决定性条件 |
| 第4章选择「独自飞升」或「带银翎飞升」 | +15 | 飞升的准备 |
| 第3章选择「坚守正道」 | +10 | 正道之心 |
| 第2章选择「摧毁洞府」（正道声望） | +8 | 拒绝了诱惑 |
| 第1章选择「拒绝墨渊」 | +7 | 坚守底线 |
| 击杀≥20个精英敌人 | +5 | 战力证明 |
| 收集≥100种不同卡牌 | +5 | 博学广识 |

**变体判定：**

| 变体 | 条件 | 尾声差异 |
|------|------|---------|
| **独自飞升·仙道孤独** | 第4章选择「独自飞升」 | 银翎留在归墟之境等待；尾声描绘林渊孤独踏上天梯的背影 |
| **携伴飞升·仙侣同行** | 第4章选择「带银翎一起飞升」 | 银翎与林渊并肩飞升；尾声描绘二人在仙界初遇的晨光 |

##### B. 留在归墟之境 —— 守护之道

> **主题**：飞升不是唯一的路——守护这片奋斗过的土地同样有意义。
> **情感基调**：沉稳、担当、余生

| 条件 | 权重 | 说明 |
|------|:----:|------|
| 第5章选择「留在归墟之境守护」 | +30 | 决定性条件 |
| 第3章选择「中立调停」 | +15 | 不偏不倚的平衡者 |
| 第2章选择「夺取枯骨秘宝」（了解黑暗面） | +10 | 了解力量的代价 |
| 第1章选择「接受墨渊条件」（了解魔道） | +8 | 了解对立面的立场 |
| 炼药/炼器≥20次 | +8 | 对归墟之境资源的深入理解 |
| 银翎存活（未在第4章死亡） | +5 | — |
| 本局身份=碎星群岛散修或丹霞谷弟子 | +5 | 非正非邪的灵活视角 |

**变体判定：**

| 变体 | 条件 | 尾声差异 |
|------|------|---------|
| **归墟之境守护者·孤身守望** | 第4章选择「独自飞升」（银翎已离开） | 林渊独自守护归墟之境各族；尾声描绘他在古魔战场遗址植下一棵灵树 |
| **建立新秩序·万族共主** | 第4章选择「带银翎一起飞升」但第5章选留归墟之境 | 林渊与银翎在归墟之境建立正道联盟；尾声描绘各族代表齐聚议事大殿 |

##### C. 归隐东域 —— 凡人之心

> **主题**：修仙千年，最终还是想回到那个青云剑宗的小屋。凡人修仙——起点也是终点。
> **情感基调**：宁静、圆满、归家

| 条件 | 权重 | 说明 |
|------|:----:|------|
| 第5章选择「归隐东域」 | +30 | 决定性条件 |
| 第1章选择「拒绝墨渊」 | +12 | 初心不改 |
| 第2章苍玄古战场后成功重修筑基 | +10 | 在跌倒的地方站起来 |
| 第3章选择任何选项（此项中性） | +5 | 东域始终是你的故乡 |
| 击杀≤10个精英敌人（更偏向发育而非战斗） | +8 | 不嗜杀 |
| 本局身份=青云剑宗外门弟子 | +10 | 在东域的根 |
| 累计轮回次数≥5 | +5 | 看尽繁华后回归 |

**变体判定：**

| 变体 | 条件 | 尾声差异 |
|------|------|---------|
| **归隐凡间·林家村** | 银翎死亡 或 无特殊角色羁绊 | 林渊以凡人之姿回到青云剑宗外门，在当年的小屋里坐下——「我回来了」 |
| **开宗立派·仙途问道** | 银翎存活 + 轮回天赋≥10个已解锁 + 累计通关≥3次 | 林渊在东域创立「仙途宗」，将毕生所学传承下去；尾声以新入门弟子的视角望向仙途峰 |

#### 3. 结局得分平局处理

如果两条线的得分相同（或有条件的分数差在5分以内）：

```
resolve_tie(scores):
  # 优先顺序（当分数差≤5时）：
  1. 飞升仙界 > 留在归墟之境 > 归隐东域（按修仙世界的价值观）
  2. 如果飞升线与守护线得分相同 → 飞升线（修仙的最终目标是飞升）
  3. 如果守护线与凡人线得分相同 → 守护线（留在修仙世界而非退回凡人生活）
  
  # 但第5章的选择直接体现玩家意志，同分时该章选择加权：
  if chapter_5_ending == "ascend":
    scores.ascend += 5  # 偏斜向飞升
  elif chapter_5_ending == "guard":
    scores.guard += 5   # 偏斜向守护
  elif chapter_5_ending == "return":
    scores.return += 5  # 偏斜向回归
```

> **设计理由**：第5章的选择是玩家面对最终抉择时的直接意志——即使「前4章」的累积偏向另一条线，第5章的最终选择仍然有扭转力。这是「此刻的选择比过去更重」的叙事哲学。

#### 4. 通关展示流程

```
game_victory_sequence():
  ① 第5章BOSS击败 → 第5章结局分支选择
  ② 屏幕渐黑 → 2秒黑屏
  ③ 【结局判定】不显示任何UI（后台计算）
  ④ 【结局展示·第1幕】静态CG + 结局标题（如「飞升仙界·仙侣同行」）
     - 持续2.5秒
     - 背景音乐：结局主题（宏大变体/沉稳变体/宁静变体）
  ⑤ 【结局展示·第2幕】尾声叙事文本（4~8句，逐句淡入）
     - 具体文本取决于结局ID + 关键story_flags
     - 每句显示2秒，共8~16秒
     - 可点击跳过
  ⑥ 【结局展示·第3幕】本局统计面板：
     - 结局路线名称
     - 5章结局选择回顾（如树形分支图）
     - 关键数值（境界、击杀、收集、炼制）
     - 轮回点获取明细
  ⑦ 【结局图鉴更新】解锁本结局的图鉴条目
  ⑧ 【轮回结算】→ 天赋树界面 → 主菜单
```

**结尾CG配置（3张静态插画）：**

| 结局线 | CG内容 |
|--------|--------|
| 飞升仙界 | 林渊（或林渊+银翎）立于云端天梯之上，远处仙界之门光芒万丈 |
| 留在归墟之境 | 林渊立于高处俯瞰归墟之境山河，各族城邦散布在大地上 |
| 归隐东域 | 林渊推开青云剑宗旧居的木门，夕阳从门缝洒入，屋内一切如旧 |

#### 5. 结局图鉴

结局图鉴是一个主菜单功能，记录玩家所有已解锁的结局：

```
EndingGallery {
  entries: [
    {
      ending_id: "ascension_solo",
      name: "飞升仙界·仙道孤独",
      unlocked: true,
      unlocked_at: "2026-08-15T14:30:00Z",
      chapter_path: [          # 解锁该结局时的5章选择路径
        "ch1: reject_mo",
        "ch2: destroy_cave",
        "ch3: defend_righteous",
        "ch4: ascend_alone",
        "ch5: ascend"
      ],
      key_stats: {             # 解锁时的关键数据
        identity: "丹霞谷弟子",
        final_realm: "化神",
        total_reincarnations: 12
      }
    },
    ...
  ]
  total_endings: 6             # 3条主线 × 2变体
  unlocked_count: 3            # 当前已解锁
}
```

**结局图鉴界面**以横向卡片排列展示所有结局，未解锁的显示为暗色剪影+「??? 」+提示文字（如「或许换个身份……」「试着拒绝墨渊的条件……」）。

**解锁后显示的完整信息：**
- 结局名称 + 线路标签
- 结局CG缩略图
- 5章选择路径（树状分支图，高亮显示该结局所走的路径）
- 解锁日期+该局身份
- 变体条件对照表（哪些条件导向了哪个变体）

#### 6. 结局对后续游戏的影响

结局系统写入 `progression.dat` 的数据：

```
# progression.dat 中的结局相关字段
{
  "endings_unlocked": ["ascension_solo", "ascension_duo", "guardian_lone"],
  "ending_statistics": {
    "ascension_solo": {unlocked_at: "...", chapter_path: [...], identity: "...", ...},
    ...
  },
  "total_completions": 3,          # 累计通关次数
  "highest_ending_tier": "ascension"  # 最佳结局线
}
```

**结局对游戏的影响：**
- 解锁结局不影响轮回点获取（轮回点计算独立于结局）
- 解锁全部6个结局 → 解锁隐藏称号「超凡入圣」（次要UI展示）
- 某些成就可能关联特定结局解锁（成就系统设计时定义）
- 结局图鉴数据不随 progression.dat 重置（永久保留——即使轮回数据损坏，结局图鉴应独立存储或作为独立校验文件 `endings.dat`）

#### 7. 叙事文本的剧情引用

尾声叙事文本中引用关键 story_flags 的技术实现：

```
generate_epilogue(ending_id, story_flags, chapter_path):
  base_text = EPILOGUE_TEXTS[ending_id]  # 通用尾声
  insertions = []  # 因 story_flags 插入的特定段落
  
  # 第1章选择引用
  if story_flags["ch1_accepted_mo_condition"]:
    insertions.append("你记得那一日在云澜城，墨渊的夺舍条件你曾动过念头……")
  
  # 第2章选择引用
  if story_flags["ch2_took_bone_secret"]:
    insertions.append("枯骨老祖的秘宝至今仍在你储物袋中——力量的代价，你已经懂了。")
  else:
    insertions.append("摧毁枯骨洞府的那一击，让你在正道中赢得了尊重——但你也知道，秘宝的诱惑从未消失。")
  
  # 第3章选择引用
  if story_flags["ch3_joined_demonic"]:
    insertions.append("东域的纷争中你选择了魔道——不是因为邪恶，而是你看到了正道的虚伪。")
  ...
  
  # 特定角色存活/死亡
  if story_flags["li_feiyu_alive"]:
    insertions.append("苏剑鸣在远处对你挥了挥手，转身消失在人群中。活着就好——修仙路上，能活着的人不多。")
  
  # 将插入段落融入尾声
  return interleave(base_text, insertions)
```

**尾声长度控制：**
- 基础尾声（无插入）：4~6句
- 每触发一个插入条件，增加1~2句
- 总长度不超过12句（约2~3分钟阅读时间）
- 超过12句的插入按优先级截断

#### 8. 结局系统与章节结局的关系

结局系统与剧情系统的职责分界：

| 职责 | 剧情系统（章节层） | 结局系统（全局层） |
|------|-----------------|------------------|
| 定义每章末的2~3选1 | ✅ | — |
| 章节选择写入 story_flags | ✅ | — |
| 聚合 story_flags 判定全局结局 | — | ✅ |
| 展示尾声叙事 | — | ✅ |
| 管理结局图鉴 | — | ✅ |
| 通关展示流程 | — | ✅ |
| 第5章末结局选择 | ✅ 提供选择（飞升/守护/回归） | ✅ 将选择纳入得分计算 |

## 公式

### 1. 结局线得分计算

```
calculate_ending_scores(story_flags, chapter_path, run_data) → {ascend: int, guard: int, return: int}:
  scores = {ascend: 0, guard: 0, return: 0}
  
  # ===== 飞升仙界线 =====
  if chapter_path["ch5"] == "ascend":        scores.ascend += 30
  if chapter_path["ch4"] in ["ascend_alone", "ascend_with_yinyue"]:
                                              scores.ascend += 15
  if chapter_path["ch3"] == "defend_righteous": scores.ascend += 10
  if chapter_path["ch2"] == "destroy_cave":   scores.ascend += 8
  if chapter_path["ch1"] == "reject_mo":      scores.ascend += 7
  if run_data.elites_killed >= 20:            scores.ascend += 5
  if run_data.unique_cards >= 100:            scores.ascend += 5
  
  # ===== 留在归墟之境线 =====
  if chapter_path["ch5"] == "guard":          scores.guard += 30
  if chapter_path["ch3"] == "neutral_mediate": scores.guard += 15
  if chapter_path["ch2"] == "take_secret":    scores.guard += 10
  if chapter_path["ch1"] == "accept_mo":      scores.guard += 8
  if run_data.craft_count >= 20:              scores.guard += 8
  if story_flags["yinyue_alive"]:             scores.guard += 5
  if run_data.identity in ["star_storm_wanderer", "yellow_maple_disciple"]:
                                              scores.guard += 5
  
  # ===== 归隐东域线 =====
  if chapter_path["ch5"] == "return":         scores.return += 30
  if chapter_path["ch1"] == "reject_mo":      scores.return += 12
  if story_flags["ch2_rebuilt_foundation"]:   scores.return += 10
  if chapter_path["ch3"] != null:             scores.return += 5
  if run_data.elites_killed <= 10:            scores.return += 8
  if run_data.identity == "seven_peaks_disciple": scores.return += 10
  if run_data.total_reincarnations >= 5:      scores.return += 5
  
  # ===== 第5章偏斜（同分时） =====
  # 此偏斜在 resolve_tie() 中应用，不在此计算
  
  return scores
```

### 2. 平局解决

```
resolve_tie(scores, chapter_5_choice) → final_line:
  # 第5章偏斜
  if chapter_5_choice == "ascend":  scores.ascend += 5
  if chapter_5_choice == "guard":   scores.guard += 5
  if chapter_5_choice == "return":  scores.return += 5
  
  max_score = max(scores.ascend, scores.guard, scores.return)
  candidates = [line for line in ["ascend", "guard", "return"] if scores[line] == max_score]
  
  if len(candidates) == 1:
    return candidates[0]  # 唯一最高分
  
  # 多人平局 → 优先级顺序
  priority = ["ascend", "guard", "return"]
  for line in priority:
    if line in candidates:
      return line
```

### 3. 变体判定

```
determine_variant(ending_line, story_flags, chapter_path, run_data) → variant_id:
  match ending_line:
    "ascend":
      if chapter_path["ch4"] == "ascend_with_yinyue":
        return "ascension_duo"   # 携伴飞升
      else:
        return "ascension_solo"  # 独自飞升
    
    "guard":
      if story_flags["yinyue_alive"] and chapter_path["ch4"] == "ascend_with_yinyue":
        return "guardian_order"  # 建立新秩序
      else:
        return "guardian_lone"   # 孤身守望
    
    "return":
      if story_flags["yinyue_alive"] and run_data.unlocked_talents_count >= 10 and run_data.total_completions >= 3:
        return "return_sect"     # 开宗立派
      else:
        return "return_home"     # 归隐凡间
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|:----:|------|
| ending_line | enum | {ascend, guard, return} | 主结局线 |
| chapter_path | dict | {"ch1": ..., "ch5": ...} | 5章结局选择 |
| story_flags | dict | 30+ flags | 全局剧情标记 |
| run_data | struct | — | 本局运行数据 |

## 边界情况

- **玩家在第5章结局选择后立即退出游戏**：第5章结局选择已写入 story_flags → 下次启动时GSM检测到 completed_chapters 包含所有5章 → 跳过探索直接进入结局展示流程
- **玩家在第4章ED前死亡（未到第5章）**：不触发结局判定——结局系统只在通关（game_victory）时运行
- **玩家所有3条线得分相同（极不可能但理论存在）**：按优先级飞升>守护>回归，同时第5章偏斜打破僵局
- **story_flags 数据异常（某些flag未定义）**：缺失的flag视为 false/null——不影响结局计算，但日志警告
- **玩家没有经历完整的5章选择路径（如果某章被跳过但系统允许了通关）**：章节推进要求 prev_chapter_completed 为硬性条件——跳过章节进不了第5章，不会触发此问题
- **结局图鉴中的某些结局因版本更新而不再可达**：图鉴中的结局条目永久保留——如果结局条件在后续更新中修改，已有解锁的玩家不受影响
- **progression.dat 损坏但 endings.dat 完好**：结局图鉴独立保存（endings.dat），不受 progression.dat 重置影响
- **结局展示中玩家断网/Alt+F4**：结局展示是纯展示（无状态写入），退出后下次启动检测到已完成5章 → 从结局展示第1幕重新开始
- **第5章结局选择是「留在归墟之境维护」但飞升线得分最高**：第5章偏斜+5分打破平局——如果加上5分后仍低，说明前4章的选择非常明确偏向飞升线。这是设计意图——「此刻的选择比过去更重，但不是绝对」
- **结局文本引用了已死亡的角色的台词**：检测该角色是否存活（story_flags中的状态），死了则跳过该插入段落
- **累计通关3次解锁「开宗立派」变体——如果 progression.dat 曾损坏导致累计被重置**：`total_completions` 来自 progression，损坏后归零。使用 endings.dat 中的 `total_completions` 作为备份校验（endings.dat 独立持久化且极少写入）
- **所有6个结局解锁后**：结局图鉴界面显示「图鉴完成·超凡入圣」+ 解锁隐藏称号。不会解锁新结局（除非DLC新增）

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |

| 每线变体数 | 2种 | 1~3 | 同线体验无差异 | 变体条件太挑剔 |

## 验收标准

- **GIVEN** 玩家击败第5章BOSS并选择「飞升仙界」，**WHEN** 前4章选择偏向飞升线（拒绝墨渊+摧毁洞府+坚守正道+独自飞升），**THEN** 结局=飞升仙界·仙道孤独
- **GIVEN** 玩家击败第5章BOSS并选择「飞升仙界」，**WHEN** 第4章选择「带银翎一起飞升」，**THEN** 结局=飞升仙界·仙侣同行
- **GIVEN** 玩家击败第5章BOSS并选择「留在归墟之境守护」，**WHEN** 前4章偏向中立/魔道，**THEN** 结局=留在归墟之境（变体根据银翎状态判定）
- **GIVEN** 玩家击败第5章BOSS并选择「归隐东域」，**WHEN** 身份=青云剑宗外门弟子+银翎存活+轮回天赋≥10+累计通关≥3，**THEN** 结局=归隐东域·开宗立派
- **GIVEN** 玩家击败第5章BOSS并选择「归隐东域」，**WHEN** 条件不满足变体条件，**THEN** 结局=归隐东域·归隐凡间
- **GIVEN** 飞升线得分=50且守护线得分=50（平局），**WHEN** 第5章选择=飞升，**THEN** 结局=飞升线（第5章偏斜+5打破平局）
- **GIVEN** 飞升线得分=55且守护线得分=50，**WHEN** 第5章选择=守护（偏斜+5后50+5=55=平局），**THEN** 结局=飞升线（优先级飞升>守护）
- **GIVEN** 玩家在第5章结局选择后退出，**WHEN** 重新启动，**THEN** 检测到5章已完成→进入结局展示流程
- **GIVEN** 结局展示中，**WHEN** 玩家点击跳过，**THEN** 跳至本局统计面板
- **GIVEN** 结局展示完成，**WHEN** 检查结局图鉴，**THEN** 该结局条目从???变为已解锁
- **GIVEN** 结局图鉴中有6个结局条目，**WHEN** 查看未解锁条目，**THEN** 显示暗色剪影+「???」+提示文字
- **GIVEN** 玩家未通关（在第3章死亡），**WHEN** 检查结局判定，**THEN** 不触发结局系统
- **GIVEN** progression.dat 损坏，**WHEN** 检查结局图鉴，**THEN** 结局图鉴仍完整（endings.dat 独立存储）
- **GIVEN** 第2章选择摧毁洞府（非夺取秘宝），**WHEN** 飞升线序幕文本生成，**THEN** 引用「摧毁枯骨洞府」的相关文本
- **GIVEN** 苏剑鸣在第1章存活，**WHEN** 归隐东域结局的尾声生成，**THEN** 包含苏剑鸣的相关插入文本
- **GIVEN** 玩家解锁全部6个结局，**WHEN** 查看结局图鉴，**THEN** 显示「图鉴完成·超凡入圣」
- **GIVEN** 第5章结局选择后，**WHEN** 通关展示流程进入CG展示，**THEN** 播放对应结局线的主题音乐
- **GIVEN** 结局图鉴详情界面，**WHEN** 查看已解锁结局，**THEN** 显示5章选择树状图（高亮该结局路径）+变体条件对照表
