# 剧情系统（章节推进）(Story System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-23
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱4「始于凡人，忠于原著」

> **快速参考** — 层级：`叙事 (Narrative)` · 优先级：`Vertical Slice` · 关键依赖：`探索系统`、`境界系统`、`事件系统`

## 概述

剧情系统管理游戏的主线故事推进——它不是一个播放过场动画的叙事引擎，而是一个**章节状态机**。每个章节定义了一组入口条件、必经事件、可选的支线故事标记和章节末结局分支。剧情系统将境界突破（玩家主动行为）与章节推进（叙事节奏）解耦——章节推进由关键事件触发，而非自动跟随境界提升。

对玩家来说，剧情系统回答了「我现在在故事里的哪个位置」和「接下来会发生什么」。它让探索不是漫无目的的刷图，而是沿着原著脉络推进的旅程——你会期待某张地图的 BOSS 战因为是「墨居仁夺舍事件」，你会因为触发「正魔大战」剧情而经历境界跌落。这些叙事节点不是简单的事件文本，而是对游戏状态产生实际影响的结构性转折。

## 玩家幻想

| 情感 | 触发场景 |
|------|---------|
| **故事的归属感** | 每章开头的引子文本——「第二话：乱星海漂泊」——熟悉的原著章节标题唤起记忆 |
| **关键剧情的高潮** | 章节末 BOSS 战前的事件链——「正魔大战即将爆发……准备好了吗？」 |
| **选择的重量** | 章节末的结局分支——「加入鬼灵门」vs「坚守黄枫谷」——选择影响下一章的剧情走向 |
| **世界对我的反应** | NPC 对话因 `story_flags` 不同而改变——上一章的选择在这一章被提及 |
| **原著彩蛋的惊喜** | 随机事件中触发原著经典桥段——「这瓶丹药……莫非是韩立的掌天瓶所炼？」 |

剧情系统的设计哲学：**故事是探索的路标，不是必须观看的电影**。玩家不会被迫看大段过场——故事通过事件选择、探索对话和章节转折自然传递。跳过剧情的玩家不会损失战力，但会损失「我为什么在这里做这件事」的上下文。

## 详细设计

### 核心规则

#### 1. 章节结构定义

游戏共 5 个主线章节，对应原著 5 个关键阶段：

```
章节模板：
ChapterTemplate {
  id: String              # 章节ID（如 "ch1_qixuan"）
  chapter_number: int     # 章节序号 [1, 5]
  title: String           # 章节标题（如 "第一话：七玄门风云"）
  subtitle: String        # 副标题（如 "凡人修仙之始"）
  
  entry_conditions: {     # 进入条件（全部满足才能开始本章）
    min_realm: int        # 最低境界要求
    prev_chapter_completed: bool  # 前一章是否完成
    required_flags: [String]      # 必须已触发的 story_flag 列表
  }
  
  required_events: [      # 本章必经事件（必须按顺序触发）
    {event_id, trigger_condition, completion_effect}
  ]
  
  chapter_boss: {         # 章末BOSS
    boss_id: String
    unlock_condition: String  # 触发BOSS战的条件
  }
  
  ending_branches: [      # 章末结局分支（2~3个选择）
    {branch_id, label, condition, consequence}
  ]
  
  completion: {           # 完成本章后的效果
    unlock_next_chapter: bool
    unlock_maps: [String]     # 解锁的新地图列表
    set_flags: [String]       # 设置的全局 story_flag
    permanent_bonus: String   # 永久加成描述
  }
  
  maps: [String]          # 本章可探索的地图列表（按难度排序）
}
```

#### 3. 章节状态机

```
chapter_state_machine:
  [未解锁] ──满足 entry_conditions──→ [进行中]
                                            ↓
                                     玩家探索本章地图
                                     触发必经事件
                                     击败章末BOSS
                                            ↓
                                     [章节末·结局选择]
                                      ↙        ↓        ↘
                                 分支A      分支B      分支C
                                      ↘        ↓        ↙
                                     [已通关] → 设置 story_flags
                                                 解锁下一章
                                                 解锁新地图
```

**章节状态存储在 GSM 中：**

```
GSM.narrative = {
  current_chapter: "ch2_luanxinghai",      # 当前所在章节ID
  current_chapter_progress: {
    completed_required_events: [            # 本章已完成的必经事件
      "ch2_event_1", "ch2_event_2"
    ],
    boss_unlocked: false,                   # 章末BOSS是否已解锁（所有必经事件完成后自动解锁）
    boss_defeated: false,                   # 章末BOSS是否已击败
    ending_chosen: null                     # 本局选择的结局分支ID
  },
  completed_chapters: ["ch1_qixuan"],       # 已完成的章节ID列表
  story_flags: {                            # 全局剧情标记（跨章节持久）
    "ch1_accepted_mo_condition": false,     # 第1章墨居仁条件：接受/拒绝
    "ch2_took_bone_secret": true,           # 第2章玄骨洞府：夺取秘宝/摧毁
    "ch3_joined_demonic": false,            # 第3章正魔阵营选择
    "ch4_took_yinyue": true                 # 第4章带银月飞升
  }
}
```

#### 4. 章节推进流程

```
advance_chapter():
  ① 检查当前章节是否满足完成条件：
     - 所有必经事件已完成
     - 章末BOSS已击败
     - 结局分支已选择
  ② 如果满足：
     a. 将当前章节ID推入 completed_chapters
     b. 设置章节结局对应的 story_flags
     c. 解锁新地图列表
     d. 广播 chapter_completed 信号
     e. 如果有下一章 → 自动设置 current_chapter = next_chapter_id
     f. 如果下一章是最后一章（ch5）→ 特殊处理（展示「最终章」）
     g. 如果当前是最后一章（ch5）→ 触发通关流程
  ③ 如果不满足：
     - 返回缺少的条件列表（UI展示"还需完成XXX"）
```

**章节切换时机：**
- 章节完成 → 展现「第X话·完」画面（含下章预告文字）
- 下一章开始前展示章节标题卡（引子文本 + 解锁内容预览）
- 章节切换可以发生在探索地图中（不需要回到主菜单）

#### 5. 必经事件 vs 支线故事标记

**必经事件（Required Events）：**
- 每个章节有 4~8 个必经事件
- 必经事件在探索地图中以特殊节点显示（金色边框/感叹号标记）
- 不完成所有必经事件 → 章末BOSS不解锁
- 必经事件的顺序有时序依赖（事件2需要在事件1后触发）

**支线故事标记（story_flags）：**
- `story_flags` 是全局布尔标记，记录玩家的叙事选择
- 来源：结局分支选择、特定事件的决策、隐藏奇遇触发
- 用途：影响后续章节的对话文本、事件选项的可用性、NPC 反应、某些卡牌的解锁条件
- `story_flags` 跨章节保留——第1章的选择在第3章仍能被读取

**story_flags 分类：**

| 类别 | 示例 | 影响范围 |
|------|------|---------|
| 阵营选择 | `joined_demonic` | 第3章Boss变化、NPC对话变化 |
| 关键决策 | `spared_mo_juren` | 后续事件中墨居仁是否出现协助 |
| 角色关系 | `saved_li_feiyu` | 厉飞雨是否在后续章节作为可招募角色出现 |
| 隐藏触发 | `found_skyvault_bottle_secret` | 解锁隐藏事件线 |
| 章节结局 | `ch1_rejected_mo` | 决定下一章可用的事件分支 |

#### 6. 剧情与探索地图的绑定

每个探索地图定义了该地图上可触发的事件池，包括普通事件和剧情事件：

```
地图剧情配置示例：越国京城
{
  map_id: "yueguo_capital",
  chapter_id: "ch1_qixuan",
  required_events: ["ch1_event_4_mo_duoshe"],   # 此地图上的必经事件
  optional_story_events: [                        # 可选剧情事件
    {event_id: "ch1_side_li_feiyu_spar", condition: null},
    {event_id: "ch1_side_medicine_hall", condition: "story_flags.ch1_accepted_mo_condition"}
  ]
}
```

**剧情事件与普通事件的区别：**
- 剧情事件有固定的叙事文本（原著引用/致敬）
- 剧情事件有 `story_flag` 前置条件（某些对话选项需要特定 flag 才出现）
- 剧情事件可能改变 `story_flags`（写入新的 flag）
- 剧情事件不可重复触发（每局一次）

#### 7. 章节末结局分支

每章末尾在击败BOSS后，展示一个 2~3 选 1 的结局选择。这不是战斗选择——是纯叙事决策：

```
章末分支界面结构：
┌──────────────────────────────────────────┐
│          第X话 · 完                        │
│                                          │
│   [简述本章的剧情回顾——1~2句话]            │
│                                          │
│   面对[关键抉择]，你决定……                 │
│                                          │
│  ┌─────────────────────────────────┐     │
│  │ A. [选项A标题]                    │     │
│  │    [选项A简述——对下章的影响]       │     │
│  └─────────────────────────────────┘     │
│  ┌─────────────────────────────────┐     │
│  │ B. [选项B标题]                    │     │
│  │    [选项B简述——对下章的影响]       │     │
│  └─────────────────────────────────┘     │
│  ┌─────────────────────────────────┐     │
│  │ C. [选项C标题]（如有）             │     │
│  │    [选项C简述]                     │     │
│  └─────────────────────────────────┘     │
│                                          │
│         [点击选项后不可撤销]               │
└──────────────────────────────────────────┘
```

**分支设计原则：**
- 每个分支提供不同的 `story_flag` 和不同的下一章初始条件
- 不存在「正确」选择——每个分支有得有失
- 分支影响后续事件内容、某些卡牌的可获取性、NPC 对话
- 分支不产生立即的战力变化（不加属性、不给卡牌——叙事选择不是奖励选择）

#### 8. 章节入口与叙境文本

每个章节开始时展示章节标题卡：

```
┌──────────────────────────────────────────┐
│                                          │
│          第 X 话                           │
│      [章节标题·大号字]                     │
│      [副标题·中号字]                       │
│                                          │
│   [引子文本——2~3句话简述本章背景，         │
│    链接上一章的结局选择]                   │
│                                          │
│   新解锁地图：                             │
│   · [地图名]（[难度]）                     │
│   · [地图名]（[难度]）                     │
│                                          │
│         [点击继续]                         │
└──────────────────────────────────────────┘
```

**引子文本示例（第2章）：**
> 「拒绝了墨居仁的夺舍条件后，你在黄枫谷站稳了根基。筑基成功让你拥有了前往更广阔天地的资本——天南正道盟邀请你参与正魔之间的纷争。然而，一场更大的风暴正在酝酿……」
>
> （如果第1章选择了接受墨居仁条件）→「你接受了墨居仁的条件，虽获得了暂时的修为提升，但魔道功法已在体内埋下隐患。你听说乱星海有解除此隐患的机缘……」

### 状态与转换

```
全局章节状态转换：

[第1章·未开始] ——新游戏开始
    ↓ 自动进入
[第1章·进行中] ——玩家探索第1章地图
    ↓ 完成必经事件 + 击败章末BOSS + 选择结局分支
[第1章·完成] + [第2章·进行中]
    ↓
[第2章·完成] + [第3章·进行中]
    ↓
[第3章·完成] + [第4章·进行中]
    ↓
[第4章·完成] + [第5章·进行中]
    ↓
[第5章·完成] → 通关
```

**章节内状态转换：**

```
[进行中·早期] —— 探索地图，触发初期必经事件
    ↓
[进行中·中期] —— 解锁更多地图，触发中期事件
    ↓
[进行中·末期] —— 必经事件全部完成→章末BOSS解锁
    ↓
[章节末·BOSS战] —— 玩家前往BOSS所在节点
    ↓ 击败
[章节末·结局选择] —— 显示分支
    ↓ 选择
[完成] → 下一章
```

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **游戏状态管理器** | 写入 `narrative.current_chapter`、`narrative.story_flags`、`narrative.completed_chapters` | 读取当前叙事状态；`chapter_completed` 信号广播 |
| **境界系统** | 章节完成时触发境界依赖事件（如第2章天南古战场的境界跌落） | 当前境界（章节入口条件验证） |
| **探索系统** | 当前章节对应的地图解锁列表 | 玩家进入的地图ID（用于触发地图上的剧情事件） |
| **事件系统** | 必经事件的触发与完成检测；章节结局分支的执行 | 事件完成事件；`set_flag` / `advance_chapter` 指令 |
| **卡牌系统** | `story_flags` 影响某些卡牌是否出现在掉落池/商店 | — |
| **战斗系统** | 章末BOSS战触发 | BOSS战结果（胜利/失败） |

## 公式

### 1. 章节进入条件验证

```
can_enter_chapter(chapter_id, state) → {allowed: bool, reason: String}:
  chapter = CHAPTER_TEMPLATES[chapter_id]
  
  # 境界条件
  if state.player.realm_level < chapter.entry_conditions.min_realm:
    return {false, "需要达到{chapter.entry_conditions.min_realm}境界"}
  
  # 前一章完成条件
  if chapter.entry_conditions.prev_chapter_completed:
    prev_chapter = get_prev_chapter(chapter_id)
    if prev_chapter not in state.narrative.completed_chapters:
      return {false, "需要先完成「{prev_chapter}」"}
  
  # story_flag 前置条件
  for flag in chapter.entry_conditions.required_flags:
    if not state.narrative.story_flags[flag]:
      return {false, "未满足剧情条件：{flag}"}
  
  return {true, ""}
```

### 2. 章末BOSS解锁判定

```
is_boss_unlocked(chapter_id, state) → bool:
  chapter = CHAPTER_TEMPLATES[chapter_id]
  completed = state.narrative.current_chapter_progress.completed_required_events
  return all(event in completed for event in chapter.required_events)
```

### 3. 章节完成判定

```
is_chapter_complete(chapter_id, state) → bool:
  prog = state.narrative.current_chapter_progress
  return (prog.boss_defeated and prog.ending_chosen != null)
```

### 4. story_flag 依赖的对话/事件选项可见性

```
is_option_visible(option, story_flags) → bool:
  if not option.required_flags: return true  # 无前置条件，始终可见
  for flag in option.required_flags:
    if story_flags[flag] != option.required_flag_values[flag]:
      return false
  return true
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| option.required_flags | string[] | — | 选项需要的 flag 列表 |
| story_flags | dict | — | 玩家当前的全局剧情标记 |
| option.required_flag_values | dict | — | 选项需要的 flag 值 |

## 边界情况

- **玩家先突破境界再推剧情**：境界满足但章节未推进→玩家可以进入更高境界的地图（探索系统解锁），但剧情事件仍按章节顺序触发。剧情不会因境界超前而跳过
- **玩家在章节中死亡（非剧情杀）**：标准死亡结算——轮回天赋保持不变，但本章进度丢失。重新开始后从本章初始状态起步（所有本章剧情事件重置）
- **剧情杀（天南古战场）与正常死亡的区别**：天南古战场的正魔大战是不可跳过的剧情战斗——战后无论胜负都触发境界跌落。如果玩家在此战前已经HP极低——进入战斗后先回满HP（剧情战保护），战后触发跌落剧情
- **读档后章节进度**：完全恢复——story_flags、completed_chapters、current_chapter_progress 全部保存
- **玩家在第2章境界跌落后，之前的高境界卡牌还能用吗**：保留——境界跌落只影响境界属性（费用上限、上场人数等），已获得的卡牌不受影响
- **章节结局选择后后悔**：不可撤销——选择即写入 story_flags。这是叙事决策的「重量」
- **跳过章节（如果第3章未完成但满足了第4章的境界条件）**：不允许——章节必须按顺序完成，entry_conditions 中的 `prev_chapter_completed` 为硬性条件
- **多周目后同一章节的不同体验**：story_flags 每局重置（不跨局保留）。轮回天赋保留但 story_flags 不保留——每局的叙事选择是独立的
- **章节中某个必经事件的触发节点未被探索**：如果玩家绕过必经事件所在的地图节点→章末BOSS不解锁。玩家必须回头探索该节点
- **所有必经事件完成但玩家尚未达到境界要求去打章末BOSS**：如果章末BOSS有境界要求（如第5章BOSS需要化神期），而玩家虽然完成了事件但境界不够→BOSS节点显示但标注「推荐境界：化神」——玩家可以尝试挑战但境界压制可能很严重

## 验收标准

- **GIVEN** 新游戏开始，**WHEN** 检查章节，**THEN** current_chapter="ch1_qixuan"，completed_chapters=[ ]
- **GIVEN** 第1章所有必经事件完成+墨居仁击败+结局选择完成，**WHEN** advance_chapter，**THEN** 第1章进 completed_chapters，current_chapter 变为 "ch2_luanxinghai"
- **GIVEN** 第1章结局选择「拒绝墨居仁」，**WHEN** 检查 story_flags，**THEN** `ch1_accepted_mo_condition` = false
- **GIVEN** 第1章结局选择「接受墨居仁」，**WHEN** 检查 story_flags，**THEN** `ch1_accepted_mo_condition` = true
- **GIVEN** 第2章天南古战场正魔大战触发，**WHEN** 战后结算，**THEN** realm_level 从筑基跌回炼气（1），cultivation 清零
- **GIVEN** 第2章天南古战场剧情进入战斗，**WHEN** 战斗开始，**THEN** 玩家HP回满（剧情战保护）
- **GIVEN** 第2章乱星海外海修炼到1000/1000，**WHEN** 触发重新筑基事件，**THEN** 境界升回筑基（剧情突破，无渡劫战）
- **GIVEN** 当前章节必经事件未全部完成，**WHEN** 检查章末BOSS，**THEN** BOSS未解锁（不可挑战）
- **GIVEN** 当前章节所有必经事件完成，**WHEN** 检查章末BOSS，**THEN** BOSS已解锁（地图上红色脉动节点）
- **GIVEN** chapter_completed 信号广播，**WHEN** 探索系统收到，**THEN** 下一章地图列表解锁
- **GIVEN** 玩家在第3章，**WHEN** 尝试跳过第2章直接进入第4章地图，**THEN** 不允许（prev_chapter_completed 不满足）
- **GIVEN** 玩家读档，**WHEN** 检查章节进度，**THEN** narrative 域完全恢复到存档时的状态
- **GIVEN** 玩家在新局中，**WHEN** 检查 story_flags，**THEN** 所有 flags 为初始值（上一局的 flag 不保留）
- **GIVEN** 章节标题卡展示，**WHEN** 玩家点击继续，**THEN** 标题卡立即关闭进入探索
- **GIVEN** 章末BOSS被击败，**WHEN** 玩家做出结局选择，**THEN** story_flags 立即更新+广播 chapter_completed
- **GIVEN** 第5章最终BOSS击败+结局选择完成，**WHEN** advance_chapter，**THEN** 触发通关流程（非 chapter_completed 而是 game_victory）
