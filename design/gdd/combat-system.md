## 概述 (Overview)

战斗系统是《凡人修仙传：掌天问道》的核心交互系统，驱动所有回合制卡牌对战的流程执行。战斗按回合进行，每回合依次经历抽牌阶段→出牌阶段→攻击阶段→敌方行动阶段。系统管理角色上场与阵位、功法/法宝绑定、阵法部署、丹药/符箓消耗等所有战斗内行为，同时处理境界压制、阵亡结算和战斗胜负判定。

对玩家来说，战斗是检验构筑的试金石——他们在战前排兵布阵（根据境界上限选择上场角色），战斗中每回合根据费用做出出牌决策，在策略执行与应变之间寻找平衡。每个决策都有分量：角色阵亡意味着绑定卡永久失去，而巧妙的阵法部署和本命协同能扭转战局。

## 玩家幻想 (Player Fantasy)

### 创意总监声明 (Creative Director's Vision)

战斗系统是玩家构筑的"试金石"——你花了几十分钟在营寨里精调卡组、规划路线，现在轮到你的构筑接受检验。战斗系统必须传递的核心情感是：**"专注的审慎"**——不是抽卡出奇迹的狂喜，也不是害怕损失的焦虑，而是一种清醒的、分秒必争的战术警觉。你知道代价沉重（角色阵亡意味着绑定卡永久失去），所以你认真对待每一手出牌；你相信自己的构筑，所以你敢于执行。

我以创意总监的身份确立以下幻想方向，作为所有下游设计（视觉效果、音效、UI、数值平衡）的战斗创意北极星。

### 1. 核心情感：专注的审慎 (Focused Deliberation)

| 情感层 | 触发场景 | 强度 |
|--------|----------|------|
| **验证的满足感** | 流派效果按预期运转——正道续航回血、魔道打出连环伤害 | 中高频——每回合出牌验证 |
| **洞悉的愉悦** | 预判敌人行为并提前布阵反制成功 | 中频——每局1-3次 |
| **取舍的清醒** | 决定牺牲一个低价值角色换取全局优势 | 低频——但每次都有分量 |
| **警觉的重新计算** | 敌方境界压制打出超出预期的伤害，需要调整计划 | 中频——每局2-3次 |
| **战败的顿悟** | "我懂了哪里需要调整，下次改进"而非"这游戏不公平" | 低频——驱动学习的动力 |

**关键区别：** 大多数卡牌Roguelike的战斗乐趣来自"我打出了好看的combo"的即时爽感。本游戏的独特之处在于：**战斗是构筑的延续，而非独立于构筑的操作环节。** 你赢是因为你组得好、算得准，而不是因为手速快或运气好。

### 2. 三层框架（按战斗阶段分层）

#### 框架A：「问剑」—— 战前排兵布阵

**一句话定义：** 战斗是一场修为检验——你的构筑有多好，打一架就知道。

**情感核心词：验证的满足感 (The Satisfaction of Proof)**

**适用场景：** 战前角色选择、卡组策略确认、流派定位

**对应支柱：** P1（策略为王）、P2（苟道成长）
**对应MDA：** 挑战(1)

**玩家心理：** "我的构筑通过了考验"——赢了证明思路正确，输了说明还有优化空间。战斗在这里变成了构筑的**回声**，而非独立的动作序列。

#### 框架B：「修士弈局」—— 战斗中交锋

**一句话定义：** 战斗是一盘步法多变的棋局——你每次落子（出牌）都改变战场的节奏，对手也会做出回应。

**情感核心词：洞悉的愉悦 (The Joy of Foresight)**

**适用场景：** 回合制出牌、费用管理、阵法部署、攻击目标选择

**对应支柱：** P1（策略为王）
**对应MDA：** 挑战(1)、表达(5)

**玩家心理：** 你不是在机械出牌，而是在做多回合规划——"这回合放阵法，下回合绑定功法，大后回合爆发"。敌人也有模式和优先级，可以被"读"，这让战斗感觉像真正在和一个修士对弈。

#### 框架C：「断尾求生」—— 战损管理

**一句话定义：** 战斗是风险管理的艺术——真正的智慧在于决定什么该保、什么该舍。

**情感核心词：取舍的清醒 (The Clarity of Sacrifice)**

**适用场景：** 角色受伤、阵亡、撤退决策、资源耗尽

**对应支柱：** P2（苟道成长）、P3（机缘巧合）
**对应MDA：** 挑战(1)、发现(2)

**玩家心理：** 永久失去机制在这个框架下不是焦虑源，而是决策维度——"这场战斗值不值得打？这个角色值不值得保？"有时候最优解是半途撤退，这在其他Roguelike卡牌中几乎不存在。这就是"苟道"的战斗体现：**不是每次都风光大胜，而是每次都活着回来。**

## 详细设计 (Detailed Design)

### 核心规则 (Core Rules)

#### 1. 完整回合流程

每回合分为8个阶段，顺序执行：

```
阶段0：准备阶段 — 触发"回合开始"效果，结算持续伤害/恢复，状态持续时间-1
阶段1：抽牌阶段 — 从牌库抽2张，触发"抽牌时"效果
阶段2：出牌阶段 — 主动阶段，玩家可任意顺序出牌
阶段3：攻击声明 — 玩家为每个己方角色指定攻击目标
阶段4：攻击结算 — 按速度顺序自动结算攻击动画和伤害
阶段5：敌方行动 — 敌方AI自动出牌和攻击
阶段6：结束阶段 — 触发"回合结束"效果，费用重置，检查战斗结束
```

> **⚠ 设计决策（2026-07-23）：原阶段3"布阵阶段"已移除。** 阵位调整改为备战阶段一次性操作（详见 `deployment-system.md §7`），以降低战斗中认知负荷。

> **⚠ 设计决策（2026-07-23）：原阶段3"布阵阶段"已移除。** 阵位调整改为备战阶段一次性操作（详见 `deployment-system.md §7`），以降低战斗中认知负荷。

#### 2. 抽牌规则

| 条件 | 抽牌数量 |
|------|:-------:|
| 先手第1回合 | 不抽牌（使用开局手牌） |
| 后手第1回合 | 抽3张（补偿） |
| 每回合基础 | 抽2张 |
| 特殊效果 | 按卡牌描述额外抽牌 |

**开局手牌：** 先手4张，后手5张
**牌库规则：** 牌库上限随境界提升（炼气20→筑基25→金丹30→元婴35→化神40）。牌库抽空后，每次需要抽牌但牌库为空时，弃牌堆随机返还1张牌到牌库底部——不洗牌不重置。

#### 3. 费用规则

- 每回合费用上限由境界固定：炼气2→筑基5→金丹8→元婴11→化神14
- 先手第1回合按境界标准费用，后手第1回合额外+1费
- 每回合全额恢复，未使用费用不累积
- 丹药可临时增加费用（可突破上限），回合结束清除

#### 4. 出牌规则

| 卡牌类型 | 使用方式 | 效果时机 | 目标 |
|---------|---------|---------|------|
| 角色卡 | 消耗费用打出上场 | 上场后"待命"（本回合不可攻击） | 己方空闲阵位 |
| 功法卡 | 消耗费用打出绑定 | 立即生效 | 场上己方角色 |
| 法宝卡 | 消耗费用打出绑定 | 立即生效 | 场上己方角色 |
| 阵法卡 | 消耗费用打出部署 | 立即生效（光环效果） | 场上阵法区（最多3个） |
| 丹药卡 | 消耗费用（0-5费）使用 | 立即生效，进弃牌堆 | 己方角色/全体 |
| 符箓卡 | 消耗费用使用 | 立即生效，进弃牌堆 | 按卡牌描述（己方/敌方） |

#### 5. 前排/后排机制

- 己方场上分为前排和后排两个区域
- 前排角色存活时，敌方不能直接攻击后排角色
- 穿透效果/范围攻击可无视前排规则
- 角色阵位在备战阶段一次性设置，战斗中不可调整（详见 `deployment-system.md §7`）
- 上场人数≤2时，所有角色自动在前排
- 上场人数≥3时：前2后1（筑基），前2后2（金丹），前3后2/3（元婴/化神）

#### 6. 攻击规则

- 每个己方角色每回合可攻击1次
- 攻击声明阶段（阶段4），玩家为每个角色分别选择目标
- 攻击顺序由速度排序（基础速度来自境界，功法可修正）
- 同速度时己方先手，完全同速按站位从左到右
- 玩家可跳过某个角色的攻击

#### 7. 角色阵亡规则

- 角色HP降至0时阵亡，触发死亡效果（如有）
- 阵亡角色的绑定功法/法宝随角色进入弃牌堆，永久失去
- 丹药「大还丹」可复活角色但不恢复绑定卡（空载复活）
- 阵亡角色从场上移除，空出阵位
- **跨战斗死亡持久性**：阵亡角色在战斗结束后转为「不可用」状态（由上场与阵位系统管理，详见 `deployment-system.md §10`）——不可用于后续战斗，需通过复活丹药或特殊事件恢复

#### 8. 境界压制规则

- 敌方角色大境界比玩家高1级：玩家所有伤害降低20%
- 敌方角色大境界比玩家高2级及以上：玩家所有伤害降低50%
- 境界压制只影响基础攻防和伤害计算，不影响卡牌天赋效果

#### 9. 战斗结束规则

**胜利条件：** 敌方全部角色阵亡
**失败条件：** 己方全部角色阵亡
> ⚠ **跨战斗影响**：阵亡角色在战斗结算后转为「不可用」状态。若所有角色均为不可用，下次战斗开始前即判定游戏结束（详见 `deployment-system.md §10`）。
**战败后果：** 保留50%本次战斗获得的修为/资源，角色绑定卡损失永久生效（阵亡的角色已失去绑定卡）
**撤退：** 战斗中可撤退，视为战败处理，但秘境中撤退可保留已获得的非绑定物品

### 状态与转换 (States and Transitions)

#### 战斗全局状态

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **备战** | 战前选择角色阶段 | 进入战斗 | 确认上场→战斗中 |
| **战斗中** | 回合循环进行中 | 备战确认后自动开始 | 达成胜利/失败条件 |
| **胜利** | 玩家获胜 | 敌方全灭 | 战利品结算→退出 |
| **战败** | 玩家战败 | 己方全灭 | 损失结算→退出 |

#### 角色状态

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **待命** | 已上场但本回合不可攻击 | 本回合上场 | 下回合开始 |
| **已行动** | 已执行完本回合动作 | 攻击/使用技能后 | 回合结束→下回合恢复 |
| **阵亡** | HP归0 | HP降至0 | 大还丹复活或永久移除 |
| **空载** | 复活后无绑定卡 | 被大还丹复活 | 重新绑定功法/法宝 |

## 公式 (Formulas)

> **设计决策：攻击必定命中，不设闪避**
> 本游戏所有攻击必定命中，无闪避（dodge）判定步骤。防御手段分为两套并行系统：(1) **DEF** —— 固定值减伤，`actual_damage = max(1, ATK - DEF)`；(2) **「受到伤害降低X%」** —— 百分比减伤，在 DEF 结算后乘算。部分卡牌提供一次性格挡/影盾效果（如鬼影步）作为等效替代，属于卡牌触发的暂时状态，而非通用的闪避属性。伤害减免叠加采用**乘算**（`伤害 × 0.9 × 0.85 ...`），上限60%。

### 1. 伤害公式

```
actual_damage = max(1, attacker_ATK - target_DEF)
```

**变量：**

| 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|
| attacker_ATK | int | [1, +∞) | 攻击方角色的当前攻击力 |
| target_DEF | int | [0, +∞) | 目标的当前防御值（初始为0，由卡牌效果提供） |
| actual_damage | int | [1, +∞) | 最终造成的伤害 |

**输出范围：** 最低1（始终造成至少1点伤害），无硬上限。

### 2. 境界压制公式

```
final_damage = floor(actual_damage × realm_penalty)
```

| 境界差距 | realm_penalty |
|:--------:|:------------:|
| 同境界或低于 | 1.0（无压制） |
| 敌方高1级 | 0.8（-20%） |
| 敌方高2级及以上 | 0.5（-50%） |

**变量：**

| 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|
| realm_penalty | float | [0.5, 1.0] | 境界压制系数 |
| final_damage | int | [1, +∞) | 经境界压制修正后的实际伤害 |

**示例：** 炼气期玩家攻击金丹期敌人，ATK=4，敌DEF=0，计算过程：
```
actual_damage = max(1, 4 - 0) = 4
realm_penalty = 0.5（高2级）
final_damage = floor(4 × 0.5) = 2
```

### 3. 攻击速度排序

```
attack_order = base_speed(realm) + speed_bonus
base_speed(realm) = L（境界层级，炼气=1, 筑基=2, ...）
```

同速度时：己方先手；完全同速：按站位从左到右。

## 边界情况 (Edge Cases)

- **费用不足时出牌**：手牌费用 > 当前可用费用 → 卡牌灰色不可用，无法拖拽
- **场上位置已满**：上场角色数已到境界上限时，不可再打出角色卡，提示已达上限
- **阵法已满**：场上已有3个阵法时，打出新阵法可覆盖旧阵法（被覆盖的进入弃牌堆）
- **阵法冲突指定**：当多个阵法在场，角色满足多个触发条件时，玩家手动指定该角色触发哪个阵法
- **绑定角色阵亡后阵法**：阵眼角色阵亡时，该阵法失效（移除出阵法区）
- **攻击目标死亡/消失**：若攻击声明后目标在结算前已死亡（被其他攻击先手击杀），该次攻击重新选择目标
- **抽空牌库**：需抽牌但牌库为空时，从弃牌堆随机返还1张到牌库底部
- **撤退后状态**：战斗中撤退保留非绑定物品，但已阵亡角色的绑定卡永久失去（不可挽回）
- **0防御时的伤害**：防御为0时，伤害 = 攻击力（按max(1, ATK - 0) = ATK）
- **最高优先级效果覆盖**：当"免疫伤害"和"受到伤害"效果同时存在时，免疫胜出

## 调优参数 (Tuning Knobs)

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 境界压制-高1级 | -20% | 10%-30% | 境界差距无感 | 越阶战斗太困难 |
| 境界压制-高2级+ | -50% | 30%-70% | 高境界可能碾压 | 完全无法挑战 |
| 后手补偿费 | +1费 | 0-2 | 后手劣势明显 | 后手优先太高 |
| 伤害最低保底 | 1点 | 1-2 | 高防可能完全免伤 | 防御价值降低 |
| 上场后待命回合 | 1回合 | 0-1 | 先手爆发过强 | 上角色节奏太慢 |
| 基础攻击速度 = L | 境界层级 | 不可调 | — | — |
| 牌库上限（炼气→化神） | 20→40 | 15-50 | 卡组策略空间小 | 抽不到核心牌 |

## 视觉/音频需求 (Visual/Audio Requirements)

### 战斗阶段视觉反馈

| 阶段 | 视觉反馈 |
|------|----------|
| 准备阶段 | 状态图标闪烁，buff/debuff更新 |
| 抽牌阶段 | 卡牌从牌库飞入手中（0.3s） |
| 出牌阶段 | 出牌卡牌从手牌滑向目标区域 |
| 布阵阶段 | 角色位置交换动画 |
| 攻击声明 | 角色指向目标的瞄准线显示 |
| 攻击结算 | 按速度顺序播放攻击动画+伤害数字飘出 |
| 敌方行动 | 敌方卡牌出场动画淡入 |
| 结束阶段 | 回合切换动画简洁过渡 |

### 音效需求

- 抽牌声、出牌声、命中/格挡/阵亡各有独立音效
- 金色/暗金卡出牌时带有更高的音调区分
- 境界压制触发时低沉的压迫音效
- 战斗胜利/失败各有明确的音乐过渡

## 验收标准 (Acceptance Criteria)

- **GIVEN** 玩家进入战斗，**WHEN** 查看上场角色，**THEN** 数量不超过当前境界上限
- **GIVEN** 炼气期战斗开始，**WHEN** 先手玩家回合，**THEN** 可用费用为2，手牌4张
- **GIVEN** 后手第1回合，**WHEN** 玩家回合开始，**THEN** 费用为3（2+1），手牌5张
- **GIVEN** 战斗回合进行中，**WHEN** 玩家打出角色卡，**THEN** 角色上场标记为"待命"
- **GIVEN** 角色上场后第1回合，**WHEN** 攻击阶段，**THEN** 该角色不可选择为攻击者
- **GIVEN** 2个敌人集火同一角色，**WHEN** 攻击结算完成，**THEN** 目标角色HP减少ATK-DEF的差值
- **GIVEN** 玩家角色的ATK=4，目标DEF=0，**WHEN** 攻击结算，**THEN** 造成4点伤害
- **GIVEN** 玩家攻击高1级敌人，**WHEN** 伤害结算，**THEN** 伤害显示为80%（取整）
- **GIVEN** 敌人高玩家2级以上，**WHEN** 伤害结算，**THEN** 伤害显示为50%（取整）
- **GIVEN** 角色阵亡，**WHEN** 检查场上，**THEN** 角色从场上移除，绑定卡进入弃牌堆
- **GIVEN** 己方全部角色阵亡，**WHEN** 检查战斗状态，**THEN** 判定为战败
- **GIVEN** 敌方全部角色阵亡，**WHEN** 检查战斗状态，**THEN** 判定为胜利
- **GIVEN** 牌库抽空，**WHEN** 需抽牌，**THEN** 从弃牌堆随机返回1张到牌库
