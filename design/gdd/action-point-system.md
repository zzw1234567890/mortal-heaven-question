> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-22

## 概述

行动力系统管理玩家在探索地图上的移动资源——每次移动（前往相邻节点）消耗1点行动力，行动力耗尽后玩家无法继续移动，必须通过特定方式回复。行动力系统是探索节奏的调控器：它迫使玩家在每个决策点做出取舍——「走这条路还是那条路？我的行动力够不够到Boss？」

对玩家来说，行动力是探索阶段最直观的资源。它不是战斗中的费用（那是费用系统的职责），而是地图上「你还能走多远」的度量。行动力系统将地图探索从一个「随意走」变成了「策略路线规划」的决策空间。

暂无相关ADR。

## 玩家幻想

行动力系统的核心体验是**路线的压迫感**——每次点击移动节点，行动力条减少一格，玩家在心里估算：到Boss还有3格，支路有个事件点，但绕路要花2格行动力……去不去？

这种「有限的脚步」将地图从静态导航变成了动态决策树。玩家不是被动看角色走路，而是在主动规划一条**最优路线**。行动力用尽时那种「就差一步」的遗憾，和刚好到达Boss节点时的「刚好够到」的满足，都是探索节奏的情感锚点。

## 详细设计

### 核心规则

#### 1. 行动力上限与初始值

| 境界 | 行动力上限 | 说明 |
|:----:|:---------:|------|
| 炼气 | 5 | 初期探索范围小 |
| 筑基 | 7 | 中期地图更大 |
| 金丹 | 9 | 有足够的灵活性探索支路 |
| 元婴 | 11 | 几乎可以探索全图 |
| 化神 | 13 | 全图探索 + 额外秘境 |

> **权威来源**：此表的数值来源于 `realm-system.md` §2 境界属性表中的 `action_points` 字段。
> 行动力系统**引用**这些值，不独立定义。若出现差异，以 `realm-system.md` 为准。
> 运行时通过 `get_realm_property(L, "action_points")` 查询当前上限。

每个地图开始时，行动力**重置为上限值**（不继承上一地图的剩余行动力）。

#### 2. 行动力消耗

- **移动1格** → 消耗1点行动力
- **部分特殊节点**（如传送阵）→ 不消耗行动力（传送类移动）
- **战斗/事件节点**：进入节点消耗行动力，事件或战斗本身不额外消耗

#### 3. 行动力回复

| 回复方式 | 回复量 | 触发条件 |
|---------|:------:|---------|
| **地图开始** | 回满 | 每次进入新地图时 |
| **境界突破** | 回满 | 渡劫成功后（P2「步步为营」支持） |
| **回复丹药** | 按丹药类型 | 玩家在探索中使用丹药 |
| **地图特殊事件** | 按事件定义 | 特定奇遇中的回复选项 |
| **行动力泉** | 全回复 | 地图上特定节点（每个地图限1次） |

#### 4. 行动力归零

- 行动力归零后，玩家**不可继续移动**到未探索节点
- 玩家可以停留在当前节点（不消耗行动力），查看已解锁信息
- 若当前位置为Boss节点，可照常挑战Boss（不消耗行动力）
- 归零后不可触发新事件（已经是尽头了）

**强制结束处理：**
- 所有节点探索完毕或行动力归零且无可移动目的地 → 战斗Boss节点（如果已到达）或结束探索阶段
- 玩家不会被困死——行动力归零但Boss节点已在当前位置时可直接开战
- 行动力归零但Boss节点在其他位置时 → 只能提前进入Boss战（跳过剩余事件）或用丹药回复行动力

#### 5. 行动力数据存储

行动力由GSM管理：

```
GSM路径：exploration.action_points
GSM路径：exploration.max_action_points
```

行动力变更通过GSM信号广播：

| 信号 | 触发时机 |
|------|---------|
| `action_points_changed` | 行动力增减时，载荷 `{current, max, delta}` |
| `action_points_depleted` | 行动力归零时 |
| `action_points_refilled` | 行动力从非满恢复到满时 |

### 状态与转换

```
行动力状态机：
  Full（有行动力）→ 移动消耗 → Partial（部分剩余）
  Partial → 继续消耗 → Depleted（归零）
  Depleted → 使用丹药/特殊事件 → Partial
  Any → 地图切换/突破 → Full（重置回满）
```

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|----------------------|---------------------|
| **游戏状态管理器** | 行动力读写（exploration.action_points） | 行动力变更信号 |
| **探索系统** | 当前行动力值（能否移动）；移动消耗 | 节点移动事件 |
| **境界系统** | — | 突破成功→回满行动力 |
| **资源系统/道具系统** | — | 回复丹药使用→增加行动力 |
| **UI系统** | 行动力当前值/上限/变更信号 | — |

## 公式

### 1. 行动力上限

```
max_action_points(realm_level) = 3 + 2 × realm_level
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| realm_level | int | [1,5] | 炼气=1, 筑基=2, 金丹=3, 元婴=4, 化神=5 |
| max_action_points | int | [5,13] | 行动力上限 |

**验证：** 炼气(L1) → 5；筑基(L2) → 7；金丹(L3) → 9；元婴(L4) → 11；化神(L5) → 13

### 2. 可移动判定

```
can_move(from_node, to_node, current_ap) → bool:
  if to_node == from_node: return false
  if not is_adjacent(from_node, to_node): return false
  if to_node.is_locked(): return false
  if to_node.is_teleport: return true                 # 传送不消耗行动力
  return current_ap >= 1
```

### 3. 移动后行动力

```
after_move(current_ap, to_node, distance):
  if to_node.is_teleport:
    return current_ap            # 传送不消耗
  return current_ap - 1          # 每格消耗1点
```

### 4. 无法移动判定

```
can_explore(current_ap, current_node, map_nodes) → bool:
  return any reachable_node in adjacent_nodes:
    can_move(current_node, reachable_node, current_ap)
```

当 `can_explore()` 返回 false 时触发探索结束流程。

## 边界情况

- **行动力刚好够到Boss节点**：允许进入Boss战，战斗本身不消耗行动力
- **行动力归零但还有未探索节点**：无法再移动。Boss节点如果已到达可开战，否则需要丹药回复行动力
- **行动力泉节点重复使用**：每个地图限1次，使用后该节点变为普通节点
- **突破回满溢出**：突破时行动力回满，即使突破前是满值也保持满值（无浪费问题）
- **地图间行动力不继承**：每个新地图独立重置行动力，不保留上一地图的剩余行动力
- **战斗中不消耗行动力**：行动力是探索资源，战斗阶段与行动力无关
- **使用丹药回复超过上限**：回复时取 `min(current + amount, max_action_points)`，不溢出
- **玩家停留在起始节点不动**：不消耗行动力，无限期停留（但游戏节奏上地图已设计为必须推进）
- **多人无意义节点反复进出**：Godot单线程，无网络问题，但探索系统应禁止回退到已探索节点
- **传送节点消耗问题**：传送节点消耗0行动力，但必须已解锁（探索系统职责）

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|
| 行动力上限基准 L1 | 5 | 3-7 | 探索节奏太紧，无法探索支路 | 无探索压力，路线无取舍 |
| 行动力上限基准 L5 | 13 | 9-15 | 高境界探索受限 | 高境界行动力过多，失去策略 |
| 每格消耗 | 1 | 1-2 | 移动成本低，鼓励多探索 | 移动成本高，探索范围太小 |
| 丹药回复量 | +2 | 1-3 | 回复性价比低 | 行动力变成无限续杯 |
| 行动力泉每图限次 | 1次 | 1-2 | — | 全图探索无压力 |

## 视觉/音频需求

| 事件 | 动画 | 时长 |
|------|------|:----:|
| 行动力消耗 | 行动力图标光粒消散或步数减少动画 | 0.2s |
| 行动力回复（丹药） | 行动力图标亮起填充动画 | 0.3s |
| 行动力归零 | 行动力图标变为灰色+闪烁 | 0.5s |

### 音效需求
- 行动力消耗：轻快的脚步/翻页声
- 行动力归零：低沉的锁定声
- 行动力回复：清脆的泉水声

## 验收标准

- **GIVEN** 炼气期，**WHEN** 检查行动力上限，**THEN** 为5
- **GIVEN** 元婴期，**WHEN** 检查行动力上限，**THEN** 为11
- **GIVEN** 行动力=5/5，**WHEN** 移动到相邻节点，**THEN** 行动力变为4/5
- **GIVEN** 行动力=1/5，**WHEN** 移动到相邻节点，**THEN** 行动力变为0/5，触发 `action_points_depleted`
- **GIVEN** 行动力=0/5，**WHEN** 尝试移动到相邻节点，**THEN** 移动被拒绝
- **GIVEN** 行动力=0/5但已在Boss节点，**WHEN** 挑战Boss，**THEN** 允许进入战斗
- **GIVEN** 使用回复丹药，**WHEN** 当前行动力=2/5，**THEN** 行动力增加（不超过上限）
- **GIVEN** 使用回复丹药但行动力已满，**WHEN** 尝试使用，**THEN** 提示"行动力已满"
- **GIVEN** 进入新地图，**WHEN** 加载完成，**THEN** 行动力重置为上限
- **GIVEN** 境界突破，**WHEN** 突破成功，**THEN** 行动力回满
- **GIVEN** 行动力变更，**WHEN** 监听信号，**THEN** 收到 `action_points_changed` 事件
