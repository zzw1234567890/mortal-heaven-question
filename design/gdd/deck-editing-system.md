# 卡组编辑系统 (Deck Editing System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-22
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱1「自由组牌，策略为王」、支柱3「机缘巧合，意外之喜」

## 概述

卡组编辑系统管理玩家在本局运行中卡组的构建与变更全过程——每一次卡牌的获得、删除、交换都通过具体的游戏行为完成，不存在独立的「自由编辑界面」。系统覆盖四种卡牌变更渠道：**战斗战利品三选一加入卡组**、**商店购买/删除卡牌**、**事件选择获得/失去卡牌**、以及**开局身份固定初始卡组**。

对玩家来说，这套系统意味着「每一张卡牌都是一次决策的结果」——开局选择身份决定了初始工具包，战斗后的三选一要求判断「这张牌值不值得加入我的构筑」，商店中花灵石删卡的每一次点击都是对卡组精炼的投资。没有免费的优化，每一次调整都有代价或机会成本。这正是 Roguelike 卡牌的核心乐趣：在限制中做选择，在选择中形成属于自己的独特构筑。

> **⚠ 设计冲突说明**：card-system.md 原有「卡组编辑在营寨中进行」「卡组容量下限30张」「可保存多套预设」等规则——这些规则基于自由编辑模型，与本系统的 StS 式渐进构筑模型冲突。本系统以用户确认的新规则为准。

## 玩家幻想

卡组编辑在此模型下对应的不是「修者锦囊」的精挑细选，而是 **「修仙路上的积累与取舍」**——你携带的每一张卡牌都是在旅途中获得并决定留下来的。卡组不是你在出发前精心打包的行囊，而是你在历经磨难后自然而然积累的家底。

情感节奏：

| 阶段 | 情感 | 触发场景 |
|------|------|---------|
| **开局** | 简朴的出身感 | 看到初始卡组中的几张基础卡——「就靠这些开始了」 |
| **战利品三选一** | 战术判断的快感 | 三张混合奖励——「这张卡能让我流派成型，但灵石可以马上去商店删几张废牌…」 |
| **商店删卡** | 心痛的优化感 | 花灵石删掉一张曾经帮过你的卡——「是时候放手了」 |
| **核心牌入手** | 豁然开朗 | 终于拿到流派最后一块拼图——「这套齐了！」 |
| **卡组超过40张** | 被迫的精炼 | 看到提示「已达上限」——「得去商店删几张了」 |
| **Boss战前看卡组** | 审视与自信 | 打开卡组查看界面，回顾自己一路积累的成果——「这些够用了」 |

核心情感不是「精炼的自信」，而是 **「机缘的积累感」**——你卡组中的每一张卡都有来处，都是你在一局游戏中某个时刻做出的选择。卡组本身就是你的冒险日志。

## 详细设计

#### 2. 战斗战利品——混合三选一

每次战斗胜利后触发战利品选择流程。**三个选项中不应全部是卡牌**——灵石是混合奖励池中的常备选项：

```
battle_reward_selection(enemy_type, is_elite, is_boss, player_state):
  ① 按战斗难度和敌人类型决定选项构成模式：
     普通战斗 → 模式A「2卡+1灵石」/ 模式B「1卡+2灵石」/ 模式C「1卡+1灵石+1消耗品」
     精英战斗 → 模式A「2卡+1灵石（大量）」/ 模式B「2卡+1灵石」/ 模式C「2卡+1消耗品」
     Boss战斗 → 模式A「2卡（含保底紫+）+1灵石」/ 模式B「2卡（含保底金）+1灵石」
  ② 卡牌选项：按卡牌系统掉落规则生成（不因同局已有卡做去重）
  ③ 灵石选项：固定数值，显示为「灵石 × N」+ 灵石图标
  ④ 消耗品选项（仅限特定模式C）：恢复血量 X 点 / 回复行动力1点 / 临时减伤等
  ⑤ UI 展示3个选项（卡牌以正面展示，灵石/消耗品以图标+数字展示）
  ⑥ 玩家选择其中1项，其余消失。额外提供「跳过」按钮——跳过获得少量灵石（普通5/精英10/Boss20）
  ⑦ 卡组已达当前境界上限时，卡牌选项标注「卡组已满」（灰色不可选）——若3个选项全部不可选，自动替换为灵石跳过补偿（普通5/精英10/Boss20）
```

**跳过与全灰保护：**
- 设计确认：保留混合奖励模型（决策 2026-07-23）。灵石选项提供与卡牌选择的策略张力——玩家需在「扩充构筑」和「积累资源」间权衡
- 全灰保护：普通战斗因模式 A/B/C 天然含灵石/消耗品，不会全灰。精英/Boss 在卡组满且走纯卡模式时（已移除纯卡模式），自动替换为灵石跳过补偿
- 「跳过」按钮始终可用——跳过的卡牌不会暂存、不会在后续战利品中重现

**灵石奖励基准值（待经济系统确认）：**

| 战斗类型 | 灵石量范围 | 说明 |
|---------|:---------:|------|
| 普通战斗（杂兵） | 10~25 | 走模式A时~15，模式B时两堆~10+~20 |
| 精英战斗 | 40~80 | 一张卡+大量灵石或2卡+少量灵石 |
| Boss战斗 | 100~200 | 高品质卡+Boss额外赏金 |

**展示规则：**
- 卡牌选项：正面展示（卡名、费用、类型、稀有度、效果简述），可悬浮查看详情
- 灵石选项：灵石堆图标 + 数字（如「灵石 × 25」），点击即确认
- 消耗品选项：对应图标 + 效果简述（如「恢复15% HP」）
- 跳过按钮：独立于三个选项之外，标注「跳过（灵石 +N）」
- NEW标识：卡牌左上角「新卡」标签（仅首次获得的卡牌）

**没有收藏池暂存机制**——选择一项即生效，未被选的选项消失。不存在「先收着以后再用」的设计。

**不选择灵石/消耗品即永久失去该机会**——错过的灵石不会累积到下次战利品。

#### 3. 坊市买卖与删卡

坊市界面提供三种卡牌相关操作，以标签页分离（买卡 / 删卡 / 售卡），避免认知混淆：

**① 购买卡牌：**
- 坊市展示3-6张随机卡牌，每张标价（按卡牌系统拆解产出的 2~3 倍定价）
- 玩家选择一张→消耗灵石→卡牌加入 `deck.current_deck`
- 灵石不足时，该卡牌显示为灰色并标注「灵石不足」
- 卡组已达当前境界上限时，所有卡牌标注「卡组已满」不可购买

**② 删除卡牌（散功）：**
- 玩家支付灵石从卡组中永久移除一张卡牌——修仙世界中称为「散功」，即主动放弃一门已修功法
- 删卡费用 = `50 + 25 × [本局已删除次数]`（本局已删除次数 = 本次删卡前已完成的删除次数；第1次删卡时次数=0，费用=50）
- 费用展示：「散功费用：50灵石」（首次）或「散功费用：75灵石」（第2次），依次类推
- 费用无上限（但一局游戏通常不会超过10次删卡）
- 灵石不足时不可操作
- 卡组仅剩5张以下时，删除按钮禁用并提示「卡组太少，无法继续散功」

**③ 出售卡牌：**
- 玩家可将卡组中的卡牌出售换取灵石
- 售价 = 卡牌系统拆解产出公式计算的值 × 0.8（坊市抽成20%）
- 出售后卡牌从卡组永久移除
- 卡组仅剩5张以下时，出售按钮禁用
- 注意：坊市出售（80%回收率）与营地拆解（100%回收率）是两个独立操作——出售提供即时流动性，拆解需在营地（地图间）进行。详见 `card-system.md` 拆解公式

> **卡组最低张数保护**：卡组任何时候不得低于5张（防止删/售光卡牌导致无法战斗）。坊市操作、事件销毁都受此限制保护。

#### 4. 事件增删卡牌

事件系统提供以下卡牌变更类型：

**获得卡牌：**
- 洞府奇遇：3选1奖励（其中可能包含卡牌选项）
- 斜月三星洞：固定奖励一张高品质卡牌（直接加入卡组）
- 坊市随机事件：免费获得一张随机卡牌
- 杀人夺宝胜利：从对方身上获得1-2张随机卡牌（可能超出当前境界上限——上限检查在事件结算后执行，若超限则触发弃牌流程）

**失去卡牌：**
- 杀人夺宝失败：随机失去1张卡牌（被对方夺走）
- 特定剧情事件：为推进剧情需要交出某张卡
- 陷阱事件：随机销毁一张丹药或符箓
- 渡劫准备事件：可选择「牺牲一张卡牌换取渡劫加成」

**事件超限预警：**
事件选项中若可能添加卡牌，展示当前卡组数量预览：
```
选项A：「获得一张随机金色卡牌 ⚠ 卡组将变为 39/40」
选项B：「获得三张随机卡牌 ⚠ 卡组将变为 41/40，需弃1张」
```

**事件导致的超限处理：**
如果事件结算后卡牌超过当前境界上限：
```
if deck.current_deck.size() > realm_deck_limit:
  ① 先展示「你获得了以下卡牌」弹窗（不可跳过），让玩家在知情情况下做弃牌决策
  ② 再弹出超限弃牌界面：「卡组超限，需散功N张至[上限]张」
  ③ 玩家选择要丢弃的卡牌（多选，弃到上限为止）
  ④ 被选中的卡牌从卡组移除——获得少量灵石补偿（每弃1张补偿5灵石，象征性回收）
  ⑤ 确认后继续游戏
```

> **设计理念**：超限弃牌不是惩罚，而是「修仙路上的取舍」——修士境界有限，承载的功法亦有限。弃牌补偿（象征性灵石）体现「放弃的功法回归天地」的修仙世界观，同时避免玩家因信息不透明而产生被系统欺骗的感觉。

#### 5. 卡组查看界面（只读）

玩家可在任何时候打开「查看卡组」界面，功能严格限制为**只读浏览**：

| 功能 | 支持 | 说明 |
|------|:----:|------|
| 浏览全部卡牌 | ✅ | 按类型分组展示（角色/功法/法宝/阵法/丹药/符箓） |
| 按稀有度筛选 | ✅ | 快速筛选白/蓝/紫/金/暗金 |
| 按费用排序 | ✅ | 升序/降序 |
| 卡牌详情弹窗 | ✅ | 点击卡牌查看大尺寸详情（含获得来源：第X章·XX事件） |
| 总张数统计 | ✅ | 「当前卡组：35张 / 上限[境界值]张」 |
| 暗金卡计数 | ✅ | 「暗金卡：1/2」 |
| 流派标签统计 | ✅ | 展示当前卡组的流派标签分布 |
| 拖拽调整顺序 | ❌ | 纯浏览，不可调整 |
| 移除卡牌 | ❌ | 不可直接移除 |
| 添加卡牌 | ❌ | 不可直接添加 |
| 保存预设 | ❌ | 不支持多套预设 |

查看界面在以下场景可打开：
- 探索地图中（点击卡组图标）
- 坊市中（在购买/删除前查看当前卡组）
- 战利品界面中（在选择前查看当前卡组）
- 战斗中：显示完整卡组 + 标注「已抽出」「仍在牌库」状态（不做限制，让玩家获取完整战术信息）

#### 6. 角色位管理

角色位单独管理，与卡组计数隔离：

- 6个角色位，开局身份决定初始角色
- 角色位中的角色卡不在卡组张数计数内
- 获得新角色卡的方式：
  - **事件获得**：特定事件奖励角色卡（如「救下李元化→获得角色卡李元化」）
  - **坊市购买**：偶有角色卡出售
  - **渡劫奖励**：突破大境界时奖励对应角色卡
- 角色位满6个后获得新角色卡时：先展示新角色→玩家选择要替换的旧角色→确认（含叙事确认步骤："确定要与[角色名]告别吗？"[确定]/[暂留]）→新角色替换旧角色，被替换的角色卡及所有绑定卡进入弃牌堆。若玩家取消，新角色卡不获得
- **主角保护**：开局身份绑定的初始角色（如青云剑宗→林渊）标记为「不可替换」，在角色位满时不受替换流程影响——这是对支柱4「忠于原著」的尊重
- 角色位中的角色可在查看界面浏览，不可在查看界面上调整

#### 7. 卡组上限——境界渐进制

卡组上限不由全局常量定义，而是由当前境界决定（数据来源：`realm-system.md`）：

| 境界 | 卡组上限 |
|------|:------:|
| 炼气期 | 20 |
| 筑基期 | 25 |
| 金丹期 | 30 |
| 元婴期 | 35 |
| 化神期 | 40 |

轮回天赋「万法归宗」（L4, cost=25）使上限+5（炼气25→化神45）。`deck_limit_modifier` 参数从 `GSM.player.talents` 查询并叠加到境界上限。

在以下场景强制执行：
| 场景 | 上限规则 |
|------|---------|
| 战利品三选一 | 卡组已达境界上限时卡牌选项不可选（灰色），仅能选灵石/消耗品或跳过 |
| 坊市购买 | 已达境界上限时不可购买 |
| 事件获得 | 超限→先展示获得物→触发弃牌流程至境界上限 |
| 开局初始卡组 | 不受上限约束（初始卡组6-7张，远低于炼气期20上限） |

#### 8. 卡组变更历史（持久化）

系统维护卡组变更日志，**序列化到存档**——这是「冒险日志」情感化设计的核心载体：

```
DeckChangeLog {
  entries: [
    {turn: int, source: enum, card_id: String, card_name: String, action: "add"|"remove", detail: String, timestamp: int}
  ]
}
```

- 每张卡牌在详情弹窗中显示获得来源（"第X章·XX事件"），与 `card-system.md` 的出处追踪保持一致
- 读档后完整恢复——玩家的冒险日志不会因读档而丢失
- 用于卡组查看界面的「历史」标签页

### 状态与转换

```
卡组状态生命周期（一局内）：

开局 → 初始卡组确定（6-7张）
  ↓
探索循环：
  战斗胜利 → 三选一（+0~1张，受境界上限约束）
  坊市 → 购买（+1张）| 散功（-1张）| 出售（-1张）
  事件 → 获得（+N张）| 失去（-N张）
  超限 → 弃牌流程（先展示获得物→强制弃至≤境界上限，每弃1张补偿5灵石）
  ↓
渡劫突破 → 可能有卡牌奖励/损失 + 境界上限提升
  ↓
继续探索 → 循环
  ↓
结局（通关/死亡）→ 卡组数据+变更日志随存档保留
```

**变更渠道汇总：**

```
三选一:    deck + [0..1] | 灵石 +N   (战利品——可选卡牌或灵石)
购买:      deck + 1        (坊市)
散功:      deck - 1        (坊市·付费)
出售:      deck - 1        (坊市·获得灵石)
事件获得:  deck + N        (多种事件)
事件失去:  deck - N        (多种事件)
超限弃牌:  deck - N        (事件过量后·每张补偿5灵石)
```

### 与其他系统的交互

## 公式

### 0. 变量定义与取整规则

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| `realm_deck_limit` | int | [20, 40] | 当前境界卡组上限（从境界系统查询） |
| `deck_limit_modifier` | int | [0, 5] | 天赋修正值（万法归宗=5，其他=0） |
| `effective_limit` | int | [20, 45] | `realm_deck_limit + deck_limit_modifier` |
| `session_remove_count` | int | [0, ∞) | 本次散功前本局已完成的散功次数（第1次散功时=0） |

**取整规则**：`floor()` 向下取整。

### 1. 卡组上限验证

```
can_add_to_deck(deck, card_count=1) → {allowed: bool, reason: String}
  effective_limit = realm_system.get_deck_limit(player.realm) + GSM.player.deck_limit_modifier
  if deck.current_deck.size() + card_count > effective_limit:
    return {allowed: false, reason: "卡组已达上限（" + effective_limit + "张）"}
  return {allowed: true, reason: ""}
```

| 境界 | `realm_deck_limit` | +万法归宗 |
|------|:---:|:---:|
| 炼气 | 20 | 25 |
| 筑基 | 25 | 30 |
| 金丹 | 30 | 35 |
| 元婴 | 35 | 40 |
| 化神 | 40 | 45 |

### 2. 坊市散功费用

```
removal_cost(session_remove_count) → int
  # session_remove_count = 本次散功前已完成的散功次数
  return 50 + 25 × session_remove_count
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| session_remove_count | int | [0, ∞) | 本次散功前本局已完成的散功次数 |
| 返回值 | int | [50, ∞) | 本次散功所需灵石 |

示例：第1次散功50灵石（次数=0），第2次75（次数=1），第3次100（次数=2）...

### 3. 坊市卡牌售价

```
sell_price(card) → int
  base = disassemble_value(card)  # 从卡牌系统获取拆解产出（100%回收率）
  return floor(base × 0.8)        # 坊市抽成20%
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| base | int | [10, 2095] | 卡牌拆解基准灵石值（营地拆解=base，坊市出售=base×0.8） |
| 返回值 | int | [8, 1676] | 坊市收购价 |

> **经济说明**：坊市出售（80%）与营地拆解（100%）是两个独立渠道。出售提供即时流动性（探索中坊市节点），拆解需在营地（地图间）进行。25%的差价是流动性的代价。

### 4. 卡组最低张数保护

```
can_remove_from_deck(deck, remove_count=1) → {allowed: bool, reason: String}
  if deck.current_deck.size() - remove_count < 5:
    return {allowed: false, reason: "卡组至少保留5张"}
  return {allowed: true, reason: ""}
```

### 5. 超限弃牌补偿

```
overflow_discard_compensation(discarded_count) → int
  return 5 × discarded_count  # 每弃1张补偿5灵石（象征性回收）
```

## 边界情况

- **三选一选了的卡发现后悔了**：不支持撤销——选了就是选了，卡牌已加入卡组。这是决策的权重所在
- **三选一跳过**：跳过的卡不会进入任何暂存区。不会在下一次战利品中再次出现
- **卡组刚开局就超过境界上限**：不可能——初始卡组固定6-7张，远低于炼气期20张上限
- **坊市散功费用过高导致玩家无法继续散功**：设计上允许——如果玩家已经删了10+次（费用275灵石/次），散功成本已成为战略决策的一部分。玩家可以选择出售卡牌而非散功
- **事件导致卡牌超限后强迫弃牌**：先展示「你获得了以下卡牌」弹窗→再弹出弃牌界面。弃牌选择界面不带撤销。玩家必须在所有卡牌中选择要丢弃的，直到≤境界上限
- **超限弃牌选择金色/暗金卡**：允许——但每弃1张补偿5灵石（象征性回收）。这不再是纯惩罚，而是「放弃的功法回归天地」的修仙世界观体现
- **卡组只剩5张时坊市操作**：散功和出售按钮禁用。事件销毁同样受保护——如果事件要销毁卡牌但卡牌≤5张，事件文本变更为替代选项
- **角色位满6个后获得新角色**：先展示新角色→选择替换目标→叙事确认步骤（"确定要与[角色名]告别吗？"）→确认后替换，取消则新角色不获得
- **林渊（开局初始角色）不可替换**：主角保护标记——这是对支柱4「始于凡人，忠于原著」的尊重
- **战斗中查看卡组**：显示完整卡组 + 标注「已抽出」「仍在牌库」状态，让玩家获取完整战术信息
- **开局初始卡组中的同名多张卡**：允许（如基础攻击×2），卡牌系统已支持同名多张
- **读档后卡组变更日志恢复**：变更日志持久化到存档——读档后完整恢复。玩家的冒险日志不会因读档而丢失
- **同时从多个渠道获得卡牌（如战斗+事件同一帧结算）**：各渠道独立处理加入验证。如果战斗三选一已使卡组达到境界上限，事件结算时即使有「获得卡牌」结果，也先展示获得物→再触发超限弃牌流程

## 调优参数

## 验收标准

### 初始卡组验证

- **GIVEN** 选择青云剑宗外门弟子身份并进入游戏，**WHEN** 检查卡组，**THEN** 卡组含枯木逢春诀(蓝)×1、烟云步(蓝)×1、影刺(白)×1、基础攻击(白)×2、简易护盾(白)×2（共7张），角色位含林渊（位1）和苏剑鸣（位2）
- **GIVEN** 选择血海殿遗孤身份，**WHEN** 检查卡组，**THEN** 卡组含万魂幡(蓝)×1、幽影步(蓝)×1、煞气斩(白)×2、血杀掌(白)×1、基础攻击(白)×2（共7张），角色位含殷若寒和屠夜
- **GIVEN** 选择碎星群岛散修身份，**WHEN** 检查卡组，**THEN** 卡组含幻花迷踪步(蓝)×1、阴魄凝魂术(蓝)×1、剑波斩(白)×2、寻宝符(蓝)×1、基础攻击(白)×2（共7张），角色位含汐音和沐瑶
- **GIVEN** 选择玄冰宫弟子身份，**WHEN** 检查卡组，**THEN** 卡组含寒玉轮回功(蓝)×1、霜魄剑诀(蓝)×1、冰棱刺(白)×2、金钟符(白)×1、基础攻击(白)×2（共7张），角色位含凌霜月和姜雪
- **GIVEN** 选择丹霞谷弟子身份，**WHEN** 检查卡组，**THEN** 卡组含三元聚气功(蓝)×1、丹霞剑气(蓝)×1、筑基丹(蓝)×1、基础攻击(白)×2、培元丹(白)×2（共7张），角色位含方灵素和石岩
- **GIVEN** 选择阵道双杰身份（已解锁），**WHEN** 检查卡组，**THEN** 卡组含万象阵典(紫)×1、七星困龙阵(蓝)×1、阴阳守御阵(白)×1、金钱镖(白)×2、基础攻击(白)×1（共6张），角色位含慕星河和云素心
- **GIVEN** 选择任意身份并确认，**WHEN** 检查卡组，**THEN** 初始卡组总张数在6-8张范围内，远低于炼气期上限20张

### 战利品选择

- **GIVEN** 普通战斗胜利，**WHEN** 进入战利品选择，**THEN** 展示3个混合奖励选项（卡牌/灵石/消耗品），至少包含1个灵石选项，且「跳过」按钮始终可用（显示"跳过（灵石 +N）"）
- **GIVEN** 精英/Boss战斗胜利且卡组未满，**WHEN** 进入战利品选择，**THEN** 展示3个混合奖励选项，至少包含1个灵石选项
- **GIVEN** 战利品选项包含卡牌和灵石，**WHEN** 玩家选择卡牌，**THEN** 该卡加入卡组，面板关闭
- **GIVEN** 战利品选项包含灵石（如25灵石），**WHEN** 玩家选择灵石，**THEN** 灵石增加对应数值，面板关闭，未选选项消失且不可回溯
- **GIVEN** 战利品选择面板，**WHEN** 玩家点击「跳过」，**THEN** 获得少量灵石（普通5/精英10/Boss20），面板关闭
- **GIVEN** 卡组已达当前境界上限且战利品有卡牌选项，**WHEN** 战利品展示，**THEN** 卡牌选项标记「卡组已满」不可选，但灵石/消耗品/跳过仍可选
- **GIVEN** 卡组已达境界上限且战利品3个选项全部不可选（极端情况），**THEN** 自动替换为灵石跳过补偿，面板关闭

### 坊市操作

- **GIVEN** 玩家在坊市选择散功，**WHEN** 灵石充足且卡组>5张，**THEN** 支付50灵石（第1次），卡组减少1张，灵石减少50
- **GIVEN** 玩家本局已散功1次（第1次50灵石），**WHEN** 第2次散功，**THEN** 费用为75灵石
- **GIVEN** 玩家本局已散功2次，**WHEN** 第3次散功，**THEN** 费用为100灵石
- **GIVEN** 玩家在坊市出售「基础攻击」（白色卡，拆解基准值10灵石），**WHEN** 确认出售，**THEN** 卡组减少1张，灵石增加 floor(10 × 0.8) = 8
- **GIVEN** 卡组已达当前境界上限，**WHEN** 尝试坊市购买卡牌，**THEN** 不可购买，提示「卡组已满（上限N张）」
- **GIVEN** 灵石少于散功费用，**WHEN** 尝试散功，**THEN** 散功按钮灰色不可操作，标注「灵石不足」
- **GIVEN** 灵石少于卡牌标价，**WHEN** 查看坊市购买界面，**THEN** 该卡显示为灰色，标注「灵石不足」
- **GIVEN** 卡组仅有5张卡，**WHEN** 尝试坊市散功/出售，**THEN** 按钮禁用，提示「卡组至少保留5张」
- **GIVEN** 玩家在坊市选择散功并选取一张卡牌，**WHEN** 确认弹窗出现后点击「取消」，**THEN** 卡牌不删除，灵石不扣减

### 超限弃牌

- **GIVEN** 卡组35张（炼气期上限20），**WHEN** 事件结算增加5张卡（合计40张>上限20），**THEN** 先展示「你获得了以下卡牌」弹窗→再触发超限弃牌流程，提示需弃至20张（弃20张），底部显示「补偿100灵石」
- **GIVEN** 超限弃牌流程中，**WHEN** 玩家选择要弃的卡牌并确认，**THEN** 卡牌移除，每弃1张获得5灵石补偿
- **GIVEN** 超限弃牌弹窗，**WHEN** 还未满足弃牌数量，**THEN** 确认按钮不可用

### 卡组查看

- **GIVEN** 玩家在探索地图中，**WHEN** 点击卡组图标，**THEN** 打开只读卡组查看界面
- **GIVEN** 玩家在坊市中，**WHEN** 点击卡组图标，**THEN** 打开只读卡组查看界面
- **GIVEN** 卡组查看界面，**WHEN** 尝试拖拽任意卡牌到界面外，**THEN** 卡牌回到原位，卡组总张数不变
- **GIVEN** 卡组查看界面的「历史」标签，**WHEN** 打开，**THEN** 展示本局卡牌变更列表，每项包含：回合数、来源渠道（战利品/坊市/事件/超限弃牌）、卡牌名称、操作类型
- **GIVEN** 读档后打开卡组查看，**WHEN** 查看「历史」，**THEN** 日志完整恢复（持久化验证）

### 角色位管理

- **GIVEN** 角色位已满6个，**WHEN** 获得新角色卡，**THEN** 弹出角色替换弹窗——先展示新角色，再选择被替换目标，含叙事确认步骤
- **GIVEN** 角色替换弹窗，**WHEN** 玩家选择被替换角色并确认，**THEN** 新角色加入，旧角色及绑定卡进入弃牌堆
- **GIVEN** 角色替换弹窗，**WHEN** 玩家点击「取消」，**THEN** 新角色卡不获得，角色位保持6个不变
- **GIVEN** 角色替换弹窗，**WHEN** 被替换目标是开局初始角色（如林渊），**THEN** 该角色标记为「不可替换」，选择被阻止

### 境界上限

- **GIVEN** 玩家境界为炼气期（上限20张），**WHEN** 卡组已有20张并尝试添加卡牌，**THEN** 被拒绝（"卡组已达上限20张"）
- **GIVEN** 玩家突破筑基期（上限25张），**WHEN** 查看卡组上限，**THEN** 显示25张
- **GIVEN** 玩家拥有轮回天赋「万法归宗」（+5上限），**WHEN** 当前境界为炼气期，**THEN** 有效上限为25张
