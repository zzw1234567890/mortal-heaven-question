## 概述

角色上场与阵位系统负责管理战斗中角色的上场、阵位分配、前后排布局以及角色离场/下场逻辑。系统定义每个战斗的"上场空间"（由境界决定总人数上限），管理6个角色位中哪些角色被选中上场、他们在前后排的分布位置、以及上场角色的行动状态（待命/已行动）。这是战斗系统的子编排层——它不执行战斗逻辑，但定义战斗的"棋局布局"。

对玩家来说，上场与阵位是每场战斗的第一个策略决策点。战前从6个角色位中选择谁上场，就是决定这场战斗的战术重心——拉上坦克还是输出？保留哪个角色应对后手？战中阵位调整（布阵阶段移动前后排）则是更精细的战术微操：把残血主力换到后排保护起来，或者把爆发角色换到前排准备下一轮攻击。上场选择决定了战斗的"牌面"，阵位调整是回合间的战术呼吸。

暂无相关ADR。

## 玩家幻想

### 创意总监声明 (Creative Director's Vision)

部署系统的核心情感不是"策略布局"——那是战斗系统「修士弈局」的领地。部署系统的独特情感是**"取舍的清醒"**：战斗系统的「断尾求生」管理的是"损失发生了怎么办"，而部署系统管理的是"在损失发生之前，你选择让谁来承担风险"。

你明知道前排会挨打，你明知道核心角色身上绑着金色功法，你还是让他上了前排——因为你需要一个能扛的人站在前面，才能让输出角色在后排安心发挥。这就是"苟道"的战斗体现：不抱怨处境，精打细算每步棋。

我确立以下幻想方向，作为所有下游设计的创意北极星。

**关键区别：** 大多数卡牌游戏的"上场"只是出牌消耗资源。本游戏的上场是多了一层空间维度的战术——**你放在前排还是后排，直接影响战斗的攻防节奏。** 这不仅是"谁上场"，更是"谁为谁挡刀"。

## 详细设计

### 核心规则

#### 1. 战场阵位布局

战场固定为6个阵位，按3×2布局排列：

```
敌方侧
─────────────────
[前1] [前2] [前3]   ← 前排（3格）
[后1] [后2] [后3]   ← 后排（3格）
─────────────────
己方侧
```

- 每个阵位在同一边界内是等价的（前1/前2/前3没有位置优劣之分）
- 阵位的唯一区别是：**前排**还是**后排**
- 上场角色从阵位1开始依次填充（无需指定具体格子编号，抽象为"前X后Y"）

#### 2. 上场人数上限

| 境界 | 最高上场人数 | 阵位分布 |
|:----:|:----------:|:--------:|
| 炼气 | 2 | 前2后0（≤2人全为前排） |
| 筑基 | 3 | 前2后1 |
| 金丹 | 4 | 前2后2 |
| 元婴 | 5 | 前3后2 |
| 化神 | 6 | 前3后3（满阵） |

**关键规则：**
- 上场人数**不可超过**境界上限（不可选超过上限的角色）
- 上场人数**可以少于**境界上限（玩家主动选择少带人）
- 前排队列填满后才能填充后排：例如金丹期上限4人，若只上场3人，则前2后1；若上场4人，则前2后2
- 当上场人数≤2时，所有人都在前排（无后排保护）

#### 3. 备战阶段（战斗开始前）

```
① 进入战斗场景
② 显示6个角色位的角色列表
③ 当前境界允许的上场人数高亮显示
④ 玩家从6个角色位中勾选/拖拽角色到阵位：
   - 自动按"前排优先"分配到具体阵位（前1→前2→前3→后1→后2→后3）
   - 玩家可手动调整角色在前后排之间的位置
⑤ 确认上场 → 进入战斗
⑥ 未上场的角色在本场战斗中不可使用（也不可被敌方攻击）
```

**自动分配规则：**
- 系统按 前1→前2→前3→后1→后2→后3 的顺序自动填充
- 玩家拖拽角色到前排区域 = 放前排，拖到后排区域 = 放后排
- 若玩家不手动调整，角色按角色位顺序自动填充

#### 4. 战中上场（补位）

战斗中打出角色卡时：

```
① 玩家打出角色卡 → 扣除费用
② 检查是否有空阵位：
   - 有空位 → 进入步骤③
   - 无空位（场上已满员6人）→ 不可打出，提示"无空余阵位"
③ 玩家选择空阵位（点击前排或后排的空位）
④ 角色上场，分配到该阵位
⑤ 角色状态设为「待命」——本回合不可攻击
⑥ 触发效果引擎：角色上场事件
```

**阵亡与空位：**
- 角色阵亡后，其阵位变为**空位**
- 空位保持到被新角色填充或战斗结束
- 空位无惩罚效果（只是位置空着）

#### 5. 待命规则

- 本回合上场的角色（包括战前部署和战中补位）状态为**「待命」**
- 待命角色：可被敌方攻击、可使用其绑定的功法/法宝效果（被动持续效果）、**不可**主动攻击、**不可**作为攻击声明对象
- 下回合开始时，该角色自动从「待命」转为**「已行动」→「可攻击」**
- 战前部署的角色在战斗第1回合也遵守此规则（先手第1回合不可攻击）

#### 6. 前后排规则

| 条件 | 规则 |
|------|------|
| 前排有存活角色 | 敌方不可直接攻击后排角色——攻击者必须先清理前排 |
| 前排无存活角色 | 后排角色暴露，敌方可直接攻击后排 |
| 穿透效果 | 无视前排保护，可直接攻击后排（如某些符箓效果） |
| 范围攻击 | 按技能描述（可能同时打前后排） |

#### 7. 布阵阶段（仅备战阶段一次性操作）

> **⚠ 设计决策（2026-07-23 review-all-gdds）：阵位调整由每回合操作改为备战阶段一次性操作，以降低战斗中的认知负荷（从7个并发主动系统降至6个）。**

- 阵位调整仅在**备战阶段**（战斗开始前）进行——玩家一次性安排所有上场角色的前后排位置
- 战斗中角色一旦上场即固定阵位（前/后排），不可在回合间移动
- 唯一例外：角色阵亡后通过「大还丹」复活，复活后可重新选择前/后排位置（因为原阵位已清空）
- 战中补位角色上场时，由系统自动分配到空阵位（前排优先）
- 每角色**每回合限移动1次**
- 移动后该角色本回合的站位立即生效（影响攻击目标选择）

#### 8. 角色下场规则

战斗中**不可主动将角色撤回**（只能通过阵亡移除或敌方击败）
- 唯一例外：战斗中撤退（全队撤退，按战败处理）
- 大还丹复活后的角色再次上场需重新打出角色卡

#### 9. 大还丹复活与重新上场

- 角色阵亡→大还丹复活→角色未在场上（需要重新打出角色卡）
- 重新上场走标准战中上场流程（打出角色卡→选空位→待命）
- 复活后角色为空载（原绑定卡已进弃牌堆）

#### 10. 角色跨战斗死亡与不可用状态

战斗中阵亡的角色在战斗结束后**不会自动复活**——死亡状态跨战斗持久保持：

**基本规则：**
- 战斗中阵亡的角色，战斗结算后标记为 **「不可用」**
- 「不可用」角色在角色位中显示为灰色 + 骷髅标记，**不可选上场**、**不可参与任何战斗**
- 「不可用」角色的绑定功法/法宝已在阵亡时进弃牌堆——**永久失去，不可恢复**
- 「不可用」状态是跨战斗持久状态，随存档保存
- 角色「不可用」后，其角色位变为空位，可供新获得的角色卡填充（替换规则见边界情况）
- 「不可用」角色的角色卡**仍留在卡组中**（作为抽牌堆中的牌），但打上场后直接进入「待命」，不受死亡状态影响——死亡状态仅限制战前选择

**恢复「不可用」角色的途径：**

| 方式 | 来源 | 说明 |
|------|------|------|
| **复活丹药**（大还丹等） | 商店购买、战利品、事件奖励 | 使用后指定角色恢复为「可用」，空载（无绑定卡） |
| **特殊事件** | 机缘事件、剧情事件 | 如「天南药谷」「韩立出手相救」等事件免费复活一名角色 |
| **轮回天赋**（未来） | 轮回天赋系统 | 特定天赋允许角色阵亡后自动消耗灵石复活（每局限次） |

**复活规则：**
- 复活后的角色回到角色位，**不继承绑定卡**（空载状态）
- 如果原角色位已被新角色替换：复活角色进入「等待队列」，下次获得新角色时优先占据
- 角色属性（等级、特性、成长）不受阵亡影响，复活后保留

**全部阵亡 → 游戏失败：**
- 若所有角色位中角色均为「不可用」（无任何可用角色），则**游戏结束**
- 判定时机：战斗开始前（检查是否有至少1个可用角色可上场）
- 例外：如果在全灭后通过事件/商店**获得了新角色卡**，即使原角色全部阵亡，只要有新角色可用即继续游戏

#### 11. 敌方阵位

- 敌方也遵循前后排规则（敌方的上场人数和阵位分配由AI系统定义）
- 玩家攻击敌方时，同样先打前排→后排暴露后才可打后排
- 敌方前排和后排的视觉区分与己方一致

#### 11. 备战阶段展示信息

在选择上场角色时，每个角色应展示：
- 角色名、HP/ATK数值
- 当前绑定的功法/法宝概览（以便玩家判断应该带谁）

### 状态与转换

#### 角色在场状态

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **未上场** | 在角色位中，但未选择上场 | 备战阶段未选中、战中未打出 | 备战选中→上场（待命） |
| **待命** | 已上场但本回合不可攻击 | 本回合上场（战前或战中） | 下回合开始→已就绪 |
| **已就绪** | 可正常行动 | 待命状态持续到下回合 | 攻击/使用技能→已行动 |
| **已行动** | 已执行完本回合动作 | 攻击后、使用技能后 | 回合结束→恢复为已就绪 |
| **阵亡** | 战斗中HP归0，仍在场上 | HP归0 | 战斗结束→不可用；大还丹→空载复活 |
| **不可用** | 跨战斗持久死亡，不可上场 | 战斗结算时阵亡角色→不可用 | 复活丹药/事件→可用；天赋复活→可用 |
| **空载复活** | 复活后不在场上，无绑定卡 | 大还丹/事件使用 | 打出角色卡上场→待命 |

#### 阵位状态

| 状态 | 含义 | 进入方式 | 离开方式 |
|------|------|----------|----------|
| **占用** | 有角色在位上 | 角色上场 | 角色阵亡→空位；角色下场→空位 |
| **空位** | 无角色 | 角色阵亡/下场/未填充 | 新角色上场→占用 |

## 公式

### 1. 上场人数上限

```
max_deploy(realm) = L + 1
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| L | int | [1,5] | 境界层级（炼气=1, 筑基=2, 金丹=3, 元婴=4, 化神=5） |
| max_deploy | int | [2,6] | 最高可上场人数 |

**验证：** 炼气 L=1 → 2、化神 L=5 → 6

### 2. 前后排填充规则（自动模式）

```
front_count = min(deployed, 3)       # 前排最多3人
back_count = max(0, deployed - 3)    # 剩余放后排
```

**约束：** 当 deployed ≤ 2 时，front_count = deployed, back_count = 0

### 3. 待命持续时间

```
ready_turn = deploy_turn + 1
```

角色在 `deploy_turn` 上场，从 `ready_turn` 开始可正常行动。

## 边界情况

- **上场人数不足上限**：玩家可主动少带人（如金丹期只上3人而非4人），空阵位保留为空
- **场上已满6人时打出角色卡**：卡牌灰色不可用，提示"战场已满，无空余阵位"
- **所有阵位空置**：开场时一个角色都不选→提示"至少选择1个角色上场"，不可确认
- **角色阵亡后空位立即可用**：同一回合阵亡后打出角色卡可立即填补该空位
- **前排全灭后排暴露**：前排3人全阵亡后，后排角色直接成为可攻击目标；若此时打出新角色上场补前排，前排恢复保护
- **前排人数不足时被打穿**：若前排只剩1人且已残血，打出穿透攻击的敌方可直接打到后排——"半残的前排比空前排更危险"（因为玩家以为还有保护，但可能被穿透越过）
- **布阵阶段移动后立即生效**：阶段3将角色从前排移到后排后，该角色的站位在本回合阶段4攻击声明时即生效
- **移动后的待命状态变更**：从后排放到前排的角色不影响其行动状态（已就绪→仍可攻击，待命→仍待命）
- **大还丹复活后角色不在场上**：复活后的角色不走"回场"流程——需要玩家重新打出角色卡填充空位
- **战斗中撤退后全队状态**：撤退后所有上场角色恢复为未上场状态，绑定卡保留（阵亡过的角色绑定卡已失去）
- **角色阵亡后不可用，但角色卡仍在卡组中**：「不可用」状态仅影响战前选择阶段，不影响抽牌堆中的角色卡——你仍然可能抽到已阵亡角色的角色卡，打出后正常上场（空载/待命）
- **全部角色不可用时进入商店/事件**：如果所有角色位均为不可用，玩家仍然可以进入商店购买角色卡、触发事件获得角色卡——只要有新角色加入，游戏继续；但如果进入战斗时仍无可用角色，则直接判定游戏失败
- **复活时原角色位已被占用**：如果阵亡角色的角色位已被新角色替换，「不可用」角色复活后进入等待队列——下次获得新角色时优先占据队列中的角色。如果等待队列中有多个角色，按阵亡顺序优先
- **暗金角色卡不可用时**：暗金角色卡的稀有度不影响不可用规则——阵亡后同样变为不可用，需要同样的复活途径
- **战斗中撤退后全队状态**：撤退后所有上场角色恢复为未上场状态，绑定卡保留（阵亡过的角色绑定卡已失去）
- **新上场角色本回合可否被治疗效果指定**：可以——新上场角色虽然不能攻击，但可以被丹药/功法指定为目标（包括治疗和buff）
- **暗金卡角色的上场限制**：暗金角色卡不卡暗金卡组上限（暗金卡上限2张只针对非角色卡）
- **敌方的前排/后排定义**：敌方AI也遵循相同的前后排规则——玩家必须先打敌方前排才能打后排

## 依赖关系

| 依赖系统 | 性质 | 说明 |
|----------|------|------|
| **战斗系统** | 硬依赖 | 接收回合阶段信号（阶段2出牌、阶段3布阵）；提供上场角色列表给攻击声明阶段 |
| **卡牌系统** | 硬依赖 | 角色卡数据（费用、属性）；角色位管理（6个角色位的读写） |

| **AI系统** | 软依赖 | 提供敌方阵位定义；己方阵位分布影响AI决策 |
| **UI系统** | 硬依赖 | 备战阶段上场选择、布阵阶段操作、阵位状态展示 |

## 调优参数

| 参数 | 默认值 | 安全范围 | 过低影响 | 过高影响 |
|------|:-----:|:--------:|---------|---------|

| 自动填充顺序 | 前1后3 | 不可调 | — | — |
| 前排保护条件 | 前排有人即可 | 不可调 | — | — |

## 验收标准

- **GIVEN** 炼气期进入战斗，**WHEN** 备战阶段，**THEN** 最多可选2人上场，自动分配为前2后0
- **GIVEN** 金丹期进入战斗，**WHEN** 备战阶段，**THEN** 最多可选4人上场，自动分配为前2后2
- **GIVEN** 化神期进入战斗，**WHEN** 备战阶段，**THEN** 最多可选6人上场，全阵位填满
- **GIVEN** 玩家只选了3人上场（金丹期上限4人），**WHEN** 确认，**THEN** 允许以3人开始战斗
- **GIVEN** 玩家一个角色都没选，**WHEN** 确认，**THEN** 提示"至少选择1个角色上场"
- **GIVEN** 战斗中打出角色卡且有空位，**WHEN** 选择空位，**THEN** 角色上场并标记为"待命"
- **GIVEN** 战斗中打出角色卡但无空位，**WHEN** 操作，**THEN** 卡牌灰色不可用
- **GIVEN** 角色上场后第1回合，**WHEN** 攻击声明阶段，**THEN** 该角色不可选为攻击者
- **GIVEN** 角色已就绪状态，**WHEN** 下回合开始，**THEN** 保持已就绪
- **GIVEN** 角色攻击后，**WHEN** 检查状态，**THEN** 变为"已行动"
- **GIVEN** 前排有存活角色，**WHEN** 敌方选择攻击目标，**THEN** 不可直接选择后排角色
- **GIVEN** 前排全灭，**WHEN** 敌方选择攻击目标，**THEN** 后排角色成为可选目标
- **GIVEN** 角色阵亡，**WHEN** 检查阵位，**THEN** 该阵位变为空位
