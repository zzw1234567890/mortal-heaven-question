# 阵营系统 (Faction System)

> **状态 (Status)**：设计中 (In Design)
> **作者 (Author)**：Claude Code + 用户
> **最后更新 (Last Updated)**：2026-07-22
> **最后验证 (Last Verified)**：—
> **实现的支柱 (Implements Pillar)**：支柱1「自由组牌，策略为王」、支柱2「苟道成长，步步为营」、支柱4「始于凡人，忠于原著」

## 概述

阵营系统是游戏的身份标识层，定义和管理所有角色卡的阵营标签及其判定规则。系统维护阵营标签库（大阵营如正道/魔道，具体门派如青云剑宗/丹霞谷等），为每个角色分配标签（最多3个），并提供实时判定接口——"场上正道角色≥3？"、"目标是否属于魔道？"——供阵法系统、卡牌效果引擎和流派系统查询使用。

对玩家来说，阵营系统是构筑策略中隐形的"第三条腿"——除了角色属性和绑定组合之外，阵营分布决定了阵法能否激活、流派能否成型。玩家在卡组编辑和上场选择时，会不自觉地在心里计数：正道够3个了吗？魔道有前排吗？这种"凑阵营"的决策过程就是阵营系统的玩法所在——它不是玩家直接操作的界面，而是贯穿所有角色选择的底层约束条件。

暂无相关ADR。

## 玩家幻想

### 创意总监声明 (Creative Director's Vision)

阵营系统不是贴在角色身上的标签——它是玩家构筑决策中的"无形之手"。玩家在卡组编辑时不是在选"带哪些角色"，而是在选"这局走正道还是魔道"。阵营系统的核心情感不是"我是谁"，而是**"我站在哪一边"**——每一张角色卡的上场选择，都在为这场战斗的阵营天平加码。

我确立以下幻想方向，作为所有下游设计的创意北极星。

---

### 1. 核心情感：阵营的筹谋 (The Faction's Calculus)

| 情感层 | 触发场景 | 强度 |
|--------|----------|------|
| **路线的抉择** | 卡组编辑时决定这局走正道路线还是魔道路线 | 每局游戏的基础决策 |
| **凑齐的满足** | 场上正道角色达到3人——阵法可以激活了 | 中频——流派成型的关键节点 |
| **散修的灵活** | 混合阵营卡组——正道+魔道各2人激活双阵营buff | 中频——表达型玩家的选择 |
| **识别的快感** | 看到敌方阵营标记——"他是魔道的，用斩皇刀有加成" | 中频——游戏知识回报 |
| **背叛的代价** | 流派成型后发现核心角色阵亡，阵营人数不够触发阵法 | 低频——构筑风险的验证 |

**关键区别：** 大多数卡牌游戏的"种族/职业"系统是静态数据库字段。本游戏的阵营系统是**动态判定**——你的阵营绑定不是写在卡面上的，而是通过你选择上场的角色实时构成的。战线推演中，你没有一个固定的"阵营"，你只有"当前场上这几个角色的标签集合"。

### 2. 两大阵营体系

游戏中的阵营按两层结构组织：

**第一层：大阵营（Broad Alignment）**
| 阵营 | 主题色 | 核心阵营机制 | 代表角色 |
|:----:|:------:|:------------|:---------|
| **正道** | 青/金 | 续航、防御、团队buff | 林渊、月清霜、赤练仙子、李元化 |
| **魔道** | 赤/紫 | 攻击、击杀回报、干扰 | 墨渊、汐音、屠夜、云煞老祖 |

**第二层：具体门派（Specific Sect）**
门派标签是阵营的子集，为正道/魔道提供更精细的判定维度：
- 青云剑宗、丹霞谷、玄冰宫、东域 → 正道分支
- 血海殿、魅影阁、轮回殿、血煞教 → 魔道分支
- 碎星群岛、归墟之境、万象阁 → 跨阵营（正道/魔道角色都可能拥有）

## 详细设计

### 核心规则

#### 1. 阵营标签结构

阵营系统维护两层标签：

```
阵营标签定义：
{
  tag_id: String                    # 唯一标识（如 "zhengdao"、"qixuanmen"）
  name: String                      # 显示名称（"正道"、"青云剑宗"）
  parent_alignment: Enum | null     # 所属大阵营（正道/魔道/null）
  is_major: Boolean                 # 是否大阵营标签
  icon: String                      # 图标资源路径
}

大阵营（Major Alignments）：
  - "正道" (zhengdao)     —— parent=null, is_major=true
  - "魔道" (modao)        —— parent=null, is_major=true

具体门派（Specific Sects）—— parent指向所属大阵营：
  - "青云剑宗"   (parent=正道)     - "丹霞谷"   (parent=正道)
  - "玄冰宫"   (parent=正道)     - "东域"     (parent=正道)
  - "星斗宗"   (parent=正道)     - "卫家"     (parent=正道)
  - "血海殿"   (parent=魔道)     - "魅影阁"   (parent=魔道)
  - "轮回殿"   (parent=魔道)     - "血煞教"   (parent=魔道)
  - "黑煞教"   (parent=魔道)     - "云蒙"     (parent=魔道)

跨阵营中立标签（Cross-alignment, parent=null）：
  - "碎星群岛"   - "归墟之境"     - "万象阁"
  - "极阴岛"
```

#### 2. 角色阵营分配规则

每张角色卡绑定最多**3个**阵营标签，组合规则：

| 规则 | 说明 | 示例 |
|------|------|------|
| 大阵营至少1个 | 每个角色至少有一个正道或魔道标签 | 林渊=正道 |
| 门派最多2个 | 具体门派标签≤2 | 林渊=青云剑宗+丹霞谷 |
| 跨阵营标签可选 | 碎星群岛/归墟之境等可作为第3标签 | 林渊（元婴）=正道+东域+碎星群岛 |
| 正魔不可同体 | 同一角色不可同时拥有正道和魔道标签 | 墨渊=魔道+血海殿（不可有正道） |

#### 3. 阵营判定接口

阵营系统对外提供以下判定接口：

```
# 基础查询
get_tags(character_id) → String[]           # 获取角色的全部阵营标签
has_tag(character_id, tag_id) → bool         # 角色是否有某标签
belongs_to_alignment(character_id, alignment) → bool  # 角色是否属于大阵营

# 场上统计（实时）
count_on_field(alignment) → int              # 场上某大阵营的角色数量
count_on_field(tag_id) → int                 # 场上某具体标签的角色数量
get_field_tags() → Dictionary               # 当前场上所有角色的标签聚合

# 条件判定（供阵法系统调用）
check_condition(requirement) → bool          # 条件如 "zhengdao>=3"
# requirement 格式: { tag_id: "zhengdao", min_count: 3 }

# 阵营关系（供卡牌效果引擎调用）
is_hostile_to(card_a, card_b) → bool         # 卡牌效果：对魔道伤害+50%
get_alignment_relation(a, b) → Enum         # 同阵营/中立/敌对
```

#### 4. 阵营buff叠加规则

- 多个阵营来源的buff（正道阵法+魔道debuff）可同时生效，无冲突
- 同一阵营的不同buff（正道阵法+正道角色的天赋）加法叠加
- 正魔混合阵容：正道+魔道角色各≥2时，可激活中立阵法"苍玄正邪盟阵"
- 阵营判定仅基于**当前场上存活角色**，阵亡角色不计入

#### 5. 阵营对卡牌效果的影响

某些卡牌效果依赖于阵营判定：

| 效果类型 | 判定方式 | 示例 |
|---------|---------|------|
| 伤害加成 | 目标是否属于某阵营 | "对魔道目标伤害翻倍" |
| 效果免疫 | 目标是否属于某阵营 | "正道角色免疫混乱效果" |
| 被动增益 | 持有者是否属于某阵营 | "全体魔道攻击+10%" |
| 限制条件 | 场上某阵营人数是否达标 | "场上正道角色HP上限+2" |

#### 6. 阵法与阵营的联动

阵法卡的激活条件依赖阵营判定：

| 阵法 | 阵营条件 | 效果 |
|:----|:---------|:------|
| 苍玄正道盟阵 | 场上≥3个正道角色 | 所有正道角色HP上限+2，防御+1 |
| 苍玄魔道盟阵 | 场上≥3个魔道角色 | 所有魔道角色攻击+2，暴击率+10% |
| 苍玄正邪盟阵 | 正道+魔道各≥2 | 全体攻防+15% |
| 魔域血海阵 | 场上≥3个魔道角色 | 魔道角色击杀后攻击永久+2，回复3血 |

#### 7. 阵营在战斗外的应用

阵营标签不仅用于战斗中的阵法判定，还用于：
- **探索事件**：某些随机事件对不同阵营角色有不同的选项和结果（正道可选"正面对抗"，魔道可选"暗中下手"）
- **商店折扣**：同一阵营的角色在坊市有折扣？
- **剧情分支**：阵营影响对话选项（后续由叙事系统决定）

### 状态与转换

阵营系统本身无运行时状态——它是纯数据查询层。场上的"阵营分布"状态由上场角色列表实时计算。

```
场上阵营分布（动态计算）：
  上场角色A（正道+丹霞谷）
  上场角色B（正道+玄冰宫）
  上场角色C（魔道+血海殿）
  → 统计结果：正道=2, 魔道=1, 丹霞谷=1, 玄冰宫=1, 血海殿=1
  → 阵法判定：正道≥3? 否 → 苍玄正道盟阵不激活
```

### 与其他系统的交互

| 系统 | 数据流入（本系统→目标） | 数据流出（目标→本系统） |
|------|--------------------------|---------------------------|
| **卡牌系统** | 阵营标签定义库 | 角色卡/阵法卡上的阵营标签字段（faction_tags, faction_requirement） |
| **阵法系统** | count_on_field() / check_condition() | 阵法部署/失效事件（触发重新判定） |
| **卡牌效果引擎** | is_hostile_to() / belongs_to_alignment() | 效果结算请求（涉及阵营判定的效果） |
| **流派系统** | 阵营标签列表（流派的路由条件之一） | 流派定义（标明所需阵营方向） |
| **探索系统** | 角色阵营标签查询 | 事件选项的阵营条件 |
| **UI系统** | 阵营标签图标、颜色、分类数据 | — |

## 公式

### 1. 阵法条件判定

```
condition_met(tag_id, min_count) = count_on_field(tag_id) >= min_count
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| tag_id | String | 阵营标签ID | 如 "zhengdao"、"modao" |
| min_count | int | [1,6] | 最低需求人数 |
| count_on_field | int | [0,6] | 当前场上符合的角色数 |

### 2. 阵营敌对判定

```
alignment_relation(a, b) = 
  a.parent == b.parent → 同阵营
  a.parent != b.parent AND both are not null → 敌对
  a.parent == null OR b.parent == null → 中立
```

### 3. 角色阵营标签上限

```
max_tags(character) = 3
max_major_tags = 1       # 正/魔只能选一个
max_sect_tags = 2        # 具体门派最多2个
```

## 边界情况

- **正魔不可同体**：同一角色不可同时标注正道和魔道——如果设计时出错，系统应在数据加载时报错而非运行时崩溃
- **角色无阵营标签**：所有角色至少有一个大阵营标签（正道或魔道），不存在无阵营角色
- **阵亡角色计入阵营统计？**：不计入——阵法激活条件只判定当前场上存活角色
- **离场保留角色计入？**：不计入——不在场上的角色不贡献阵营计数
- **敌方阵营是对称系统**：敌方角色也有阵营标签，玩家可用针对性的卡牌效果（如"对魔道伤害+50%"）攻击敌方
- **跨阵营角色（如碎星群岛）的阵法适配**：碎星群岛标签不属于正道或魔道，不贡献大阵营计数，但可被针对碎星群岛的效果指定
- **场上只有一个角色时**：如果只有一个正道角色，阵法条件（≥3）不满足，但该角色仍享受其自己的正道buff（如果有）
- **阵营buff冲突**：正魔混搭时，正道阵法buff和魔道阵法buff可同时存在——因为判定条件各自独立
- **流派和阵营的关系**：流派走正道方向≠玩家被锁定在正道阵营——玩家可混搭，但阵法收益需要集中投资

## 验收标准

- **GIVEN** 林渊角色卡，**WHEN** 读取阵营标签，**THEN** 包含「正道」和具体门派（如青云剑宗、丹霞谷）
- **GIVEN** 任何角色卡，**WHEN** 检查阵营标签数量，**THEN** ≤3个
- **GIVEN** 任何角色卡，**WHEN** 检查正魔标签，**THEN** 不可同时包含正道和魔道
- **GIVEN** 场上正道角色≥3，**WHEN** 阵法系统调用 check_condition("zhengdao", 3)，**THEN** 返回true
- **GIVEN** 场上正道角色=2，**WHEN** 阵法系统调用 check_condition("zhengdao", 3)，**THEN** 返回false
- **GIVEN** 卡牌效果"对魔道伤害翻倍"，**WHEN** 目标角色为魔道，**THEN** 伤害加成生效
- **GIVEN** 卡牌效果"对魔道伤害翻倍"，**WHEN** 目标角色为正道，**THEN** 伤害加成不生效
- **GIVEN** 角色阵亡，**WHEN** 统计场上阵营分布，**THEN** 阵亡角色不计入
- **GIVEN** 场上正道角色全部阵亡，**WHEN** 检查阵法条件，**THEN** 正道阵法失效
- **GIVEN** 敌方角色有魔道标签，**WHEN** 玩家使用针对性卡牌，**THEN** 效果正确作用于敌方
