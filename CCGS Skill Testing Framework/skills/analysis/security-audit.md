
# 技能测试规格：/security-audit（安全审计）

## 技能概述（Skill Summary）

`/security-audit` 对游戏进行安全风险审计，包括存档数据完整性、网络通信、反作弊暴露和数据隐私。它读取 `src/` 中的源文件以检查安全模式，并验证敏感数据是否得到正确处理。不调用任何主管关卡（Director Gate）。该技能不写入文件（仅生成发现结果报告）。裁决结果（Verdict）：SECURE（安全）、CONCERNS（关注）或 VULNERABILITIES FOUND（发现漏洞）。

---

## 测试用例（Test Cases）

### 用例 1：正常路径 —— 存档数据已加密，无硬编码凭据

**测试夹具（Fixture）：**
- `src/core/save_system.gd` 使用 `Crypto` 类在写入存档前加密数据
- 在任何 `src/` 文件中均无硬编码的 API 密钥、密码或凭据
- 客户端面向用户的输出中未暴露版本号或内部构建 ID

**输入：** `/security-audit`

**预期行为：**
1. 技能扫描 `src/` 中的安全模式：加密使用情况、硬编码凭据、内部信息暴露
2. 所有检查通过：存档数据已加密，未发现凭据，未暴露内部信息
3. 发现结果报告显示所有检查 PASS
4. 裁决结果为 SECURE

**断言：**
- [ ] 技能检查存档数据处理中加密的使用情况
- [ ] 技能扫描硬编码凭据（API 密钥、密码、令牌）
- [ ] 技能检查暴露给玩家的版本/构建号
- [ ] 发现结果报告中显示所有检查
- [ ] 当所有检查通过时，裁决结果为 SECURE

---

### 用例 2：发现漏洞 —— 未加密的存档数据和暴露的版本信息

**测试夹具：**
- `src/core/save_system.gd` 以纯 JSON 格式写入存档数据（未加密）
- `src/ui/debug_overlay.gd` 包含：`label.text = "Build: " + ProjectSettings.get("application/config/version")`
  （将内部构建版本暴露给玩家）

**输入：** `/security-audit`

**预期行为：**
1. 技能扫描 `src/` —— 发现 `save_system.gd` 中未加密的存档写入
2. 技能发现 `debug_overlay.gd` 中暴露的版本字符串
3. 两个发现结果均标记为 VULNERABILITIES（漏洞）
4. 裁决结果为 VULNERABILITIES FOUND
5. 技能为每个漏洞提供修复建议

**断言：**
- [ ] 未加密的存档数据被标记为漏洞，并附有文件和大约行号
- [ ] 暴露的版本字符串被标记为漏洞
- [ ] 为每个漏洞提供修复建议
- [ ] 当检测到任何漏洞时，裁决结果为 VULNERABILITIES FOUND
- [ ] 不写入或修改任何文件

---

### 用例 3：在线功能无身份验证 —— CONCERNS

**测试夹具：**
- `src/networking/lobby.gd` 存在，包含函数：`join_lobby()`、`send_chat()`
- 在 `send_chat()` 之前未发现身份验证检查 —— 玩家可在未经验证的情况下调用
- 游戏具有在线多人功能（从文件存在推断）

**输入：** `/security-audit`

**预期行为：**
1. 技能扫描 `src/networking/` —— 检测到在线功能代码
2. 技能检查网络调用前的身份验证守卫 —— 发现 `send_chat()` 缺少守卫
3. 标记："在线功能缺少身份验证检查 —— CONCERNS"
4. 裁决结果为 CONCERNS（而非 VULNERABILITIES FOUND，因为这是缺少控制，而非漏洞利用）

**断言：**
- [ ] 技能通过扫描网络源文件来检测在线功能
- [ ] 网络操作前缺少身份验证检查被标记
- [ ] 对于缺失的身份验证守卫，裁决结果为 CONCERNS（建议级别严重程度）
- [ ] 输出建议在添加网络调用前进行身份验证

---

### 用例 4：边界情况 —— 无源文件可分析

**测试夹具：**
- `src/` 目录不存在或完全为空

**输入：** `/security-audit`

**预期行为：**
1. 技能尝试扫描 `src/` —— 未找到文件
2. 技能输出错误："在 `src/` 中未找到源文件 —— 无可审计内容"
3. 不生成发现结果报告
4. 不发出裁决结果

**断言：**
- [ ] 当 `src/` 为空或不存在时，技能不崩溃
- [ ] 输出明确指出未找到源文件
- [ ] 不发出裁决结果（无内容可评估）
- [ ] 技能建议验证 `src/` 目录路径

---

### 用例 5：关卡合规性 —— 无关卡；安全工程师单独调用

**测试夹具：**
- 源文件存在；检测到 1 个 CONCERNS 级别的发现结果（发布版本中启用了调试日志）
- `review-mode.txt` 包含 `full`

**输入：** `/security-audit`

**预期行为：**
1. 技能扫描源码；发现在发布路径中调试日志处于活动状态
2. 无论审核模式如何，均不调用任何主管关卡
3. 裁决结果为 CONCERNS
4. 输出说明："如需正式安全审查，请考虑启用安全工程师（security-engineer）智能体"
5. 发现结果以只读报告形式呈现；不写入文件

**断言：**
- [ ] 在任何审核模式下均不调用主管关卡
- [ ] 建议（而非强制）咨询安全工程师
- [ ] 不写入任何文件
- [ ] 对于建议级别的安全发现结果，裁决结果为 CONCERNS

## 协议合规性（Protocol Compliance）

- [ ] 在审计前读取 `src/` 中的源文件
- [ ] 检查存档数据加密、硬编码凭据、内部信息暴露、身份验证守卫
- [ ] 为每个发现结果提供修复建议
- [ ] 不写入任何文件（只读技能）
- [ ] 不调用任何主管关卡
- [ ] 裁决结果为以下之一：SECURE、CONCERNS、VULNERABILITIES FOUND

---

## 覆盖范围说明（Coverage Notes）

- 反作弊分析（客户端值验证、服务器权威性）未在此处明确测试；它根据严重程度遵循 CONCERNS 或 VULNERABILITIES 模式。
- 数据隐私合规性（GDPR、COPPA）不在本规格范围内；这些需要代码扫描之外的法律审查。
