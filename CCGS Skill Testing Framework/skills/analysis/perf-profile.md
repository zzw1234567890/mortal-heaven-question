
# 技能测试规格：/perf-profile（性能分析）

## 技能概述（Skill Summary）

`/perf-profile` 是一个结构化的性能分析工作流，用于识别瓶颈并推荐优化方案。如果提供了分析器数据或性能日志，它会直接进行分析。如果没有提供，它会引导用户完成一份手动分析清单。不调用任何主管关卡（Director Gate）。该技能在持久化报告前会询问"我可以写入 `production/qa/perf-[date].md` 吗？"。裁决结果（Verdict）：WITHIN BUDGET（预算内）、CONCERNS（关注）或 OVER BUDGET（超预算）。

---

## 静态断言（结构）（Static Assertions）

由 `/skill-test static` 自动验证 —— 无需测试夹具。

- [ ] 具有所需的前置元数据字段：`name`、`description`、`argument-hint`、`user-invocable`、`allowed-tools`
- [ ] 具有 ≥2 个阶段标题
- [ ] 包含裁决关键词：WITHIN BUDGET、CONCERNS、OVER BUDGET
- [ ] 包含"May I write"用语（技能写入性能报告）
- [ ] 具有下一步交接说明（审查性能发现结果后的操作建议）

## 主管关卡检查（Director Gate Checks）

无。性能分析是一项建议性分析技能；不调用任何关卡。

## 测试用例（Test Cases）

### 用例 1：正常路径 —— 提供了帧数据，发现绘制调用峰值

**测试夹具（Fixture）：**
- 用户提供 `production/qa/profiler-export-2026-03-15.json`，包含帧时间数据
- 数据显示：平均帧时间 14ms（在 16.6ms 预算内），但第 42–48 帧峰值达到 28ms
- 峰值与一个包含 450 次绘制调用（预算：200）的场景相关

**输入：** `/perf-profile production/qa/profiler-export-2026-03-15.json`

**预期行为：**
1. 技能读取分析器数据
2. 技能识别出平均帧时间在预算内
3. 技能识别出第 42–48 帧的绘制调用峰值（450 次调用 vs 预算 200）
4. 裁决结果为 CONCERNS（平均值正常，但峰值表明存在问题）
5. 技能建议对识别出的场景进行批处理或裁剪
6. 技能询问"我可以写入 `production/qa/perf-2026-04-06.md` 吗？"

**断言：**
- [ ] 峰值帧按帧号标识
- [ ] 绘制调用次数和预算进行明确比较
- [ ] 即使平均值正常，当峰值超出预算时裁决结果为 CONCERNS
- [ ] 至少给出一条具体的优化建议
- [ ] 在写入报告前出现"May I write"提示

---

### 用例 2：无分析器数据 —— 输出手动清单

**测试夹具：**
- 用户运行 `/perf-profile`，不带任何参数
- `production/qa/` 中不存在分析器数据文件

**输入：** `/perf-profile`

**预期行为：**
1. 技能未找到分析器数据
2. 技能输出一份手动分析清单供用户逐步操作：
   - 启用 Godot 分析器或目标引擎的分析器
   - 记录一次 60 秒的游戏会话
   - 导出帧时间数据
   - 记录任何丢帧或卡顿
3. 技能要求用户在收集数据后提供数据再运行分析

**断言：**
- [ ] 当未提供数据时，技能不崩溃或不发出裁决结果
- [ ] 输出手动分析清单（可操作的步骤，而非仅错误信息）
- [ ] 不发出裁决结果（尚无评估内容）
- [ ] 不写入任何文件

---

### 用例 3：超预算 —— 目标平台的帧预算被超出

**测试夹具：**
- 分析器数据显示一致的 22ms 帧时间（目标：60fps 对应 16.6ms）
- 所有帧均超出预算；无单一峰值 —— 系统性问题
- `technical-preferences.md` 指定目标平台：PC，60fps

**输入：** `/perf-profile production/qa/profiler-export-2026-03-20.json`

**预期行为：**
1. 技能读取分析器数据和技术偏好中的性能预算
2. 所有帧均超过 16.6ms 预算
3. 裁决结果为 OVER BUDGET
4. 技能输出一个按优先级排列的优化列表（例如细节层次系统 LOD、着色器复杂度、物理更新频率）
5. 技能在写入报告前询问"May I write"

**断言：**
- [ ] 当所有或大多数帧超出预算时，裁决结果为 OVER BUDGET
- [ ] 目标帧预算从 `technical-preferences.md` 读取（非硬编码）
- [ ] 提供优化优先级列表，而非仅原始裁决结果
- [ ] 在报告写入前出现"May I write"提示

---

### 用例 4：存在之前的性能报告 —— 增量对比

**测试夹具：**
- `production/qa/perf-2026-03-28.md` 存在，包含先前结果（平均 15ms，最大 19ms）
- 新的分析器导出显示：平均 13ms，最大 17ms
- 两份报告针对同一场景

**输入：** `/perf-profile production/qa/profiler-export-2026-04-05.json`

**预期行为：**
1. 技能读取新的分析器数据
2. 技能检测到同一场景的先前报告
3. 技能计算增量：平均改善 2ms，最大改善 2ms
4. 技能呈现回归检查：未检测到回归
5. 裁决结果为 WITHIN BUDGET；报告说明自上次分析以来有所改善

**断言：**
- [ ] 技能在写入前检查 `production/qa/` 中的先前性能报告
- [ ] 显示增量对比（关键指标的前次 vs 当前）
- [ ] 当当前指标在预算内时，裁决结果为 WITHIN BUDGET
- [ ] 在报告中正面说明改善趋势

---

### 用例 5：关卡合规性 —— 无关卡；性能分析师单独处理

**测试夹具：**
- 分析器数据显示 CONCERNS 级别的发现结果（一些峰值）
- `review-mode.txt` 包含 `full`

**输入：** `/perf-profile production/qa/profiler-export-2026-04-01.json`

**预期行为：**
1. 技能分析分析器数据；裁决结果为 CONCERNS
2. 无论审核模式如何，均不调用任何主管关卡
3. 输出说明："如需深入分析，请考虑使用性能分析师（performance-analyst）智能体运行 `/perf-profile`"
4. 技能询问"May I write"，并在用户批准后写入报告

**断言：**
- [ ] 在任何审核模式下均不调用主管关卡
- [ ] 建议（而非强制）咨询性能分析师
- [ ] 在报告写入前出现"May I write"提示
- [ ] 对于基于峰值的发现结果，裁决结果为 CONCERNS

## 协议合规性（Protocol Compliance）

- [ ] 在提供分析器数据时读取数据；未提供时输出操作清单
- [ ] 读取 `technical-preferences.md` 中的目标平台帧预算
- [ ] 检查先前的性能报告以支持增量对比
- [ ] 在写入报告前始终询问"May I write"
- [ ] 不调用任何主管关卡
- [ ] 裁决结果为以下之一：WITHIN BUDGET、CONCERNS、OVER BUDGET

---

## 覆盖范围说明（Coverage Notes）

- 平台特定的分析工作流（主机、移动端）未在此处测试；用例 2 中的清单输出在实践中将因平台而异。
- 用例 4 中的增量对比假定报告覆盖同一场景；跨场景比较未进行明确处理。
