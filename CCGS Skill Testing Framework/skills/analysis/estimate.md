
# 技能测试规格：/estimate（估算）

## 技能概述（Skill Summary）

`/estimate` 使用相对规模量表（S / M / L / XL）估算任务或故事的工时，依据为故事复杂度、验收标准数量以及过往冲刺（Sprint）文件中的历史冲刺速度。估算结果为建议性质，从不自动写入。不调用任何主管关卡（Director Gate）。结果输出为工时范围，而非通过/失败 —— 每次运行都会产生估算。

---

## 静态断言（结构）（Static Assertions）

由 `/skill-test static` 自动验证 —— 无需测试夹具。

- [ ] 具有所需的前置元数据字段：`name`、`description`、`argument-hint`、`user-invocable`、`allowed-tools`
- [ ] 具有 ≥2 个阶段标题
- [ ] 包含规模标签：S、M、L、XL（该技能的"裁决结果"等效项）
- [ ] 不需要"May I write"用语（仅限建议性输出）
- [ ] 具有下一步交接说明（如何在冲刺规划中使用估算结果）

---

## 主管关卡检查（Director Gate Checks）

无。估算是一项建议性信息类技能；不调用任何关卡。

---

## 测试用例（Test Cases）

### 用例 1：正常路径 —— 清晰的故事且技术栈已知

**测试夹具（Fixture）：**
- `production/epics/combat/story-hitbox-detection.md` 存在，且：
  - 4 条清晰的验收标准（Acceptance Criteria）
  - 架构决策记录（Architecture Decision Record, ADR）引用（状态为 Accepted）
  - 故事正文中无"unknown"或"TBD"用语
- `production/sprints/sprint-003.md` 至 `sprint-005.md` 存在且包含速度数据
- 技术栈为 GDScript（根据冲刺历史，团队对此非常熟悉）

**输入：** `/estimate production/epics/combat/story-hitbox-detection.md`

**预期行为：**
1. 技能读取故事文件 —— 评估清晰度、验收标准数量、技术栈
2. 技能读取冲刺历史以确定平均速度
3. 技能输出估算：M（1–2 天）并附推理说明
4. 不写入任何文件

**断言：**
- [ ] 对于一个清晰、范围明确且技术已知的故事，估算为 M
- [ ] 推理说明引用验收标准数量、技术栈熟悉程度和速度数据
- [ ] 估算以范围形式呈现（例如"1–2 天"），而非单一数值
- [ ] 不写入任何文件

### 用例 2：高度不确定 —— 系统未知，尚无 ADR

**测试夹具：**
- `production/epics/online/story-lobby-matchmaking.md` 存在，且：
  - 2 条含糊的验收标准（使用了"should"和"TBD"）
  - 无 ADR 引用 —— 匹配架构尚未决定
  - 引用新子系统（"online/matchmaking"），无现有源文件

**输入：** `/estimate production/epics/online/story-lobby-matchmaking.md`

**预期行为：**
1. 技能读取故事 —— 发现含糊的验收标准、无 ADR、无现有源码
2. 技能标记多个不确定性因素
3. 估算为 L–XL，并附明确的风险说明："由于架构未知，估算范围较宽"
4. 技能建议在开发开始前创建 ADR

**断言：**
- [ ] 当存在重大未知因素时，估算为 L 或 XL（而非 S 或 M）
- [ ] 风险说明解释了导致范围较宽的具体未知因素
- [ ] 输出建议先解决架构问题
- [ ] 不写入任何文件

---

### 用例 3：无冲刺速度数据 —— 使用保守默认值

**测试夹具：**
- 故事文件存在且定义良好
- `production/sprints/` 为空 —— 无历史冲刺

**输入：** `/estimate production/epics/core/story-save-load.md`

**预期行为：**
1. 技能读取故事 —— 评估复杂度
2. 技能尝试读取冲刺速度数据 —— 未找到
3. 技能说明："未找到冲刺历史 —— 使用速度的保守默认值"
4. 使用默认假设生成估算（例如 1 个故事点 = 1 天）
5. 不写入任何文件

**断言：**
- [ ] 当无冲刺历史时技能不会报错
- [ ] 输出明确说明正在使用保守默认值
- [ ] 仍然生成估算（不因缺少速度数据而阻塞）
- [ ] 保守默认值产生更高（而非更低）的估算范围

---

### 用例 4：多个故事 —— 逐个估算并给出冲刺总计

**测试夹具：**
- 用户提供冲刺文件：`production/sprints/sprint-007.md`，包含 4 个故事
- 冲刺历史存在（3 个历史冲刺）

**输入：** `/estimate production/sprints/sprint-007.md`

**预期行为：**
1. 技能读取冲刺文件 —— 识别出 4 个故事
2. 技能逐个估算每个故事：S、M、M、L
3. 技能计算冲刺总计：约 6–8 个故事点
4. 技能逐个展示故事的估算结果，随后展示冲刺总计
5. 不写入任何文件

**断言：**
- [ ] 每个故事获得自己的估算标签
- [ ] 冲刺总计在单个估算之后展示
- [ ] 总计是由各个范围相加得出的总和范围
- [ ] 技能处理冲刺文件（而非仅单个故事文件）作为输入

**测试夹具：**

**预期行为：**

2. 在任何审核模式下均不调用主管关卡
3. 估算仅以建议性输出形式呈现
4. 技能说明："在 `/sprint-plan` 中选择下个冲刺的故事时使用此估算"

**断言：**
- [ ] 无论审核模式如何，均不调用主管关卡
- [ ] 输出纯粹为信息参考 —— 无需批准或写入提示
- [ ] 下一步建议引用 `/sprint-plan`
- [ ] 估算不因审核模式而改变

## 协议合规性（Protocol Compliance）

- [ ] 在估算前读取故事文件
- [ ] 在可用时读取冲刺速度历史
- [ ] 生成工时范围（S/M/L/XL），而非单一数值
- [ ] 不写入任何文件
- [ ] 不调用任何主管关卡
- [ ] 始终生成估算（不会因缺少数据而阻塞；使用默认值）

---

## 覆盖范围说明（Coverage Notes）

- 该技能不生成 PASS/FAIL 裁决结果；此处的"裁决结果"即为工时时长范围本身。测试断言侧重于范围的准确性和推理说明的质量，而非二元结果。
- 团队特定的速度校准（"M"对该团队意味着什么）是一个未在此处测试的实现细节；它通过冲刺历史进行配置。
