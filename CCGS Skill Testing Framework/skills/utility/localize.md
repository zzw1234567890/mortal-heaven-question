
# 技能测试规格：/localize

## 技能概述 (Skill Summary)

`/localize` 管理完整的本地化管线：它从源文件中提取所有面向玩家的字符串，管理 `assets/localization/` 中的翻译文件，并验证所有语言环境文件的完整性。对于新语言，它会创建一个包含所有当前字符串作为键（key）和空值（value）的语言环境文件骨架。对于现有的语言环境文件，它生成一个差异报告（diff），显示新增、删除和变更的键。

翻译文件在征询"我可以写入吗？"（May I write）后写入 `assets/localization/[locale-code].csv`（或引擎适用的格式）。无需主管关口（director gate）。判定结果（Verdicts）：LOCALIZATION COMPLETE（所有语言环境文件完整）或 GAPS FOUND（至少一个语言环境文件缺少字符串键）。

## 测试用例 (Test Cases)

### 用例 1：新语言——字符串提取并创建语言环境文件骨架

**测试夹具 (Fixture)：**
- `src/` 中的源代码包含面向玩家的字符串（UI 文本、教程信息）
- 现有语言环境文件：`assets/localization/en.csv`
- 不存在法语（French）语言环境文件

**输入：** `/localize fr`

**预期行为：**
1. 技能从源文件中提取所有面向玩家的字符串
2. 技能找到 `en.csv` 中的相同字符串作为参考
3. 技能生成 `fr.csv` 骨架文件，包含所有字符串键和空值
4. 技能询问"我可以写入 `assets/localization/fr.csv` 吗？"
5. 批准后写入文件；判定结果为 GAPS FOUND（文件已创建但值为空）
6. 技能注明："fr.csv 已创建——发送给翻译人员填写值"

**断言 (Assertions)：**
- [ ] `en.csv` 中的所有字符串键都出现在 `fr.csv` 中
- [ ] `fr.csv` 中的所有值均为空（未从英文复制）
- [ ] 在创建文件前询问"May I write"
- [ ] 判定结果为 GAPS FOUND（文件已创建但未翻译）

### 用例 2：现有语言环境文件的差异报告——列出新增、删除和变更项

**测试夹具：**
- `assets/localization/fr.csv` 存在，包含 20 个已翻译的字符串键
- 源代码已变更：新增 3 个字符串，删除 1 个字符串，2 个字符串的英文源文本已更改

**输入：** `/localize fr`

**预期行为：**
1. 技能从源文件中提取当前字符串
2. 技能与现有的 `fr.csv` 进行差异对比
3. 技能生成差异报告：
   - 3 个新键（需要翻译——在 fr.csv 中列为空）
   - 1 个已删除键（标记为已弃用——建议删除）
   - 2 个已变更键（英文源已更改——法语翻译可能需要更新，已标记）
4. 技能询问"我可以更新 `assets/localization/fr.csv` 吗？"
5. 文件更新，新增了空键，标记了已弃用的键；判定结果为 GAPS FOUND

**断言：**
- [ ] 新键在更新后的文件中显示为空（不自动翻译）
- [ ] 已删除的键被标记为已弃用（不会静默删除）
- [ ] 已更改的源字符串被标记为需要翻译人员审查
- [ ] 判定结果为 GAPS FOUND（存在新的空键）

### 用例 3：一个语言环境文件中缺少字符串——GAPS FOUND 并列出缺失键列表

**测试夹具：**
- 存在 3 个语言环境文件：`en.csv`、`fr.csv`、`de.csv`
- `de.csv` 缺少 4 个在 `en.csv` 和 `fr.csv` 中都存在的键

**输入：** `/localize`

**预期行为：**
1. 技能读取所有 3 个语言环境文件并交叉引用（cross-reference）键
2. `de.csv` 缺少 4 个键
3. 技能生成 GAPS FOUND 报告，按语言环境列出 4 个缺失键：
   "de.csv 缺少：[key1]、[key2]、[key3]、[key4]"
4. 技能主动提出将缺失键作为空值添加到 `de.csv` 中
5. 批准后：文件已更新；判定仍为 GAPS FOUND（值仍为空）

**断言：**
- [ ] 缺失键被明确列出（不仅仅是数量）
- [ ] 缺失键归属于特定的语言环境文件
- [ ] 判定结果为 GAPS FOUND（而非 LOCALIZATION COMPLETE）
- [ ] 缺失键被添加为空值（不自动从英文翻译）

### 用例 4：翻译文件存在语法错误——带行号引用的错误信息

**测试夹具：**
- `assets/localization/fr.csv` 在第 47 行存在格式错误（缺少引号闭合）

**输入：** `/localize fr`

**预期行为：**
1. 技能读取 `fr.csv`，在第 47 行遇到解析错误
2. 技能输出："fr.csv 第 47 行存在解析错误：[错误详情]"
3. 在错误修复之前，技能无法对该文件进行差异对比或验证
4. 技能不尝试覆盖或自动修复格式错误的文件
5. 技能建议手动修复文件并重新运行 `/localize`

**断言：**
- [ ] 错误信息包含行号（第 47 行）
- [ ] 错误详情描述了解析错误的性质
- [ ] 技能不覆盖或修改格式错误的文件
- [ ] 建议手动修复并重新运行作为补救措施

## 协议合规性 (Protocol Compliance)

- [ ] 在操作语言环境文件之前，从源文件中提取字符串
- [ ] 创建所有键值为空的新语言环境文件（不自动翻译）
- [ ] 将现有语言环境文件与当前源字符串进行差异对比
- [ ] 按语言环境和键名标记缺失的键
- [ ] 在创建或更新任何语言环境文件前询问"May I write"
- [ ] 判定结果为 LOCALIZATION COMPLETE（所有语言环境文件完全翻译）或 GAPS FOUND

## 覆盖说明 (Coverage Notes)

- LOCALIZATION COMPLETE 仅在所有语言环境文件的所有键具有非空值时才能达到；新语言文件骨架创建始终导致 GAPS FOUND。
- 引擎特定的语言环境格式（Godot `.translation`、Unity `.po` 文件）由技能主体处理；`.csv` 在测试中用作规范格式。
- 源字符串以极高速率变更的情况（新 UI 文本的持续集成）未测试；差异对比逻辑可处理此情况。
