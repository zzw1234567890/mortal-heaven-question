## 测试用例

### 用例 1：领域内请求 —— 合适的输出
**输入：** "在 Godot 中为敌人死亡编写一个溶解效果着色器。"
**预期行为：**
- 产出有效的 Godot 着色语言代码（非 HLSL，也非直接 GLSL）
- 按需使用 `shader_type spatial;` 或 `canvas_item`
- 定义 `uniform float dissolve_amount : hint_range(0.0, 1.0);`
- 采样噪声纹理以确定每个像素的溶解阈值
- 对低于阈值的像素使用 `discard;`
- 可选地在溶解边界附近添加边缘发光效果
- 代码在 Godot 着色语言的语法上正确

### 用例 2：HLSL 重定向
**输入：** "为此溶解效果编写一个 HLSL 计算着色器。"
**预期行为：**
- 不产出 HLSL 代码
- 明确说明："Godot 不直接使用 HLSL；它使用自己的着色语言（GLSL 的衍生版本）"
- 将 HLSL 意图转换为等效的 Godot 着色器方法
- 注明 Godot 4 中可使用 RenderingDevice 计算着色器，但属于底层 API，并酌情标记

### 用例 3：知识截止日期后的 API 变更 —— 纹理采样（Godot 4.4）
**输入：** "在着色器中使用带有 sampler2D 的 `texture()` 对噪声纹理进行采样。"
**预期行为：**
- 检查版本参考：Godot 4.4 更改了纹理采样器类型声明
- 标记潜在的 API 变更：`sampler2D` 语法和 `texture()` 调用行为可能与 4.4 之前版本不同
- 提供项目锁定版本（4.6）的正确语法，如迁移说明中所述
- 不标记版本风险的情况下不使用 4.4 之前的纹理采样语法

### 用例 4：片段着色器 LOD 策略
**输入：** "水面的片段着色器有 8 个纹理采样，在中端硬件上造成了 GPU 瓶颈。"
**预期行为：**
- 识别每个片段的纹理采样数是主要的性能成本驱动因素
- 提出 LOD（细节层级）策略：
  - 距离较远时减少采样数（基于距离的着色器变体或 LOD 层级）
  - 离线预烘焙某些纹理组合
  - 对远距离采样使用较低分辨率的噪声纹理
- 提供实现 LOD 方法的着色器代码修改
- 不改变水系统的玩法行为

### 用例 5：上下文传递 —— Godot 4.6 发光效果重做
**输入：** 引擎版本上下文：Godot 4.6。请求："为场景添加泛光/发光后处理效果。"
**预期行为：**
- 引用 VERSION.md 中的说明：Godot 4.6 包含发光效果重做 (glow rework)
- 使用 4.6 的 WorldEnvironment 方法（而非 4.6 之前的 API）产出发光配置指导
- 明确注明在 4.6 发光效果重做中哪些属性或参数发生了变更
- 标记任何 LLM 训练数据可能因知识截止日期而包含错误信息的属性
