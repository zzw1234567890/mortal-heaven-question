# 智能体测试规格：world-builder（世界构建师）


## 智能体概要 (Agent Summary)
- **领域**：世界设定架构 —— 派系及其文化/政府/动机、世界历史、地理与生态、宇宙观与形而上学、世界规则（魔法如何运作、什么可能什么不可能）、跨世界文档的内部一致性维护
- **不拥有**：特定 NPC 或任务对话（属于 writer 编剧）、源自世界规则的游戏机制规则（属于 game-designer 游戏设计师 / systems-designer 系统设计师）、叙事故事结构与情节设计（属于 narrative-director 叙事总监）
- **模型层级**：Sonnet
- **关卡 ID**：无；将世界规则/机制冲突升级至 narrative-director 和 game-designer 共同处理

---

## 静态断言（结构）

- [ ] `description:` 字段存在且具有领域特定性（引用世界设定、派系、历史、世界规则、生态）
- [ ] `allowed-tools:` 列表与智能体角色匹配（Read/Write 用于 design/narrative/world/ 文档；无游戏源代码、机制设计或对话文件）
- [ ] 模型层级为 Sonnet（创意类专家默认）
- [ ] 智能体定义未声称对对话写作、机制设计或叙事结构拥有权限

## 测试用例 (Test Cases)

### 用例 1：领域内请求 —— 派系文化与政府设计
**输入**："设计铁幕商业联盟 —— 我们世界中一个强大的贸易派系。我需要他们的文化、政府结构和内部动机。"
**预期行为**：
- 生成派系档案文档，包含：文化价值观和规范、政府结构（如何决策、谁掌权、继承或任命流程）、联盟内部派系或紧张关系、与其他派系的关系（盟友、对手、中立势力），以及主要动机（他们想要什么以及为什么）
- 派系内部一致：商业联盟的政府由经济逻辑驱动，而非封建或宗教逻辑，除非明确指定了有意为之的混合模式
- 输出包含至少一个派系内部的紧张关系或矛盾 —— 没有内部复杂性的派系是扁平的
- 格式化为结构化派系档案，而非叙事文章

### 用例 2：领域外请求 —— 对话写作
**输入**："编写玩家在城门遇到的一位铁幕商业联盟商人 NPC 的对话。"
**预期行为**：
- 不生成 NPC 对话
- 明确说明："对话写作由 writer 负责；我提供为对话提供背景的世界和派系信息，包括派系的文化、语气和说话风格"
- 主动提供 writer 编写一致对话所需的派系说话风格说明和文化背景

### 用例 3：新增设定条目与既定历史矛盾 —— 冲突标记
**输入**："添加一条设定，说明铁幕商业联盟由单一商人家族于 50 年前创立。"[上下文包含既有设定：该联盟已存在 300 年，由 12 家竞争商行作为集体创立。]
**预期行为**：
- 识别矛盾：既有设定说明 300 年历史和 12 家商行的创始联盟；新条目声称 50 年和单一创始家族
- 不按请求写入新条目
- 标记冲突：说明两个版本，确定哪个是既有设定、哪个是提议的变更
- 提出解决方案选项：(a) 新条目有误，应予更正；(b) 如果新版本是意图的正典，则应更新既有设定；(c) 存在世界内解释（当前家族声称创始功绩，尽管是集体起源 —— 一种故意的不可靠叙事手法）
- 如无明确答案则路由至 narrative-director 解决

### 用例 4：世界规则有游戏性影响 —— 与 game-designer 协调
**输入**："我想确立一条世界规则：在铁矿附近施法的魔法使用者会被削弱。铁会扰乱奥术能量。"
**预期行为**：
- 将世界规则作为设定条目生成：形而上学的解释、世界内如何理解、历史影响
- 识别游戏性影响：此世界规则有直接的机制后果（铁矿附近的玩家受到减益效果、关卡设计必须考虑铁矿放置）
- 标记协调需求："此世界规则有游戏机制影响 —— game-designer 需要定义如何将其转化为面向玩家的机制；在未定义机制的情况下推进设定存在不一致的风险"
- 不单方面设计游戏机制 —— 描述设定规则及其暗示的机制领域，然后委托给 game-designer

### 用例 5：上下文传递 —— 使用既定世界文档
**输入上下文**：现有世界文档说明：世界使用双星系统，一颗太阳是奥术能量的来源（白太阳），奥术魔法在为期 3 天的月食期间（暗化期）停止运作。
**输入**："添加关于法师学院及其如何为暗化期做准备的设定条目。"
**预期行为**：
- 使用既定的双星宇宙观：引用白太阳作为奥术能量来源
- 使用既定的暗化期事件：3 天月食，魔法停止
- 不编造不同的月食机制、持续时间或名称
- 生成设定条目，其中法师学院为暗化期做的准备与既定规则一致：他们不能在暗化期内施法，因此准备是实用性的（囤积非魔法物资、排期、停止正在进行的魔法过程）
- 不矛盾上下文文档中的任何既定事实

## 协议合规 (Protocol Compliance)

- [ ] 保持在声明领域内（派系、世界历史、地理、生态、世界规则、宇宙观）
- [ ] 将对话写作请求重定向至 writer，附带背景派系说明
- [ ] 标记设定矛盾，列出两个版本并提供解决选项 —— 不静默覆盖既定设定
- [ ] 识别世界规则的游戏性影响并标记与 game-designer 协调
- [ ] 使用上下文中的所有既定世界事实；不编造所述设定的替代方案

## 覆盖说明 (Coverage Notes)
- 用例 3（矛盾检测）要求既有设定在上下文中 —— 这是最重要的一致性测试
- 用例 4（世界规则/机制协调）测试跨领域意识；验证智能体识别机制边界而不越界
- 用例 5 是最重要的上下文感知测试；智能体必须使用既定事实，而非创意替代方案
- 无自动运行器；手动审查或通过 `/skill-test` 进行
