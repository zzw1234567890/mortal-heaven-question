# 智能体测试规格：sound-designer（音效设计师）


## 智能体概要 (Agent Summary)
领域：音效规格、音频事件、混音参数和声音分类定义。
不拥有：作曲方向（属于 audio-director 音频总监）、音频系统的代码实现。
模型层级：Sonnet（默认）。
未分配关卡 ID。

---

## 静态断言（结构）

- [ ] `description:` 字段存在且具有领域特定性（引用音效 / 音频事件 / 混音）
- [ ] `allowed-tools:` 列表包含 Read, Write, Edit, Glob, Grep —— 不包含引擎代码执行工具
- [ ] 模型层级为 Sonnet（专家默认）
- [ ] 智能体定义未声称对作曲方向或音频代码实现拥有权限

---

## 测试用例 (Test Cases)

### 用例 1：领域内请求 —— 正确输出
**输入：** "为剑挥击攻击创建音效规格。"
**预期行为：**
- 生成完整的音频事件规格，包括：
  - 事件名称（例如：`sfx_combat_sword_swing`）
  - 变体数量（至少 3 个，避免听觉疲劳）
  - 音高范围（例如：±8% 随机化）
  - 音量范围和归一化目标（例如：-12 dBFS）
  - 声音分类（例如：`combat_sfx`）
  - 建议的分层说明（呼啸层 + 冲击瞬态）
- 输出遵循已建立的项目音频命名规范（如有）

### 用例 2：领域外请求 —— 正确重定向
**输入：** "为森林关卡创作一首循环环境音乐轨道。"
**预期行为：**
- 不生成作曲方向或音乐简报
- 明确声明音乐方向属于 `audio-director`
- 将请求重定向至 `audio-director`
- 可以说明一旦音乐方向确定，可以提供音效环境层规格（风声、野生动物）以补充音乐

### 用例 3：动态参数 —— 衰减曲线规格
**输入：** "剑挥击音效需要距离衰减，以便在整个竞技场中听起来不同。"
**预期行为：**
- 生成动态参数规格，包括：
  - 参数名称（例如：`distance` 或 `listener_distance`）
  - 衰减曲线类型（例如：对数、线性、自定义）
  - 近/远距离阈值及对应的音量和高频衰减值
  - 遮挡覆盖行为（如适用）
- 不编写音频引擎集成代码（委托给对应的程序员）

### 用例 4：命名规范冲突
**输入：** "为近战系统添加一个名为 `SWORD_HIT_1` 的新音效事件。"
**预期行为：**
- 识别出 `SWORD_HIT_1` 与已建立的命名规范冲突（应为蛇形命名法加分类前缀，例如：`sfx_combat_sword_hit`）
- 不静默注册不符合规范的事件名
- 向 `audio-director` 标记冲突并提出合规的替代方案
- 待 audio-director 确认后按更正名称继续

### 用例 5：上下文传递 —— 使用音频风格指南
**输入：** 上下文中提供的音频风格指南指定："粗粝、扎实、混响尾音不超过 1.5 秒，参考：《巫师 3》战斗音频。"请求："为整套近战战斗套件创建音效规格。"
**预期行为：**
- 在规格说明中引用"粗粝、扎实"的调性描述
- 将所有混响尾音规格限制在 1.5 秒以内
- 注明参考资料（《巫师 3》）作为混音水平和瞬态设计的基准
- 不生成与风格指南矛盾的规格（例如：不生成空灵或重度混响处理的规格）

## 协议合规 (Protocol Compliance)

- [ ] 保持在声明领域内（音效规格、事件定义、混音参数）
- [ ] 将音乐方向请求重定向至 audio-director
- [ ] 返回结构化的音频事件规格（事件名称、变体、音高、音量、分类）
- [ ] 不生成音频系统实现的代码
- [ ] 标记命名规范违规而非静默接受不符合规范的事件名
- [ ] 在所有规格输出中引用提供的风格指南和约束

---

## 覆盖说明 (Coverage Notes)
- 音效规格格式（用例 1）应与音频中间件（Wwise/FMOD/内置）所需的事件模式相匹配
- 衰减曲线（用例 3）验证智能体生成可供实施的参数规格
- 风格指南合规（用例 5）确认智能体读取提供的上下文并据此约束输出
