
# 安全策略

## 受支持的版本

只有 `main` 分支会接收安全修复。Fork 和旧版本不受支持。

## 报告漏洞

**不要通过公开的 GitHub issue 报告安全漏洞。**

请改用 GitHub 的私密漏洞报告：

**[报告漏洞 →](https://github.com/Donchitos/Claude-Code-Game-Studios/security/advisories/new)**

请包含尽可能多的细节：
- 漏洞描述以及它影响什么
- 复现步骤
- 潜在影响和攻击场景
- 任何建议的缓解措施

**预期流程：**
- 在 **48 小时**内确认收到
- 在 **7 天**内提供状态更新
- 对于确认的漏洞，在 **90 天**内解决

## 范围内内容

CCGS 是一个**本地开发工具**——它安装 shell 钩子并协调直接在你机器上运行的 AI 智能体。安全问题主要涉及在用户不知情的情况下在用户环境中执行的贡献代码。

### 高严重性
- 钩子（`.claude/hooks/*.sh`）在用户机器上执行恶意或未披露的 shell 命令
- 技能或智能体泄露环境变量、API 密钥或机密
- 通过技能或智能体定义进行提示注入 (prompt injection)，导致 Claude 绕过安全措施或采取未经授权的破坏性操作
- 贡献以用户无法审计的方式静默更改行为

### 中严重性
- 技能发起未披露的出站网络请求
- 智能体定义升级权限或绕过用户确认提示
- 钩子模式在不同平台上表现不同以隐藏行为
- 技能在没有明确用户批准步骤的情况下写入其文档化范围之外

### 范围外
- Claude 或 Claude Code CLI 本身的行为（请报告给 [Anthropic](https://www.anthropic.com/security)）
- 用户的 Claude Code 安装或编辑器扩展中的 bug
- 没有现实攻击路径的理论漏洞
- 需要物理访问用户机器的问题

## 贡献者安全指南

在贡献钩子、技能或智能体时：

- **钩子必须 POSIX 兼容**——使用 `grep -E`，而非 `grep -P`；避免在不同操作系统上行为不同的平台特定语法
- **不得有静默网络调用**——钩子或技能不得发起，除非明确文档化并由用户选择加入
- **不得读取机密或环境变量**——超出最低必需且在技能头部明确文档化的范围
- **技能不得写入其文档化范围之外**——没有明确用户确认步骤不得写入

## 披露策略

我们遵循 **90 天协调披露**时间线：

1. 你私密地提交漏洞
2. 我们在 48 小时内确认
3. 我们在 7 天内确认并评估严重性
4. 我们开发并测试修复
5. 我们在任何公开披露之前通知你
6. 公开披露在修复发布后，或在 90 天时进行——以先到者为准

除非你希望保持匿名，否则我们会在发布说明中致谢报告者。
