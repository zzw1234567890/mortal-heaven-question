# 设计文档规则 (Design Document Rules)

- 每个设计文档**必须**包含以下 8 个部分：概述 (Overview)、玩家幻想 (Player Fantasy)、详细规则 (Detailed Rules)、公式 (Formulas)、边缘情况 (Edge Cases)、依赖关系 (Dependencies)、调优旋钮 (Tuning Knobs)、验收标准 (Acceptance Criteria)
- 公式必须包含变量定义、预期值范围和计算示例
- 边缘情况必须明确说明会发生什么，而非仅写"优雅处理"
- 依赖关系必须是双向的 —— 如果系统 A 依赖于 B，B 的文档必须提及 A
- 调优旋钮必须指定安全范围及其影响的游戏方面
- 验收标准必须是可测试的 —— QA 测试员必须能够验证通过/失败
- 不得含糊其辞："系统应该感觉良好"不是有效的规范
- 平衡数值必须链接到其来源公式或基本原理
- 设计文档**必须**增量编写：先创建骨架，然后在用户批准下一个章节一个章节地填充。每个已批准的章节立即写入文件以持久化决策并管理上下文。
