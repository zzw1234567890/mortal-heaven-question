> **仅限显式调用**：此技能仅在用户通过 `/launch-checklist` 显式请求时运行。请勿基于上下文匹配自动调用。

## 阶段 1：解析参数

读取启动日期或 `dry-run`（试运行）模式的参数。试运行模式将生成检查清单，但不会创建签署条目或写入文件。

---

## 阶段 2：收集项目上下文

- 读取 `CLAUDE.md` 以了解技术栈、目标平台和团队结构
- 读取 `production/milestones/` 中最新的里程碑文件
- 读取 `production/releases/` 中已有的发布检查清单
- 如果存在，读取 `design/live-ops/content-calendar.md` 中的内容日历

---

## 阶段 3：扫描代码库健康状态

- 统计 `TODO`、`FIXME`、`HACK` 注释的数量及其位置
- 检查生产代码中是否遗留了 `console.log`、`print()` 或调试输出
- 检查占位资源（搜索 `placeholder`、`temp_`、`WIP_`）
- 检查硬编码的测试/开发环境值（localhost、测试凭据、调试标志）

---

## 阶段 4：生成发布检查清单

```markdown
# 发布检查清单：[游戏名称]
目标发布日期：[日期 或 试运行]
生成日期：[日期]

---

## 1. 代码就绪

### 构建健康
- [ ] 所有目标平台上的干净构建
- [ ] 零编译器警告
- [ ] 所有单元测试通过
- [ ] 所有集成测试通过
- [ ] 性能基准测试达标
- [ ] 无内存泄漏（经扩展浸泡测试验证）
- [ ] 构建体积在平台限制范围内
- [ ] 构建版本号在源码控制中正确设置并标记

### 代码质量
- [ ] TODO 数量：[N]（发布时需归零，或记录例外情况）
- [ ] FIXME 数量：[N]（必须归零）
- [ ] HACK 数量：[N]（每项必须有书面理由）
- [ ] 生产代码中无调试输出
- [ ] 无硬编码的开发/测试环境值
- [ ] 所有功能开关设置为生产环境值
- [ ] 错误处理覆盖所有关键路径
- [ ] 崩溃报告已集成并验证

### 安全
- [ ] 源码中无暴露的 API 密钥或凭据
- [ ] 存档数据已加密
- [ ] 网络通信已加密（TLS/DTLS）
- [ ] 反作弊措施已启用（若为多人游戏）
- [ ] 所有服务器端点均有输入验证（若为多人游戏）
- [ ] 隐私政策合规性已验证

---

## 2. 内容就绪

### 资产
- [ ] 所有占位美术已替换为最终资源
- [ ] 所有占位音频已替换为最终音频
- [ ] 混音已完成并由音频总监批准
- [ ] 所有视觉效果已打磨并通过性能验证
- [ ] 无缺失或损坏的资源引用
- [ ] 资源命名规范已执行

### 文本与本地化
- [ ] 所有面向玩家的文本已完成校对
- [ ] 无硬编码字符串（全部外部化以支持本地化）
- [ ] 所有支持的语言已翻译并验证
- [ ] 文本在所有语言中均适配 UI（文本适配通过）
- [ ] 所有支持语言的字体覆盖已验证
- [ ] 制作人员名单完整、准确且为最新

### 游戏内容
- [ ] 所有关卡/地图可从头到尾游玩
- [ ] 教程流程完整并经过新玩家测试
- [ ] 所有成就/奖杯已实现并测试
- [ ] 存档/读档在所有游戏状态下均正常工作
- [ ] 难度设置已平衡并测试
- [ ] 结局/制作人员名单序列已完成

---

## 3. 质量保证

### 测试
- [ ] 完整回归测试套件通过
- [ ] 零个 S1（严重）缺陷未关闭
- [ ] 零个 S2（重大）缺陷未关闭（或已有书面例外说明）
- [ ] 浸泡测试通过（连续游戏 8 小时以上）
- [ ] 多人游戏压力测试通过（如适用）
- [ ] 每个平台上的所有关键用户路径均已测试
- [ ] 边缘情况已测试（存储空间满、无网络、挂起/恢复）

### 平台认证
- [ ] PC：满足 Steam/Epic/GOG SDK 要求
- [ ] 主机：TRC/TCR/Lotcheck 提审材料已准备就绪
- [ ] 移动端：符合 App Store/Play Store 指南
- [ ] 无障碍：满足最低标准（按键重映射、文本缩放、色盲模式）
- [ ] 年龄分级已获取（ESRB、PEGI、地区分级）

### 性能
- [ ] 在最低规格硬件上达到目标帧率
- [ ] 所有平台上的加载时间在预算范围内
- [ ] 所有平台上的内存使用在预算范围内
- [ ] 网络带宽在目标范围内（若为多人游戏）
- [ ] 关键游戏时刻无帧卡顿

---

## 4. 商店与分发

### 商店页面
- [ ] 商店页面文案已定稿并通过校对
- [ ] 截图为最新版本并按平台分辨率调整
- [ ] 预告片为最新版本并已批准
- [ ] 主视觉图和胶囊图已定稿
- [ ] 系统需求准确（PC）
- [ ] 所有地区的定价已配置
- [ ] 预购/心愿单活动已启用（如适用）

### 法务
- [ ] 最终用户许可协议已定稿并获法务批准
- [ ] 隐私政策已发布并链接
- [ ] 第三方许可声明完整
- [ ] 音乐/音频许可已验证
- [ ] 商标/知识产权确认无误
- [ ] GDPR/CCPA 合规性已验证（数据收集、同意、删除）

---

## 5. 基础设施

### 服务器（若为多人/在线游戏）
- [ ] 生产环境服务器已配置并通过负载测试
- [ ] 自动扩展已配置并测试
- [ ] 数据库备份已配置
- [ ] CDN 已配置用于内容分发
- [ ] DDoS 防护已启用
- [ ] 监控和告警已配置

### 分析与监控
- [ ] 分析管道已验证并能接收数据
- [ ] 崩溃报告已启用且仪表板可访问
- [ ] 服务器监控仪表板已上线
- [ ] 跟踪关键指标：DAU、会话时长、留存率、崩溃率
- [ ] 已为关键阈值配置告警

---

## 6. 社区与营销

### 社区就绪
- [ ] 社区准则已发布
- [ ] 审核团队已沟通到位，工具已就绪
- [ ] Discord/论坛/社交频道已建立
- [ ] FAQ 和已知问题页面已准备就绪
- [ ] 支持邮箱/工单系统已启用

### 营销
- [ ] 发布预告片已发布
- [ ] 媒体/主播评测密钥已分发
- [ ] 社交媒体发布帖子已排期
- [ ] 发布日博客文章/开发者更新已起草
- [ ] 发布版本的补丁说明已发布

---

## 7. 运营

### 团队就绪
- [ ] 发布后前 72 小时的值班排班已安排
- [ ] 事件响应预案已由团队审阅
- [ ] 回滚方案已记录并测试
- [ ] 热修复流程已测试（可在 4 小时内发布紧急修复）
- [ ] 发布问题沟通方案已制定（谁发布、在哪里发布、响应速度）

### 首日计划
- [ ] 首日补丁已准备就绪（如需）
- [ ] 服务器解锁/上线流程已记录
- [ ] 所有负责人已收藏发布监控仪表板
- [ ] 发布日的作战室/频道已建立

---

## 通过 / 否决决策

**总体状态**：[已就绪 / 未就绪 / 有条件就绪]

### 阻塞项
[列出发布前必须解决的问题]

### 条件项
[列出已有书面变通方案或已接受风险的项目]

### 所需签署
- [ ] 创意总监 — 内容和体验质量
- [ ] 技术总监 — 技术健康与稳定性
- [ ] QA 主管 — 质量与测试覆盖
- [ ] 制作人 — 排期与整体就绪度
- [ ] 发布经理 — 构建与部署就绪
```

---

## 阶段 5：保存检查清单

向用户展示完成的检查清单和摘要（总项目数、阻塞项数量、条件项数量、未完成部分的部门）。

如果不是试运行模式，询问："我可以将此文件写入 `production/releases/launch-checklist-[date].md` 吗？"

如果同意，写入文件，根据需要创建目录。

---

## 阶段 6：后续步骤

- 运行 `/gate-check` 以在发布前获得正式的通过/关注/失败裁决。
- 通过 `/team-release` 协调签署事宜。
