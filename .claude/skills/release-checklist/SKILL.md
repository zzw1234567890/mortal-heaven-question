> **仅限显式调用**：此技能仅在用户使用 `/release-checklist` 显式请求时运行。不要基于上下文匹配自动调用。

## 阶段 1：解析参数

读取目标平台参数（`pc`、`console`、`mobile` 或 `all`）。如果未指定平台，默认使用 `all`。

---

## 阶段 2：加载项目上下文

- 读取 `CLAUDE.md` 以获取项目上下文、版本信息和平台目标。
- 从 `production/milestones/` 读取当前里程碑，以了解此版本应包含哪些功能和内容。

---

## 阶段 3：扫描代码库

扫描未解决的问题：

- 统计 `TODO` 注释数量
- 统计 `FIXME` 注释数量
- 统计 `HACK` 注释数量
- 记录它们的位置和严重程度

检查测试输出目录或 CI 日志中是否包含测试结果（如有）。

---

## 阶段 4：生成发布清单

```markdown
## 发布清单：[版本号] -- [平台]
生成日期：[日期]

### 代码库健康度
- TODO 数量：[N]（如果较多，列出前 5 个）
- FIXME 数量：[N]（列出所有——这些是潜在的阻塞项）
- HACK 数量：[N]（列出所有——这些需要审查）

### 构建验证
- [ ] 在所有目标平台上都能成功清理构建
- [ ] 无编译器警告（零警告策略）
- [ ] 所有资产已包含并正确加载
- [ ] 构建体积在预算内（[目标大小]）
- [ ] 构建版本号正确设置（[版本号]）
- [ ] 构建可从标记的提交重现

### 质量门禁
- [ ] 零个 S1（严重）Bug
- [ ] 零个 S2（主要）Bug——或经制作人批准的有文档记录的例外
- [ ] 所有关键路径功能均已测试并由 QA 签署通过
- [ ] 性能在预算范围内：
  - [ ] 在最低规格硬件上达到目标 FPS
  - [ ] 内存使用在预算内
  - [ ] 加载时间在预算内
  - [ ] 长时间游戏会话无内存泄漏
- [ ] 无回归问题
- [ ] 浸泡测试通过（连续游戏 4 小时以上）

### 内容完整性
- [ ] 所有占位资源已替换为最终版本
- [ ] 内容文件中的所有 TODO/FIXME 已解决或记录
- [ ] 所有面向玩家的文本已校对
- [ ] 所有文本已准备好本地化（无硬编码字符串）
- [ ] 音频混音已最终确定并获得批准
- [ ] 演职员表完整且准确
```

根据参数添加平台特定部分：

**对于 `pc`：**
```markdown
### 平台要求：PC
- [ ] 最低和推荐配置已验证并记录
- [ ] 键盘+鼠标控制功能完整
- [ ] 已测试手柄支持（Xbox、PlayStation、通用）
- [ ] 已测试分辨率缩放（1080p、1440p、4K、超宽屏）
- [ ] 窗口模式、无边框模式和全屏模式正常工作
- [ ] 图形设置正确保存和加载
- [ ] Steam/Epic/GOG SDK 已集成并测试
- [ ] 成就功能正常
- [ ] 云存档功能正常
- [ ] Steam Deck 兼容性已验证（如果作为目标平台）
```

**对于 `console`：**
```markdown
### 平台要求：主机
- [ ] TRC/TCR/Lotcheck 要求清单完整
- [ ] 平台特定手柄提示正确显示
- [ ] 暂停/恢复功能正常工作
- [ ] 用户切换处理正确
- [ ] 网络连接丢失处理优雅
- [ ] 存储空间已满场景已处理
- [ ] 家长控制功能受尊重
- [ ] 平台特定成就/奖杯集成已测试
- [ ] 第一方认证提交通道已准备就绪
```

**对于 `mobile`：**
```markdown
### 平台要求：移动端
- [ ] 应用商店指南合规性已验证
- [ ] 所有必需的设备权限已说明并记录
- [ ] 隐私政策已链接且准确
- [ ] 数据安全/营养标签已完成
- [ ] 触控控制在多种屏幕尺寸上测试
- [ ] 电池使用在可接受范围内
- [ ] 后台行为正确（暂停、恢复、终止）
- [ ] 推送通知权限处理正确
- [ ] 应用内购买流程已测试（如适用）
- [ ] 应用体积在商店限制内
```

**商店和发布部分（所有平台）：**
```markdown
### 商店/分发
- [ ] 商店页面元数据完整且已校对
  - [ ] 简短描述
  - [ ] 详细描述
  - [ ] 功能列表
  - [ ] 系统要求（PC）
- [ ] 截图已更新并满足各平台分辨率要求
- [ ] 预告片已更新
- [ ] 主视觉图和胶囊图已更新
- [ ] 年龄分级已获取并配置：
  - [ ] ESRB
  - [ ] PEGI
  - [ ] 其他地区分级（按要求）
- [ ] 法律声明、EULA 和隐私政策已就位
- [ ] 第三方许可声明完整
- [ ] 所有地区的定价已配置

### 发布就绪
- [ ] 分析/遥测已验证并接收数据
- [ ] 崩溃报告已配置且仪表板可访问
- [ ] 首日补丁已准备并测试（如需）
- [ ] 值班团队前 72 小时排班已确定
- [ ] 社区发布公告草稿已完成
- [ ] 媒体/KOL 密钥已准备分发
- [ ] 客服团队已了解已知问题和 FAQ
- [ ] 回滚计划已记录（如发现关键问题后发布）

### 批准/不批准：[就绪 / 未就绪]

**理由：**
[准备情况评估摘要。列出发布前必须解决的任何阻塞项。
如果未就绪，列出需要解决的具体事项及预计解决时间。]

**所需签字：**
- [ ] QA 负责人
- [ ] 技术总监
- [ ] 制作人
- [ ] 创意总监
```

---

## 阶段 5：保存清单

向用户展示清单：清单项总数、已知阻塞项数量（FIXME/HACK 数量、已知 Bug）。

询问："我可以将此文件写入 `production/releases/release-checklist-[version].md` 吗？"

如果同意，写入文件，并在需要时创建目录。

---

## 阶段 6：后续步骤

- 在进入发布前运行 `/gate-check` 以获取正式阶段门禁裁决。
- 通过 `/team-release` 协调最终签字。
