# 技能改进工具

在单个技能上运行改进循环：
测试 → 修复 → 重新测试 → 保留或还原。

---

## 阶段 1：解析参数

从第一个参数读取技能名称。如果缺失，输出用法并停止：

```
用法：/skill-improve [skill-name]
示例：/skill-improve tech-debt
```

验证 `.claude/skills/[name]/SKILL.md` 是否存在。如果不存在，停止并显示：
"未找到技能 '[name]'。"

---

## 阶段 2：基准测试

运行 `/skill-test static [name]` 并记录基准分数：
- FAIL 数量
- WARN 数量
- 哪些具体检查失败（检查 1–7）

向用户显示：
```
静态基准：   [N] 个失败，[M] 个警告
失败项：检查 4（无先问再写），检查 5（无交接）
```

如果基准是 0 个 FAIL 和 0 个 WARN，记录下来并继续到阶段 2b。

### 阶段 2b：类别基准

在 `CCGS Skill Testing Framework/catalog.yaml` 中查找技能的 `category:` 字段。

如果未找到 `category:` 字段，显示：
"类别：尚未分配 — 跳过类别检查。"
并跳到阶段 3。

如果找到了类别，运行 `/skill-test category [name]` 并记录类别基准：
- FAIL 数量
- WARN 数量
- 哪些具体的类别评分指标失败

向用户显示：
```
类别基准：[N] 个失败，[M] 个警告（[category] 评分标准）
```

如果静态和类别基准都是 0 个 FAIL 和 0 个 WARN，停止：
"此技能已通过所有静态和类别检查。无需改进。"

---

## 阶段 3：诊断

读取完整的技能文件 `.claude/skills/[name]/SKILL.md`。

对于每个失败或警告的**静态**检查，确定确切差距：

- **检查 1 失败** → 哪个 frontmatter 字段缺失
- **检查 2 失败** → 找到的阶段数与最低要求
- **检查 3 失败** → 技能正文中没有任何裁决关键字
- **检查 4 失败** → allowed-tools 中有 Write 或 Edit，但没有先问再写的语言
- **检查 5 警告** → 末尾没有后续或下一步部分
- **检查 6 警告** → `context: fork` 已设置但找到的阶段少于 5 个
- **检查 7 警告** → argument-hint 为空或不匹配文档中的模式

对于每个失败或警告的**类别**检查（如果在阶段 2b 中分配了类别），在技能文本中确定确切差距。例如：
- 如果 G2 失败（门禁模式，未生成所有主管）：技能正文从未引用所有 4 个 PHASE-GATE 主管提示
- 如果 A2 失败（创作，没有逐节的"我可以写吗"）：技能只在末尾询问一次，而不是在每个章节写入前
- 如果 T3 失败（团队，未上报 BLOCKED）：技能在代理被阻塞时不停顿相关依赖工作

在提议任何更改前，向用户显示完整的综合诊断。

---

## 阶段 4：提议修复

为每个失败和警告编写针对性的修复。以清晰标记的修改前后块显示提议的更改。只更改失败的部分——不要重写通过的部分。

询问："我可以将此改进版本写入 `.claude/skills/[name]/SKILL.md` 吗？"

如果用户说否，在此停止。

---

## 阶段 5：写入并重新测试

记录技能文件的当前内容（以便在需要时还原）。

将改进后的技能写入 `.claude/skills/[name]/SKILL.md`。

重新运行 `/skill-test static [name]` 并记录新的静态分数。
如果分配了类别，也重新运行 `/skill-test category [name]` 并记录新的类别分数。

显示比较结果：
```
静态：   之前 [N] 个失败，[M] 个警告  →  之后 [N'] 个失败，[M'] 个警告
类别：   之前 [N] 个失败，[M] 个警告  →  之后 [N'] 个失败，[M'] 个警告（如适用）
综合变化：改进 / 无变化 / 变差
```

---

## 阶段 6：裁决

计算合并失败总数：静态 FAIL + 类别 FAIL + 静态 WARN + 类别 WARN。

**如果综合分数提高（合并失败数低于基准）：**
报告："分数提高。更改已保留。"
显示每个维度修复的内容摘要。

**如果综合分数相同或更差：**
报告："综合分数未提高。"
显示更改的内容以及为什么可能没有帮助。
询问："我可以使用 git checkout 还原 `.claude/skills/[name]/SKILL.md` 吗？"
如果同意：运行 `git checkout -- .claude/skills/[name]/SKILL.md`

---

## 阶段 7：后续步骤

- 运行 `/skill-test static all` 以查找下一个有失败的技能。
- 运行 `/skill-improve [next-name]` 以在另一个技能上继续循环。
- 运行 `/skill-test audit` 以查看整体覆盖进度。
